<h1>ct_slave</h1> <h2 class="modsummary">Common Test Framework functions for starting and stopping nodes for Large Scale Testing.</h2> <div class="description"> <p>Common Test Framework functions for starting and stopping nodes for Large Scale Testing.</p> <p>This module exports functions which are used by the Common Test Master to start and stop "slave" nodes. It is the default callback module for the <code>{init, node_start}</code> term of the Test Specification.</p></div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="start/1">start(Node) -&gt; Result</h3> <ul class="type"> <li><code>Node = atom()</code></li><li><code>Result = {ok, NodeName} | {error, Reason, NodeName}</code></li><li><code>Reason = already_started | started_not_connected | boot_timeout | init_timeout | startup_timeout | not_alive</code></li><li><code>NodeName = atom()</code></li></ul> <div class="description"><span id="start-1"> </span> <p>Starts an Erlang node with name <code>Node</code> on the local host.</p> <p><em>See also:</em> <a href="#start-3" class="seealso">start/3</a>.</p> </div></div> <div class="function"> <h3 id="start/2">start(HostOrNode, NodeOrOpts) -&gt; Result</h3> <ul class="type"> <li><code>HostOrNode = atom()</code></li><li><code>NodeOrOpts = atom() | list()</code></li><li><code>Result = {ok, NodeName} | {error, Reason, NodeName}</code></li><li><code>Reason = already_started | started_not_connected | boot_timeout | init_timeout | startup_timeout | not_alive</code></li><li><code>NodeName = atom()</code></li></ul> <div class="description"><span id="start-2"> </span> <p>Starts an Erlang node with default options on a specified host, or on the local host with specified options. That is, the call is interpreted as <code>start(Host, Node)</code> when the second argument is atom-valued and <code>start(Node, Opts)</code> when it's list-valued.</p> <p><em>See also:</em> <a href="#start-3" class="seealso">start/3</a>.</p> </div></div> <div class="function"> <h3 id="start/3">start(Host, Node, Opts) -&gt; Result</h3> <ul class="type"> <li><code>Node = atom()</code></li><li><code>Host = atom()</code></li><li><code>Opts = [OptTuples]</code></li><li><code>OptTuples = {username, Username} | {password, Password} | {boot_timeout, BootTimeout} | {init_timeout, InitTimeout} | {startup_timeout, StartupTimeout} | {startup_functions, StartupFunctions} | {monitor_master, Monitor} | {kill_if_fail, KillIfFail} | {erl_flags, ErlangFlags} | {env, [{EnvVar, Value}]}</code></li><li><code>Username = string()</code></li><li><code>Password = string()</code></li><li><code>BootTimeout = integer()</code></li><li><code>InitTimeout = integer()</code></li><li><code>StartupTimeout = integer()</code></li><li><code>StartupFunctions = [StartupFunctionSpec]</code></li><li><code>StartupFunctionSpec = {Module, Function, Arguments}</code></li><li><code>Module = atom()</code></li><li><code>Function = atom()</code></li><li><code>Arguments = [term]</code></li><li><code>Monitor = bool()</code></li><li><code>KillIfFail = bool()</code></li><li><code>ErlangFlags = string()</code></li><li><code>EnvVar = string()</code></li><li><code>Value = string()</code></li><li><code>Result = {ok, NodeName} | {error, Reason, NodeName}</code></li><li><code>Reason = already_started | started_not_connected | boot_timeout | init_timeout | startup_timeout | not_alive</code></li><li><code>NodeName = atom()</code></li></ul> <div class="description"><span id="start-3"> </span> <p>Starts an Erlang node with name <code>Node</code> on host <code>Host</code> as specified by the combination of options in <code>Opts</code>.</p> <p>Options <code>Username</code> and <code>Password</code> will be used to log in onto the remote host <code>Host</code>. Username, if omitted, defaults to the current user name, and password is empty by default.</p> <p>A list of functions specified in the <code>Startup</code> option will be executed after startup of the node. Note that all used modules should be present in the code path on the <code>Host</code>.</p> <p>The timeouts are applied as follows: <list> <item> <code>BootTimeout</code> - time to start the Erlang node, in seconds. Defaults to 3 seconds. If node does not become pingable within this time, the result <code>{error, boot_timeout, NodeName}</code> is returned; </item> <item> <code>InitTimeout</code> - time to wait for the node until it calls the internal callback function informing master about successfull startup. Defaults to one second. In case of timed out message the result <code>{error, init_timeout, NodeName}</code> is returned; </item> <item> <code>StartupTimeout</code> - time to wait intil the node finishes to run the <code>StartupFunctions</code>. Defaults to one second. If this timeout occurs, the result <code>{error, startup_timeout, NodeName}</code> is returned. </item> </list></p> <p>Option <code>monitor_master</code> specifies, if the slave node should be stopped in case of master node stop. Defaults to false.</p> <p>Option <code>kill_if_fail</code> specifies, if the slave node should be killed in case of a timeout during initialization or startup. Defaults to true. Note that node also may be still alive it the boot timeout occurred, but it will not be killed in this case.</p> <p>Option <code>erlang_flags</code> specifies, which flags will be added to the parameters of the <code>erl</code> executable.</p> <p>Option <code>env</code> specifies a list of environment variables that will extended the environment.</p> <p>Special return values are: <list> <item><code>{error, already_started, NodeName}</code> - if the node with the given name is already started on a given host;</item> <item><code>{error, started_not_connected, NodeName}</code> - if node is started, but not connected to the master node.</item> <item><code>{error, not_alive, NodeName}</code> - if node on which the <code>ct_slave:start/3</code> is called, is not alive. Note that <code>NodeName</code> is the name of current node in this case.</item> </list></p> </div></div> <div class="function"> <h3 id="stop/1">stop(Node) -&gt; Result</h3> <ul class="type"> <li><code>Node = atom()</code></li><li><code>Result = {ok, NodeName} | {error, Reason, NodeName}</code></li><li><code>Reason = not_started | not_connected | stop_timeout</code></li></ul> <div class="description"><span id="stop-1"> </span> <p>Stops the running Erlang node with name <code>Node</code> on the localhost.</p> </div></div> <div class="function"> <h3 id="stop/2">stop(Host, Node) -&gt; Result</h3> <ul class="type"> <li><code>Host = atom()</code></li><li><code>Node = atom()</code></li><li><code>Result = {ok, NodeName} | {error, Reason, NodeName}</code></li><li><code>Reason = not_started | not_connected | stop_timeout</code></li><li><code>NodeName = atom()</code></li></ul> <div class="description"><span id="stop-2"> </span> <p>Stops the running Erlang node with name <code>Node</code> on host <code>Host</code>.</p> </div></div></div> <authors> <aname> </aname> <email> </email></authors>
