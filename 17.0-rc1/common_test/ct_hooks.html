<h1>ct_hooks</h1> <h2 class="modsummary">A callback interface on top of Common Test</h2> <div class="description"> <p>The <em>Common Test Hook</em> (henceforth called CTH) framework allows extensions of the default behaviour of Common Test by means of callbacks before and after all test suite calls. It is meant for advanced users of Common Test which want to abstract out behaviour which is common to multiple test suites. </p> <p>In brief, Common Test Hooks allows you to:</p> <list> <item>Manipulate the runtime config before each suite configuration call</item> <item>Manipulate the return of all suite configuration calls and in extension the result of the test themselves.</item> </list> <p>The following sections describe the mandatory and optional CTH functions Common Test will call during test execution. For more details see <a href="ct_hooks_chapter.html" class="seealso">Common Test Hooks</a> in the User's Guide.</p> <p>For information about how to add a CTH to your suite see <a href="ct_hooks_chapter.html#installing" class="seealso">Installing a CTH </a> in the User's Guide.</p> <div class="note"><h2>Note!</h2><p>See the <a href="ct_hooks_chapter.html#example" class="seealso">Example CTH</a> in the User's Guide for a minimal example of a CTH. </p></div> </div> <div class="section"> <h4>CALLBACK FUNCTIONS</h4> <p>The following functions define the callback interface for a Common Test Hook.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="init/2">Module:init(Id, Opts) -&gt; {ok, State} | {ok, State, Priority}</h3> <ul class="type"> <li><code>Id = reference() | term()</code></li> <li><code>Opts = term()</code></li> <li><code>State = term()</code></li> <li><code>Priority = integer()</code></li> </ul> <div class="description"> <p> MANDATORY </p> <p>Always called before any other callback function. Use this to initiate any common state. It should return a state for this CTH.</p> <p><code>Id</code> is the return value of <a href="../#Module/id-1.html" class="seealso">id/1</a>, or a <code>reference</code> (created using <a href="../erts/erlang.html#make_ref-0" class="seealso">make_ref/0</a>) if <a href="../#Module/id-1.html" class="seealso">id/1</a> is not implemented. </p> <p><code>Priority</code> is the relative priority of this hook. Hooks with a lower priority will be executed first. If no priority is given, it will be set to 0. </p> <p>For details about when init is called see <a href="ct_hooks_chapter.html#scope" class="seealso">scope</a> in the User's Guide.</p> </div> </div> <div class="function"> <h3 id="pre_init_per_suite/3">Module:pre_init_per_suite(SuiteName, InitData, CTHState) -&gt; Result</h3> <ul class="type"> <li><code>SuiteName = atom()</code></li> <li><code>InitData = Config | SkipOrFail</code></li> <li><code>Config = NewConfig = [{Key,Value}]</code></li> <li><code>CTHState = NewCTHState = term()</code></li> <li><code>Result = {Return, NewCTHState}</code></li> <li><code>Return = NewConfig | SkipOrFail</code></li> <li><code>SkipOrFail = {fail, Reason} | {skip, Reason}</code></li> <li><code>Key = atom()</code></li> <li><code>Value = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called before <a href="../common_test#Module/init_per_suite-1.html" class="seealso"> init_per_suite</a> if it exists. It typically contains initialization/logging which needs to be done before init_per_suite is called. If <code>{skip,Reason}</code> or <code>{fail,Reason}</code> is returned, init_per_suite and all test cases of the suite will be skipped and Reason printed in the overview log of the suite.</p> <p><code>SuiteName</code> is the name of the suite to be run.</p> <p><code>InitData</code> is the original config list of the test suite, or a <code>SkipOrFail</code> tuple if a previous CTH has returned this.</p> <p><code>CTHState</code> is the current internal state of the CTH.</p> <p><code>Return</code> is the result of the init_per_suite function. If it is <code>{skip,Reason}</code> or <code>{fail,Reason}</code> <a href="../common_test#Module/init_per_suite-1.html" class="seealso">init_per_suite </a> will never be called, instead the initiation is considered to be skipped/failed respectively. If a <code>NewConfig</code> list is returned, <a href="../common_test#Module/init_per_suite-1.html" class="seealso"> init_per_suite</a> will be called with that <code>NewConfig</code> list. See <a href="ct_hooks_chapter.html#pre" class="seealso"> Pre Hooks</a> in the User's Guide for more details.</p> <p>Note that this function is only called if the CTH has been added before init_per_suite is run, see <a href="ct_hooks_chapter.html#scope" class="seealso">CTH Scoping</a> in the User's Guide for details.</p> </div> </div> <div class="function"> <h3 id="post_init_per_suite/4">Module:post_init_per_suite(SuiteName, Config, Return, CTHState) -&gt; Result</h3> <ul class="type"> <li><code>SuiteName = atom()</code></li> <li><code>Config = [{Key,Value}]</code></li> <li><code>Return = NewReturn = Config | SkipOrFail | term()</code></li> <li><code>SkipOrFail = {fail, Reason} | {skip, Reason} | term()</code></li> <li><code>CTHState = NewCTHState = term()</code></li> <li><code>Result = {NewReturn, NewCTHState}</code></li> <li><code>Key = atom()</code></li> <li><code>Value = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called after <a href="../common_test#Module/init_per_suite-1.html" class="seealso"> init_per_suite</a> if it exists. It typically contains extra checks to make sure that all the correct dependencies have been started correctly.</p> <p><code>Return</code> is what <a href="../common_test#Module/init_per_suite-1.html" class="seealso">init_per_suite </a> returned, i.e. {fail,Reason}, {skip,Reason}, a <code>Config</code> list or a term describing how <a href="../common_test#Module/init_per_suite-1.html" class="seealso">init_per_suite </a> failed.</p> <p><code>NewReturn</code> is the possibly modified return value of <a href="../common_test#Module/init_per_suite-1.html" class="seealso">init_per_suite </a>. It is here possible to recover from a failure in <a href="../common_test#Module/init_per_suite-1.html" class="seealso">init_per_suite </a> by returning the <code>ConfigList</code> with the <code>tc_status</code> element removed. See <a href="ct_hooks_chapter.html#post" class="seealso"> Post Hooks</a> in the User's Guide for more details.</p> <p><code>CTHState</code> is the current internal state of the CTH.</p> <p>Note that this function is only called if the CTH has been added before or in init_per_suite, see <a href="ct_hooks_chapter.html#scope" class="seealso">CTH Scoping</a> in the User's Guide for details.</p> </div> </div> <div class="function"> <h3 id="pre_init_per_group/3">Module:pre_init_per_group(GroupName, InitData, CTHState) -&gt; Result</h3> <ul class="type"> <li><code>GroupName = atom()</code></li> <li><code>InitData = Config | SkipOrFail</code></li> <li><code>Config = NewConfig = [{Key,Value}]</code></li> <li><code>CTHState = NewCTHState = term()</code></li> <li><code>Result = {NewConfig | SkipOrFail, NewCTHState}</code></li> <li><code>SkipOrFail = {fail,Reason} | {skip, Reason}</code></li> <li><code>Key = atom()</code></li> <li><code>Value = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called before <a href="../common_test#Module/init_per_group-2.html" class="seealso"> init_per_group</a> if it exists. It behaves the same way as <a href="../ct_hooks#Module/pre_init_per_suite-3.html" class="seealso"> pre_init_per_suite</a>, but for the <a href="../common_test#Module/init_per_group-2.html" class="seealso"> init_per_group</a> instead.</p> </div> </div> <div class="function"> <h3 id="post_init_per_group/4">Module:post_init_per_group(GroupName, Config, Return, CTHState) -&gt; Result</h3> <ul class="type"> <li><code>GroupName = atom()</code></li> <li><code>Config = [{Key,Value}]</code></li> <li><code>Return = NewReturn = Config | SkipOrFail | term()</code></li> <li><code>SkipOrFail = {fail,Reason} | {skip, Reason}</code></li> <li><code>CTHState = NewCTHState = term()</code></li> <li><code>Result = {NewReturn, NewCTHState}</code></li> <li><code>Key = atom()</code></li> <li><code>Value = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called after <a href="../common_test#Module/init_per_group-2.html" class="seealso"> init_per_group</a> if it exists. It behaves the same way as <a href="../ct_hooks#Module/post_init_per_suite-4.html" class="seealso"> post_init_per_suite</a>, but for the <a href="../common_test#Module/init_per_group-2.html" class="seealso"> init_per_group</a> instead.</p> </div> </div> <div class="function"> <h3 id="pre_init_per_testcase/3">Module:pre_init_per_testcase(TestcaseName, InitData, CTHState) -&gt; Result</h3> <ul class="type"> <li><code>TestcaseName = atom()</code></li> <li><code>InitData = Config | SkipOrFail</code></li> <li><code>Config = NewConfig = [{Key,Value}]</code></li> <li><code>CTHState = NewCTHState = term()</code></li> <li><code>Result = {NewConfig | SkipOrFail, NewCTHState}</code></li> <li><code>SkipOrFail = {fail,Reason} | {skip, Reason}</code></li> <li><code>Key = atom()</code></li> <li><code>Value = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called before <a href="../common_test#Module/init_per_testcase-2.html" class="seealso"> init_per_testcase</a> if it exists. It behaves the same way as <a href="../ct_hooks#Module/pre_init_per_suite-3.html" class="seealso"> pre_init_per_suite</a>, but for the <a href="../common_test#Module/init_per_testcase-2.html" class="seealso"> init_per_testcase</a> function instead.</p> <p>Note that it is not possible to add CTH's here right now, that feature might be added later, but it would right now break backwards compatibility.</p> </div> </div> <div class="function"> <h3 id="post_end_per_testcase/4">Module:post_end_per_testcase(TestcaseName, Config, Return, CTHState) -&gt; Result</h3> <ul class="type"> <li><code>TestcaseName = atom()</code></li> <li><code>Config = [{Key,Value}]</code></li> <li><code>Return = NewReturn = Config | SkipOrFail | term()</code></li> <li><code>SkipOrFail = {fail,Reason} | {skip, Reason}</code></li> <li><code>CTHState = NewCTHState = term()</code></li> <li><code>Result = {NewReturn, NewCTHState}</code></li> <li><code>Key = atom()</code></li> <li><code>Value = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called after <a href="../common_test#Module/end_per_testcase-2.html" class="seealso"> end_per_testcase</a> if it exists. It behaves the same way as <a href="../ct_hooks#Module/post_init_per_suite-4.html" class="seealso"> post_init_per_suite</a>, but for the <a href="../common_test#Module/end_per_testcase-2.html" class="seealso"> end_per_testcase</a> function instead.</p> </div> </div> <div class="function"> <h3 id="pre_end_per_group/3">Module:pre_end_per_group(GroupName, EndData, CTHState) -&gt; Result</h3> <ul class="type"> <li><code>GroupName = atom()</code></li> <li><code>EndData = Config | SkipOrFail</code></li> <li><code>Config = NewConfig = [{Key,Value}]</code></li> <li><code>CTHState = NewCTHState = term()</code></li> <li><code>Result = {NewConfig | SkipOrFail, NewCTHState}</code></li> <li><code>SkipOrFail = {fail,Reason} | {skip, Reason}</code></li> <li><code>Key = atom()</code></li> <li><code>Value = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called before <a href="../common_test#Module/end_per_group-2.html" class="seealso"> end_per_group</a> if it exists. It behaves the same way as <a href="../ct_hooks#Module/pre_init_per_suite-3.html" class="seealso"> pre_init_per_suite</a>, but for the <a href="../common_test#Module/end_per_group-2.html" class="seealso"> end_per_group</a> function instead.</p> </div> </div> <div class="function"> <h3 id="post_end_per_group/4">Module:post_end_per_group(GroupName, Config, Return, CTHState) -&gt; Result</h3> <ul class="type"> <li><code>GroupName = atom()</code></li> <li><code>Config = [{Key,Value}]</code></li> <li><code>Return = NewReturn = Config | SkipOrFail | term()</code></li> <li><code>SkipOrFail = {fail,Reason} | {skip, Reason}</code></li> <li><code>CTHState = NewCTHState = term()</code></li> <li><code>Result = {NewReturn, NewCTHState}</code></li> <li><code>Key = atom()</code></li> <li><code>Value = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called after <a href="../common_test#Module/end_per_group-2.html" class="seealso"> end_per_group</a> if it exists. It behaves the same way as <a href="../ct_hooks#Module/post_init_per_suite-4.html" class="seealso"> post_init_per_suite</a>, but for the <a href="../common_test#Module/end_per_group-2.html" class="seealso"> end_per_group</a> function instead.</p> </div> </div> <div class="function"> <h3 id="pre_end_per_suite/3">Module:pre_end_per_suite(SuiteName, EndData, CTHState) -&gt; Result</h3> <ul class="type"> <li><code>SuiteName = atom()</code></li> <li><code>EndData = Config | SkipOrFail</code></li> <li><code>Config = NewConfig = [{Key,Value}]</code></li> <li><code>CTHState = NewCTHState = term()</code></li> <li><code>Result = {NewConfig | SkipOrFail, NewCTHState}</code></li> <li><code>SkipOrFail = {fail,Reason} | {skip, Reason}</code></li> <li><code>Key = atom()</code></li> <li><code>Value = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called before <a href="../common_test#Module/end_per_suite-1.html" class="seealso"> end_per_suite</a> if it exists. It behaves the same way as <a href="../ct_hooks#Module/pre_init_per_suite-3.html" class="seealso"> pre_init_per_suite</a>, but for the <a href="../common_test#Module/end_per_suite-1.html" class="seealso"> end_per_suite</a> function instead.</p> </div> </div> <div class="function"> <h3 id="post_end_per_suite/4">Module:post_end_per_suite(SuiteName, Config, Return, CTHState) -&gt; Result</h3> <ul class="type"> <li><code>SuiteName = atom()</code></li> <li><code>Config = [{Key,Value}]</code></li> <li><code>Return = NewReturn = Config | SkipOrFail | term()</code></li> <li><code>SkipOrFail = {fail,Reason} | {skip, Reason}</code></li> <li><code>CTHState = NewCTHState = term()</code></li> <li><code>Result = {NewReturn, NewCTHState}</code></li> <li><code>Key = atom()</code></li> <li><code>Value = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called after <a href="../common_test#Module/end_per_suite-1.html" class="seealso"> end_per_suite</a> if it exists. It behaves the same way as <a href="../ct_hooks#Module/post_init_per_suite-4.html" class="seealso"> post_init_per_suite</a>, but for the <a href="../common_test#Module/end_per_suite-1.html" class="seealso"> end_per_suite</a> function instead.</p> </div> </div> <div class="function"> <h3 id="on_tc_fail/3">Module:on_tc_fail(TestcaseName, Reason, CTHState) -&gt; NewCTHState</h3> <ul class="type"> <li><code>TestcaseName = init_per_suite | end_per_suite | init_per_group | end_per_group | atom()</code></li> <li><code>Reason = term()</code></li> <li><code>CTHState = NewCTHState = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called whenever a testcase fails. It is called after the post function has been called for the testcase which failed. i.e. if init_per_suite fails this function is called after <a href="../#Module/post_init_per_suite-4.html" class="seealso"> post_init_per_suite</a>, and if a testcase fails it is called after <a href="../#Module/post_end_per_testcase-4.html" class="seealso"> post_end_per_testcase</a>.</p> <p>The data which comes with the Reason follows the same format as the <a href="event_handler_chapter.html#failreason" class="seealso">FailReason </a> in the <a href="event_handler_chapter.html#tc_done" class="seealso">tc_done</a> event. See <a href="event_handler_chapter.html#events" class="seealso">Event Handling </a> in the User's Guide for details.</p> </div> </div> <div class="function"> <h3 id="on_tc_skip/3">Module:on_tc_skip(TestcaseName, Reason, CTHState) -&gt; NewCTHState</h3> <ul class="type"> <li><code>TestcaseName = end_per_suite | {init_per_group,GroupName} | {end_per_group,GroupName} | atom()</code></li> <li><code>GroupName = atom()</code></li> <li><code>Reason = {tc_auto_skip | tc_user_skip, term()}</code></li> <li><code>CTHState = NewCTHState = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called whenever a testcase is skipped. It is called after the post function has been called for the testcase which was skipped. i.e. if init_per_group is skipped this function is called after <a href="../#Module/post_init_per_suite-4.html" class="seealso">post_init_per_group </a>, and if a testcase is skipped it is called after <a href="../#Module/post_end_per_testcase-4.html" class="seealso">post_end_per_testcase </a>.</p> <p>The data which comes with the Reason follows the same format as <a href="event_handler_chapter.html#tc_auto_skip" class="seealso">tc_auto_skip </a> and <a href="event_handler_chapter.html#tc_user_skip" class="seealso"> tc_user_skip</a> events. See <a href="event_handler_chapter.html#events" class="seealso">Event Handling </a> in the User's Guide for details.</p> </div> </div> <div class="function"> <h3 id="terminate/1">Module:terminate(CTHState)</h3> <ul class="type"> <li><code>CTHState = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called at the end of a CTH's <a href="ct_hooks_chapter.html#scope" class="seealso">scope</a>. </p> </div> </div> <div class="function"> <h3 id="id/1">Module:id(Opts) -&gt; Id</h3> <ul class="type"> <li><code>Opts = term()</code></li> <li><code>Id = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>The <code>Id</code> is used to uniquely identify a CTH instance, if two CTH's return the same <code>Id</code> the second CTH is ignored and subsequent calls to the CTH will only be made to the first instance. For more information see <a href="ct_hooks_chapter.html#installing" class="seealso">Installing a CTH </a> in the User's Guide. </p> <p>This function should NOT have any side effects as it might be called multiple times by Common Test.</p> <p>If not implemented the CTH will act as if this function returned a call to <code>make_ref/0</code>.</p> </div> </div> </div>
