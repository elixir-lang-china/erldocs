<h1>common_test</h1> <h2 class="modsummary">A framework for automated testing of arbitrary target nodes</h2> <div class="description"> <p>The <em>Common Test</em> framework is an environment for implementing and performing automatic and semi-automatic execution of test cases. Common Test uses the OTP Test Server as engine for test case execution and logging.</p> <p>In brief, Common Test supports:</p> <list> <item>Automated execution of test suites (sets of test cases).</item> <item>Logging of the events during execution.</item> <item>HTML presentation of test suite results.</item> <item>HTML presentation of test suite code.</item> <item>Support functions for test suite authors.</item> <item>Step by step execution of test cases.</item> </list> <p>The following sections describe the mandatory and optional test suite functions Common Test will call during test execution. For more details see <a href="write_test_chapter.html" class="seealso">Common Test User's Guide.</a> </p> </div> <div class="section"> <h4>TEST CASE CALLBACK FUNCTIONS</h4> <p>The following functions define the callback interface for a test suite.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="all/0">Module:all() -&gt; Tests | {skip,Reason} </h3> <ul class="type"> <li><code>Tests = [TestCase | {group,GroupName} | {group,GroupName,Properties} | {group,GroupName,Properties,SubGroups}]</code></li> <li><code>TestCase = atom()</code></li> <li><code>GroupName = atom()</code></li> <li><code>Properties = [parallel | sequence | Shuffle | {RepeatType,N}] | default</code></li> <li><code>SubGroups = [{GroupName,Properties} | {GroupName,Properties,SubGroups}]</code></li> <li><code>Shuffle = shuffle | {shuffle,Seed}</code></li> <li><code>Seed = {integer(),integer(),integer()}</code></li> <li><code>RepeatType = repeat | repeat_until_all_ok | repeat_until_all_fail | repeat_until_any_ok | repeat_until_any_fail</code></li> <li><code>N = integer() | forever</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p> MANDATORY </p> <p>This function must return the list of all test cases and test case groups in the test suite module that are to be executed. This list also specifies the order the cases and groups will be executed by Common Test. A test case is represented by an atom, the name of the test case function. A test case group is represented by a <code>group</code> tuple, where <code>GroupName</code>, an atom, is the name of the group (defined in <a href="../#Module/groups-0.html" class="seealso"><code>groups/0</code></a>). Execution properties for groups may also be specified, both for a top level group and for any of its sub-groups. Group execution properties specified here, will override properties in the group definition (see <a href="../#Module/groups-0.html" class="seealso"><code>groups/0</code></a>). (With value <code>default</code>, the group definition properties will be used).</p> <p> If <code>{skip,Reason}</code> is returned, all test cases in the module will be skipped, and the <code>Reason</code> will be printed on the HTML result page.</p> <p>For details on groups, see <a href="write_test_chapter.html#test_case_groups" class="seealso">Test case groups</a> in the User's Guide.</p> </div> </div> <div class="function"> <h3 id="groups/0">Module:groups() -&gt; GroupDefs</h3> <ul class="type"> <li><code>GroupDefs = [Group]</code></li> <li><code>Group = {GroupName,Properties,GroupsAndTestCases}</code></li> <li><code>GroupName = atom()</code></li> <li><code>Properties = [parallel | sequence | Shuffle | {RepeatType,N}]</code></li> <li><code>GroupsAndTestCases = [Group | {group,GroupName} | TestCase]</code></li> <li><code>TestCase = atom()</code></li> <li><code>Shuffle = shuffle | {shuffle,Seed}</code></li> <li><code>Seed = {integer(),integer(),integer()}</code></li> <li><code>RepeatType = repeat | repeat_until_all_ok | repeat_until_all_fail | repeat_until_any_ok | repeat_until_any_fail</code></li> <li><code>N = integer() | forever</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>Function for defining test case groups. Please see <a href="write_test_chapter.html#test_case_groups" class="seealso">Test case groups</a> in the User's Guide for details.</p> </div> </div> <div class="function"> <h3 id="suite/0">Module:suite() -&gt; [Info] </h3> <ul class="type"> <li><code> Info = {timetrap,Time} | {require,Required} | {require,Name,Required} | {userdata,UserData} | {silent_connections,Conns} | {stylesheet,CSSFile} | {ct_hooks, CTHs}</code></li> <li><code> Time = TimeVal | TimeFunc</code></li> <li><code> TimeVal = MilliSec | {seconds,integer()} | {minutes,integer()} | {hours,integer()}</code></li> <li><code> TimeFunc = {Mod,Func,Args} | Fun</code></li> <li><code> MilliSec = integer()</code></li> <li><code> Mod = atom()</code></li> <li><code> Func = atom()</code></li> <li><code> Args = list()</code></li> <li><code> Fun = fun()</code></li> <li><code> Required = Key | {Key,SubKeys} | {Key,SubKey} | {Key,SubKey,SubKeys}</code></li> <li><code> Key = atom()</code></li> <li><code> SubKeys = SubKey | [SubKey]</code></li> <li><code> SubKey = atom()</code></li> <li><code> Name = atom()</code></li> <li><code> UserData = term()</code></li> <li><code> Conns = [atom()]</code></li> <li><code> CSSFile = string()</code></li> <li><code> CTHs = [CTHModule |</code></li> <li><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;{CTHModule, CTHInitArgs} |</code></li> <li><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;{CTHModule, CTHInitArgs, CTHPriority}]</code></li> <li><code> CTHModule = atom()</code></li> <li><code> CTHInitArgs = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This is the test suite info function. It is supposed to return a list of tagged tuples that specify various properties related to the execution of this test suite (common for all test cases in the suite).</p> <p>The <code>timetrap</code> tag sets the maximum time each test case is allowed to execute (including <a href="../#Module/init_per_testcase-2.html" class="seealso"><code>init_per_testcase/2</code></a> and <a href="../#Module/end_per_testcase-2.html" class="seealso"><code>end_per_testcase/2</code></a>). If the timetrap time is exceeded, the test case fails with reason <code>timetrap_timeout</code>. A <code>TimeFunc</code> function can be used to set a new timetrap by returning a <code>TimeVal</code>. It may also be used to trigger a timetrap timeout by, at some point, returning a value other than a <code>TimeVal</code>. (See the <a href="write_test_chapter.html#timetraps" class="seealso">User's Guide</a> for details). </p> <p>The <code>require</code> tag specifies configuration variables that are required by test cases (and/or configuration functions) in the suite. If the required configuration variables are not found in any of the configuration files, all test cases are skipped. For more information about the 'require' functionality, see the reference manual for the function <a href="ct.html#require-1" class="seealso"><code>ct:require/1/2</code></a>.</p> <p>With <code>userdata</code>, it is possible for the user to specify arbitrary test suite related information which can be read by calling <a href="ct.html#userdata-2" class="seealso"><code>ct:userdata/2</code></a>.</p> <p>The <code>ct_hooks</code> tag specifies which <a href="ct_hooks_chapter.html" class="seealso">Common Test Hooks</a> are to be run together with this suite.</p> <p>Other tuples than the ones defined will simply be ignored.</p> <p>For more information about the test suite info function, see <a href="write_test_chapter.html#suite" class="seealso">Test suite info function</a> in the User's Guide.</p> </div> </div> <div class="function"> <h3 id="init_per_suite/1">Module:init_per_suite(Config) -&gt; NewConfig | {skip,Reason} | {skip_and_save,Reason,SaveConfig}</h3> <ul class="type"> <li><code> Config = NewConfig = SaveConfig = [{Key,Value}]</code></li> <li><code> Key = atom()</code></li> <li><code> Value = term()</code></li> <li><code> Reason = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This configuration function is called as the first function in the suite. Whatever is returned from this function is given as <code>Config</code> to all configuration functions and test cases in the suite. If <code>{skip,Reason}</code> is returned, all test cases in the suite will be skipped and <code>Reason</code> printed in the overview log for the suite.</p> <p>For information on <code>save_config</code> and <code>skip_and_save</code>, please see <a href="dependencies_chapter.html#save_config" class="seealso">Dependencies between Test Cases and Suites</a> in the User's Guide.</p> </div> </div> <div class="function"> <h3 id="end_per_suite/1">Module:end_per_suite(Config) -&gt; void() | {save_config,SaveConfig}</h3> <ul class="type"> <li><code> Config = SaveConfig = [{Key,Value}]</code></li> <li><code> Key = atom()</code></li> <li><code> Value = term()</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called as the last test case in the suite. The return value of this function is given as <code>Config</code> to all test cases and sub-groups in the group. If <code>{skip,Reason}</code> is returned, all test cases in the group will be skipped and <code>Reason</code> printed in the overview log for the group.</p> <p>For information about test case groups, please see <a href="write_test_chapter.html#test_case_groups" class="seealso">Test case groups</a> chapter in the User's Guide.</p> </div> </div> <div class="function"> <h3 id="end_per_group/2">Module:end_per_group(GroupName, Config) -&gt; void() | {return_group_result,Status}</h3> <ul class="type"> <li><code> GroupName = atom()</code></li> <li><code> Config = [{Key,Value}]</code></li> <li><code> Key = atom()</code></li> <li><code> Value = term()</code></li> <li><code> Status = ok | skipped | failed</code></li> </ul> <div class="description"> <p> OPTIONAL </p> <p>This function is called after the execution of a test case group is finished.
