<h1>unix_telnet</h1> <h2 class="modsummary">Callback module for ct_telnet for talking telnet to a unix host.</h2> <div class="description"> <p>Callback module for ct_telnet for talking telnet to a unix host.</p> <p>It requires the following entry in the config file:</p> <pre class="sh_erlang"> {unix,[{telnet,HostNameOrIpAddress}, {port,PortNum}, % optional {username,UserName}, {password,Password}, {keep_alive,Bool}]}. % optional</pre> <p>To talk telnet to the host specified by <code>HostNameOrIpAddress</code>, use the interface functions in <code>ct</code>, e.g. <code>open(Name), cmd(Name,Cmd), ...</code>.</p> <p><code>Name</code> is the name you allocated to the unix host in your <code>require</code> statement. E.g.</p> <pre class="sh_erlang"> suite() -&gt; [{require,Name,{unix,[telnet,username,password]}}].</pre> <p>or</p> <pre class="sh_erlang"> ct:require(Name,{unix,[telnet,username,password]}).</pre> <p>The "keep alive" activity (i.e. that Common Test sends NOP to the server every 10 seconds if the connection is idle) may be enabled or disabled for one particular connection as described here. It may be disabled for all connections using <code>telnet_settings</code> (see <code>ct_telnet</code>).</p> <p>Note that the <code>{port,PortNum}</code> tuple is optional and if omitted, default telnet port 23 will be used. Also the <code>keep_alive</code> tuple is optional, and the value defauls to true (enabled).</p> </div> <div class="section"><h4>See also</h4><p><a href="ct.html" class="seealso">ct</a>, <a href="ct_telnet.html" class="seealso">ct_telnet</a></p></div> <authors> <aname> </aname> <email> </email></authors>
