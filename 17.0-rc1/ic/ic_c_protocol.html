<h1>ic_c_protocol</h1> <libsummary>IC C Protocol Functions</libsummary> <div class="description"> <p>This manual page lists some of the functions of the IC C runtime library that are used internally for the IC protocol. </p> <p>The listed functions are used internally by generated C client and server code. They are documented here for <em>the advanced user</em> that want to replace the default protocol (Erlang distribution + gen_server) by his own protocol, For each set of client or sever functions below with prefix <code>oe</code>, the user has to implement his own set of functions, the names of which are obtained by replacing the <code>oe</code> prefix by <code>Prefix</code>. The <code>Prefix</code> has to be set with the option <code>{user_protocol, Prefix}</code> at compile time.</p> <p>The following terminology is used (reflected in names of functions): a <em>notification</em> is a message send from client to server, without any reply back (i.e. a <em>oneway</em> operation); a <em>request</em> is a message sent from client to server, and where a <em>reply</em> message is sent back from the server to the client.</p> <p>In order to understand how the functions work and what they do the user <em>must</em> study their implementation in the IC C library (source file is <code>ic.c</code>), and also consider how they are used in the C code of ordinary generated client stubs or server skeletons.</p> <p /> </div> <div class="section"> <h4>Client Protocol Functions</h4> <p>The following functions are used internally by generated C client code.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <pre>int oe_prepare_notification_encoding(CORBA_Environment *env)</pre> <div class="description"> <p>The result of this function is the beginning of a binary of in external format of the tuple <code>{'$gen_cast', X}</code> where <code>X</code> is not yet filled in. </p> <p>In generated client code this function is the first to be called in the encoding function for each oneway operation.</p> </div> </div> <div class="function"> <pre>int oe_send_notification(CORBA_Environment *env)</pre> <pre>int oe_send_notification_tmo(CORBA_Environment *env, unsigned int send_ms)</pre> <div class="description"> <p>Sends a client notification to a server according to the Erlang distribution + gen_server protocol.</p> <p>The <code>send_ms</code> parameter specified a timeout in milliseconds.</p> </div> </div> <div class="function"> <pre>int oe_prepare_request_encoding(CORBA_Environment *env)</pre> <div class="description"> <p>The result of this function is the beginning of a binary in the external format of the tuple <code>{'$gen_call', {Pid, Ref}, X}</code> where <code>X</code> is not yet filled in.</p> <p>In generated client code this function is the first to be called in the encoding function for each twoway operation.</p> </div> </div> <div class="function"> <pre>int oe_send_request_and_receive_reply(CORBA_Environment *env)</pre> <pre>int oe_send_request_and_receive_reply_tmo(CORBA_Environment *env, unsigned int send_ms, unsigned int recv_ms)</pre> <div class="description"> <p>Sends a client request and receives the reply according to the Erlang distribution + gen_server protocol. This function calls the <code>oe_prepare_reply_decoding</code> function in order to obtain the gen_server reply. </p> <p><code>send_ms</code> and <code>recv_ms</code> specify timeouts for send and receive, respectively, in milliseconds.</p> </div> </div> <div class="function"> <pre>int oe_prepare_reply_decoding(CORBA_Environment *env)</pre> <div class="description"> <p>Decodes the binary version of the tuple <code>{Ref, X}</code>, where <code>X</code> is to be decoded later by the specific client decoding function.</p> </div> </div> </div> <div class="section"> <h4>Server Protocol Functions</h4> <p>The following functions are used internally by generated C server code.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <pre>int oe_prepare_request_decoding(CORBA_Environment *env)</pre> <div class="description"> <p>Decodes the binary version of the tuple <code>{'$gen_cast', Op}</code> (<code>Op</code> an atom), or the tuple <code>{'$gen_cast', {Op, X}}</code>, where <code>Op</code> is the operation name, and where <code>X</code> is to be decoded later by the specific operation decoding function; or</p> <p>decodes the binary version of the tuple <code>{'$gen_call', {Pid, Ref}, Op}</code> (<code>Op</code> an atom), or the tuple <code>{'$gen_call', {Pid, Ref}, {Op, X}}</code>, where <code>Op&gt;</code> is the operation name, and <code>X</code> is to be decode later by the specific operation decoding function.</p> </div> </div> <div class="function"> <pre>int oe_prepare_reply_encoding(CORBA_Environment *env)</pre> <div class="description"> <p>Encodes the beginning of the binary version of the tuple <code>{{Ref,X}</code>, where <code>X</code> is to be filled in by the specific server encoding function.</p> </div> </div> </div> <div class="section"> <h4>SEE ALSO</h4> <p>ic(3), ic_clib(3), <a href="ch_ic_protocol.html" class="seealso">IC Protocol</a></p> </div>
