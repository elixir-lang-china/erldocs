<h1>i</h1> <h2 class="modsummary">Debugger/Interpreter Interface</h2> <div class="description"> <p>The module <code>i</code> provides short forms for some of the functions used by the graphical Debugger and some of the functions in the <code>int</code> module, the Erlang interpreter. </p> <p>This module also provides facilities for displaying status information about interpreted processes and break points.</p> <p>It is possible to attach to interpreted processes by giving the corresponding process identity only. By default, an attachment window pops up. Processes at other Erlang nodes can be attached manually or automatically.</p> <p>By preference, these functions can be included in the module <code>shell_default</code>. By default, they are.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="im/0">im() -&gt; pid()</h3> <div class="description"> <p>Starts a new graphical monitor. This is the Monitor window, the main window of the Debugger. All of the Debugger and interpreter functionality is accessed from the Monitor window. The Monitor window displays the status of all processes that have been/are executing interpreted modules.</p> </div> </div> <div class="function"> <h3 id="ii/1">ii(AbsModules) -&gt; ok</h3> <h3 id="ii/1-1">ii(AbsModule) -&gt; {module, Module} | error</h3> <h3 id="ini/1">ini(AbsModules) -&gt; ok</h3> <h3 id="ini/1-1">ini(AbsModule) -&gt; {module, Module} | error</h3> <ul class="type"> <li><code>AbsModules = [AbsModule]</code></li> <li><code>AbsModule = Module | File</code></li> <li><code>&nbsp;Module = atom()</code></li> <li><code>&nbsp;File = string()</code></li> </ul> <div class="description"> <p>Interprets the specified module(s). <code>ii/1</code> interprets the module(s) only at the current node, see <a href="int.html#i/1" class="seealso">int:i/1</a>. <code>ini/1</code> interprets the module(s) at all known nodes, see <a href="int.html#ni/1" class="seealso">int:ni/1</a>.</p> </div> </div> <div class="function"> <h3 id="iq/1">iq(AbsModule) -&gt; ok</h3> <h3 id="inq/1">inq(AbsModule) -&gt; ok</h3> <ul class="type"> <li><code>AbsModule = Module | File</code></li> <li><code>&nbsp;Module = atom()</code></li> <li><code>&nbsp;File = string()</code></li> </ul> <div class="description"> <p>Stops interpreting the specified module. <code>iq/1</code> stops interpreting the module only at the current node. <code>inq/1</code> stops interpreting the module at all known nodes.</p> </div> </div> <div class="function"> <h3 id="il/0">il() -&gt; ok</h3> <div class="description"> <p>Makes a printout of all interpreted modules. Modules are printed together with the full path name of the corresponding source code file.</p> </div> </div> <div class="function"> <h3 id="ip/0">ip() -&gt; ok</h3> <div class="description"> <p>Makes a printout of the current status of all interpreted processes.</p> </div> </div> <div class="function"> <h3 id="ic/0">ic() -&gt; ok</h3> <div class="description"> <p>Clears information about processes executing interpreted code byt removing all information about terminated processes.</p> </div> </div> <div class="function"> <h3 id="iaa/1">iaa(Flags) -&gt; true</h3> <h3 id="iaa/2">iaa(Flags, Function) -&gt; true</h3> <ul class="type"> <li><code>Flags = [init | break | exit]</code></li> <li><code>Function = {Module,Name,Args}</code></li> <li><code>&nbsp;Module = Name = atom()</code></li> <li><code>&nbsp;Args = [term()]</code></li> </ul> <div class="description"> <p>Sets when and how to automatically attach to a debugged process, see <a href="int.html#auto_attach/0" class="seealso">int:auto_attach/2</a>. <code>Function</code> defaults to the standard function used by the Debugger.</p> </div> </div> <div class="function"> <h3 id="ist/1">ist(Flag) -&gt; true</h3> <ul class="type"> <li><code>Flag = all | no_tail | false</code></li> </ul> <div class="description"> <p>Sets how to save call frames in the stack, see <a href="int.html#stack_trace/0" class="seealso">int:stack_trace/1</a>. </p> </div> </div> <div class="function"> <h3 id="ia/1">ia(Pid) -&gt; ok | no_proc</h3> <ul class="type"> <li><code>Pid = pid()</code></li> </ul> <div class="description"> <p>Attaches to the debugged process <code>Pid</code>. A Debugger Attach Process window is opened for the process.</p> </div> </div> <div class="function"> <h3 id="ia/3">ia(X,Y,Z) -&gt; ok | no_proc</h3> <ul class="type"> <li><code>X = Y = Z = int()</code></li> </ul> <div class="description"> <p>Same as <code>ia(Pid)</code>, where <code>Pid</code> is the result of calling the shell function <code>pid(X,Y,Z)</code>.</p> </div> </div> <div class="function"> <h3 id="ia/2">ia(Pid, Function) -&gt; ok | no_proc</h3> <ul class="type"> <li><code>Pid = pid()</code></li> <li><code>Function = {Module,Name}</code></li> <li><code>&nbsp;Module = Name = atom()</code></li> </ul> <div class="description"> <p>Attaches to the debugged process <code>Pid</code>. The interpreter will call <code>spawn(Module, Name, [Pid])</code> (and ignore the result).</p> </div> </div> <div class="function"> <h3 id="ia/4">ia(X,Y,Z, Function) -&gt; ok | no_proc</h3> <ul class="type"> <li><code>X = Y = Z = int()</code></li> <li><code>Function = {Module,Name}</code></li> <li><code>&nbsp;Module = Name = atom()</code></li> </ul> <div class="description"> <p>Same as <code>ia(Pid, Function)</code>, where <code>Pid</code> is the result of calling the shell function <code>pid(X,Y,Z)</code>. An attached process is expected to call the unofficial <code>int:attached(Pid)</code> function and to be able to handle messages from the interpreter, see <code>dbg_ui_trace.erl</code> for an example.</p> </div> </div> <div class="function"> <h3 id="ib/2">ib(Module, Line) -&gt; ok</h3> <div class="description"> <p>Deletes all breakpoints.</p> </div> </div> <div class="function"> <h3 id="ir/1">ir(Module) -&gt; ok</h3> <ul class="type"> <li><code>Module = atom()</code></li> </ul> <div class="description"> <p>Deletes all breakpoints in <code>Module</code>.</p> </div> </div> <div class="function"> <h3 id="ir/2">ir(Module, Line) -&gt; ok</h3> <ul class="type"> <li><code>Module = atom()</code></li> <li><code>Line = int()</code></li> </ul> <div class="description"> <p>Deletes the breakpoint located at <code>Line</code> in <code>Module</code>.</p> </div> </div> <div class="function"> <h3 id="ir/3">ir(Module, Name, Arity) -&gt; ok | {error, function_not_found} </h3> <ul class="type"> <li><code>Module = Name = atom()</code></li> <li><code>Arity = int()</code></li> </ul> <div class="description"> <p>Deletes the breakpoints at the first line of every clause of the <code>Module:Name/Arity</code> function.</p> </div> </div> <div class="function"> <h3 id="ibd/2">ibd(Module, Line) -&gt; ok</h3> <ul class="type"> <li><code>Module = atom()</code></li> <li><code>Line = int()</code></li> </ul> <div class="description"> <p>Makes the breakpoint at <code>Line</code> in <code>Module</code> inactive.</p> </div> </div> <div class="function"> <h3 id="ibe/2">ibe(Module, Line) -&gt; ok</h3> <ul class="type"> <li><code>Module = atom()</code></li> <li><code>Line = int()</code></li> </ul> <div class="description"> <p>Makes the breakpoint at <code>Line</code> in <code>Module</code> active. </p> </div> </div> <div class="function"> <h3 id="iba/3">iba(Module, Line, Action) -&gt; ok</h3> <ul class="type"> <li><code>Module = atom()</code></li> <li><code>Line = int()</code></li> <li><code>Action = enable | disable | delete</code></li> </ul> <div class="description"> <p>Sets the trigger action of the breakpoint at <code>Line</code> in <code>Module</code> to <code>Action</code>.</p> </div> </div> <div class="function"> <h3 id="ibc/3">ibc(Module, Line, Function) -&gt; ok</h3> <ul class="type"> <li><code>Module = atom()</code></li> <li><code>Line = int()</code></li> <li><code>Function = {Module,Name}</code></li> <li><code>&nbsp;Name = atom()</code></li> </ul> <div class="description"> <p>Sets the conditional test of the breakpoint at <code>Line</code> in <code>Module</code> to <code>Function</code>.</p> <p>The conditional test is performed by calling <code>Module:Name(Bindings)</code>, where <code>Bindings</code> is the current variable bindings. The function must return <code>true</code> (break) or <code>false</code> (do not break). Use <code>int:get_binding(Var, Bindings)</code> to retrieve the value of a variable <code>Var</code>.</p> </div> </div> <div class="function"> <h3 id="ipb/0">ipb() -&gt; ok</h3> <div class="description"> <p>Makes a printout of all existing breakpoints.</p> </div> </div> <div class="function"> <h3 id="ipb/1">ipb(Module) -&gt; ok</h3> <ul class="type"> <li><code>Module = atom()</code></li> </ul> <div class="description"> <p>Makes a printout of all existing breakpoints in <code>Module</code>.</p> </div> </div> <div class="function"> <h3 id="iv/0">iv() -&gt; atom()</h3> <div class="description"> <p>Returns the current version number of the interpreter.
