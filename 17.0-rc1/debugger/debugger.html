<h1>debugger</h1> <h2 class="modsummary">Erlang Debugger</h2> <div class="description"> <p>Erlang Debugger for debugging and testing of Erlang programs.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="start/0">start()</h3> <h3 id="start/1">start(File)</h3> <h3 id="start/1-1">start(Mode)</h3> <h3 id="start/2">start(Mode, File)</h3> <ul class="type"> <li><code>Mode = local | global</code></li> <li><code>File = string()</code></li> </ul> <div class="description"> <p>Starts Debugger.</p> <p>If given a file name as argument, Debugger will try to load its settings from this file. Refer to Debugger User's Guide for more information about settings.</p> <p>If given <code>local</code> as argument, Debugger will interpret code only at the current node. If given <code>global</code> as argument, Debugger will interpret code at all known nodes, this is the default.</p> </div> </div> <div class="function"> <h3 id="quick/3">quick(Module, Name, Args)</h3> <ul class="type"> <li><code>Module = Name = atom()</code></li> <li><code>Args = [term()]</code></li> </ul> <div class="description"> <p>This function can be used to debug a single process. The module <code>Module</code> is interpreted and <code>apply(Module,Name,Args)</code> is called. This will open an Attach Process window, refer to Debugger User's Guide for more information.</p> </div> </div> </div>
