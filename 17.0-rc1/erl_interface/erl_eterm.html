<h1>erl_eterm</h1> <libsummary>Functions for Erlang Term Construction</libsummary> <div class="description"> <p>This module contains functions for creating and manipulating Erlang terms. </p> <p>An Erlang term is represented by a C structure of type <code>ETERM</code>. Applications should not reference any fields in this structure directly, because it may be changed in future releases to provide faster and more compact term storage. Instead, applications should us the macros and functions provided. </p> <p>The following macros each take a single ETERM pointer as an argument. They return a non-zero value if the test is true, and 0 otherwise:</p> <taglist> <dt><code>ERL_IS_INTEGER(t)</code></dt> <item>True if <code>t</code> is an integer.</item> <dt><code>ERL_IS_UNSIGNED_INTEGER(t)</code></dt> <item>True if <code>t</code> is an integer.</item> <dt><code>ERL_IS_FLOAT(t)</code></dt> <item>True if <code>t</code> is a floating point number.</item> <dt><code>ERL_IS_ATOM(t)</code></dt> <item>True if <code>t</code> is an atom.</item> <dt><code>ERL_IS_PID(t)</code></dt> <item>True if <code>t</code> is a Pid (process identifier).</item> <dt><code>ERL_IS_PORT(t)</code></dt> <item>True if <code>t</code> is a port.</item> <dt><code>ERL_IS_REF(t)</code></dt> <item>True if <code>t</code> is a reference.</item> <dt><code>ERL_IS_TUPLE(t)</code></dt> <item>True if <code>t</code> is a tuple.</item> <dt><code>ERL_IS_BINARY(t)</code></dt> <item>True if <code>t</code> is a binary.</item> <dt><code>ERL_IS_LIST(t)</code></dt> <item>True if <code>t</code> is a list with zero or more elements.</item> <dt><code>ERL_IS_EMPTY_LIST(t)</code></dt> <item>True if <code>t</code> is an empty list.</item> <dt><code>ERL_IS_CONS(t)</code></dt> <item>True if <code>t</code> is a list with at least one element.</item> </taglist> <p>The following macros can be used for retrieving parts of Erlang terms. None of these do any type checking; results are undefined if you pass an ETERM* containing the wrong type. For example, passing a tuple to ERL_ATOM_PTR() will likely result in garbage. </p> <taglist> <dt><code>char *ERL_ATOM_PTR(t)</code></dt> <dt><code>char *ERL_ATOM_PTR_UTF8(t)</code></dt> <item>A string representing atom <code>t</code>. </item> <dt><code>int ERL_ATOM_SIZE(t)</code></dt> <dt><code>int ERL_ATOM_SIZE_UTF8(t)</code></dt> <item>The length (in bytes) of atom t.</item> <dt><code>void *ERL_BIN_PTR(t)</code></dt> <item>A pointer to the contents of <code>t</code></item> <dt><code>int ERL_BIN_SIZE(t)</code></dt> <item>The length (in bytes) of binary object <code>t</code>.</item> <dt><code>int ERL_INT_VALUE(t)</code></dt> <item>The integer of <code>t</code>.</item> <dt><code>unsigned int ERL_INT_UVALUE(t)</code></dt> <item>The unsigned integer value of <code>t</code>.</item> <dt><code>double ERL_FLOAT_VALUE(t)</code></dt> <item>The floating point value of <code>t</code>.</item> <dt><code>ETERM *ERL_PID_NODE(t)</code></dt> <dt><code>ETERM *ERL_PID_NODE_UTF8(t)</code></dt> <item>The Node in pid <code>t</code>.</item> <dt><code>int ERL_PID_NUMBER(t)</code></dt> <item>The sequence number in pid <code>t</code>.</item> <dt><code>int ERL_PID_SERIAL(t)</code></dt> <item>The serial number in pid <code>t</code>.</item> <dt><code>int ERL_PID_CREATION(t)</code></dt> <item>The creation number in pid <code>t</code>.</item> <dt><code>int ERL_PORT_NUMBER(t)</code></dt> <item>The sequence number in port <code>t</code>.</item> <dt><code>int ERL_PORT_CREATION(t)</code></dt> <item>The creation number in port <code>t</code>.</item> <dt><code>ETERM *ERL_PORT_NODE(t)</code></dt> <dt><code>ETERM *ERL_PORT_NODE_UTF8(t)</code></dt> <item>The node in port <code>t</code>.</item> <dt><code>int ERL_REF_NUMBER(t)</code></dt> <item>The first part of the reference number in ref <code>t</code>. Use only for compatibility.</item> <dt><code>int ERL_REF_NUMBERS(t)</code></dt> <item>Pointer to the array of reference numbers in ref <code>t</code>.</item> <dt><code>int ERL_REF_LEN(t)</code></dt> <item>The number of used reference numbers in ref <code>t</code>.</item> <dt><code>int ERL_REF_CREATION(t)</code></dt> <item>The creation number in ref <code>t</code>.</item> <dt><code>int ERL_TUPLE_SIZE(t)</code></dt> <item>The number of elements in tuple <code>t</code>.</item> <dt><code>ETERM *ERL_CONS_HEAD(t)</code></dt> <item>The head element of list <code>t</code>.</item> <dt><code>ETERM *ERL_CONS_TAIL(t)</code></dt> <item>A List representing the tail elements of list <code>t</code>.</item> </taglist> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <pre>ETERM * erl_cons(head, tail)</pre> <ul class="type"> <li><code>ETERM *head;</code></li> <li><code>ETERM *tail;</code></li> </ul> <div class="description"> <p>This function concatenates two Erlang terms, prepending <code>head</code> onto <code>tail</code> and thereby creating a <code>cons</code> cell.
