<h1>registry</h1> <libsummary>Store and backup key-value pairs</libsummary> <div class="description"> <p>This module provides support for storing key-value pairs in a table known as a registry, backing up registries to Mnesia in an atomic manner, and later restoring the contents of a registry from Mnesia.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <pre>ei_reg * ei_reg_open(size)</pre> <ul class="type"> <li><code>int size;</code></li> </ul> <div class="description"> <p>Open (create) a registry. The registry will be initially empty. Use <code>ei_reg_close()</code> to close the registry later. </p> <p><code>size</code> is the approximate number of objects you intend to store in the registry. Since the registry uses a hash table with collision chaining, there is no absolute upper limit on the number of objects that can be stored in it. However for reasons of efficiency, it is a good idea to choose a number that is appropriate for your needs. It is possible to use <code>ei_reg_resize()</code> to change the size later. Note that the number you provide will be increased to the nearest larger prime number. </p> <p>On success, an empty registry will be returned. On failure, NULL will be returned.</p> </div> </div> <div class="function"> <pre>int ei_reg_resize(reg,newsize)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>int newsize;</code></li> </ul> <div class="description"> <p>Change the size of a registry. </p> <p><code>newsize</code> is the new size to make the registry. The number will be increased to the nearest larger prime number. </p> <p>On success, the registry will be resized, all contents rehashed, and the function will return 0. On failure, the registry will be left unchanged and the function will return -1.</p> </div> </div> <div class="function"> <pre>int ei_reg_close(reg)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> </ul> <div class="description"> <p>A registry that has previously been created with <code>ei_reg_open()</code> is closed, and all the objects it contains are freed. </p> <p><code>reg</code> is the registry to close. </p> <p>The function returns 0.</p> </div> </div> <div class="function"> <pre>int ei_reg_setival(reg,key,i)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> <li><code>int i;</code></li> </ul> <div class="description"> <p>Create a key-value pair with the specified <code>key</code> and integer value <code>i</code>. If an object already existed with the same <code>key</code>, the new value replaces the old one. If the previous value was a binary or string, it is freed with <code>free()</code>. </p> <p><code>reg</code> is the registry where the object should be placed. </p> <p><code>key</code> is the name of the object. </p> <p><code>i</code> is the integer value to assign. </p> <p>The function returns 0 on success, or -1 on failure.</p> </div> </div> <div class="function"> <pre>int ei_reg_setfval(reg,key,f)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> <li><code>double f;</code></li> </ul> <div class="description"> <p>Create a key-value pair with the specified <code>key</code> and floating point value <code>f</code>. If an object already existed with the same <code>key</code>, the new value replaces the old one. If the previous value was a binary or string, it is freed with <code>free()</code>. </p> <p><code>reg</code> is the registry where the object should be placed. </p> <p><code>key</code> is the name of the object. </p> <p><code>f</code> is the floating point value to assign. </p> <p>The function returns 0 on success, or -1 on failure.</p> </div> </div> <div class="function"> <pre>int ei_reg_setsval(reg,key,s)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> <li><code>const char *s;</code></li> </ul> <div class="description"> <p>Create a key-value pair with the specified <code>key</code> whose "value" is the specified string <code>s</code>. If an object already existed with the same <code>key</code>, the new value replaces the old one. If the previous value was a binary or string, it is freed with <code>free()</code>. </p> <p><code>reg</code> is the registry where the object should be placed. </p> <p><code>key</code> is the name of the object. </p> <p><code>s</code> is the string to assign. The string itself must have been created through a single call to <code>malloc()</code> or similar function, so that the registry can later delete it if necessary by calling <code>free()</code>. </p> <p>The function returns 0 on success, or -1 on failure.</p> </div> </div> <div class="function"> <pre>int ei_reg_setpval(reg,key,p,size)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> <li><code>const void *p;</code></li> <li><code>int size;</code></li> </ul> <div class="description"> <p>Create a key-value pair with the specified <code>key</code> whose "value" is the binary object pointed to by <code>p</code>. If an object already existed with the same <code>key</code>, the new value replaces the old one. If the previous value was a binary or string, it is freed with <code>free()</code>. </p> <p><code>reg</code> is the registry where the object should be placed. </p> <p><code>key</code> is the name of the object. </p> <p><code>p</code> is a pointer to the binary object. The object itself must have been created through a single call to <code>malloc()</code> or similar function, so that the registry can later delete it if necessary by calling <code>free()</code>. </p> <p><code>size</code> is the length in bytes of the binary object. </p> <p>The function returns 0 on success, or -1 on failure.</p> </div> </div> <div class="function"> <pre>int ei_reg_setval(reg,key,flags,v,...)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> <li><code>int flags;</code></li> <li><code>v (see below)</code></li> </ul> <div class="description"> <p>Create a key-value pair with the specified <code>key</code> whose value is specified by <code>v</code>. If an object already existed with the same <code>key</code>, the new value replaces the old one. If the previous value was a binary or string, it is freed with <code>free()</code>. </p> <p><code>reg</code> is the registry where the object should be placed. </p> <p><code>key</code> is the name of the object. </p> <p><code>flags</code> indicates the type of the object specified by <code>v</code>. Flags must be one of EI_INT, EI_FLT, EI_STR and EI_BIN, indicating whether <code>v</code> is <code>int</code>, <code>double</code>, <code>char*</code> or <code>void*</code>. If <code>flags</code> is EI_BIN, then a fifth argument <code>size</code> is required, indicating the size in bytes of the object pointed to by <code>v</code>. </p> <p>If you wish to store an arbitrary pointer in the registry, specify a <code>size</code> of 0. In this case, the object itself will not be transferred by an <code>ei_reg_dump()</code> operation, just the pointer value. </p> <p>The function returns 0 on success, or -1 on failure.</p> </div> </div> <div class="function"> <pre>int ei_reg_getival(reg,key)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> </ul> <div class="description"> <p>Get the value associated with <code>key</code> in the registry. The value must be an integer. </p> <p><code>reg</code> is the registry where the object will be looked up. </p> <p><code>key</code> is the name of the object to look up. </p> <p>On success, the function returns the value associated with <code>key</code>. If the object was not found or it was not an integer object, -1 is returned. To avoid problems with in-band error reporting (i.e. if you cannot distinguish between -1 and a valid result) use the more general function <code>ei_reg_getval()</code> instead.</p> </div> </div> <div class="function"> <pre>double ei_reg_getfval(reg,key)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> </ul> <div class="description"> <p>Get the value associated with <code>key</code> in the registry. The value must be a floating point type. </p> <p><code>reg</code> is the registry where the object will be looked up. </p> <p><code>key</code> is the name of the object to look up. </p> <p>On success, the function returns the value associated with <code>key</code>. If the object was not found or it was not a floating point object, -1.0 is returned. To avoid problems with in-band error reporting (i.e. if you cannot distinguish between -1.0 and a valid result) use the more general function <code>ei_reg_getval()</code> instead.</p> </div> </div> <div class="function"> <pre>const char * ei_reg_getsval(reg,key)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> </ul> <div class="description"> <p>Get the value associated with <code>key</code> in the registry. The value must be a string. </p> <p><code>reg</code> is the registry where the object will be looked up. </p> <p><code>key</code> is the name of the object to look up. </p> <p>On success, the function returns the value associated with <code>key</code>. If the object was not found or it was not a string, NULL is returned. To avoid problems with in-band error reporting (i.e. if you cannot distinguish between NULL and a valid result) use the more general function <code>ei_reg_getval()</code> instead.</p> </div> </div> <div class="function"> <pre>const void * ei_reg_getpval(reg,key,size)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> <li><code>int size;</code></li> </ul> <div class="description"> <p>Get the value associated with <code>key</code> in the registry. The value must be a binary (pointer) type. </p> <p><code>reg</code> is the registry where the object will be looked up. </p> <p><code>key</code> is the name of the object to look up. </p> <p><code>size</code> will be initialized to contain the length in bytes of the object, if it is found. </p> <p>On success, the function returns the value associated with <code>key</code> and indicates its length in <code>size</code>. If the object was not found or it was not a binary object, NULL is returned. To avoid problems with in-band error reporting (i.e. if you cannot distinguish between NULL and a valid result) use the more general function <code>ei_reg_getval()</code> instead.</p> </div> </div> <div class="function"> <pre>int ei_reg_getval(reg,key,flags,v,...)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> <li><code>int flags;</code></li> <li><code>void *v (see below)</code></li> </ul> <div class="description"> <p>This is a general function for retrieving any kind of object from the registry. </p> <p><code>reg</code> is the registry where the object will be looked up. </p> <p><code>key</code> is the name of the object to look up. </p> <p><code>flags</code> indicates the type of object that you are looking for. If <code>flags</code> is 0, then any kind of object will be returned. If <code>flags</code> is one of EI_INT, EI_FLT, EI_STR or EI_BIN, then only values of that kind will be returned. The buffer pointed to by <code>v</code> must be large enough to hold the return data, i.e. it must be a pointer to one of <code>int</code>, <code>double</code>, <code>char*</code> or <code>void*</code>, respectively. Also, if <code>flags</code> is EI_BIN, then a fifth argument <code>int *size</code> is required, so that the size of the object can be returned. </p> <p>If the function succeeds, <code>v</code> (and <code>size</code> if the object is binary) will be initialized with the value associated with <code>key</code>, and the function will return one of EI_INT, EI_FLT, EI_STR or EI_BIN, indicating the type of object. On failure the function will return -1 and the arguments will not be updated.</p> </div> </div> <div class="function"> <pre>int ei_reg_markdirty(reg,key)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> </ul> <div class="description"> <p>Mark a registry object as dirty. This will ensure that it is included in the next backup to Mnesia. Normally this operation will not be necessary since all of the normal registry 'set' functions do this automatically. However if you have retrieved the value of a string or binary object from the registry and modified the contents, then the change will be invisible to the registry and the object will be assumed to be unmodified. This function allows you to make such modifications and then let the registry know about them. </p> <p><code>reg</code> is the registry containing the object. </p> <p><code>key</code> is the name of the object to mark. </p> <p>The function returns 0 on success, or -1 on failure.</p> </div> </div> <div class="function"> <pre>int ei_reg_delete(reg,key)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> </ul> <div class="description"> <p>Delete an object from the registry. The object is not actually removed from the registry, it is only marked for later removal so that on subsequent backups to Mnesia, the corresponding object can be removed from the Mnesia table as well. If another object is later created with the same key, the object will be reused. </p> <p>The object will be removed from the registry after a call to <code>ei_reg_dump()</code> or <code>ei_reg_purge()</code>. </p> <p><code>reg</code> is the registry containing <code>key</code>. </p> <p><code>key</code> is the object to remove. </p> <p>If the object was found, the function returns 0 indicating success. Otherwise the function returns -1.</p> </div> </div> <div class="function"> <pre>int ei_reg_stat(reg,key,obuf)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>const char *key;</code></li> <li><code>struct ei_reg_stat *obuf;</code></li> </ul> <div class="description"> <p>Return information about an object. </p> <p><code>reg</code> is the registry containing the object. </p> <p><code>key</code> is the name of the object. </p> <p><code>obuf</code> is a pointer to an <code>ei_reg_stat</code> structure, defined below: </p> <pre class="sh_erlang"> struct ei_reg_stat { int attr; int size; }; </pre> <p>In <code>attr</code> the object's attributes are stored as the logical OR of its type (one of EI_INT, EI_FLT, EI_BIN and EI_STR), whether it is marked for deletion (EI_DELET) and whether it has been modified since the last backup to Mnesia (EI_DIRTY). </p> <p>The <code>size</code> field indicates the size in bytes required to store EI_STR (including the terminating 0) and EI_BIN objects, or 0 for EI_INT and EI_FLT. </p> <p>The function returns 0 and initializes <code>obuf</code> on success, or returns -1 on failure.</p> </div> </div> <div class="function"> <pre>int ei_reg_tabstat(reg,obuf)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> <li><code>struct ei_reg_tabstat *obuf;</code></li> </ul> <div class="description"> <p>Return information about a registry. Using information returned by this function, you can see whether the size of the registry is suitable for the amount of data it contains. </p> <p><code>reg</code> is the registry to return information about. </p> <p><code>obuf</code> is a pointer to an <code>ei_reg_tabstat</code> structure, defined below: </p> <pre class="sh_erlang"> struct ei_reg_tabstat { int size; int nelem; int npos; int collisions; }; </pre> <p>The <code>size</code> field indicates the number of hash positions in the registry. This is the number you provided when you created or last resized the registry, rounded up to the nearest prime. </p> <p><code>nelem</code> indicates the number of elements stored in the registry. It includes objects that are deleted but not purged. </p> <p><code>npos</code> indicates the number of unique positions that are occupied in the registry. </p> <p><code>collisions</code> indicates how many elements are sharing positions in the registry. </p> <p>On success, the function returns 0 and <code>obuf</code> is initialized to contain table statistics. On failure, the function returns -1.</p> </div> </div> <div class="function"> <pre>int ei_reg_dump(fd,reg,mntab,flags)</pre> <ul class="type"> <li><code>int fd;</code></li> <li><code>ei_reg *reg;</code></li> <li><code>const char *mntab;</code></li> <li><code>int flags;</code></li> </ul> <div class="description"> <p>Dump the contents of a registry to a Mnesia table in an atomic manner, i.e. either all data will be updated, or none of it will. If any errors are encountered while backing up the data, the entire operation is aborted. </p> <p><code>fd</code> is an open connection to Erlang. Mnesia 3.0 or later must be running on the Erlang node. </p> <p><code>reg</code> is the registry to back up. </p> <p><code>mntab</code> is the name of the Mnesia table where the backed up data should be placed. If the table does not exist, it will be created automatically using configurable defaults. See your Mnesia documentation for information about configuring this behaviour. </p> <p>If <code>flags</code> is 0, the backup will include only those objects which have been created, modified or deleted since the last backup or restore (i.e. an incremental backup). After the backup, any objects that were marked dirty are now clean, and any objects that had been marked for deletion are deleted. </p> <p>Alternatively, setting flags to EI_FORCE will cause a full backup to be done, and EI_NOPURGE will cause the deleted objects to be left in the registry afterwards. These can be bitwise ORed together if both behaviours are desired. If EI_NOPURGE was specified, you can use <code>ei_reg_purge()</code> to explicitly remove the deleted items from the registry later. </p> <p>The function returns 0 on success, or -1 on failure.</p> </div> </div> <div class="function"> <pre>int ei_reg_restore(fd,reg,mntab)</pre> <ul class="type"> <li><code>int fd;</code></li> <li><code>ei_reg *reg;</code></li> <li><code>const char *mntab;</code></li> </ul> <div class="description"> <p>The contents of a Mnesia table are read into the registry. </p> <p><code>fd</code> is an open connection to Erlang. Mnesia 3.0 or later must be running on the Erlang node. </p> <p><code>reg</code> is the registry where the data should be placed. </p> <p><code>mntab</code> is the name of the Mnesia table to read data from. </p> <p>Note that only tables of a certain format can be restored, i.e. those that have been created and backed up to with <code>ei_reg_dump()</code>. If the registry was not empty before the operation, then the contents of the table are added to the contents of the registry. If the table contains objects with the same keys as those already in the registry, the registry objects will be overwritten with the new values. If the registry contains objects that were not in the table, they will be unchanged by this operation. </p> <p>After the restore operation, the entire contents of the registry is marked as unmodified. Note that this includes any objects that were modified before the restore and not overwritten by the restore. </p> <p>The function returns 0 on success, or -1 on failure.</p> </div> </div> <div class="function"> <pre>int ei_reg_purge(reg)</pre> <ul class="type"> <li><code>ei_reg *reg;</code></li> </ul> <div class="description"> <p>Remove all objects marked for deletion. When objects are deleted with <code>ei_reg_delete()</code> they are not actually removed from the registry, only marked for later removal. This is so that on a subsequent backup to Mnesia, the objects can also be removed from the Mnesia table. If you are not backing up to Mnesia then you may wish to remove the objects manually with this function. </p> <p><code>reg</code> is a registry containing objects marked for deletion. </p> <p>The function returns 0 on success, or -1 on failure.</p> </div> </div> </div>
