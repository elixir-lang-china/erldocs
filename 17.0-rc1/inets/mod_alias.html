<h1>mod_alias</h1> <h2 class="modsummary">URL aliasing.</h2> <div class="description"> <p>Erlang Webserver Server internal API for handling of things such as interaction data exported by the mod_alias module.</p> <span id="default_index"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="default_index/2">default_index(ConfigDB, Path) -&gt; NewPath</h3> <ul class="type"> <li><code>ConfigDB = config_db()</code></li> <li><code>Path = NewPath = string()</code></li> </ul> <div class="description"> <span id="default_index"> </span> <p>If <code>Path</code> is a directory, <code>default_index/2</code>, it starts searching for resources or files that are specified in the config directive DirectoryIndex. If an appropriate resource or file is found, it is appended to the end of <code>Path</code> and then returned. <code>Path</code> is returned unaltered, if no appropriate file is found, or if <code>Path</code> is not a directory. <code>config_db()</code> is the server config file in ETS table format as described in <a href="http_server.html" class="seealso">Inets Users Guide.</a>.</p> <span id="path"> </span> </div> </div> <div class="function"> <h3 id="path/3">path(PathData, ConfigDB, RequestURI) -&gt; Path</h3> <ul class="type"> <li><code>PathData = interaction_data()</code></li> <li><code>ConfigDB = config_db()</code></li> <li><code>RequestURI = Path = string()</code></li> </ul> <div class="description"> <span id="path"> </span> <p><code>path/3</code> returns the actual file <code>Path</code> in the <code>RequestURI</code> (See RFC 1945). If the interaction data <code>{real_name,{Path,AfterPath}}</code> has been exported by mod_alias; <code>Path</code> is returned. If no interaction data has been exported, ServerRoot is used to generate a file <code>Path</code>. <code>config_db()</code> and <code>interaction_data()</code> are as defined in <a href="http_server.html" class="seealso">Inets Users Guide</a>.</p> <span id="real_name"> </span> </div> </div> <div class="function"> <h3 id="real_name/3">real_name(ConfigDB, RequestURI, Aliases) -&gt; Ret</h3> <ul class="type"> <li><code>ConfigDB = config_db()</code></li> <li><code>RequestURI = string()</code></li> <li><code>Aliases = [{FakeName,RealName}]</code></li> <li><code>Ret = {ShortPath,Path,AfterPath}</code></li> <li><code>ShortPath = Path = AfterPath = string()</code></li> </ul> <div class="description"> <span id="real_name"> </span> <p><code>real_name/3</code> traverses <code>Aliases</code>, typically extracted from <code>ConfigDB</code>, and matches each <code>FakeName</code> with <code>RequestURI</code>. If a match is found <code>FakeName</code> is replaced with <code>RealName</code> in the match. The resulting path is split into two parts, that is <code>ShortPath</code> and <code>AfterPath</code> as defined in <a href="httpd_util.html#split_path" class="seealso">httpd_util:split_path/1</a>. <code>Path</code> is generated from <code>ShortPath</code>, that is the result from <a href="#default_index" class="seealso">default_index/2</a> with <code>ShortPath</code> as an argument. <code>config_db()</code> is the server config file in ETS table format as described in <a href="http_server.html" class="seealso">Inets User Guide.</a>. </p> <span id="real_script_name"> </span> </div> </div> <div class="function"> <h3 id="real_script_name/3">real_script_name(ConfigDB, RequestURI, ScriptAliases) -&gt; Ret</h3> <ul class="type"> <li><code>ConfigDB = config_db()</code></li> <li><code>RequestURI = string()</code></li> <li><code>ScriptAliases = [{FakeName,RealName}]</code></li> <li><code>Ret = {ShortPath,AfterPath} | not_a_script</code></li> <li><code>ShortPath = AfterPath = string()</code></li> </ul> <div class="description"> <span id="real_script_name"> </span> <p><code>real_script_name/3</code> traverses <code>ScriptAliases</code>, typically extracted from <code>ConfigDB</code>, and matches each <code>FakeName</code> with <code>RequestURI</code>. If a match is found <code>FakeName</code> is replaced with <code>RealName</code> in the match. If the resulting match is not an executable script <code>not_a_script</code> is returned. If it is a script the resulting script path is in two parts, that is <code>ShortPath</code> and <code>AfterPath</code> as defined in <a href="httpd_util.html#split_script_path" class="seealso">httpd_util:split_script_path/1</a>. <code>config_db()</code> is the server config file in ETS table format as described in <a href="http_server.html" class="seealso">Inets Users Guide.</a>.</p> </div> </div> </div>
