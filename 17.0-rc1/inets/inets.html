<h1>inets</h1> <h2 class="modsummary">The inets services API</h2> <div class="description"> <p>This module provides the most basic API to the clients and servers, that are part of the Inets application, such as start and stop. </p> <span id="common_data_types"> </span> </div> <div class="section"> <h4>COMMON DATA TYPES </h4> <p>Type definitions that are used more than once in this module: </p> <p><code> service() = ftpc | tftp | httpc | httpd</code></p> <p><code> property() = atom() </code></p> <span id="functions"> </span> <span id="services"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="services/0">services() -&gt; [{Service, Pid}]</h3> <ul class="type"> <li><code>Service = service()</code></li> <li><code>Pid = pid()</code></li> </ul> <div class="description"> <p>Returns a list of currently running services.</p> <div class="note"><h2>Note!</h2> <p>Services started as <code>stand_alone</code> will not be listed.</p> </div> <span id="services_info"> </span> </div> </div> <div class="function"> <h3 id="services_info/0">services_info() -&gt; [{Service, Pid, Info}]</h3> <ul class="type"> <li><code>Service = service()</code></li> <li><code>Pid = pid()</code></li> <li><code>Info = [{Option, Value}]</code></li> <li><code>Option = property()</code></li> <li><code>Value = term()</code></li> </ul> <div class="description"> <p>Returns a list of currently running services where each service is described by a [{Option, Value}] list. The information given in the list is specific for each service and it is probable that each service will have its own info function that gives you even more details about the service.</p> <span id="service_names"> </span> </div> </div> <div class="function"> <h3 id="service_names/0">service_names() -&gt; [Service] </h3> <ul class="type"> <li><code>Service = service()</code></li> </ul> <div class="description"> <p>Returns a list of available service names.</p> <span id="start"> </span> </div> </div> <div class="function"> <h3 id="start/0">start() -&gt; </h3> <h3 id="start/1">start(Type) -&gt; ok | {error, Reason}</h3> <ul class="type"> <li><code>Type = permanent | transient | temporary</code></li> </ul> <div class="description"> <p>Starts the Inets application. Default type is temporary. See also <a href="../kernel/application.html" class="seealso">application(3)</a>. </p> <span id="stop"> </span> </div> </div> <div class="function"> <h3 id="stop/0">stop() -&gt; ok </h3> <div class="description"> <p>Stops the inets application. See also <a href="../kernel/application.html" class="seealso">application(3)</a>. </p> <span id="start2"> </span> </div> </div> <div class="function"> <h3 id="start/2">start(Service, ServiceConfig) -&gt; {ok, Pid} | {error, Reason}</h3> <h3 id="start/3">start(Service, ServiceConfig, How) -&gt; {ok, Pid} | {error, Reason}</h3> <ul class="type"> <li><code>Service = service()</code></li> <li><code>ServiceConfig = [{Option, Value}]</code></li> <li><code>Option = property()</code></li> <li><code>Value = term()</code></li> <li><code>How = inets | stand_alone - default is inets</code></li> </ul> <div class="description"> <p>Dynamically starts an inets service after the inets application has been started. </p> <div class="note"><h2>Note!</h2> <p>Dynamically started services will not be handled by application takeover and failover behavior when inets is run as a distributed application. Nor will they be automatically restarted when the inets application is restarted, but as long as the inets application is up and running they will be supervised and may be soft code upgraded. Services started as <code>stand_alone</code>, e.i. the service is not started as part of the inets application, will lose all OTP application benefits such as soft upgrade. The "stand_alone-service" will be linked to the process that started it. In most cases some of the supervision functionality will still be in place and in some sense the calling process has now become the top supervisor.</p> </div> <span id="stop2"> </span> </div> </div> <div class="function"> <h3 id="stop/2">stop(Service, Reference) -&gt; ok | {error, Reason} </h3> <ul class="type"> <li><code>Service = service() | stand_alone</code></li> <li><code>Reference = pid() | term() - service specified reference</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Stops a started service of the inets application or takes down a "stand_alone-service" gracefully. When the <code>stand_alone</code> option is used in start, only the pid is a valid argument to stop.</p> <span id="see_also"> </span> </div> </div> </div> <div class="section"> <h4>SEE ALSO</h4> <p><a href="ftp.html" class="seealso">ftp(3)</a>, <a href="httpc.html" class="seealso">httpc(3)</a>, <a href="httpd.html" class="seealso">httpd(3)</a>, <a href="tftp.html" class="seealso">tftp(3)</a></p> </div>
