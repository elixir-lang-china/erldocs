<h1>make</h1> <h2 class="modsummary">A Make Utility for Erlang</h2> <div class="description"> <p>The module <code>make</code> provides a set of functions similar to the UNIX type <code>Make</code> functions.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="all/0">all() -&gt; up_to_date | error</h3> <h3 id="all/1">all(Options) -&gt; up_to_date | error</h3> <ul class="type"> <li><code>Options = [Option]</code></li> <li><code>&nbsp;Option = noexec | load | netload | &lt;compiler option&gt;</code></li> </ul> <div class="description"> <p>This function first looks in the current working directory for a file named <code>Emakefile</code> (see below) specifying the set of modules to compile and the compile options to use. If no such file is found, the set of modules to compile defaults to all modules in the current working directory.</p> <p>Traversing the set of modules, it then recompiles every module for which at least one of the following conditions apply:</p> <list type="bulleted"> <item>there is no object file, or</item> <item>the source file has been modified since it was last compiled, or,</item> <item>an include file has been modified since the source file was last compiled.</item> </list> <p>As a side effect, the function prints the name of each module it tries to compile. If compilation fails for a module, the make procedure stops and <code>error</code> is returned.</p> <p><code>Options</code> is a list of make- and compiler options. The following make options exist:</p> <list type="bulleted"> <item><code>noexec</code> <br /> No execution mode. Just prints the name of each module that needs to be compiled.</item> <item><code>load</code> <br /> Load mode. Loads all recompiled modules.</item> <item><code>netload</code> <br /> Net load mode. Loads all recompiled modules an all known nodes.</item> </list> <p>All items in <code>Options</code> that are not make options are assumed to be compiler options and are passed as-is to <code>compile:file/2</code>. <code>Options</code> defaults to <code>[]</code>.</p> </div> </div> <div class="function"> <h3 id="files/1">files(ModFiles) -&gt; up_to_date | error</h3> <h3 id="files/2">files(ModFiles, Options) -&gt; up_to_date | error</h3> <ul class="type"> <li><code>ModFiles = [Module | File]</code></li> <li><code>&nbsp;Module = atom()</code></li> <li><code>&nbsp;File = string()</code></li> <li><code>Options = [Option]</code></li> <li><code>&nbsp;Option = noexec | load | netload | &lt;compiler option&gt;</code></li> </ul> <div class="description"> <p><code>files/1,2</code> does exactly the same thing as <code>all/0,1</code> but for the specified <code>ModFiles</code>, which is a list of module or file names. The file extension <code>.erl</code> may be omitted.</p> <p>The <code>Emakefile</code> (if it exists) in the current directory is searched for compiler options for each module. If a given module does not exist in <code>Emakefile</code> or if <code>Emakefile</code> does not exist, the module is still compiled.</p> </div> </div> </div> <div class="section"> <h4>Emakefile</h4> <p><code>make:all/0,1</code> and <code>make:files/1,2</code> looks in the current working directory for a file named <code>Emakefile</code>. If it exists, <code>Emakefile</code> should contain elements like this:</p> <pre class="sh_erlang"> Modules. {Modules,Options}. </pre> <p><code>Modules</code> is an atom or a list of atoms. It can be </p> <list type="bulleted"> <item>a module name, e.g. <code>file1</code></item> <item>a module name in another directory, e.g. <code>../foo/file3</code></item> <item>a set of modules specified with a wildcards, e.g. <code>'file*'</code></item> <item>a wildcard indicating all modules in current directory, i.e. <code>'*'</code></item> <item>a list of any of the above, e.g. <code>['file*','../foo/file3','File4']</code></item> </list> <p><code>Options</code> is a list of compiler options. </p> <p><code>Emakefile</code> is read from top to bottom. If a module matches more than one entry, the first match is valid. For example, the following <code>Emakefile</code> means that <code>file1</code> shall be compiled with the options <code>[debug_info,{i,"../foo"}]</code>, while all other files in the current directory shall be compiled with only the <code>debug_info</code> flag.</p> <pre class="sh_erlang"> {'file1',[debug_info,{i,"../foo"}]}. {'*',[debug_info]}. </pre> <p /> </div>
