<h1>cover</h1> <h2 class="modsummary">A Coverage Analysis Tool for Erlang</h2> <div class="description"> <p>The module <code>cover</code> provides a set of functions for coverage analysis of Erlang programs, counting how many times each <em>executable line</em> of code is executed when a program is run. <br /> An executable line contains an Erlang expression such as a matching or a function call. A blank line or a line containing a comment, function head or pattern in a <code>case</code>- or <code>receive</code> statement is not executable.</p> <p>Coverage analysis can be used to verify test cases, making sure all relevant code is covered, and may also be helpful when looking for bottlenecks in the code.</p> <p>Before any analysis can take place, the involved modules must be <em>Cover compiled</em>. This means that some extra information is added to the module before it is compiled into a binary which then is loaded. The source file of the module is not affected and no <code>.beam</code> file is created.</p> <p>Each time a function in a Cover compiled module is called, information about the call is added to an internal database of Cover. The coverage analysis is performed by examining the contents of the Cover database. The output <code>Answer</code> is determined by two parameters, <code>Level</code> and <code>Analysis</code>.</p> <list type="bulleted"> <item> <p><code>Level = module</code></p> <p><code>Answer = {Module,Value}</code>, where <code>Module</code> is the module name.</p> </item> <item> <p><code>Level = function</code></p> <p><code>Answer = [{Function,Value}]</code>, one tuple for each function in the module. A function is specified by its module name <code>M</code>, function name <code>F</code> and arity <code>A</code> as a tuple <code>{M,F,A}</code>.</p> </item> <item> <p><code>Level = clause</code></p> <p><code>Answer = [{Clause,Value}]</code>, one tuple for each clause in the module. A clause is specified by its module name <code>M</code>, function name <code>F</code>, arity <code>A</code> and position in the function definition <code>C</code> as a tuple <code>{M,F,A,C}</code>.</p> </item> <item> <p><code>Level = line</code></p> <p><code>Answer = [{Line,Value}]</code>, one tuple for each executable line in the module. A line is specified by its module name <code>M</code> and line number in the source file <code>N</code> as a tuple <code>{M,N}</code>.</p> </item> <item> <p><code>Analysis = coverage</code></p> <p><code>Value = {Cov,NotCov}</code> where <code>Cov</code> is the number of executable lines in the module, function, clause or line that have been executed at least once and <code>NotCov</code> is the number of executable lines that have not been executed.</p> </item> <item> <p><code>Analysis = calls</code></p> <p><code>Value = Calls</code> which is the number of times the module, function, or clause has been called. In the case of line level analysis, <code>Calls</code> is the number of times the line has been executed.</p> </item> </list> <p><em>Distribution</em></p> <p>Cover can be used in a distributed Erlang system. One of the nodes in the system must then be selected as the <em>main node</em>, and all Cover commands must be executed from this node. The error reason <code>not_main_node</code> is returned if an interface function is called on one of the remote nodes.</p> <p>Use <code>cover:start/1</code> and <code>cover:stop/1</code> to add or remove nodes. The same Cover compiled code will be loaded on each node, and analysis will collect and sum up coverage data results from all nodes.</p> <p>To only collect data from remote nodes without stopping <code>cover</code> on those nodes, use <code>cover:flush/1</code></p> <p>If the connection to a remote node goes down, the main node will mark it as lost. If the node comes back it will be added again. If the remote node was alive during the disconnected periode, cover data from before and during this periode will be included in the analysis.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="start/0">start() -&gt; {ok,Pid} | {error,Reason}</h3> <ul class="type"> <li><code>Pid = pid()</code></li> <li><code>Reason = {already_started,Pid}</code></li> </ul> <div class="description"> <p>Starts the Cover server which owns the Cover internal database.
