<h1>erlang.el</h1> <h2 class="modsummary">Erlang mode for Emacs </h2> <div class="description"> <p>Possibly the most important feature of an editor designed for programmers is the ability to indent a line of code in accordance with the structure of the programming language. The Erlang mode does, of course, provide this feature. The layout used is based on the common use of the language. The mode also provides things as syntax highlighting, electric commands, module name verification, comment support including paragraph filling, skeletons, tags support etc.</p> <p>In the following descriptions the use of the word <em>Point</em> means: "Point can be seen as the position of the cursor. More precisely, the point is the position between two characters while the cursor is drawn over the character following the point".</p> </div> <div class="section"> <h4>Indent</h4> <p>The following command are directly available for indentation.</p> <list type="bulleted"> <item><em><code>TAB</code></em> (<code>erlang-indent-command</code>) - Indents the current line of code. </item> <item><em><code>M-C-\</code></em> (<code>indent-region</code>) - Indents all lines in the region. </item> <item><em><code>M-l</code></em> (<code>indent-for-comment</code>) - Insert a comment character to the right of the code on the line (if any).</item> </list> <p>Lines containing comment are indented differently depending on the number of %-characters used: </p> <list type="bulleted"> <item>Lines with one %-character is indented to the right of the code. The column is specified by the variable <code>comment-column</code>, by default column 48 is used.</item> <item>Lines with two %-characters will be indented to the same depth as code would have been in the same situation. </item> <item>Lines with three of more %-characters are indented to the left margin.</item> <item><em><code>C-c C-q</code></em> (<code>erlang-indent-function</code>) - Indents the current Erlang function. </item> <item><em><code>M-x erlang-indent-clause RET</code></em> <br /> -Indent the current Erlang clause.</item> <item><em><code>M-x erlang-indent-current-buffer RET</code></em> - Indent the entire buffer. </item> </list> </div> <div class="section"> <h4>Edit - Fill Comment </h4> <p>When editing normal text in text mode you can let Emacs reformat the text by the <code>fill-paragraph</code> command. This command will not work for comments since it will treat the comment characters as words.</p> <p>The Erlang editing mode provides a command that knows about the Erlang comment structure and can be used to fill text paragraphs in comments. Ex:</p> <pre class="sh_erlang"> %% This is just a very simple test to show %% how the Erlang fill %% paragraph command works.</pre> <p>Clearly, the text is badly formatted. Instead of formatting this paragraph line by line, let's try <code>erlang-fill-paragraph</code> by pressing <em><code>M-q</code></em>. The result is:</p> <pre class="sh_erlang"> %% This is just a very simple test to show how the Erlang fill %% paragraph command works.</pre> </div> <div class="section"> <h4>Edit - Comment/Uncomment Region </h4> <p><em><code>C-c C-c</code></em> will put comment characters at the beginning of all lines in a marked region. If you want to have two comment characters instead of one you can do <em><code>C-u 2 C-c C-c</code></em></p> <p><em><code>C-c C-u</code></em> will undo a comment-region command. </p> </div> <div class="section"> <h4>Edit - Moving the marker </h4> <list type="bulleted"> <item><em><code>C-a M-a </code></em> (<code>erlang-beginning-of-function</code>) - Move the point to the beginning of the current or preceding Erlang function. With an numeric argument (ex <em><code>C-u 2 C-a M-a</code></em>) the function skips backwards over this many Erlang functions. Should the argument be negative the point is moved to the beginning of a function below the current function. </item> <item><em><code>M-C-a </code></em> (<code>erlang-beginning-of-clause</code>) - As above but move point to the beginning of the current or preceding Erlang clause.</item> <item><em><code>C-a M-e </code></em> (<code>erlang-end-of-function</code>) - Move to the end of the current or following Erlang function. With an numeric argument (ex <em><code>C-u 2 C-a M-e</code></em>) the function skips backwards over this many Erlang functions. Should the argument be negative the point is moved to the end of a function below the current function.</item> <item><em><code>M-C-e </code></em> (<code>erlang-end-of-clause</code>) - As above but move point to the end of the current or following Erlang clause.</item> </list> </div> <div class="section"> <h4>Edit - Marking </h4> <list type="bulleted"> <item><em><code>C-c M-h</code></em> (<code>erlang-mark-function</code>) - Put the region around the current Erlang function. The point is placed in the beginning and the mark at the end of the function.</item> <item><em><code>M-C-h </code></em> (<code>erlang-mark-clause</code>) Put the region around the current Erlang clause. The point is placed in the beginning and the mark at the end of the function. </item> </list> </div> <div class="section"> <h4>Edit - Function Header Commands </h4> <list type="bulleted"> <item><em><code>C-c C-j</code></em> (<code>erlang-generate-new-clause</code>) - Create a new clause in the current Erlang function. The point is placed between the parentheses of the argument list.</item> <item><em><code>C-c C-y</code></em> (<code>erlang-clone-arguments</code>) - Copy the function arguments of the preceding Erlang clause. This command is useful when defining a new clause with almost the same argument as the preceding.</item> </list> </div> <div class="section"> <h4>Edit - Arrows</h4> <list type="bulleted"> <item> <p><em><code>C-c C-a</code></em> (<code>erlang-align-arrows</code>) - aligns arrows after clauses inside a region.</p> <pre class="sh_erlang"> Example: sum(L) -&gt; sum(L, 0). sum([H|T], Sum) -&gt; sum(T, Sum + H); sum([], Sum) -&gt; Sum. becomes: sum(L) -&gt; sum(L, 0). sum([H|T], Sum) -&gt; sum(T, Sum + H); sum([], Sum) -&gt; Sum.</pre> </item> </list> </div> <div class="section"> <h4>Syntax highlighting</h4> <p>The syntax highlighting can be activated from the Erlang menu. There are four different alternatives:</p> <list type="bulleted"> <item>Off: Normal black and white display. </item> <item>Level 1: Function headers, reserved words, comments, strings, quoted atoms, and character constants will be colored. </item> <item>Level 2: The above, attributes, Erlang bif:s, guards, and words in comments enclosed in single quotes will be colored.</item> <item>Level 3: The above, variables, records, and macros will be colored. (This level is also known as the Christmas tree level.) </item> </list> </div> <div class="section"> <h4>Tags</h4> <p>For the tag commands to work it requires that you have generated a tag file. See <a href="erlang_mode_chapter.html#tags" class="seealso">Erlang mode users guide</a></p> <p /> <list type="bulleted"> <item><em><code>M-. </code></em> (<code>find-tag</code>) - Find a function definition. The default value is the function name under the point. </item> <item>Find Tag (<code>erlang-find-tag</code>) - Like the Elisp-function `find-tag'. Capable of retrieving Erlang modules. Tags can be given on the forms `tag', `module:', `module:tag'.</item> <item><em><code>M-+</code></em> (<code>erlang-find-next-tag</code>) - Find the next occurrence of tag.</item> <item><em><code>M-TAB</code></em> (<code>erlang-complete-tag</code>) - Perform completion on the tag entered in a tag search. Completes to the set of names listed in the current tags table.</item> <item>Tags aprops (<code>tags-apropos</code>) - Display list of all tags in tags table REGEXP matches. </item> <item><em><code>C-x t s</code></em> (<code>tags-search</code>) - Search through all files listed in tags table for match for REGEXP. Stops when a match is found.</item> </list> </div> <div class="section"> <h4>Skeletons</h4> <p>A skeleton is a piece of pre-written code that can be inserted into the buffer. Erlang mode comes with a set of predefined skeletons. The skeletons can be accessed either from the Erlang menu of from commands named <code>tempo-template-erlang-*</code>, as the skeletons is defined using the standard Emacs package "tempo". Here follows a brief description of the available skeletons:</p> <list type="bulleted"> <item>Simple skeletons: If, Case, Receive, Receive After, Receive Loop - Basic code constructs. </item> <item>Header elements: Module, Author - These commands insert lines on the form <code>-module(</code>xxx<code>). </code> and <code>-author('my@home').</code>. They can be used directly, but are also used as part of the full headers described below.</item> <item>Full Headers: Small (minimum requirement), Medium (with fields for basic information about the module), and Large Header (medium header with some extra layout structure).</item> <item>Small Server - skeleton for a simple server not using OTP.</item> <item>Application - skeletons for the OTP application behavior</item> <item>Supervisor - skeleton for the OTP supervisor behavior</item> <item>Supervisor Bridge - skeleton for the OTP supervisor bridge behavior </item> <item>gen_server - skeleton for the OTP gen_server behavior</item> <item>gen_event - skeleton for the OTP gen_event behavior</item> <item>gen_fsm - skeleton for the OTP gen_fsm behavior</item> <item>Library module - skeleton for a module that does not implement a process.</item> <item>Corba callback - skeleton for a Corba callback module.</item> <item>Erlang test suite - skeleton for a callback module for the erlang test server.</item> </list> </div> <div class="section"> <h4>Shell</h4> <list type="bulleted"> <item>New shell (<code>erlang-shell</code>) - Starts a new Erlang shell.</item> <item><em><code>C-c C-z,</code></em> (<code>erlang-shell-display </code>) - Displays an Erlang shell, or starts a new one if there is no shell started.</item> </list> </div> <div class="section"> <h4>Compile</h4> <list type="bulleted"> <item><em><code>C-c C-k,</code></em> (<code>erlang-compile</code>) - Compiles the Erlang module in the current buffer. You can also use <em><code>C-u C-c C-k</code></em> to debug compile the module with the debug options <code>debug_info</code> and <code>export_all</code>.</item> <item><em><code>C-c C-l,</code></em> (<code>erlang-compile-display</code>) - Display compilation output.</item> <item><em><code>C-u C-x`</code></em> Start parsing the compiler output from the beginning. This command will place the point on the line where the first error was found.</item> <item><em><code>C-x`</code></em> (<code>erlang-next-error</code>) - Move the point on to the next error. The buffer displaying the compilation errors will be updated so that the current error will be visible.</item> </list> </div> <div class="section"> <h4>Man</h4> <p>On unix you can view the manual pages in emacs. In order to find the manual pages, the variable `erlang-root-dir' should be bound to the name of the directory containing the Erlang installation. The name should not include the final slash. Practically, you should add a line on the following form to your ~/.emacs,</p> <pre class="sh_erlang"> (setq erlang-root-dir "/the/erlang/root/dir/goes/here")</pre> </div> <div class="section"> <h4>Starting IMenu</h4> <list type="bulleted"> <item><em><code>M-x imenu-add-to-menubar RET</code></em> - This command will create the IMenu menu containing all the functions in the current buffer.The command will ask you for a suitable name for the menu. Not supported by Xemacs.</item> </list> </div> <div class="section"> <h4>Version</h4> <list type="bulleted"> <item><em><code>M-x erlang-version RET</code></em> - This command displays the version number of the Erlang editing mode. Remember to always supply the version number when asking questions about the Erlang mode.</item> </list> </div>
