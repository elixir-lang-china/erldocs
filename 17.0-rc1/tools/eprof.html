<h1>eprof</h1> <h2 class="modsummary">A Time Profiling Tool for Erlang</h2> <div class="description"> <p>The module <code>eprof</code> provides a set of functions for time profiling of Erlang programs to find out how the execution time is used. The profiling is done using the Erlang trace BIFs. Tracing of local function calls for a specified set of processes is enabled when profiling is begun, and disabled when profiling is stopped.</p> <p>When using Eprof expect a slowdown in program execution.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="start/0">start() -&gt; {ok,Pid} | {error,Reason}</h3> <ul class="type"> <li><code>Pid = pid()</code></li> <li><code>Reason = {already_started,Pid}</code></li> </ul> <div class="description"> <p>Starts the Eprof server which holds the internal state of the collected data.</p> </div> </div> <div class="function"> <h3 id="start_profiling/1">start_profiling(Rootset) -&gt; profiling | {error, Reason}</h3> <h3 id="start_profiling/2">start_profiling(Rootset,Pattern) -&gt; profiling | {error, Reason}</h3> <h3 id="start_profiling/3">start_profiling(Rootset,Pattern,Options) -&gt; profiling | {error, Reason}</h3> <ul class="type"> <li><code>Rootset = [atom() | pid()]</code></li> <li><code>Pattern = {Module, Function, Arity}</code></li> <li><code>Module = Function = atom()</code></li> <li><code>Arity = integer()</code></li> <li><code>Options = [set_on_spawn]</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Starts profiling for the processes in <code>Rootset</code> (and any new processes spawned from them). Information about activity in any profiled process is stored in the Eprof database.</p> <p><code>Rootset</code> is a list of pids and registered names.</p> <p>The function returns <code>profiling</code> if tracing could be enabled for all processes in <code>Rootset</code>, or <code>error</code> otherwise.</p> <p>A pattern can be selected to narrow the profiling. For instance a specific module can be selected, and only the code executed in that module will be profiled.</p> <p>The <code>set_on_spawn</code> option will active call time tracing for all processes spawned by processes in the rootset. This is the default behaviour.</p> </div> </div> <div class="function"> <h3 id="stop_profiling/0">stop_profiling() -&gt; profiling_stopped | profiling_already_stopped</h3> <div class="description"> <p>Stops profiling started with <code>start_profiling/1</code> or <code>profile/1</code>.</p> </div> </div> <div class="function"> <h3 id="profile/1">profile(Fun) -&gt; profiling | {error, Reason}</h3> <h3 id="profile/2">profile(Fun, Options) -&gt; profiling | {error, Reason}</h3> <h3 id="profile/1-1">profile(Rootset) -&gt; profiling | {error, Reason}</h3> <h3 id="profile/2-1">profile(Rootset,Fun) -&gt; {ok, Value} | {error,Reason}</h3> <h3 id="profile/3">profile(Rootset,Fun,Pattern) -&gt; {ok, Value} | {error, Reason}</h3> <h3 id="profile/4">profile(Rootset,Module,Function,Args) -&gt; {ok, Value} | {error, Reason}</h3> <h3 id="profile/5">profile(Rootset,Module,Function,Args,Pattern) -&gt; {ok, Value} | {error, Reason}</h3> <h3 id="profile/6">profile(Rootset,Module,Function,Args,Pattern,Options) -&gt; {ok, Value} | {error, Reason}</h3> <ul class="type"> <li><code>Rootset = [atom() | pid()]</code></li> <li><code>Fun = fun() -&gt; term() end</code></li> <li><code>Pattern = {Module, Function, Arity}</code></li> <li><code>Module = Function = atom()</code></li> <li><code>Args = [term()]</code></li> <li><code>Arity = integer()</code></li> <li><code>Options = [set_on_spawn]</code></li> <li><code>Value = Reason = term()</code></li> </ul> <div class="description"> <p>This function first spawns a process <code>P</code> which evaluates <code>Fun()</code> or <code>apply(Module,Function,Args)</code>. Then, it starts profiling for <code>P</code> and the processes in <code>Rootset</code> (and any new processes spawned from them). Information about activity in any profiled process is stored in the Eprof database.</p> <p><code>Rootset</code> is a list of pids and registered names.</p> <p>If tracing could be enabled for <code>P</code> and all processes in <code>Rootset</code>, the function returns <code>{ok,Value}</code> when <code>Fun()</code>/<code>apply</code> returns with the value <code>Value</code>, or <code>{error,Reason}</code> if <code>Fun()</code>/<code>apply</code> fails with exit reason <code>Reason</code>. Otherwise it returns <code>{error, Reason}</code> immediately.</p> <p>The <code>set_on_spawn</code> option will active call time tracing for all processes spawned by processes in the rootset. This is the default behaviour.</p> <p>The programmer must ensure that the function given as argument is truly synchronous and that no work continues after the function has returned a value.</p> </div> </div> <div class="function"> <h3 id="analyze/0">analyze() -&gt; ok</h3> <h3 id="analyze/1">analyze(Type) -&gt; ok</h3> <h3 id="analyze/2">analyze(Type,Options) -&gt; ok</h3> <ul class="type"> <li><code>Type = procs | total</code></li> <li><code>Options = [{filter, Filter} | {sort, Sort}</code></li> <li><code>Filter = [{calls, integer()} | {time, float()}]</code></li> <li><code>Sort = time | calls | mfa</code></li> </ul> <div class="description"> <p>Call this function when profiling has been stopped to display the results per process, that is:</p> <list type="bulleted"> <item>how much time has been used by each process, and</item> <item>in which function calls this time has been spent.</item> </list> <p>Call <code>analyze</code> with <code>total</code> option when profiling has been stopped to display the results per function call, that is in which function calls the time has been spent.</p> <p>Time is shown as percentage of total time and as absolute time.</p> </div> </div> <div class="function"> <h3 id="log/1">log(File) -&gt; ok</h3> <ul class="type"> <li><code>File = atom() | string()</code></li> </ul> <div class="description"> <p>This function ensures that the results displayed by <code>analyze/0,1,2</code> are printed both to the file <code>File</code> and the screen.</p> </div> </div> <div class="function"> <h3 id="stop/0">stop() -&gt; stopped</h3> <div class="description"> <p>Stops the Eprof server.</p> </div> </div> </div>
