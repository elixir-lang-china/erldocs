<h1>zlib</h1> <h2 class="modsummary">Zlib Compression interface.</h2> <div class="description"> <p>The zlib module provides an API for the zlib library (http://www.zlib.org). It is used to compress and decompress data. The data format is described by RFCs 1950 to 1952.</p> <p>A typical (compress) usage looks like:</p> <pre class="sh_erlang"> Z = zlib:open(), ok = zlib:deflateInit(Z,default), Compress = fun(end_of_data, _Cont) -&gt; []; (Data, Cont) -&gt; [zlib:deflate(Z, Data)|Cont(Read(),Cont)] end, Compressed = Compress(Read(),Compress), Last = zlib:deflate(Z, [], finish), ok = zlib:deflateEnd(Z), zlib:close(Z), list_to_binary([Compressed|Last])</pre> <p>In all functions errors, <code>{'EXIT',{Reason,Backtrace}}</code>, might be thrown, where <code>Reason</code> describes the error. Typical reasons are:</p> <taglist> <dt><code>badarg</code></dt> <item> <p>Bad argument</p> </item> <dt><code>data_error</code></dt> <item> <p>The data contains errors</p> </item> <dt><code>stream_error</code></dt> <item> <p>Inconsistent stream state</p> </item> <dt><code>einval</code></dt> <item> <p>Bad value or wrong function called</p> </item> <dt><code>{need_dictionary,Adler32}</code></dt> <item> <p>See <code>inflate/2</code></p> </item> </taglist> </div> <datatypes> <datatype> <name name="zstream" /> <div class="description"> <p>A zlib <h3 id="deflateInit/6">deflateInit/6</h3> <div class="description"> <p>Initiates a zlib stream for compression.</p> <p>The <code><anno>Level</anno></code> parameter decides the compression level to be used, 0 (<code>none</code>), gives no compression at all, 1 (<code>best_speed</code>) gives best speed and 9 (<code>best_compression</code>) gives best compression.</p> <p>The <code><anno>Method</anno></code> parameter decides which compression method to use, currently the only supported method is <code>deflated</code>.</p> <p>The <code><anno>WindowBits</anno></code> parameter is the base two logarithm of the window size (the size of the history buffer). It should be in the range 9 through 15. Larger values of this parameter result in better compression at the expense of memory usage. The default value is 15 if <code>deflateInit/2</code>. A negative <code><anno>WindowBits</anno></code> value suppresses the zlib header (and checksum) from the stream. Note that the zlib source mentions this only as a undocumented feature.</p> <p>The <code><anno>MemLevel</anno></code> parameter specifies how much memory should be allocated for the internal compression state. <code><anno>MemLevel</anno></code>=1 uses minimum memory but is slow and reduces compression ratio; <code><anno>MemLevel</anno></code>=9 uses maximum memory for optimal speed. The default value is 8.</p> <p>The <code><anno>Strategy</anno></code> parameter is used to tune the compression algorithm. Use the value <code>default</code> for normal data, <code>filtered</code> for data produced by a filter (or predictor), <code>huffman_only</code> to force Huffman encoding only (no string match), or <code>rle</code> to limit match distances to one (run-length encoding). Filtered data consists mostly of small values with a somewhat random distribution. In this case, the compression algorithm is tuned to compress them better. The effect of <code>filtered</code>is to force more Huffman coding and less string matching; it is somewhat intermediate between <code>default</code> and <code>huffman_only</code>. <code>rle</code> is designed to be almost as fast as <code>huffman_only</code>, but give better compression for PNG image data. The <code><anno>Strategy</anno></code> parameter only affects the compression ratio but not the correctness of the compressed output even if it is not set appropriately.</p> </div> </div> <div class="function"> <h3 id="deflate/2">deflate/2</h3> <div class="description"> <p>Same as <code>deflate(<anno>Z</anno>, <anno>Data</anno>, none)</code>.</p> </div> </div> <div class="function"> <h3 id="deflate/3">deflate/3</h3> <div class="description"> <p><code>deflate/3</code> compresses as much data as possible, and stops when the input buffer becomes empty. It may introduce some output latency (reading input without producing any output) except when forced to flush.</p> <p>If the parameter <code><anno>Flush</anno></code> is set to <code>sync</code>, all pending output is flushed to the output buffer and the output is aligned on a byte boundary, so that the decompressor can get all input data available so far. Flushing may degrade compression for some compression algorithms and so it should be used only when necessary.</p> <p>If <code><anno>Flush</anno></code> is set to <code>full</code>, all output is flushed as with <code>sync</code>, and the compression state is reset so that decompression can restart from this point if previous compressed data has been damaged or if random access is desired. Using <code>full</code> too often can seriously degrade the compression.</p> <p>If the parameter <code><anno>Flush</anno></code> is set to <code>finish</code>, pending input is processed, pending output is flushed and <code>deflate/3</code> returns. Afterwards the only possible operations on the stream are <code>deflateReset/1</code> or <code>deflateEnd/1</code>.</p> <p><code><anno>Flush</anno></code> can be set to <code>finish</code> immediately after <code>deflateInit</code> if all compression is to be done in one step.</p> <pre class="sh_erlang"> zlib:deflateInit(Z), B1 = zlib:deflate(Z,Data), B2 = zlib:deflate(Z,&lt;&lt; &gt;&gt;,finish), zlib:deflateEnd(Z), list_to_binary([B1,B2])</pre> </div> </div> <div class="function"> <h3 id="deflateSetDictionary/2">deflateSetDictionary/2</h3> <div class="description"> <p>Initializes the compression dictionary from the given byte sequence without producing any compressed output. This function must be called immediately after <code>deflateInit/[1|2|6]</code> or <code>deflateReset/1</code>, before any call of <code>deflate/3</code>. The compressor and decompressor must use exactly the same dictionary (see <code>inflateSetDictionary/2</code>). The adler checksum of the dictionary is returned.</p> </div> </div> <div class="function"> <h3 id="deflateReset/1">deflateReset/1</h3> <div class="description"> <p>This function is equivalent to <code>deflateEnd/1</code> followed by <code>deflateInit/[1|2|6]</code>, but does not free and reallocate all the internal compression state. The stream will keep the same compression level and any other attributes.</p> </div> </div> <div class="function"> <h3 id="deflateParams/3">deflateParams/3</h3> <div class="description"> <p>Dynamically update the compression level and compression strategy. The interpretation of <code><anno>Level</anno></code> and <code><anno>Strategy</anno></code> is as in <code>deflateInit/6</code>. This can be used to switch between compression and straight copy of the input data, or to switch to a different kind of input data requiring a different strategy. If the compression level is changed, the input available so far is compressed with the old level (and may be flushed); the new level will take effect only at the next call of <code>deflate/3</code>.</p> <p>Before the call of <code>deflateParams</code>, the stream state must be set as for a call of <code>deflate/3</code>, since the currently available input may have to be compressed and flushed.</p> </div> </div> <div class="function"> <h3 id="deflateEnd/1">deflateEnd/1</h3> <div class="description"> <p>End the deflate session and cleans all data used. Note that this function will throw an <code>data_error</code> exception if the last call to <code>deflate/3</code> was not called with <code>Flush</code> set to <code>finish</code>.</p> </div> </div> <div class="function"> <h3 id="inflateInit/1">inflateInit/1</h3> <div class="description"> <p>Initialize a zlib stream for decompression.</p> </div> </div> <div class="function"> <h3 id="inflateInit/2">inflateInit/2</h3> <div class="description"> <p>Initialize decompression session on zlib stream.</p> <p>The <code><anno>WindowBits</anno></code> parameter is the base two logarithm of the maximum window size (the size of the history buffer). It should be in the range 9 through 15. The default value is 15 if <code>inflateInit/1</code> is used. If a compressed stream with a larger window size is given as input, inflate() will throw the <code>data_error</code> exception. A negative <code><anno>WindowBits</anno></code> value makes zlib ignore the zlib header (and checksum) from the stream. Note that the zlib source mentions this only as a undocumented feature.</p> </div> </div> <div class="function"> <h3 id="inflate/2">inflate/2</h3> <div class="description"> <p><code>inflate/2</code> decompresses as much data as possible. It may some introduce some output latency (reading input without producing any output).</p> <p>If a preset dictionary is needed at this point (see <code>inflateSetDictionary</code> below), <code>inflate/2</code> throws a <code>{need_dictionary,Adler}</code> exception where <code>Adler</code> is the adler32 checksum of the dictionary chosen by the compressor.</p> </div> </div> <div class="function"> <h3 id="inflateSetDictionary/2">inflateSetDictionary/2</h3> <div class="description"> <p>Initializes the decompression dictionary from the given uncompressed byte sequence. This function must be called immediately after a call of <code>inflate/2</code> if this call threw a <code>{need_dictionary,Adler}</code> exception. The dictionary chosen by the compressor can be determined from the Adler value thrown by the call to <code>inflate/2</code>. The compressor and decompressor must use exactly the same dictionary (see <code>deflateSetDictionary/2</code>).</p> <p>Example:</p> <pre class="sh_erlang"> unpack(Z, Compressed, Dict) -&gt; case catch zlib:inflate(Z, Compressed) of {'EXIT',{{need_dictionary,DictID},_}} -&gt; zlib:inflateSetDictionary(Z, Dict), Uncompressed = zlib:inflate(Z, []); Uncompressed -&gt; Uncompressed end.</pre> </div> </div> <div class="function"> <h3 id="inflateReset/1">inflateReset/1</h3> <div class="description"> <p>This function is equivalent to <code>inflateEnd/1</code> followed by <code>inflateInit/1</code>, but does not free and reallocate all the internal decompression state. The stream will keep attributes that may have been set by <code>inflateInit/[1|2]</code>.</p> </div> </div> <div class="function"> <h3 id="inflateEnd/1">inflateEnd/1</h3> <div class="description"> <p>End the inflate session and cleans all data used.
