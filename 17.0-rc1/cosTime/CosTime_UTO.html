The time base is <em>15 october 1582 00:00</em>. Comparing two time objects which use different time base is, by obvious reasons, pointless. Raises DATA_CONVERSION if causes an overflow. This operation is only useful if the target object represents a relative time.</p> </div> </div> <div class="function"> <h3 id="compare_time/3">compare_time(UTO, ComparisonType, OtherUTO) -&gt; Reply</h3> <ul class="type"> <li><code>UTO = OtherUTO = #objref</code></li> <li><code>ComparisonType = 'IntervalC' | 'MidC'</code></li> <li><code>Reply = 'TCEqualTo' | 'TCLessThan' | 'TCGreaterThan' | 'TCIndeterminate'</code></li> </ul> <div class="description"> <p>This operation compares the time associated with the target object and the given UTO object. The different <code>ComparisonType</code> are:</p> <p /> <list type="bulleted"> <item>'MidC' - only compare the time represented by each object. Furthermore, the target object is always used as the first parameter in the comparison, i.e., if the target object's time is larger 'TCGreaterThan' will be returned.</item> <item>'IntervalC' - also takes the inaccuracy into consideration, i.e., if the two objects interval overlaps 'TCIndeterminate' is returned, otherwise the as for 'MidC'.</item> </list> </div> </div> <div class="function"> <h3 id="time_to_interval/2">time_to_interval(UTO, OtherUTO) -&gt; TIO</h3> <ul class="type"> <li><code>UTO = OtherUTO = TIO = #objref</code></li> </ul> <div class="description"> <p>This operation returns a TIO representing the interval between the target object and the given UTO midpoint times. The inaccuracy in the objects are not taken into consideration.</p> </div> </div> <div class="function"> <h3 id="interval/1">interval(UTO) -&gt; TIO</h3> <ul class="type"> <li><code>UTO = TIO = #objref</code></li> </ul> <div class="description"> <p>This operation creates a TIO object representing the error interval around the time value represented by the target object, i.e., <code>TIO.upper_bound = UTO.time+UTO.inaccuracy</code> and <code>TIO.lower_bound = UTO.time-UTO.inaccuracy</code>.</p> </div> </div> </div>
