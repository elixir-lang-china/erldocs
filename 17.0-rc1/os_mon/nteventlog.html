<h1>nteventlog</h1> <h2 class="modsummary">Interface to Windows Event Log</h2> <div class="description"> <p><code>nteventlog</code> provides a generic interface to the Windows event log. It is part of the OS_Mon application, see <a href="os_mon_app.html" class="seealso">os_mon(6)</a>. Available for Windows versions where the event log is available. That is, not for Windows 98 and some other older Windows versions, but for most (all?) newer Windows versions.</p> <p>This module is used as the Windows backend for <code>os_sup</code>, see <a href="os_sup.html" class="seealso">os_sup(3)</a>.</p> <p>To retain backwards compatibility, this module can also be used to start a standalone <code>nteventlog</code> process which is not part of the OS_Mon supervision tree. When starting such a process, the user has to supply an identifier as well as a callback function to handle the messages.</p> <p>The identifier, an arbitrary string, should be reused whenever the same application (or node) wants to start the process. <code>nteventlog</code> is informed about all events that have arrived to the eventlog since the last accepted message for the current identifier. As long as the same identifier is used, the same eventlog record will not be sent to <code>nteventlog</code> more than once (with the exception of when graved system failures arise, in which case the last records written before the failure may be sent to Erlang again after reboot).</p> <p>If the event log is configured to wrap around automatically, records that have arrived to the log and been overwritten when <code>nteventlog</code> was not running are lost. It however detects this state and loses no records that are not overwritten.</p> <p>The callback function works as described in <code>os_sup(3)</code>.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="start/2">start(Identifier, MFA) -&gt; Result</h3> <h3 id="start_link/2">start_link(Identifier, MFA) -&gt; Result</h3> <ul class="type"> <li><code>Identifier = string() | atom()</code></li> <li><code>MFA = {Mod, Func, Args}</code></li> <li><code>&nbsp;Mod = Func = atom()</code></li> <li><code>&nbsp;Args = [term()]</code></li> <li><code>Result = {ok, Pid} | {error, {already_started, Pid}}</code></li> <li><code>Pid = pid()</code></li> </ul> <div class="description"> <p>This function starts the standalone <code>nteventlog</code> process and, if <code>start_link/2</code> is used, links to it.</p> <p><code>Identifier</code> is an identifier as described above.</p> <p><code>MFA</code> is the supplied callback function. When <code>nteventlog</code> receives information about a new event, this function will be called as <code>apply(Mod, Func, [Event|Args])</code> where <code>Event</code> is a tuple</p> </div> </div> <div class="function"> <h3 id="stop/0">stop() -&gt; stopped</h3> <ul class="type"> <li><code>Result = stopped</code></li> </ul> <div class="description"> <p>Stops <code>nteventlog</code>.
