<h1>disksup</h1> <h2 class="modsummary">A Disk Supervisor Process</h2> <div class="description"> <p><code>disksup</code> is a process which supervises the available disk space in the system. It is part of the OS_Mon application, see <a href="os_mon_app.html" class="seealso">os_mon(6)</a>. Available for Unix and Windows.</p> <p>Periodically checks the disks. For each disk or partition which uses more than a certain amount of the available space, the alarm <code>{{disk_almost_full, MountedOn}, []}</code> is set.</p> <taglist> <dt>On Unix</dt> <item> <p>All (locally) mounted disks are checked, including the swap disk if it is present.</p> </item> <dt>On WIN32</dt> <item> <p>All logical drives of type "FIXED_DISK" are checked.</p> </item> </taglist> <p>Alarms are reported to the SASL alarm handler, see <a href="../sasl/alarm_handler.html" class="seealso">alarm_handler(3)</a>. To set an alarm, <code>alarm_handler:set_alarm(Alarm)</code> is called where <code>Alarm</code> is the alarm specified above.</p> <p>The alarms are cleared automatically when the alarm cause is no longer valid.</p> </div> <div class="section"> <span id="config"> </span> <h4>Configuration</h4> <p>The following configuration parameters can be used to change the default values for time interval and threshold:</p> <taglist> <dt><code>disk_space_check_interval = int()&gt;0</code></dt> <item> <p>The time interval, in minutes, for the periodic disk space check. The default is 30 minutes.</p> </item> <dt><code>disk_almost_full_threshold = float()</code></dt> <item> <p>The threshold, as percentage of total disk space, for how much disk can be utilized before the <code>disk_almost_full</code> alarm is set. The default is 0.80 (80%).</p> </item> </taglist> <p>See <a href="../kernel/config.html" class="seealso">config(4)</a> for information about how to change the value of configuration parameters.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="get_disk_data/0">get_disk_data() -&gt; [DiskData]</h3> <ul class="type"> <li><code>DiskData = {Id, KByte, Capacity}</code></li> <li><code>&nbsp;Id = string()</code></li> <li><code>&nbsp;KByte = int()</code></li> <li><code>&nbsp;Capacity = int()</code></li> </ul> <div class="description"> <p>Returns the result of the latest disk check. <code>Id</code> is a string that identifies the disk or partition. <code>KByte</code> is the total size of the disk or partition in kbytes. <code>Capacity</code> is the percentage of disk space used.</p> <p>The function is asynchronous in the sense that it does not invoke a disk check, but returns the latest available value.</p> <p>Returns <code>[{"none",0,0}]</code> if <code>disksup</code> is not available.</p> </div> </div> <div class="function"> <h3 id="get_check_interval/0">get_check_interval() -&gt; MS</h3> <ul class="type"> <li><code>MS = int()</code></li> </ul> <div class="description"> <p>Returns the time interval, in milliseconds, for the periodic disk space check.</p> </div> </div> <div class="function"> <h3 id="set_check_interval/1">set_check_interval(Minutes) -&gt; ok</h3> <ul class="type"> <li><code>Minutes = int()&gt;=1</code></li> </ul> <div class="description"> <p>Changes the time interval, given in minutes, for the periodic disk space check.</p> <p>The change will take effect after the next disk space check and is non-persist. That is, in case of a process restart, this value is forgotten and the default value will be used. See <a href="#config" class="seealso">Configuration</a> above.</p> </div> </div> <div class="function"> <h3 id="get_almost_full_threshold/0">get_almost_full_threshold() -&gt; Percent</h3> <ul class="type"> <li><code>Percent = int()</code></li> </ul> <div class="description"> <p>Returns the threshold, in percent, for disk space utilization.</p> </div> </div> <div class="function"> <h3 id="set_almost_full_threshold/1">set_almost_full_threshold(Float) -&gt; ok</h3> <ul class="type"> <li><code>Float = float(), 0=&lt;Float=&lt;1</code></li> </ul> <div class="description"> <p>Changes the threshold, given as a float, for disk space utilization.</p> <p>The change will take effect during the next disk space check and is non-persist. That is, in case of a process restart, this value is forgotten and the default value will be used. See <a href="#config" class="seealso">Configuration</a> above.</p> </div> </div> </div> <div class="section"> <h4>See Also</h4> <p><a href="../sasl/alarm_handler.html" class="seealso">alarm_handler(3)</a>, <a href="os_mon_app.html" class="seealso">os_mon(3)</a></p> </div>
