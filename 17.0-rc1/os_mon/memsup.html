<h1>memsup</h1> <h2 class="modsummary">A Memory Supervisor Process</h2> <div class="description"> <p><code>memsup</code> is a process which supervises the memory usage for the system and for individual processes. It is part of the OS_Mon application, see <a href="os_mon_app.html" class="seealso">os_mon(6)</a>. Available for Unix, Windows and VxWorks.</p> <p>Periodically performs a memory check:</p> <list type="bulleted"> <item> <p>If more than a certain amount of available system memory is allocated, as reported by the underlying operating system, the alarm <code>{system_memory_high_watermark, []}</code> is set.</p> </item> <item> <p>If any Erlang process <code>Pid</code> in the system has allocated more than a certain amount of total system memory, the alarm <code>{process_memory_high_watermark, Pid}</code> is set.</p> </item> </list> <p>Alarms are reported to the SASL alarm handler, see <a href="../sasl/alarm_handler.html" class="seealso">alarm_handler(3)</a>. To set an alarm, <code>alarm_handler:set_alarm(Alarm)</code> is called where <code>Alarm</code> is either of the alarms specified above.</p> <p>The alarms are cleared automatically when the alarm cause is no longer valid.</p> <p>The function <a href="#get_memory_data/0" class="seealso">get_memory_data()</a> can be used to retrieve the result of the latest periodic memory check.</p> <p>There is also a interface to system dependent memory data, <a href="#get_system_memory_data/0" class="seealso">get_system_memory_data()</a>. The result is highly dependent on the underlying operating system and the interface is targeted primarily for systems without virtual memory (e.g. VxWorks). The output on other systems is however still valid, although sparse.</p> <p>A call to <code>get_system_memory_data/0</code> is more costly than a call to <code>get_memory_data/0</code> as data is collected synchronously when this function is called.</p> <p>The total system memory reported under UNIX is the number of physical pages of memory times the page size, and the available memory is the number of available physical pages times the page size. This is a reasonable measure as swapping should be avoided anyway, but the task of defining total memory and available memory is difficult because of virtual memory and swapping.</p> </div> <div class="section"> <span id="config"> </span> <h4>Configuration</h4> <p>The following configuration parameters can be used to change the default values for time intervals and thresholds:</p> <taglist> <dt><code>memory_check_interval = int()&gt;0</code></dt> <item> <p>The time interval, in minutes, for the periodic memory check. The default is one minute.</p> </item> <dt><code>system_memory_high_watermark = float()</code></dt> <item> <p>The threshold, as percentage of system memory, for how much system memory can be allocated before the corresponding alarm is set. The default is 0.80 (80%).</p> </item> <dt><code>process_memory_high_watermark = float()</code></dt> <item> <p>The threshold, as percentage of system memory, for how much system memory can be allocated by one Erlang process before the corresponding alarm is set. The default is 0.05 (5%).</p> </item> <dt><code>memsup_helper_timeout = int()&gt;0</code></dt> <item> <p>A timeout, in seconds, for how long the <code>memsup</code> process should wait for a result from a memory check. If the timeout expires, a warning message <code>"OS_MON (memsup) timeout"</code> is issued via <code>error_logger</code> and any pending, synchronous client calls will return a dummy value. Normally, this situation should not occur. There have been cases on Linux, however, where the pseudo file from which system data is read is temporarily unavailable when the system is heavily loaded.</p> <p>The default is 30 seconds. </p> </item> <dt><code>memsup_system_only = bool()</code></dt> <item> <p>Specifies whether the <code>memsup</code> process should only check system memory usage (<code>true</code>) or not. The default is <code>false</code>, meaning that information regarding both system memory usage and Erlang process memory usage is collected.</p> <p>It is recommended to set this parameter to <code>false</code> on systems with many concurrent processes, as each process memory check makes a traversal of the entire list of processes.</p> </item> </taglist> <p>See <a href="../kernel/config.html" class="seealso">config(4)</a> for information about how to change the value of configuration parameters.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="get_memory_data/0">get_memory_data() -&gt; {Total,Allocated,Worst}</h3> <ul class="type"> <li><code>Total = Allocated = int()</code></li> <li><code>Worst = {Pid, PidAllocated} | undefined</code></li> <li><code>&nbsp;Pid = pid()</code></li> <li><code>&nbsp;PidAllocated = int()</code></li> </ul> <div class="description"> <p>Returns the result of the latest memory check, where <code>Total</code> is the total memory size and <code>Allocated</code> the allocated memory size, in bytes.</p> <p><code>Worst</code> is the pid and number of allocated bytes of the largest Erlang process on the node. If <code>memsup</code> should not collect process data, that is if the configuration parameter <code>memsup_system_only</code> was set to <code>true</code>, <code>Worst</code> is <code>undefined</code>.</p> <p>The function is normally asynchronous in the sense that it does not invoke a memory check, but returns the latest available value. The one exception if is the function is called before a first memory check is finished, in which case it does not return a value until the memory check is finished.</p> <p>Returns <code>{0,0,{pid(),0}}</code> or <code>{0,0,undefined}</code> if <code>memsup</code> is not available, or if all memory checks so far have timed out.</p> </div> </div> <div class="function"> <h3 id="get_system_memory_data/0">get_system_memory_data() -&gt; MemDataList</h3> <ul class="type"> <li><code>MemDataList = [{Tag, Size}]</code></li> <li><code>&nbsp;Tag = atom()</code></li> <li><code>&nbsp;Size = int()</code></li> </ul> <div class="description"> <p>Invokes a memory check and returns the resulting, system dependent, data as a list of tagged tuples, where <code>Tag</code> can be one of the following:</p> <taglist> <dt><code>total_memory</code></dt> <item>The total amount of memory available to the Erlang emulator, allocated and free. May or may not be equal to the amount of memory configured in the system.</item> <dt><code>free_memory</code></dt> <item>The amount of free memory available to the Erlang emulator for allocation.</item> <dt><code>system_total_memory</code></dt> <item>The amount of memory available to the whole operating system. This may well be equal to <code>total_memory</code> but not necessarily.</item> <dt><code>largest_free</code></dt> <item>The size of the largest contiguous free memory block available to the Erlang emulator.</item> <dt><code>number_of_free</code></dt> <item>The number of free blocks available to the Erlang runtime system. This gives a fair indication of how fragmented the memory is.</item> <dt><code>buffered_memory</code></dt> <item> The amount of memory the system uses for temporary storing raw disk blocks. </item> <dt><code>cached_memory</code></dt> <item> The amount of memory the system uses for cached files read from disk. </item> <dt><code>total_swap</code></dt> <item> The amount of total amount of memory the system has available for disk swap. </item> <dt><code>free_swap</code></dt> <item> The amount of memory the system has available for disk swap. </item> </taglist> <p>All memory sizes are presented as number of <em>bytes</em>.</p> <p>The <code>largest_free</code> and <code>number_of_free</code> tags are currently only returned on a VxWorks system.</p> <p>Returns the empty list [] if <code>memsup</code> is not available, or if the memory check times out.</p> <div class="note"><h2>Note!</h2><p> On linux the memory available to the emulator is <code>cached_memory</code> and <code>buffered_memory</code> in addition to <code>free_memory</code>.</p> </div> </div> </div> <div class="function"> <h3 id="get_os_wordsize/0">get_os_wordsize() -&gt; Wordsize</h3> <ul class="type"> <li><code>Wordsize = 32 | 64 | unsupported_os</code></li> </ul> <div class="description"> <p>Returns the wordsize of the current running operating system. </p> </div> </div> <div class="function"> <h3 id="get_check_interval/0">get_check_interval() -&gt; MS</h3> <ul class="type"> <li><code>MS = int()</code></li> </ul> <div class="description"> <p>Returns the time interval, in milliseconds, for the periodic memory check.</p> </div> </div> <div class="function"> <h3 id="set_check_interval/1">set_check_interval(Minutes) -&gt; ok</h3> <ul class="type"> <li><code>Minutes = int()&gt;0</code></li> </ul> <div class="description"> <p>Changes the time interval, given in minutes, for the periodic memory check.</p> <p>The change will take effect after the next memory check and is non-persistent.
