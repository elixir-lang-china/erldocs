The <code>Type</code> parameter determines which type we want the VFS to represent. 'FTP' maps to the <code>INETS</code> ftp implementation, while {'NATIVE', 'cosFileTransferNATIVE_file'} uses the <code>file</code> module. It is also possible to implement own mappings which are activated by supplying {'NATIVE', MyModule}. The MyModule module must export the same functions and behave in the same way as the INETS ftp module, and an operation named <code>open(Host, Port)</code>, which shall return <code>{ok, Pid}</code> or <code>{error, Reason}</code>.</p> <p>If no <code>Options</code> are supplied the default setting will be used, i.e., tcp and 60 seconds.</p> <p>The <code>Content</code> parameter is currently ignored by must be supplied as an empty list.</p> </div> </div> <div class="function"> <h3 id="ssl_server_certfile/0">ssl_server_certfile() -&gt; string()</h3> <div class="description"> <p>This function returns a path to a file containing a chain of PEM encoded certificates for the cosFileTransfer as target.
