<h1>cerl_trees</h1> <h2 class="modsummary">Basic functions on Core Erlang abstract syntax trees.</h2> <div class="description"> <p>Basic functions on Core Erlang abstract syntax trees.</p> <p>Syntax trees are defined in the module <a href="cerl.html" class="seealso">cerl</a>.</p> </div> <div class="section"><h4>DATA TYPES</h4><span id="types"> </span> <taglist> <dt><code>cerl() = cerl() (see module cerl)</code></dt> <item><span id="type-cerl"> </span> </item> </taglist></div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="depth/1">depth(Tree::cerl()) -&gt; integer()</h3> <div class="description"><span id="depth-1"> </span> <p>Returns the length of the longest path in the tree. A leaf node has depth zero, the tree representing "<code>{foo, bar}</code>" has depth one, etc.</p> </div></div> <div class="function"> <h3 id="size/1">size(Tree::cerl()) -&gt; integer()</h3> <div class="description"><span id="size-1"> </span> <p>Returns the number of nodes in <code>Tree</code>.</p> </div></div> <div class="function"> <h3 id="map/2">map(F::Function, Tree::cerl()) -&gt; cerl()</h3> <ul class="type"> <li><code>Function = (cerl()) -&gt; cerl()</code></li></ul> <div class="description"><span id="map-2"> </span> <p>Maps a function onto the nodes of a tree. This replaces each node in the tree by the result of applying the given function on the original node, bottom-up. </p> <p><em>See also:</em> <a href="#mapfold-3" class="seealso">mapfold/3</a>.</p> </div></div> <div class="function"> <h3 id="fold/2">fold(F::Function, Unit::term(), Tree::cerl()) -&gt; term()</h3> <ul class="type"> <li><code>Function = (cerl(), term()) -&gt; term()</code></li></ul> <div class="description"><span id="fold-3"> </span> <p>Does a fold operation over the nodes of the tree. The result is the value of <code>Function(X1, Function(X2, ... Function(Xn, Unit) ... ))</code>, where <code>X1, ..., Xn</code> are the nodes of <code>Tree</code> in a post-order traversal. </p> <p><em>See also:</em> <a href="#mapfold-3" class="seealso">mapfold/3</a>.</p> </div></div> <div class="function"> <h3 id="mapfold/2">mapfold(F::Function, Initial::term(), Tree::cerl()) -&gt; {cerl(), term()}</h3> <ul class="type"> <li><code>Function = (cerl(), term()) -&gt; {cerl(), term()}</code></li></ul> <div class="description"><span id="mapfold-3"> </span> <p>Does a combined map/fold operation on the nodes of the tree. This is similar to <code>map/2</code>, but also propagates a value from each application of <code>Function</code> to the next, starting with the given value <code>Initial</code>, while doing a post-order traversal of the tree, much like <code>fold/3</code>. </p> <p><em>See also:</em> <a href="#fold-3" class="seealso">fold/3</a>, <a href="#map-2" class="seealso">map/2</a>.</p> </div></div> <div class="function"> <h3 id="variables/1">variables(Tree::cerl()) -&gt; [var_name()]</h3> <ul class="type"> <li><code>integer() | atom() | {atom(), integer()}</code></li></ul> <div class="description"><span id="variables-1"> </span> <p>Returns an ordered-set list of the names of all variables in the syntax tree. (This includes function name variables.) An exception is thrown if <code>Tree</code> does not represent a well-formed Core Erlang syntax tree. </p> <p><em>See also:</em> <a href="#free_variables-1" class="seealso">free_variables/1</a>.</p> </div></div> <div class="function"> <h3 id="free_variables/1">free_variables(Tree::cerl()) -&gt; [var_name()]</h3> <div class="description"><span id="free_variables-1"> </span> <p>Like <code>variables/1</code>, but only includes variables that are free in the tree. </p> <p><em>See also:</em> <a href="#variables-1" class="seealso">variables/1</a>.</p> </div></div> <div class="function"> <h3 id="label/1">label(T::cerl() (see module cerl)) -&gt; {cerl() (see module cerl), integer()}</h3> <div class="description"><span id="label-1"> </span> <p>Equivalent to <a href="#label-2" class="seealso">label(Tree, 0)</a>.</p> </div></div> <div class="function"> <h3 id="label/1-1">label(Tree::cerl(), N::integer()) -&gt; {cerl(), integer()}</h3> <div class="description"><span id="label-2"> </span> <p>Labels each expression in the tree. A term <code>{label, L}</code> is prefixed to the annotation list of each expression node, where L is a unique number for every node, except for variables (and function name variables) which get the same label if they represent the same variable. Constant literal nodes are not labeled.</p> <p>The returned value is a tuple <code>{NewTree, Max}</code>, where <code>NewTree</code> is the labeled tree and <code>Max</code> is 1 plus the largest label value used. All previous annotation terms on the form <code>{label, X}</code> are deleted.</p> <p>The values of L used in the tree is a dense range from <code>N</code> to <code>Max - 1</code>, where <code>N =&lt; Max =&lt; N + size(Tree)</code>. Note that it is possible that no labels are used at all, i.e., <code>N = Max</code>.</p> <p>Note: All instances of free variables will be given distinct labels.</p> <p><em>See also:</em> <a href="#label-1" class="seealso">label/1</a>, <a href="#size-1" class="seealso">size/1</a>.</p> </div></div> <div class="function"> <h3 id="get_label/1">get_label(T::cerl() (see module cerl)) -&gt; top | integer()</h3> <div class="description"><span id="get_label-1"> </span> </div></div></div> <authors> <aname> </aname> <email> </email></authors>
