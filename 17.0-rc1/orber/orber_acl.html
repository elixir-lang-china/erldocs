<h1>orber_acl</h1> <h2 class="modsummary">Orber ACL operations</h2> <div class="description"> <p>This module contains functions intended for analyzing Access Control List (ACL) filters. The filters uses a extended format of Classless Inter Domain Routing (CIDR). For example, <code>"123.123.123.10"</code> limits the connection to that particular host, while <code>"123.123.123.10/17"</code> allows connections to or from any host equal to the 17 most significant bits. Orber also allow the user to specify a certain port or port range, for example, <code>"123.123.123.10/17#4001"</code> and <code>"123.123.123.10/17#4001/5001"</code> respectively. IPv4 or none compressed IPv6 strings are accepted.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="match/2">match(IP, Direction) -&gt; boolean()</h3> <h3 id="match/3">match(IP, Direction, GetInfo) -&gt; Reply</h3> <ul class="type"> <li><code>IP = tuple() | [integer()]</code></li> <li><code>Direction = tcp_in | ssl_in | tcp_out | ssl_out</code></li> <li><code>GetInfo = boolean()</code></li> <li><code>Reply = boolean() | {boolean(), [Interface], PortInfo}</code></li> <li><code>Interface = string()</code></li> <li><code>PortInfo = integer() | {integer(), integer()}</code></li> </ul> <div class="description"> <p>If <code>GetInfo</code> is not supplied or set to false, this operation returns a boolean which tells if the IPv4 or IPv6 address would pass the ACL filter, defined by the <code>iiop_acl</code> configuration parameter, or not. When <code>GetInfo</code> is set to true, a tuple which, besides the boolean that tells if access was granted, also include the defined interfaces and port(s). This operation requires that Orber is running and can be used on a live node to determine if Orber has been properly configured.</p> </div> </div> <div class="function"> <h3 id="verify/3">verify(IP, Filter, Family) -&gt; Reply</h3> <ul class="type"> <li><code>IP = string()</code></li> <li><code>Filter = string()</code></li> <li><code>Family = inet | inet6</code></li> <li><code>Reply = true | {false, From, To} | {error, string()}</code></li> <li><code>From = string()</code></li> <li><code>To = string()</code></li> </ul> <div class="description"> <p>This operation returns true if the IPv4 or IPv6 address would pass the supplied ACL. If that is not the case, a tuple containing the accepted range is returned. This operation should only be used for test purposes.</p> </div> </div> <div class="function"> <h3 id="range/2">range(Filter, Family) -&gt; Reply</h3> <ul class="type"> <li><code>Filter = string()</code></li> <li><code>Family = inet | inet6</code></li> <li><code>Reply = {ok, From, To} | {error, string()}</code></li> <li><code>From = string()</code></li> <li><code>To = string()</code></li> </ul> <div class="description"> <p>Returns the range of accepted IP addresses based on the supplied filter. This operation should only be used for test purposes.</p> </div> </div> </div>
