<h1>corba_object</h1> <h2 class="modsummary">The CORBA Object interface functions</h2> <div class="description"> <p>This module contains the CORBA Object interface functions that can be called for all objects.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="get_interface/1">get_interface(Object) -&gt; InterfaceDef</h3> <ul class="type"> <li><code>Object = #objref</code></li> <li><code>InterfaceDef = term()</code></li> </ul> <div class="description"> <p>This function returns the full interface description for an object.</p> </div> </div> <div class="function"> <h3 id="is_nil/1">is_nil(Object) -&gt; boolean()</h3> <ul class="type"> <li><code>Object = #objref</code></li> </ul> <div class="description"> <p>This function checks if the object reference has a nil object value, which denotes no object. It is the reference that is tested and no object implementation is involved in the test. </p> </div> </div> <div class="function"> <h3 id="is_a/2">is_a(Object, Logical_type_id) -&gt; Return</h3> <h3 id="is_a/3">is_a(Object, Logical_type_id, Contexts) -&gt; Return</h3> <ul class="type"> <li><code>Object = #objref</code></li> <li><code>Logical_type_id = string()</code></li> <li><code>Contexts = [Context]</code></li> <li><code>Context = #'IOP_ServiceContext'{context_id = CtxId, context_data = CtxData}</code></li> <li><code>CtxId = ?ORBER_GENERIC_CTX_ID</code></li> <li><code>CtxData = {interface, Interface} | {userspecific, term()} | {configuration, Options}</code></li> <li><code>Interface = string()</code></li> <li><code>Options = [{Key, Value}]</code></li> <li><code>Key = ssl_client_options</code></li> <li><code>Value = allowed value associated with the given key</code></li> <li><code>Return = boolean() | {'EXCEPTION', E}</code></li> </ul> <div class="description"> <p>The <em>Logical_type_id</em> is a string that is a share type identifier (repository id). The function returns true if the object is an instance of that type or an ancestor of the "most derived" type of that object.</p> <p>The <em>configuration</em> context is used to override the global SSL client side <a href="ch_install.html#config" class="seealso">configuration</a>.</p> <p>Note: Other ORB suppliers may not support this function completely according to the OMG specification. Thus, a <em>is_a</em> call may raise an exception or respond unpredictable if the Object is located on a remote node.</p> </div> </div> <div class="function"> <h3 id="is_remote/1">is_remote(Object) -&gt; boolean()</h3> <ul class="type"> <li><code>Object = #objref</code></li> </ul> <div class="description"> <p>This function returns true if an object reference is remote otherwise false. </p> </div> </div> <div class="function"> <h3 id="non_existent/1">non_existent(Object) -&gt; Return</h3> <h3 id="non_existent/2">non_existent(Object, Contexts) -&gt; Return</h3> <ul class="type"> <li><code>Object = #objref</code></li> <li><code>Contexts = [Context]</code></li> <li><code>Context = #'IOP_ServiceContext'{context_id = CtxId, context_data = CtxData}</code></li> <li><code>CtxId = ?ORBER_GENERIC_CTX_ID</code></li> <li><code>CtxData = {interface, Interface} | {userspecific, term()} | {configuration, Options}</code></li> <li><code>Interface = string()</code></li> <li><code>Options = [{Key, Value}]</code></li> <li><code>Key = ssl_client_options</code></li> <li><code>Value = allowed value associated with the given key</code></li> <li><code>Return = boolean() | {'EXCEPTION', E}</code></li> </ul> <div class="description"> <p>This function can be used to test if the object has been destroyed. It does this without invoking any application level code. The ORB returns true if it knows that the object is destroyed otherwise false.</p> <p>The <em>configuration</em> context is used to override the global SSL client side <a href="ch_install.html#config" class="seealso">configuration</a>.</p> <p>Note: The OMG have specified two different operators, <code>_not_existent</code> (CORBA version 2.0 and 2.2) and <code>_non_existent</code> (CORBA version 2.3), to be used for this function. It is not mandatory to support both versions. Thus, a <em>non_existent</em> call may raise an exception or respond unpredictable if the Object is located on a remote node. Depending on which version, ORB:s you intend to communicate with supports, you can either use this function or <code>not_existent/1</code>.</p> </div> </div> <div class="function"> <h3 id="not_existent/1">not_existent(Object) -&gt; Return</h3> <h3 id="not_existent/2">not_existent(Object, Contexts) -&gt; Return</h3> <ul class="type"> <li><code>Object = #objref</code></li> <li><code>Contexts = [Context]</code></li> <li><code>Context = #'IOP_ServiceContext'{context_id = CtxId, context_data = CtxData}</code></li> <li><code>CtxId = ?ORBER_GENERIC_CTX_ID</code></li> <li><code>CtxData = {interface, Interface} | {userspecific, term()} | {configuration, Options}</code></li> <li><code>Interface = string()</code></li> <li><code>Options = [{Key, Value}]</code></li> <li><code>Key = ssl_client_options</code></li> <li><code>Value = allowed value associated with the given key</code></li> <li><code>Return = boolean() | {'EXCEPTION', E}</code></li> </ul> <div class="description"> <p>This function is implemented due to Interoperable purposes. Behaves as <code>non_existent</code> except the operator <code>_not_existent</code> is used when communicating with other ORB:s.</p> <p>The <em>configuration</em> context is used to override the global SSL client side <a href="ch_install.html#config" class="seealso">configuration</a>.</p> </div> </div> <div class="function"> <h3 id="is_equivalent/2">is_equivalent(Object, OtherObject) -&gt; boolean()</h3> <ul class="type"> <li><code>Object = #objref</code></li> <li><code>OtherObject = #objref</code></li> </ul> <div class="description"> <p>This function is used to determine if two object references are equivalent so far the ORB easily can determine. It returns <em>true</em> if the target object reference is equal to the other object reference and <em>false</em> otherwise.</p> </div> </div> <div class="function"> <h3 id="hash/2">hash(Object, Maximum) -&gt; int()</h3> <ul class="type"> <li><code>Object = #objref</code></li> <li><code>Maximum = int()</code></li> </ul> <div class="description"> <p>This function returns a hash value based on the object reference that not will change during the lifetime of the object.
