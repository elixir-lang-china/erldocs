<h1>CosNaming_NamingContext</h1> <h2 class="modsummary">This interface supports different bind and access functions for names in a context.</h2> <div class="description"> <p>This is the object that defines name scopes, names must be unique within a naming context. Objects may have multiple names and may exist in multiple naming contexts. Name context may be named in other contexts and cycles are permitted.</p> <p>The type <code>NameComponent</code> used below is defined as:</p> <pre class="sh_erlang"> -record('CosNaming_NameComponent', {id, kind=""}). </pre> <p>where <code>id</code> and <code>kind</code> are strings. </p> <p>The type <code>Binding</code> used below is defined as:</p> <pre class="sh_erlang"> -record('CosNaming_Binding', {binding_name, binding_type}). </pre> <p>where <code>binding_name</code> is a Name and <code>binding_type</code> is an enum which has the values <code>nobject</code> and <code>ncontext</code>.</p> <p>Both these records are defined in the file <code>CosNaming.hrl</code> and it is included with:</p> <pre class="sh_erlang"> -include_lib("orber/COSS/CosNaming/CosNaming.hrl"). </pre> <p>There are a number of exceptions that can be returned from functions in this interface.</p> <list type="bulleted"> <item> <p>NotFound is defined as </p> <pre class="sh_erlang"> -record('CosNaming_NamingContext_NotFound', {rest_of_name, why}). </pre> </item> <item> <p>CannotProceed is defined as </p> <pre class="sh_erlang"> -record('CosNaming_NamingContext_CannotProceed', {rest_of_name, cxt}). </pre> </item> <item> <p>InvalidName is defined as </p> <pre class="sh_erlang"> -record('CosNaming_NamingContext_InvalidName', {}). </pre> </item> <item> <p>NotFound is defined as </p> <pre class="sh_erlang"> -record('CosNaming_NamingContext_NotFound', {}). </pre> </item> <item> <p>AlreadyBound is defined as </p> <pre class="sh_erlang"> -record('CosNaming_NamingContext_AlreadyBound', {}). </pre> </item> <item> <p>NotEmpty is defined as </p> <pre class="sh_erlang"> -record('CosNaming_NamingContext_NotEmpty', {). </pre> </item> </list> <p>These exceptions are defined in the file <code>CosNaming_NamingContext.hrl</code> and it is included with:</p> <pre class="sh_erlang"> -include_lib("orber/COSS/CosNaming/CosNaming_NamingContext.hrl"). </pre> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="bind/3">bind(NamingContext, Name, Object) Naming contexts that are bound using <em>bind()</em> do not participate in name resolution.</p> </div> </div> <div class="function"> <h3 id="rebind/3">rebind(NamingContext, Name, Object) -&gt; Return</h3> <ul class="type"> <li><code>NamingContext = #objref</code></li> <li><code>Name = [NameComponent]</code></li> <li><code>Object = #objref</code></li> <li><code>Return = ok</code></li> </ul> <div class="description"> <p>Creates a binding of a name and an object in the naming context even if the name is already bound.
