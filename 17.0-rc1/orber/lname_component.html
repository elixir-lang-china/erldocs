<h1>lname_component</h1> <h2 class="modsummary">Interface that supports the name pseudo-objects.</h2> <div class="description"> <p>This interface is a part of the name library, which is used to hide the representation of names. In Orbers Erlang mapping the pseudo-object names and the real IDL names have the same representation but it is desirable that the clients uses the names library so they will not be dependent of the representation. The lname_component interface supports handling of name components e.g. set and get of the struct members.</p> <p>Note that the lname_component interface in orber does not contain a destroy function because the NameComponents are represented as Erlang records and therefor will be removed by the garbage collector when not in use.</p> <p>The type <code>NameComponent</code> used below is defined as:</p> <pre class="sh_erlang"> -record('CosNaming_NameComponent', {id, kind=""}). </pre> <p><code>id</code> and <code>kind</code> are strings. </p> <p>The record is defined in the file <code>CosNaming.hrl</code> and it is included with:</p> <pre class="sh_erlang"> -include_lib("orber/COSS/CosNaming/CosNaming.hrl"). </pre> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="create/0">create() -&gt;
