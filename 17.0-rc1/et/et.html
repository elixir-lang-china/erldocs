<h1>et</h1> <h2 class="modsummary">Main API of the Event Trace (ET) application</h2> <div class="description"> <p>Interface module for the Event Trace (ET) application</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="trace_me/5">trace_me(DetailLevel, From, To, Label, Contents) -&gt; hopefully_traced</h3> <ul class="type"> <li><code>DetailLevel = integer(X) when X =&lt; 0, X &gt;= 100</code></li> <li><code>From = actor()</code></li> <li><code>To = actor()</code></li> <li><code>Label = atom() | string() | term()</code></li> <li><code>Contents = [{Key, Value}] | term()</code></li> <li><code>actor() = term()</code></li> </ul> <div class="description"> <p>A function that is intended to be traced.</p> <p>This function is intended to be invoked at strategic places in user applications in order to enable simplified tracing. The functions are extremely light weight as they do nothing besides returning an atom. The functions are designed for being traced. The global tracing mechanism in <code>et_collector</code> defaults to set its trace pattern to these functions.</p> <p>The label is intended to provide a brief summary of the event. It is preferred to use an atom but a string would also do.</p> <p>The contents can be any term but in order to simplify post processing of the traced events, a plain list of {Key, Value} tuples is preferred.</p> <p>Some events, such as messages, are directed from some actor to another. Other events (termed actions) may be undirected and only have one actor.</p> </div> </div> <div class="function"> <h3 id="trace_me/4">trace_me(DetailLevel, FromTo, Label, Contents) -&gt; hopefully_traced</h3> <div class="description"> <p>Invokes <code>et:trace_me/5</code> with both <code>From</code> and <code>To</code> set to <code>FromTo</code>.</p> </div> </div> <div class="function"> <h3 id="phone_home/4">phone_home(DetailLevel, FromTo, Label, Contents) -&gt; hopefully_traced</h3> <h3 id="phone_home/5">phone_home(DetailLevel, From, To, Label, Contents) -&gt; hopefully_traced</h3> <div class="description"> <p>These functions sends a signal to the outer space and the caller hopes that someone is listening. In other words, they invoke <code>et:trace_me/4</code> and <code>et:trace_me/5</code> respectively.</p> </div> </div> <div class="function"> <h3 id="report_event/4">report_event(DetailLevel, FromTo, Label, Contents) -&gt; hopefully_traced</h3> <h3 id="report_event/5">report_event(DetailLevel, From, To, Label, Contents) -&gt; hopefully_traced</h3> <div class="description"> <p>Deprecated functions which for the time being are kept for backwards compatibility.
