<h1>et_selector</h1> <h2 class="modsummary">Define event transforms and trace patterns</h2> <div class="description"> <p /> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="make_pattern/1">make_pattern(RawPattern) -&gt; TracePattern</h3> <ul class="type"> <li><code>RawPattern = detail_level()</code></li> <li><code>TracePattern = erlang_trace_pattern_match_spec()</code></li> <li><code>detail_level() = min | max | integer(X) when X &gt;= 0, X =&lt; 100</code></li> </ul> <div class="description"> <p>Makes a trace pattern suitable to feed change_pattern/1</p> <p>Min detail level deactivates tracing of calls to <code>et:trace_me/4,5</code></p> <p>Max detail level activates tracing of all calls to <code>et:trace_me/4,5</code></p> <p>integer(X) detail level activates tracing of all calls to <code>et:trace_me/4,5</code> whose detail level argument is lesser than X.</p> <p>See also <code>erlang:trace_pattern/2</code> for more info about its <code>match_spec()</code></p> </div> </div> <div class="function"> <h3 id="change_pattern/1">change_pattern(Pattern) -&gt; ok</h3> <ul class="type"> <li><code>Pattern = detail_level() | empty_match_spec() | erlang_trace_pattern_match_spec()</code></li> <li><code>detail_level() = min | max | integer(X) when X &gt;= 0, X =&lt; 100</code></li> <li><code>empty_match_spec() = [] </code></li> </ul> <div class="description"> <p>Activates/deactivates tracing by changing the current trace pattern.</p> <p><code>min</code> detail level deactivates tracing of calls to <code>et:trace_me/4,5</code></p> <p><code>max</code> detail level activates tracing of all calls to <code>et:trace_me/4,5</code></p> <p><code>integer(X)</code> detail level activates tracing of all calls to <code>et:trace_me/4,5</code> whose detail level argument is lesser than <code>X</code>.</p> <p>An empty match spec deactivates tracing of calls to <code>et:trace_me/4,5</code></p> <p>Other match specs activates tracing of calls to <code>et:trace_me/4,5</code> accordingly with <code>erlang:trace_pattern/2</code>.</p> </div> </div> <div class="function"> <h3 id="parse_event/2">parse_event(Mod, ValidTraceData) -&gt; false | true | {true, Event}</h3> <ul class="type"> <li><code>Mod = module_name() | undefined &lt;v&gt;module_name() = atom() &lt;v&gt;ValidTraceData = erlang_trace_data() | record(event)</code></li> <li><code>erlang_trace_data() = {trace, Pid, Label, Info} | {trace, Pid, Label, Info, Extra} | {trace_ts, Pid, Label, Info, ReportedTS} | {trace_ts, Pid, Label, Info, Extra, ReportedTS} | {seq_trace, Label, Info} | {seq_trace, Label, Info, ReportedTS} | {drop, NumberOfDroppedItems}</code></li> </ul> <div class="description"> <p>Transforms trace data and makes an event record out of it.</p> <p>See <code>erlang:trace/3</code> for more info about the semantics of the trace data.</p> <p>An event record consists of the following fields:</p> <taglist> <dt><em>detail_level</em></dt> <item><p>Noise has a high level as opposed to essentials.</p></item> <dt><em>trace_ts</em></dt> <item><p>Time when the trace was generated. Same as event_ts if omitted in trace data.</p></item> <dt><em>event_ts</em></dt> <item><p>Time when the event record was created.</p></item> <dt><em>from</em></dt> <item><p>From actor, such as sender of a message.</p></item> <dt><em>to</em></dt> <item><p>To actor, such as receiver of message.</p></item> <dt><em>label</em></dt> <item><p>Label intended to provide a brief event summary.</p></item> <dt><em>contents</em></dt> <item><p>All nitty gritty details of the event.</p></item> </taglist> <p>See <code>et:trace_me/4</code>and <code>et:trace_me/5</code> for details.</p> <p>Returns:</p> <taglist> <dt><em>{true, Event}</em></dt> <item><p>where Event is an #event{} record representing the trace data</p></item> <dt><em>true</em></dt> <item><p>means that the trace data already is an event record and that it is valid as it is. No transformation is needed.</p></item> <dt><em>false</em></dt> <item><p>means that the trace data is uninteresting and should be dropped</p></item> </taglist> </div> </div> </div>
