<h1>etop</h1> <h2 class="modsummary">Erlang Top is a tool for presenting information about erlang processes similar to the information presented by "top" in UNIX.</h2> <div class="description"> <p><code>etop</code> should be started with the provided scripts <code>etop</code>. This will start a hidden erlang node which connects to the node to be measured. The measured node is given with the <code>-node</code> option. If the measured node has a different cookie than the default cookie for the user who invokes the script, the cookie must be explicitly given witht the <code>-setcookie</code> option.</p> <p>Under Windows the batch file <code>etop.bat</code> can be used.</p> <p>The following configuration parameters exist for the <code>etop</code> tool. When executing the <code>etop</code> script, these parameters can be given as command line options, e.g. <code>etop -node testnode@myhost -setcookie MyCookie</code>.</p> <taglist> <dt>node</dt> <item>The measured node. <br /> Value: atom() <br /> Mandatory</item> <dt>setcookie</dt> <item>Cookie to use for the etop node - must be the same as the cookie on the measured node. <br /> Value: atom()</item> <dt>lines</dt> <item>Number of lines (processes) to display. <br /> Value: integer() <br /> Default: 10</item> <dt>interval</dt> <item>The time interval (in seconds) between each update of the display. <br /> Value: integer() <br /> Default: 5</item> <dt>accumulate</dt> <item>If <code>true</code> the execution time and reductions are accumulated. <br /> Value: boolean() <br /> Default: <code>false</code></item> <dt>sort</dt> <item>Identifies what information to sort by. <br /> Value: <code>runtime | reductions | memory | msg_q</code> <br /> Default: <code>runtime</code> (<code>reductions</code> if <code>tracing=off</code>)</item> <dt>tracing</dt> <item><code>etop</code> uses the erlang trace facility, and thus no other tracing is possible on the measured node while <code>etop</code> is running, unless this option is set to <code>off</code>. Also helpful if the <code>etop</code> tracing causes too high load on the measured node. With tracing off, runtime is not measured. <br /> Value: <code>on | off</code> <br /> Default: <code>on</code></item> </taglist> <p>See the <a href="etop_ug.html" class="seealso">user's guide</a> for more information about the <code>etop</code> tool.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="start/0">start() -&gt; ok</h3> <div class="description"> <p>This function starts <code>etop</code>.
