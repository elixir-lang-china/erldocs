<h1>net_adm</h1> <h2 class="modsummary">Various Erlang Net Administration Routines</h2> <div class="description"> <p>This module contains various network utility functions.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="dns_hostname/1">dns_hostname/1</h3> <div class="description"> <p>Returns the official name of <code><anno>Host</anno></code>, or <code>{error, <anno>Host</anno>}</code> if no such name is found. See also <code>inet(3)</code>.</p> </div> </div> <div class="function"> <h3 id="host_file/0">host_file/0</h3> <div class="description"> <p>Reads the <code>.hosts.erlang</code> file, see the section <em>Files</em> below. Returns the hosts in this file as a list, or returns <code>{error, <anno>Reason</anno>}</code> if the file could not be read or the Erlang terms on the file could not be interpreted.</p> </div> </div> <div class="function"> <h3 id="localhost/0">localhost/0</h3> <div class="description"> <p>Returns the name of the local host. If Erlang was started with the <code>-name</code> command line flag, <code><anno>Name</anno></code> is the fully qualified name.</p> </div> </div> <div class="function"> <h3 id="names/0">names/0</h3> <h3 id="names/1">names/1</h3> <div class="description"> <p>Similar to <code>epmd -names</code>, see <code>epmd(1)</code>. <code><anno>Host</anno></code> defaults to the local host. Returns the names and associated port numbers of the Erlang nodes that <code>epmd</code> at the specified host has registered.</p> <p>Returns <code>{error, address}</code> if <code>epmd</code> is not running.</p> <pre class="sh_erlang"> (arne@dunn)1&gt; <code>net_adm:names().</code> {ok,[{"arne",40262}]}</pre> </div> </div> <div class="function"> <h3 id="ping/1">ping/1</h3> <div class="description"> <p>Tries to set up a connection to <code><anno>Node</anno></code>. Returns <code>pang</code> if it fails, or <code>pong</code> if it is successful.</p> </div> </div> <div class="function"> <h3 id="world/0">world/0</h3> <h3 id="world/1">world/1</h3> <type name="verbosity" /> <div class="description"> <p>This function calls <code>names(Host)</code> for all hosts which are specified in the Erlang host file <code>.hosts.erlang</code>, collects the replies and then evaluates <code>ping(Node)</code> on all those nodes. Returns the list of all nodes that were, successfully pinged.</p> <p><code><anno>Arg</anno></code> defaults to <code>silent</code>. If <code><anno>Arg</anno> == verbose</code>, the function writes information about which nodes it is pinging to stdout.</p> <p>This function can be useful when a node is started, and the names of the other nodes in the network are not initially known.</p> <p>Failure: <code>{error, Reason}</code> if <code>host_file()</code> returns <code>{error, Reason}</code>.</p> </div> </div> <div class="function"> <h3 id="world_list/1">world_list/1</h3> <h3 id="world_list/2">world_list/2</h3> <type name="verbosity" /> <div class="description"> <p>As <code>world/0,1</code>, but the hosts are given as argument instead of being read from <code>.hosts.erlang</code>.</p> </div> </div> </div> <div class="section"> <h4>Files</h4> <p>The <code>.hosts.erlang</code> file consists of a number of host names written as Erlang terms. It is looked for in the current work directory, the user's home directory, and <code>$OTP_ROOT</code> (the root directory of Erlang/OTP), in that order.</p> <p>The format of the <code>.hosts.erlang</code> file must be one host name per line. The host names must be within quotes as shown in the following example:</p> <pre class="sh_erlang"> 'super.eua.ericsson.se'. 'renat.eua.ericsson.se'. 'grouse.eua.ericsson.se'. 'gauffin1.eua.ericsson.se'. ^ (new line)</pre> </div>
