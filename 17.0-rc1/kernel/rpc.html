<h1>rpc</h1> <h2 class="modsummary">Remote Procedure Call Services</h2> <div class="description"> <p>This module contains services which are similar to remote procedure calls. It also contains broadcast facilities and parallel evaluators. A remote procedure call is a method to call a function on a remote node and collect the answer. It is used for collecting information on a remote node, or for running a function with some specific side effects on the remote node.</p> </div> <datatypes> <datatype> <name name="key" /> <div class="description"> <p>As returned by <a href="#async_call/4" class="seealso"> <code>async_call/4</code>.</a></p> </div> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="call/4">call/4</h3> <div class="description"> <p>Evaluates <code>apply(<anno>Module</anno>, <anno>Function</anno>, <anno>Args</anno>)</code> on the node <code><anno>Node</anno></code> and returns the corresponding value <code><anno>Res</anno></code>, or <code>{badrpc, <anno>Reason</anno>}</code> if the call fails.</p> </div> </div> <div class="function"> <h3 id="call/5">call/5</h3> <div class="description"> <p>Evaluates <code>apply(<anno>Module</anno>, <anno>Function</anno>, <anno>Args</anno>)</code> on the node <code><anno>Node</anno></code> and returns the corresponding value <code><anno>Res</anno></code>, or <code>{badrpc, <anno>Reason</anno>}</code> if the call fails. <code><anno>Timeout</anno></code> is a timeout value in milliseconds. If the call times out, <code><anno>Reason</anno></code> is <code>timeout</code>.</p> <p>If the reply arrives after the call times out, no message will contaminate the caller's message queue, since this function spawns off a middleman process to act as (a void) destination for such an orphan reply. This feature also makes this function more expensive than <code>call/4</code> at the caller's end.</p> </div> </div> <div class="function"> <h3 id="block_call/4">block_call/4</h3> <div class="description"> <p>Like <code>call/4</code>, but the RPC server at <code><anno>Node</anno></code> does not create a separate process to handle the call. Thus, this function can be used if the intention of the call is to block the RPC server from any other incoming requests until the request has been handled. The function can also be used for efficiency reasons when very small fast functions are evaluated, for example BIFs that are guaranteed not to suspend.</p> </div> </div> <div class="function"> <h3 id="block_call/5">block_call/5</h3> <div class="description"> <p>Like <code>block_call/4</code>, but with a timeout value in the same manner as <code>call/5</code>.</p> </div> </div> <div class="function"> <h3 id="async_call/4">async_call/4</h3> <div class="description"> <p>Implements <em>call streams with promises</em>, a type of RPC which does not suspend the caller until the result is finished. Instead, a key is returned which can be used at a later stage to collect the value. The key can be viewed as a promise to deliver the answer.</p> <p>In this case, the key <code><anno>Key</anno></code> is returned, which can be used in a subsequent call to <code>yield/1</code> or <code>nb_yield/1,2</code> to retrieve the value of evaluating <code>apply(<anno>Module</anno>, <anno>Function</anno>, <anno>Args</anno>)</code> on the node <code><anno>Node</anno></code>.</p> </div> </div> <div class="function"> <h3 id="yield/1">yield/1</h3> <div class="description"> <p>Returns the promised answer from a previous <code>async_call/4</code>. If the answer is available, it is returned immediately. Otherwise, the calling process is suspended until the answer arrives from <code>Node</code>.</p> </div> </div> <div class="function"> <h3 id="nb_yield/1">nb_yield/1</h3> <div class="description"> <p>Equivalent to <code>nb_yield(<anno>Key</anno>, 0)</code>.</p> </div> </div> <div class="function"> <h3 id="nb_yield/2">nb_yield/2</h3> <div class="description"> <p>This is a non-blocking version of <code>yield/1</code>. It returns the tuple <code>{value, <anno>Val</anno>}</code> when the computation has finished, or <code>timeout</code> when <code><anno>Timeout</anno></code> milliseconds has elapsed.</p> </div> </div> <div class="function"> <h3 id="multicall/3">multicall/3</h3> <div class="description"> <p>Equivalent to <code>multicall([node()|nodes()], <anno>Module</anno>, <anno>Function</anno>, <anno>Args</anno>, infinity)</code>.</p> </div> </div> <div class="function"> <name name="multicall" arity="4" clause_i="1" /> <div class="description"> <p>Equivalent to <code>multicall(<anno>Nodes</anno>, <anno>Module</anno>, <anno>Function</anno>, <anno>Args</anno>, infinity)</code>.</p> </div> </div> <div class="function"> <name name="multicall" arity="4" clause_i="2" /> <div class="description"> <p>Equivalent to <code>multicall([node()|nodes()], <anno>Module</anno>, <anno>Function</anno>, <anno>Args</anno>, <anno>Timeout</anno>)</code>.</p> </div> </div> <div class="function"> <h3 id="multicall/5">multicall/5</h3> <div class="description"> <p>In contrast to an RPC, a multicall is an RPC which is sent concurrently from one client to multiple servers. This is useful for collecting some information from a set of nodes, or for calling a function on a set of nodes to achieve some side effects. It is semantically the same as iteratively making a series of RPCs on all the nodes, but the multicall is faster as all the requests are sent at the same time and are collected one by one as they come back.</p> <p>The function evaluates <code>apply(<anno>Module</anno>, <anno>Function</anno>, <anno>Args</anno>)</code> on the specified nodes and collects the answers. It returns <code>{<anno>ResL</anno>, <anno>BadNodes</anno>}</code>, where <code><anno>BadNodes</anno></code> is a list of the nodes that terminated or timed out during computation, and <code><anno>ResL</anno></code> is a list of the return values. <code><anno>Timeout</anno></code> is a time (integer) in milliseconds, or <code>infinity</code>.</p> <p>The following example is useful when new object code is to be loaded on all nodes in the network, and also indicates some side effects RPCs may produce:</p> <pre class="sh_erlang"> %% Find object code for module Mod {Mod, Bin, File} = code:get_object_code(Mod), %% and load it on all nodes including this one {ResL, _} = rpc:multicall(code, load_binary, [Mod, File, Bin]), %% and then maybe check the ResL list.</pre> </div> </div> <div class="function"> <h3 id="cast/4">cast/4</h3> <div class="description"> <p>Evaluates <code>apply(<anno>Module</anno>, <anno>Function</anno>, <anno>Args</anno>)</code> on the node <code><anno>Node</anno></code>. No response is delivered and the calling process is not suspended until the evaluation is complete, as is the case with <code>call/4,5</code>.</p> </div> </div> <div class="function"> <h3 id="eval_everywhere/3">eval_everywhere/3</h3> <div class="description"> <p>Equivalent to <code>eval_everywhere([node()|nodes()], <anno>Module</anno>, <anno>Function</anno>, <anno>Args</anno>)</code>.</p> </div> </div> <div class="function"> <h3 id="eval_everywhere/4">eval_everywhere/4</h3> <div class="description"> <p>Evaluates <code>apply(<anno>Module</anno>, <anno>Function</anno>, <anno>Args</anno>)</code> on the specified nodes. No answers are collected.</p> </div> </div> <div class="function"> <h3 id="abcast/2">abcast/2</h3> <div class="description"> <p>Equivalent to <code>abcast([node()|nodes()], <anno>Name</anno>, <anno>Msg</anno>)</code>.</p> </div> </div> <div class="function"> <h3 id="abcast/3">abcast/3</h3> <div class="description"> <p>Broadcasts the message <code><anno>Msg</anno></code> asynchronously to the registered process <code><anno>Name</anno></code> on the specified nodes.</p> </div> </div> <div class="function"> <h3 id="sbcast/2">sbcast/2</h3> <div class="description"> <p>Equivalent to <code>sbcast([node()|nodes()], <anno>Name</anno>, <anno>Msg</anno>)</code>.</p> </div> </div> <div class="function"> <h3 id="sbcast/3">sbcast/3</h3> <div class="description"> <p>Broadcasts the message <code><anno>Msg</anno></code> synchronously to the registered process <code><anno>Name</anno></code> on the specified nodes.</p> <p>Returns <code>{<anno>GoodNodes</anno>, <anno>BadNodes</anno>}</code>, where <code><anno>GoodNodes</anno></code> is the list of nodes which have <code><anno>Name</anno></code> as a registered process.</p> <p>The function is synchronous in the sense that it is known that all servers have received the message when the call returns. It is not possible to know that the servers have actually processed the message.</p> <p>Any further messages sent to the servers, after this function has returned, will be received by all servers after this message.</p> </div> </div> <div class="function"> <h3 id="server_call/4">server_call/4</h3> <div class="description"> <p>This function can be used when interacting with a server called <code><anno>Name</anno></code> at node <code><anno>Node</anno></code>. It is assumed that the server receives messages in the format <code>{From, <anno>Msg</anno>}</code> and replies using <code>From ! {<anno>ReplyWrapper</anno>, <anno>Node</anno>, <anno>Reply</anno>}</code>. This function makes such a server call and ensures that the entire call is packed into an atomic transaction which either succeeds or fails. It never hangs, unless the server itself hangs.</p> <p>The function returns the answer <code><anno>Reply</anno></code> as produced by the server <code><anno>Name</anno></code>, or <code>{error, <anno>Reason</anno>}</code>.</p> </div> </div> <div class="function"> <h3 id="multi_server_call/2">multi_server_call/2</h3> <div class="description"> <p>Equivalent to <code>multi_server_call([node()|nodes()], <anno>Name</anno>, <anno>Msg</anno>)</code>.</p> </div> </div> <div class="function"> <h3 id="multi_server_call/3">multi_server_call/3</h3> <div class="description"> <p>This function can be used when interacting with servers called <code><anno>Name</anno></code> on the specified nodes. It is assumed that the servers receive messages in the format <code>{From, <anno>Msg</anno>}</code> and reply using <code>From ! {<anno>Name</anno>, Node, <anno>Reply</anno>}</code>, where <code>Node</code> is the name of the node where the server is located. The function returns <code>{<anno>Replies</anno>, <anno>BadNodes</anno>}</code>, where <code><anno>Replies</anno></code> is a list of all <code><anno>Reply</anno></code> values and <code><anno>BadNodes</anno></code> is a list of the nodes which did not exist, or where the server did not exist, or where the server terminated before sending any reply.</p> </div> </div> <div class="function"> <h3 id="safe_multi_server_call/2">safe_multi_server_call/2</h3> <h3 id="safe_multi_server_call/3">safe_multi_server_call/3</h3> <div class="description"> <div class="warning"><h2>Warning!</h2> <p>This function is deprecated. Use <code>multi_server_call/2,3</code> instead.</p> </div> <p>In Erlang/OTP R6B and earlier releases, <code>multi_server_call/2,3</code> could not handle the case where the remote node exists, but there is no server called <code><anno>Name</anno></code>. Instead this function had to be used. In Erlang/OTP R7B and later releases, however, the functions are equivalent, except for this function being slightly slower.</p> </div> </div> <div class="function"> <h3 id="parallel_eval/1">parallel_eval/1</h3> <div class="description"> <p>For every tuple in <code><anno>FuncCalls</anno></code>, evaluates <code>apply(<anno>Module</anno>, <anno>Function</anno>, <anno>Args</anno>)</code> on some node in the network. Returns the list of return values, in the same order as in <code><anno>FuncCalls</anno></code>.</p> </div> </div> <div class="function"> <h3 id="pmap/3">pmap/3</h3> <div class="description"> <p>Evaluates <code>apply(<anno>Module</anno>, <anno>Function</anno>, [<anno>Elem</anno>|<anno>ExtraArgs</anno>])</code>, for every element <code><anno>Elem</anno></code> in <code><anno>List1</anno></code>, in parallel. Returns the list of return values, in the same order as in <code><anno>List1</anno></code>.</p> </div> </div> <div class="function"> <h3 id="pinfo/1">pinfo/1</h3> <div class="description"> <p>Location transparent version of the BIF <a href="../erts/erlang.html#process_info/1" class="seealso"> <code>process_info/1</code></a>.</p> </div> </div> <div class="function"> <h3 id="pinfo/2">pinfo/2</h3> <div class="description"> <p>Location transparent version of the BIF <a href="../erts/erlang.html#process_info/2" class="seealso"> <code>process_info/2</code></a>.</p> </div> </div> </div>
