<h1>wrap_log_reader</h1> <h2 class="modsummary">A function to read internally formatted wrap disk logs</h2> <div class="description"> <p><code>wrap_log_reader</code> is a function to read internally formatted wrap disk logs, refer to disk_log(3). <code>wrap_log_reader</code> does not interfere with disk_log activities; there is however a known bug in this version of the <code>wrap_log_reader</code>, see chapter <code>bugs</code> below. </p> <p>A wrap disk log file consists of several files, called index files. A log file can be opened and closed. It is also possible to open just one index file separately. If an non-existent or a non-internally formatted file is opened, an error message is returned. If the file is corrupt, no attempt to repair it will be done but an error message is returned. </p> <p>If a log is configured to be distributed, there is a possibility that all items are not loggen on all nodes. <code>wrap_log_reader</code> does only read the log on the called node, it is entirely up to the user to be sure that all items are read. </p> </div> <datatypes> <datatype> <name name="continuation" /> <div class="description"><p>Continuation returned by <code>open/1,2</code> or <code>chunk/1,2</code>.</p> </div> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="chunk/1">chunk/1</h3> <h3 id="chunk/2">chunk/2</h3> <type name="chunk_ret" /> <div class="description"> <p>This function makes it possible to efficiently read the terms which have been appended to a log. It minimises disk I/O by reading large 8K chunks from the file. </p> <p>The first time <code>chunk</code> is called an initial continuation returned from the <code>open/1</code>, <code>open/2</code> must be provided. </p> <p>When <code>chunk/3</code> is called, <code><anno>N</anno></code> controls the maximum number of terms that are read from the log in each chunk. Default is <code>infinity</code>, which means that all the terms contained in the 8K chunk are read. If less than <code><anno>N</anno></code> terms are returned, this does not necessarily mean that end of file is reached. </p> <p>The <code>chunk</code> function returns a tuple <code>{<anno>Continuation2</anno>, <anno>Terms</anno>}</code>, where <code><anno>Terms</anno></code> is a list of terms found in the log. <code><anno>Continuation2</anno></code> is yet another continuation which must be passed on into any subsequent calls to <code>chunk</code>. With a series of calls to <code>chunk</code> it is then possible to extract all terms from a log. </p> <p>The <code>chunk</code> function returns a tuple <code>{<anno>Continuation2</anno>, <anno>Terms</anno>, <anno>Badbytes</anno>}</code> if the log is opened in read only mode and the read chunk is corrupt. <code><anno>Badbytes</anno></code> indicates the number of non-Erlang terms found in the chunk. Note also that the log is not repaired. </p> <p><code>chunk</code> returns <code>{<anno>Continuation2</anno>, eof}</code> when the end of the log is reached, and <code>{error, <anno>Reason</anno>}</code> if an error occurs. </p> <p>The returned continuation may or may not be valid in the next call to <code>chunk</code>. This is because the log may wrap and delete the file into which the continuation points. To make sure this does not happen, the log can be blocked during the search. </p> </div> </div> <div class="function"> <h3 id="close/1">close/1</h3> <div class="description"> <p>This function closes a log file properly. </p> </div> </div> <div class="function"> <h3 id="open/1">open/1</h3> <h3 id="open/2">open/2</h3> <type name="open_ret" /> <div class="description"> <p><code><anno>Filename</anno></code> specifies the name of the file which is to be read. </p> <p><code><anno>N</anno></code> specifies the index of the file which is to be read. If <code><anno>N</anno></code> is omitted the whole wrap log file will be read; if it is specified only the specified index file will be read. </p> <p>The <code>open</code> function returns <code>{ok, <anno>Continuation</anno>}</code> if the log/index file was successfully opened. The <code><anno>Continuation</anno></code> is to be used when chunking or closing the file. </p> <p>The function returns <code>{error, <anno>Reason</anno>}</code> for all errors. </p> </div> </div> </div> <div class="section"> <h4>Bugs</h4> <p>This version of the <code>wrap_log_reader</code> does not detect if the <code>disk_log</code> wraps to a new index file between a <code>wrap_log_reader:open</code> and the first <code>wrap_log_reader:chunk</code>. In this case the chuck will actually read the last logged items in the log file, because the opened index file was truncated by the <code>disk_log</code>. </p> </div> <div class="section"> <h4>See Also</h4> <p><a href="disk_log.html" class="seealso">disk_log(3)</a></p> </div>
