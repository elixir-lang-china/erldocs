<h1>application</h1> <h2 class="modsummary">Generic OTP application functions</h2> <div class="description"> <p>In OTP, <em>application</em> denotes a component implementing some specific functionality, that can be started and stopped as a unit, and which can be re-used in other systems as well. This module interfaces the <em>application controller</em>, a process started at every Erlang runtime system, and contains functions for controlling applications (for example starting and stopping applications), and functions to access information about applications (for example configuration parameters).</p> <p>An application is defined by an <em>application specification</em>. The specification is normally located in an <em>application resource file</em> called <code>Application.app</code>, where <code>Application</code> is the name of the application. Refer to <a href="app.html" class="seealso">app(4)</a> for more information about the application specification.</p> <p>This module can also be viewed as a behaviour for an application implemented according to the OTP design principles as a supervision tree. The definition of how to start and stop the tree should be located in an <em>application callback module</em> exporting a pre-defined set of functions.</p> <p>Refer to <a href="../doc/design_principles/des_princ.html" class="seealso">OTP Design Principles</a> for more information about applications and behaviours.</p> </div> <datatypes> <datatype> <name name="start_type" /> </datatype> <datatype> <name name="restart_type" /> </datatype> <datatype> <div class="description"><p>A tuple where the elements are of type <code>T</code>.</p></div> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="get_all_env/0">get_all_env/0</h3> <h3 id="get_all_env/1">get_all_env/1</h3> <div class="description"> <p>Returns the configuration parameters and their values for <code><anno>Application</anno></code>. If the argument is omitted, it defaults to the application of the calling process.</p> <p>If the specified application is not loaded, or if the process executing the call does not belong to any application, the function returns <code>[]</code>.</p> </div> </div> <div class="function"> <h3 id="get_all_key/0">get_all_key/0</h3> <h3 id="get_all_key/1">get_all_key/1</h3> <div class="description"> <p>Returns the application specification keys and their values for <code><anno>Application</anno></code>. If the argument is omitted, it defaults to the application of the calling process.</p> <p>If the specified application is not loaded, the function returns <code>undefined</code>. If the process executing the call does not belong to any application, the function returns <code>[]</code>.</p> </div> </div> <div class="function"> <h3 id="get_application/0">get_application/0</h3> <h3 id="get_application/1">get_application/1</h3> <div class="description"> <p>Returns the name of the application to which the process <code><anno>Pid</anno></code> or the module <code><anno>Module</anno></code> belongs. Providing no argument is the same as calling <code>get_application(self())</code>.</p> <p>If the specified process does not belong to any application, or if the specified process or module does not exist, the function returns <code>undefined</code>.</p> </div> </div> <div class="function"> <h3 id="get_env/1">get_env/1</h3> <h3 id="get_env/2">get_env/2</h3> <div class="description"> <p>Returns the value of the configuration parameter <code><anno>Par</anno></code> for <code><anno>Application</anno></code>. If the application argument is omitted, it defaults to the application of the calling process.</p> <p>If the specified application is not loaded, or the configuration parameter does not exist, or if the process executing the call does not belong to any application, the function returns <code>undefined</code>.</p> </div> </div> <div class="function"> <h3 id="get_env/3">get_env/3</h3> <div class="description"> <p>Works like <a href="#get_env/2" class="seealso">get_env/2</a> but returns <code><anno>Def</anno></code> value when configuration parameter <code><anno>Par</anno></code> does not exist.</p> </div> </div> <div class="function"> <h3 id="get_key/1">get_key/1</h3> <h3 id="get_key/2">get_key/2</h3> <div class="description"> <p>Returns the value of the application specification key <code><anno>Key</anno></code> for <code><anno>Application</anno></code>. If the application argument is omitted, it defaults to the application of the calling process.</p> <p>If the specified application is not loaded, or the specification key does not exist, or if the process executing the call does not belong to any application, the function returns <code>undefined</code>.</p> </div> </div> <div class="function"> <h3 id="load/1">load/1</h3> <h3 id="load/2">load/2</h3> <type name="application_spec" /> <type name="application_opt" /> <div class="description"> <p>Loads the application specification for an application into the application controller. It will also load the application specifications for any included applications. Note that the function does not load the actual Erlang object code.</p> <p>The application can be given by its name <code><anno>Application</anno></code>. In this case the application controller will search the code path for the application resource file <code><anno>Application</anno>.app</code> and load the specification it contains.</p> <p>The application specification can also be given directly as a tuple <code><anno>AppSpec</anno></code>. This tuple should have the format and contents as described in <code>app(4)</code>.</p> <p>If <code><anno>Distributed</anno> == {<anno>Application</anno>,[<anno>Time</anno>,]<anno>Nodes</anno>}</code>, the application will be distributed. The argument overrides the value for the application in the Kernel configuration parameter <code>distributed</code>. <code><anno>Application</anno></code> must be the name of the application (same as in the first argument). If a node crashes and <code><anno>Time</anno></code> has been specified, then the application controller will wait for <code><anno>Time</anno></code> milliseconds before attempting to restart the application on another node. If <code><anno>Time</anno></code> is not specified, it will default to 0 and the application will be restarted immediately.</p> <p><code><anno>Nodes</anno></code> is a list of node names where the application may run, in priority from left to right. Node names can be grouped using tuples to indicate that they have the same priority. Example:</p> <pre class="sh_erlang"> Nodes = [cp1@cave, {cp2@cave, cp3@cave}]</pre> <p>This means that the application should preferably be started at <code>cp1@cave</code>. If <code>cp1@cave</code> is down, the application should be started at either <code>cp2@cave</code> or <code>cp3@cave</code>.</p> <p>If <code>Distributed == default</code>, the value for the application in the Kernel configuration parameter <code>distributed</code> will be used.</p> </div> </div> <div class="function"> <h3 id="loaded_applications/0">loaded_applications/0</h3> <div class="description"> <p>Returns a list with information about the applications which have been loaded using <code>load/1,2</code>, also included applications. <code><anno>Application</anno></code> is the application name. <code><anno>Description</anno></code> and <code><anno>Vsn</anno></code> are the values of its <code>description</code> and <code>vsn</code> application specification keys, respectively.</p> </div> </div> <div class="function"> <h3 id="permit/2">permit/2</h3> <div class="description"> <p>Changes the permission for <code><anno>Application</anno></code> to run at the current node. The application must have been loaded using <code>load/1,2</code> for the function to have effect.</p> <p>If the permission of a loaded, but not started, application is set to <code>false</code>, <code>start</code> will return <code>ok</code> but the application will not be started until the permission is set to <code>true</code>.</p> <p>If the permission of a running application is set to <code>false</code>, the application will be stopped. If the permission later is set to <code>true</code>, it will be restarted.</p> <p>If the application is distributed, setting the permission to <code>false</code> means that the application will be started at, or moved to, another node according to how its distribution is configured (see <code>load/2</code> above).</p> <p>The function does not return until the application is started, stopped or successfully moved to another node. However, in some cases where permission is set to <code>true</code> the function may return <code>ok</code> even though the application itself has not started. This is true when an application cannot start because it has dependencies to other applications which have not yet been started. When they have been started, <code>Application</code> will be started as well.</p> <p>By default, all applications are loaded with permission <code>true</code> on all nodes. The permission is configurable by using the Kernel configuration parameter <code>permissions</code>.</p> </div> </div> <div class="function"> <h3 id="set_env/3">set_env/3</h3> <h3 id="set_env/4">set_env/4</h3> <div class="description"> <p>Sets the value of the configuration parameter <code><anno>Par</anno></code> for <code><anno>Application</anno></code>.</p> <p><code>set_env/3</code> uses the standard <code>gen_server</code> timeout value (5000 ms). A <code><anno>Timeout</anno></code> argument can be provided if another timeout value is useful, for example, in situations where the application controller is heavily loaded.</p> <div class="warning"><h2>Warning!</h2> <p>Use this function only if you know what you are doing, that is, on your own applications. It is very application and configuration parameter dependent when and how often the value is read by the application, and careless use of this function may put the application in a weird, inconsistent, and malfunctioning state. </p> </div> </div> </div> <div class="function"> <h3 id="ensure_started/1">ensure_started/1</h3> <h3 id="ensure_started/2">ensure_started/2</h3> <div class="description"> <p>Equivalent to <a href="#start/2" class="seealso"><code>application:start/1,2</code></a> except it returns <code>ok</code> for already started applications.</p> </div> </div> <div class="function"> <h3 id="ensure_all_started/1">ensure_all_started/1</h3> <h3 id="ensure_all_started/2">ensure_all_started/2</h3> <div class="description"> <p>Equivalent to calling <a href="#start/2" class="seealso"><code>application:start/1,2</code></a> repeatedly on all dependencies that have not yet been started for an application. The function returns <code>{ok, AppNames}</code> for a successful start or for an already started application (which are however omitted from the <code>AppNames</code> list), and reports <code>{error, {AppName,Reason}}</code> for errors, where <code>Reason</code> is any possible reason returned by <a href="#start/2" class="seealso"><code>application:start/1,2</code></a> when starting a specific dependency. In case of an error, the applications that were started by the function are stopped to bring the set of running applications back to its initial state.</p> </div> </div> <div class="function"> <h3 id="start/1">start/1</h3> <h3 id="start/2">start/2</h3> <div class="description"> <p>Starts <code><anno>Application</anno></code>. If it is not loaded, the application controller will first load it using <code>load/1</code>. It will make sure any included applications are loaded, but will not start them. That is assumed to be taken care of in the code for <code><anno>Application</anno></code>.</p> <p>The application controller checks the value of the application specification key <code>applications</code>, to ensure that all applications that should be started before this application are running. If not, <code>{error,{not_started,App}}</code> is returned, where <code>App</code> is the name of the missing application.</p> <p>The application controller then creates an <em>application master</em> for the application. The application master is the group leader of all the processes in the application. The application master starts the application by calling the application callback function <code>Module:start/2</code> as defined by the application specification key <code>mod</code>.</p> <p>The <code><anno>Type</anno></code> argument specifies the type of the application. If omitted, it defaults to <code>temporary</code>.</p> <list type="bulleted"> <item>If a permanent application terminates, all other applications and the entire Erlang node are also terminated.</item> <item>If a transient application terminates with <code>Reason == normal</code>, this is reported but no other applications are terminated. If a transient application terminates abnormally, all other applications and the entire Erlang node are also terminated.</item> <item>If a temporary application terminates, this is reported but no other applications are terminated.</item> </list> <p>Note that it is always possible to stop an application explicitly by calling <code>stop/1</code>. Regardless of the type of the application, no other applications will be affected.</p> <p>Note also that the transient type is of little practical use, since when a supervision tree terminates, the reason is set to <code>shutdown</code>, not <code>normal</code>.</p> </div> </div> <div class="function"> <h3 id="start_type/0">start_type/0</h3> <div class="description"> <p>This function is intended to be called by a process belonging to an application, when the application is being started, to determine the start type which is either <code><anno>StartType</anno></code> or <code>local</code>.</p> <p>See <a href="#start_type" class="seealso"><code>Module:start/2</code></a> for a description of <code><anno>StartType</anno></code>.</p> <p><code>local</code> is returned if only parts of the application is being restarted (by a supervisor), or if the function is called outside a startup.</p> <p>If the process executing the call does not belong to any application, the function returns <code>undefined</code>.</p> </div> </div> <div class="function"> <h3 id="stop/1">stop/1</h3> <div class="description"> <p>Stops <code><anno>Application</anno></code>. The application master calls <code>Module:prep_stop/1</code>, if such a function is defined, and then tells the top supervisor of the application to shutdown (see <code>supervisor(3)</code>). This means that the entire supervision tree, including included applications, is terminated in reversed start order. After the shutdown, the application master calls <code>Module:stop/1</code>. <code>Module</code> is the callback module as defined by the application specification key <code>mod</code>.</p> <p>Last, the application master itself terminates. Note that all processes with the application master as group leader, i.e. processes spawned from a process belonging to the application, thus are terminated as well.</p> <p>When stopped, the application is still loaded.</p> <p>In order to stop a distributed application, <code>stop/1</code> has to be called on all nodes where it can execute (that is, on all nodes where it has been started). The call to <code>stop/1</code> on the node where the application currently executes will stop its execution. The application will not be moved between nodes due to <code>stop/1</code> being called on the node where the application currently executes before <code>stop/1</code> is called on the other nodes.</p> </div> </div> <div class="function"> <h3 id="takeover/2">takeover/2</h3> <div class="description"> <p>Performs a takeover of the distributed application <code><anno>Application</anno></code>, which executes at another node <code>Node</code>. At the current node, the application is restarted by calling <code>Module:start({takeover,Node},StartArgs)</code>. <code>Module</code> and <code>StartArgs</code> are retrieved from the loaded application specification. The application at the other node is not stopped until the startup is completed, i.e. when <code>Module:start/2</code> and any calls to <code>Module:start_phase/3</code> have returned.</p> <p>Thus two instances of the application will run simultaneously during the takeover, which makes it possible to transfer data from the old to the new instance. If this is not acceptable behavior, parts of the old instance may be shut down when the new instance is started. Note that the application may not be stopped entirely however, at least the top supervisor must remain alive.</p> <p>See <code>start/1,2</code> for a description of <code>Type</code>.</p> </div> </div> <div class="function"> <h3 id="unload/1">unload/1</h3> <div class="description"> <p>Unloads the application specification for <code><anno>Application</anno></code> from the application controller. It will also unload the application specifications for any included applications. Note that the function does not purge the actual Erlang object code.</p> </div> </div> <div class="function"> <h3 id="unset_env/2">unset_env/2</h3> <h3 id="unset_env/3">unset_env/3</h3> <div class="description"> <p>Removes the configuration parameter <code><anno>Par</anno></code> and its value for <code><anno>Application</anno></code>.</p> <p><code>unset_env/2</code> uses the standard <code>gen_server</code> timeout value (5000 ms). A <code><anno>Timeout</anno></code> argument can be provided if another timeout value is useful, for example, in situations where the application controller is heavily loaded.</p> <div class="warning"><h2>Warning!</h2> <p>Use this function only if you know what you are doing, that is, on your own applications. It is very application and configuration parameter dependent when and how often the value is read by the application, and careless use of this function may put the application in a weird, inconsistent, and malfunctioning state. </p> </div> </div> </div> <div class="function"> <h3 id="which_applications/0">which_applications/0</h3> <h3 id="which_applications/1">which_applications/1</h3> <div class="description"> <p>Returns a list with information about the applications which are currently running. <code><anno>Application</anno></code> is the application name. <code><anno>Description</anno></code> and <code><anno>Vsn</anno></code> are the values of its <code>description</code> and <code>vsn</code> application specification keys, respectively.</p> <p><code>which_applications/0</code> uses the standard <code>gen_server</code> timeout value (5000 ms). A <code><anno>Timeout</anno></code> argument can be provided if another timeout value is useful, for example, in situations where the application controller is heavily loaded.</p> </div> </div> </div> <div class="section"> <h4>CALLBACK MODULE</h4> <p>The following functions should be exported from an <code>application</code> callback module.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="start/2">Module:start(StartType, StartArgs) -&gt; {ok, Pid} | {ok, Pid, State} | {error, Reason}</h3> <ul class="type"> <li><code>StartType = normal | {takeover,Node} | {failover,Node}</code></li> <li><code>&nbsp;Node = node()</code></li> <li><code>StartArgs = term()</code></li> <li><code>Pid = pid()</code></li> <li><code>State = term()</code></li> </ul> <div class="description"> <p>This function is called whenever an application is started using <code>application:start/1,2</code>, and should start the processes of the application. If the application is structured according to the OTP design principles as a supervision tree, this means starting the top supervisor of the tree.</p> <p><span id="start_type"> </span><code>StartType</code> defines the type of start:</p> <list type="bulleted"> <item><code>normal</code> if it's a normal startup.</item> <item><code>normal</code> also if the application is distributed and started at the current node due to a failover from another node, and the application specification key <code>start_phases == undefined</code>.</item> <item><code>{takeover,Node}</code> if the application is distributed and started at the current node due to a takeover from <code>Node</code>, either because <code>application:takeover/2</code> has been called or because the current node has higher priority than <code>Node</code>.</item> <item><code>{failover,Node}</code> if the application is distributed and started at the current node due to a failover from <code>Node</code>, and the application specification key <code>start_phases /= undefined</code>.</item> </list> <p><code>StartArgs</code> is the <code>StartArgs</code> argument defined by the application specification key <code>mod</code>.</p> <p>The function should return <code>{ok,Pid}</code> or <code>{ok,Pid,State}</code> where <code>Pid</code> is the pid of the top supervisor and <code>State</code> is any term. If omitted, <code>State</code> defaults to <code>[]</code>. If later the application is stopped, <code>State</code> is passed to <code>Module:prep_stop/1</code>.</p> </div> </div> <div class="function"> <h3 id="start_phase/3">Module:start_phase(Phase, StartType, PhaseArgs) -&gt; ok | {error, Reason}</h3> <ul class="type"> <li><code>Phase = atom()</code></li> <li><code>StartType = <a href="#type-start_type" class="seealso">start_type()</a></code></li> <li><code>PhaseArgs = term()</code></li> <li><code>Pid = pid()</code></li> <li><code>State = state()</code></li> </ul> <div class="description"> <p>This function is used to start an application with included applications, when there is a need for synchronization between processes in the different applications during startup.</p> <p>The start phases is defined by the application specification key <code>start_phases == [{Phase,PhaseArgs}]</code>. For included applications, the set of phases must be a subset of the set of phases defined for the including application.</p> <p>The function is called for each start phase (as defined for the primary application) for the primary application and all included applications, for which the start phase is defined.</p> <p>See <code>Module:start/2</code> for a description of <code>StartType</code>.</p> </div> </div> <div class="function"> <h3 id="prep_stop/1">Module:prep_stop(State) -&gt; NewState</h3> <ul class="type"> <li><code>State = NewState = term()</code></li> </ul> <div class="description"> <p>This function is called when an application is about to be stopped, before shutting down the processes of the application.</p> <p><code>State</code> is the state returned from <code>Module:start/2</code>, or <code>[]</code> if no state was returned. <code>NewState</code> is any term and will be passed to <code>Module:stop/1</code>.</p> <p>The function is optional. If it is not defined, the processes will be terminated and then <code>Module:stop(State)</code> is called.</p> </div> </div> <div class="function"> <h3 id="stop/1">Module:stop(State)</h3> <ul class="type"> <li><code>State = term()</code></li> </ul> <div class="description"> <p>This function is called whenever an application has stopped. It is intended to be the opposite of <code>Module:start/2</code> and should do any necessary cleaning up. The return value is ignored.</p> <p><code>State</code> is the return value of <code>Module:prep_stop/1</code>, if such a function exists. Otherwise <code>State</code> is taken from the return value of <code>Module:start/2</code>.</p> </div> </div> <div class="function"> <h3 id="config_change/3">Module:config_change(Changed, New, Removed) -&gt; ok</h3> <ul class="type"> <li><code>Changed = [{Par,Val}]</code></li> <li><code>New = [{Par,Val}]</code></li> <li><code>Removed = [Par]</code></li> <li><code>&nbsp;Par = atom()</code></li> <li><code>&nbsp;Val = term()</code></li> </ul> <div class="description"> <p>This function is called by an application after a code replacement, if there are any changes to the configuration parameters.</p> <p><code>Changed</code> is a list of parameter-value tuples with all configuration parameters with changed values, <code>New</code> is a list of parameter-value tuples with all configuration parameters that have been added, and <code>Removed</code> is a list of all parameters that have been removed.</p> </div> </div> </div> <div class="section"> <h4>SEE ALSO</h4> <p><a href="../doc/design_principles/des_princ.html" class="seealso">OTP Design Principles</a>, <a href="kernel_app.html" class="seealso">kernel(6)</a>, <a href="app.html" class="seealso">app(4)</a></p> </div>
