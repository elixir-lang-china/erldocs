<h1>erl_boot_server</h1> <h2 class="modsummary">Boot Server for Other Erlang Machines</h2> <div class="description"> <p>This server is used to assist diskless Erlang nodes which fetch all Erlang code from another machine.</p> <p>This server is used to fetch all code, including the start script, if an Erlang runtime system is started with the <code>-loader inet</code> command line flag. All hosts specified with the <code>-hosts Host</code> command line flag must have one instance of this server running.</p> <p>This server can be started with the <code>kernel</code> configuration parameter <code>start_boot_server</code>.</p> <p>The <code>erl_boot_server</code> can both read regular files as well as files in archives. See <a href="code.html" class="seealso">code(3)</a> and <a href="../erts/erl_prim_loader.html" class="seealso">erl_prim_loader(3)</a>.</p> <div class="warning"><h2>Warning!</h2><p>The support for loading of code from archive files is experimental. The sole purpose of releasing it before it is ready is to obtain early feedback. The file format, semantics, interfaces etc. may be changed in a future release.</p></div> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="start/1">start/1</h3> <div class="description"> <p>Starts the boot server. <code><anno>Slaves</anno></code> is a list of IP addresses for hosts which are allowed to use this server as a boot server.</p> </div> </div> <div class="function"> <h3 id="start_link/1">start_link/1</h3> <div class="description"> <p>Starts the boot server and links to the caller.
