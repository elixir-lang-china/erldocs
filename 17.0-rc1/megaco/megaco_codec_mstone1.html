<h1>megaco_codec_mstone1</h1> <h2 class="modsummary">This module implements a simple megaco codec-based performance tool.</h2> <div class="description"> <p>This module implements the <em>mstone1</em> tool, a simple megaco codec-based performance tool.</p> <p>The results, the mstone value(s), are written to stdout.</p> <p><em>Note</em> that this module is <em>not</em> included in the runtime part of the application.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="start/0">start() -&gt; void()</h3> <h3 id="start/1">start(MessagePackage) -&gt; void()</h3> <h3 id="start/2">start(MessagePackage, Factor) -&gt; void()</h3> <ul class="type"> <li><code>MessagePackage = message_package()</code></li> <li><code>message_package() = atom()</code></li> <li><code>Factor() = integer() &gt; 0</code></li> </ul> <div class="description"> <p>This function starts the <em>mstone1</em> performance test with all codec configs. <code>Factor</code> (defaults to <code>1</code>) processes are started for every supported codec config. </p> <p>Each process encodes and decodes their messages. The number of messages processed in total (for all processes) is the mstone value.</p> </div> </div> <div class="function"> <h3 id="start_flex/0">start_flex() -&gt; void()</h3> <h3 id="start_flex/1">start_flex(MessagePackage) -&gt; void()</h3> <h3 id="start_flex/2">start_flex(MessagePackage, Factor) -&gt; void()</h3> <ul class="type"> <li><code>MessagePackage = message_package()</code></li> <li><code>message_package() = atom()</code></li> <li><code>Factor() = integer() &gt; 0</code></li> </ul> <div class="description"> <p>This function starts the <em>mstone1</em> performance test with only the flex codec configs (i.e. <code>pretty</code> and <code>compact</code> with <code>flex</code>). The same number of processes are started as when running the standard test (using the <code>start/0,1</code> function). Each process encodes and decodes their messages. The number of messages processed in total (for all processes) is the mstone value.</p> </div> </div> <div class="function"> <h3 id="start_only_drv/0">start_only_drv() -&gt; void()</h3> <h3 id="start_only_drv/1">start_only_drv(MessagePackage) -&gt; void()</h3> <h3 id="start_only_drv/2">start_only_drv(MessagePackage, Factor) -&gt; void()</h3> <ul class="type"> <li><code>MessagePackage = message_package()</code></li> <li><code>message_package() = atom()</code></li> <li><code>Factor() = integer() &gt; 0</code></li> </ul> <div class="description"> <p>This function starts the <em>mstone1</em> performance test with only the driver using codec configs (i.e. <code>pretty</code> and <code>compact</code> with <code>flex</code>, and <code>ber</code> and <code>per</code> with <code>driver</code> and <code>erlang</code> with <code>compressed</code>). The same number of processes are started as when running the standard test (using the <code>start/0,1</code> function). Each process encodes and decodes their messages. The number of messages processed in total (for all processes) is the mstone value.</p> </div> </div> <div class="function"> <h3 id="start_no_drv/0">start_no_drv() -&gt; void()</h3> <h3 id="start_no_drv/1">start_no_drv(MessagePackage) -&gt; void()</h3> <h3 id="start_no_drv/2">start_no_drv(MessagePackage, Factor) -&gt; void()</h3> <ul class="type"> <li><code>MessagePackage = message_package()</code></li> <li><code>message_package() = atom()</code></li> <li><code>Factor() = integer() &gt; 0</code></li> </ul> <div class="description"> <p>This function starts the <em>mstone1</em> performance test with codec configs not using any drivers (i.e. <code>pretty</code> and <code>compact</code> without <code>flex</code>, <code>ber</code> and <code>per</code> without <code>driver</code> and <code>erlang</code> without <code>compressed</code>). The same number of processes are started as when running the standard test (using the <code>start/0,1</code> function). Each process encodes and decodes their messages. The number of messages processed in total (for all processes) is the mstone value.</p> </div> </div> </div>
