<h1>megaco_transport</h1> <h2 class="modsummary">Megaco transport behaviour.</h2> <div class="description"> <p>The following functions should be exported from a <code>megaco_transport</code> callback module:</p> <list type="bulleted"> <item> <p><a href="#send_message" class="seealso">send_message/2</a> [<code>mandatory</code>]</p> </item> <item> <p><a href="#send_message" class="seealso">send_message/3</a> [<code>optional</code>]</p> </item> <item> <p><a href="#resend_message" class="seealso">resend_message/2</a> [<code>optional</code>]</p> </item> <item> </item> </list> <span id="send_message"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="send_message/2">Module:send_message(Handle, Msg) -&gt; ok | {cancel, Reason} | Error</h3> <h3 id="send_message/3">Module:send_message(Handle, Msg, Resend) -&gt; ok | {cancel, Reason} | Error</h3> <ul class="type"> <li><code>Handle = term()</code></li> <li><code>Msg = binary() | iolist()</code></li> <li><code>Resend = boolean()</code></li> <li><code>Reason = term()</code></li> <li><code>Error = term()</code></li> </ul> <div class="description"> <p>Send a megaco message. </p> <p>If the function returns <code>{cancel, Reason}</code>, this means the transport module decided not to send the message. This is <em>not</em> an error. No error messages will be issued and no error counters incremented. What actions this will result in depends on what kind of message was sent. </p> <p>In the case of requests, megaco will cancel the message in much the same way as if <code>megaco:cancel</code> had been called (after a successfull send). The information will be propagated back to the user differently depending on how the request(s) where issued: For requests issued using <a href="megaco.html#call" class="seealso">megaco:call</a>, the info will be delivered in the return value. For requests issued using <code>megaco:cast</code> the info will be delivered via a call to the callback function <a href="megaco_user.html#trans_reply" class="seealso">handle_trans_reply</a>. </p> <p>In the case of reply, megaco will cancel the reply and information of this will be returned to the user via a call to the callback function <a href="megaco_user.html#trans_ack" class="seealso">handle_trans_ack</a>. </p> <p>The function <code>send_message/3</code> will only be called if the <a href="megaco.html#ui_resend_indication" class="seealso">resend_indication</a> config option has been set to the value <code>flag</code>. The third argument, <code>Resend</code> then indicates if the message send is a resend or not. </p> <span id="resend_message"> </span> </div> </div> <div class="function"> <h3 id="resend_message/2">Module:resend_message(Handle, Msg) -&gt; ok | {cancel, Reason} | Error</h3> <ul class="type"> <li><code>Handle = term()</code></li> <li><code>Msg = binary() | iolist()</code></li> <li><code>Reason = term()</code></li> <li><code>Error = term()</code></li> </ul> <div class="description"> <p>Re-send a megaco message. </p> <p>Note that this function will only be called if the user has set the <a href="megaco.html#ui_resend_indication" class="seealso">resend_indication</a> config option to <code>true</code><em>and</em> it is in fact a message resend. If not <em>both</em> of these condition's are meet, <code>send_message</code> will be called. </p> <p>If the function returns <code>{cancel, Reason}</code>, this means the transport module decided not to send the message. This is <em>not</em> an error. No error messages will be issued and no error counters incremented. What actions this will result in depends on what kind of message was sent. </p> <p>In the case of requests, megaco will cancel the message in much the same way as if <code>megaco:cancel</code> had been called (after a successfull send). The information will be propagated back to the user differently depending on how the request(s) where issued: For requests issued using <a href="megaco.html#call" class="seealso">megaco:call</a>, the info will be delivered in the return value. For requests issued using <code>megaco:cast</code> the info will be delivered via a call to the callback function <a href="megaco_user.html#trans_reply" class="seealso">handle_trans_reply</a>. </p> <p>In the case of reply, megaco will cancel the reply and information of this will be returned to the user via a call to the callback function <a href="megaco_user.html#trans_ack" class="seealso">handle_trans_ack</a>. </p> </div> </div> </div>
