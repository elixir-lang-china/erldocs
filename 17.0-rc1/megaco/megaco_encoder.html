<h1>megaco_encoder</h1> <h2 class="modsummary">Megaco encoder behaviour.</h2> <div class="description"> <p>The following functions should be exported from a <code>megaco_encoder</code> callback module:</p> </div> <div class="section"> <h4>DATA TYPES</h4> <pre class="sh_erlang"> megaco_message() = #'MegacoMessage{}' transaction() = {transactionRequest, transaction_request()} | {transactionPending, transaction_reply()} | {transactionReply, transaction_pending()} | {transactionResponseAck, transaction_response_ack()} | {segmentReply, segment_reply()} transaction_request() = #'TransactionRequest'{} transaction_pending() = #'TransactionPending'{} transaction_reply() = #'TransactionReply'{} transaction_response_ack() = [transaction_ack()] transaction_ack() = #'TransactionAck'{} segment_reply() = #'SegmentReply'{} action_request() = #'ActionRequest'{} action_reply() = #'ActionReply'{} </pre> <span id="encode_message"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="encode_message/3">Module:encode_message(EncodingConfig, Version, Message) -&gt; {ok, Bin} | Error</h3> <ul class="type"> <li><code>EncodingConfig = list()</code></li> <li><code>Version = integer()</code></li> <li><code>Message = megaco_message()</code></li> <li><code>Bin = binary()</code></li> <li><code>Error = term()</code></li> </ul> <div class="description"> <p>Encode a megaco message.</p> <span id="decode_message"> </span> </div> </div> <div class="function"> <h3 id="decode_message/3">Module:decode_message(EncodingConfig, Version, Bin) -&gt; {ok, Message} | Error</h3> <ul class="type"> <li><code>EncodingConfig = list()</code></li> <li><code>Version = integer() | dynamic</code></li> <li><code>Message = megaco_message()</code></li> <li><code>Bin = binary()</code></li> <li><code>Error = term()</code></li> </ul> <div class="description"> <p>Decode a megaco message. </p> <p>Note that if the Version argument is <code>dynamic</code>, the decoder should try to figure out the actual version from the message itself and then use the proper decoder, e.g. version 1. <br />If on the other hand the Version argument is an integer, it means that this is the expected version of the message and the decoder for that version should be used.</p> <span id="decode_mini_message"> </span> </div> </div> <div class="function"> <h3 id="decode_mini_message/3">Module:decode_mini_message(EncodingConfig, Version, Bin) -&gt; {ok, Message} | Error</h3> <ul class="type"> <li><code>EncodingConfig = list()</code></li> <li><code>Version = integer() | dynamic</code></li> <li><code>Message = megaco_message()</code></li> <li><code>Bin = binary()</code></li> <li><code>Error = term()</code></li> </ul> <div class="description"> <p>Perform a minimal decode of a megaco message. </p> <p>The purpose of this function is to do a minimal decode of Megaco message. A successfull result is a <code>'MegacoMessage'</code> in which only version and mid has been initiated. This function is used by the megaco_messenger module when the <code>decode_message/3</code> function fails to figure out the mid (the actual sender) of the message.</p> <p>Note again that a successfull decode only returns a partially initiated message.</p> <span id="encode_transaction"> </span> </div> </div> <div class="function"> <h3 id="encode_transaction/3">Module:encode_transaction(EncodingConfig, Version, Transaction) -&gt; OK | Error</h3> <ul class="type"> <li><code>EncodingConfig = list()</code></li> <li><code>Version = integer()</code></li> <li><code>Transaction = transaction()</code></li> <li><code>OK = {ok, Bin}</code></li> <li><code>Bin = binary()</code></li> <li><code>Error = {error, Reason}</code></li> <li><code>Reason = not_implemented | OtherReason</code></li> <li><code>OtherReason = term()</code></li> </ul> <div class="description"> <p>Encode a megaco transaction. If this, for whatever reason, is not supported, the function should return the error reason <code>not_implemented</code>. </p> <p>This functionality is used both when the transaction sender is used and for segmentation. So, for either of those to work, this function <em>must</em> be fully supported! </p> <span id="encode_action_requests"> </span> </div> </div> <div class="function"> <h3 id="encode_action_requests/3">Module:encode_action_requests(EncodingConfig, Version, ARs) -&gt; OK | Error</h3> <ul class="type"> <li><code>EncodingConfig = list()</code></li> <li><code>Version = integer()</code></li> <li><code>ARs = action_requests()</code></li> <li><code>action_requests() = [action_request()]</code></li> <li><code>OK = {ok, Bin}</code></li> <li><code>Bin = binary()</code></li> <li><code>Error = {error, Reason}</code></li> <li><code>Reason = not_implemented | OtherReason</code></li> <li><code>OtherReason = term()</code></li> </ul> <div class="description"> <p>Encode megaco action requests. This function is called when the user calls the function <a href="megaco.html#encode_actions" class="seealso">encode_actions/3</a>. If that function is never used or if the codec cannot support this (the encoding of individual actions), then return with error reason <code>not_implemented</code>. </p> <span id="encode_action_reply"> </span> </div> </div> <div class="function"> <h3 id="encode_action_reply/3">Module:encode_action_reply(EncodingConfig, Version, AR) -&gt; OK | Error</h3> <ul class="type"> <li><code>EncodingConfig = list()</code></li> <li><code>Version = integer()</code></li> <li><code>AR = action_reply()</code></li> <li><code>OK = {ok, Bin}</code></li> <li><code>Bin = binary()</code></li> <li><code>Error = {error, Reason}</code></li> <li><code>Reason = not_implemented | OtherReason</code></li> <li><code>OtherReason = term()</code></li> </ul> <div class="description"> <p>Encode a megaco action reply. If this, for whatever reason, is not supported, the function should return the error reason <code>not_implemented</code>. </p> <p>This function is used when segmentation has been configured. So, for this to work, this function <em>must</em> be fully supported! </p> </div> </div> </div>
