<h1>webtool</h1> <h2 class="modsummary">WebTool is a tool used to simplify the implementation of web based tools with Erlang/OTP.</h2> <div class="description"> <p>WebTool makes it easy to use web based tools with Erlang/OTP. WebTool configures and starts the webserver httpd.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="start/0">start()-&gt; {ok,Pid}| {stop,Reason}</h3> <div class="description"> <p>Start WebTool with default data, i.e. port 8888, ip-number 127.0.0.1, and server-name <code>localhost</code>. If port 8888 is in use, port 8889 is tried instead. If 8889 is also in use, 8890 is tried and so on. Max number of ports tried is 256. </p> <p>The <code>mime.types</code> file and WebTool's own HTML files are assumed to be in the directory <code>webtool-&lt;vsn&gt;/priv/root/conf</code>.</p> </div> </div> <div class="function"> <h3 id="start/2">start(Path,Data)-&gt;{ok,Pid}|{stop,Reason}</h3> <ul class="type"> <li><code>Path = string() | standard_path</code></li> <li><code>Data = [Port,Address,Name] | PortNumber | standard_data</code></li> <li><code>Port = {port,PortNumber}</code></li> <li><code>Address = {bind_address,IpNumber}</code></li> <li><code>Name = {server_name,ServerName}</code></li> <li><code>PortNumber = integer()</code></li> <li><code>IpNumber = tuple(), e.g. {127,0,0,1}</code></li> <li><code>ServerName = string()</code></li> <li><code>Pid = pid()</code></li> </ul> <div class="description"> <p>Use this function to start WebTool if the default port, ip-number,servername or path can not be used.</p> <p><code>Path</code> is the directory where the <code>mime.types</code> file and WebTool's own HTML files are located. By default this is <code>webtool-&lt;vsn&gt;/priv</code>, and in most cases there is no need to change this. If <code>Path</code> is set to <code>standard_path</code> the default will be used.</p> <p>If <code>Data</code> is set to <code>PortNumber</code>, the default data will be used for ip-number (<code>127.0.0.1</code>) and server name (<code>localhost</code>).</p> </div> </div> <div class="function"> <h3 id="stop/0">stop()-&gt;void</h3> <div class="description"> <p>Stop WebTool and the tools started by WebTool.</p> </div> </div> <div class="function"> <h3 id="debug_app/1">debug_app(Module)-&gt;void</h3> <ul class="type"> <li><code>Module = atom()</code></li> </ul> <div class="description"> <p>Debug a WebTool application by tracing all functions in the given module which are called from WebTool.</p> </div> </div> <div class="function"> <h3 id="stop_debug/0">stop_debug()-&gt;void</h3> <div class="description"> <p>Stop the tracing started by <code>debug_app/1</code>, and format the trace log.</p> </div> </div> </div> <div class="section"> <h4>CALLBACK FUNCTIONS</h4> <p>The following callback function must be implemented by each web based tool that will be used via WebTool. When started, WebTool searches the Erlang code path for *.tool files to locate all web based tools and their callback functions. See the <a href="webtool_chapter.html" class="seealso">WebTool User's Guide</a> for more information about the *.tool files.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="Func/1">Module:Func(Data)-&gt; {Name,WebData}|error</h3> <ul class="type"> <li><code>Data = term()</code></li> <li><code>Name = atom()</code></li> <li><code>WebData = [WebOptions]</code></li> <li><code>WebOptions = LinkData | Alias | Start</code></li> <li><code>LinkData = {web_data,{ToolName,Url}}</code></li> <li><code>Alias = {alias,{VirtualPath,RealPath}} | {alias,{erl_alias,Path,[Modules]}</code></li> <li><code>Start = {start,StartData}</code></li> <li><code>ToolName = Url = VirtualPath = RealPath = Path = string()</code></li> <li><code>Modules = atom()</code></li> <li><code>StartData = AppData | ChildSpec | Func</code></li> <li><code>AppData = {app,AppName}</code></li> <li><code>ChildSpec = {child,child_spec()}</code></li> <d>See the Reference Manual for the module supervisor in the STDLIB application for details about child_spec().</d> <li><code>Func = {func,{StartMod,StartFunc,StartArg}, {StopMod,StopFunc,StopArg}}</code></li> <li><code>AppName = StartMod = StartFunc = StopMod = StopFunc =atom()</code></li> <li><code>StartArg = StopArg = [term()]</code></li> </ul> <div class="description"> <p>This is the configuration function (<code>config_func</code>) which must be stated in the <code>*.tool</code> file.</p> <p>The function is called by WebTool at startup to retrieve the data needed to start and configure the tool. <code>LinkData</code> is used by WebTool to create the link to the tool. <code>Alias</code> is used to create the aliases needed by the webserver. <code>Start</code> is used to start and stop the tool.</p> </div> </div> </div> <div class="section"> <h4>See Also</h4> <p><a href="start_webtool.html" class="seealso">start_webtool(1)</a>, <a href="webtool_chapter.html" class="seealso">WebTool User's Guide</a></p> </div>
