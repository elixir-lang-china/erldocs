<h1>CosTransactions_SubtransactionAwareResource</h1> <h2 class="modsummary">This module implements the OMG CosTransactions::SubtransactionAwareResource interface.</h2> <div class="description"> <p>This interface inherits the CosTransactions::Resource interface. Hence, it must also support all operations defined in the Resource interface.</p> <p>To get access to the record definitions for the structures use: <br /> <code>-include_lib("cosTransactions/include/CosTransactions.hrl").</code></p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="commit_subtransaction/2">commit_subtransaction(SubtransactionAwareResource, Coordinator) -&gt; Return</h3> <ul class="type"> <li><code>SubtransactionAwareResource = #objref</code></li> <li><code>Coordinator = #objref</code></li> <li><code>Return = ok</code></li> </ul> <div class="description"> <p>If the <code>SubtransactionAwareResource</code> have been registered with a <em>subtransaction</em> using the operation <code>CosTransactions_Coordinator:register_subtran_aware/2</code>, it will be notified when the transaction has committed.</p> <div class="note"><h2>Note!</h2> <p>The results of a committed subtransaction are relative to the completion of its ancestor transactions, that is, these results can be undone if any ancestor transaction is rolled back.</p> </div> </div> </div> <div class="function"> <h3 id="rollback_subtransaction/1">rollback_subtransaction(SubtransactionAwareResource) -&gt; Return</h3> <ul class="type"> <li><code>SubtransactionAwareResource = #objref</code></li> <li><code>Return = ok</code></li> </ul> <div class="description"> <p>If the <code>SubtransactionAwareResource</code> have been registered with a transactions using the operation <code>CosTransactions_Coordinator:register_subtran_aware/2</code> it will be notified when the transaction has rolled back.</p> </div> </div> </div>
