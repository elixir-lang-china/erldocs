<h1>erl_comment_scan</h1> <h2 class="modsummary">Functions for reading comment lines from Erlang source code.</h2> <div class="description"> <p>Functions for reading comment lines from Erlang source code.</p></div> <div class="section"><h4>DATA TYPES</h4><span id="types"> </span> <taglist> <dt><code>comment() = {integer(), integer(), integer(), [string()]}</code></dt> <item><span id="type-comment"> </span> </item> </taglist></div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="file/1">file(FileName::filename() (see module file)) -&gt; [Comment]</h3> <ul class="type"> <li><code>Comment = {Line, Column, Indentation, Text}</code></li><li><code>Line = integer()</code></li><li><code>Column = integer()</code></li><li><code>Indentation = integer()</code></li><li><code>Text = [string()]</code></li></ul> <div class="description"><span id="file-1"> </span> <p>Extracts comments from an Erlang source code file. Returns a list of entries representing <em>multi-line</em> comments, listed in order of increasing line-numbers. For each entry, <code>Text</code> is a list of strings representing the consecutive comment lines in top-down order; the strings contain <em>all</em> characters following (but not including) the first comment-introducing <code>%</code> character on the line, up to (but not including) the line-terminating newline.</p> <p>Furthermore, <code>Line</code> is the line number and <code>Column</code> the left column of the comment (i.e., the column of the comment-introducing <code>%</code> character). <code>Indent</code> is the indentation (or padding), measured in character positions between the last non-whitespace character before the comment (or the left margin), and the left column of the comment. <code>Line</code> and <code>Column</code> are always positive integers, and <code>Indentation</code> is a nonnegative integer.</p> <p>Evaluation exits with reason <code>{read, Reason}</code> if a read error occurred, where <code>Reason</code> is an atom corresponding to a Posix error code; see the module <url href="http://www.erlang.org/edoc/doc/kernel/doc/file.xml">//kernel/file</url> for details.</p> </div></div> <div class="function"> <h3 id="string/1">string(Text::string()) -&gt; [Comment]</h3> <ul class="type"> <li><code>Comment = {Line, Column, Indentation, Text}</code></li><li><code>Line = integer()</code></li><li><code>Column = integer()</code></li><li><code>Indentation = integer()</code></li><li><code>Text = [string()]</code></li></ul> <div class="description"><span id="string-1"> </span> <p>Extracts comments from a string containing Erlang source code. Except for reading directly from a string, the behaviour is the same as for <a href="#file-1" class="seealso">file/1</a>. </p> <p><em>See also:</em> <a href="#file-1" class="seealso">file/1</a>.</p> </div></div> <div class="function"> <h3 id="scan_lines/1">scan_lines(Text::string()) -&gt; [CommentLine]</h3> <ul class="type"> <li><code>CommentLine = {Line, Column, Indent, Text}</code></li><li><code>Line = integer()</code></li><li><code>Column = integer()</code></li><li><code>Indent = integer()</code></li><li><code>Text = string()</code></li></ul> <div class="description"><span id="scan_lines-1"> </span> <p>Extracts individual comment lines from a source code string. Returns a list of comment lines found in the text, listed in order of <em>decreasing</em> line-numbers, i.e., the last comment line in the input is first in the resulting list. <code>Text</code> is a single string, containing all characters following (but not including) the first comment-introducing <code>%</code> character on the line, up to (but not including) the line-terminating newline. For details on <code>Line</code>, <code>Column</code> and <code>Indent</code>, see <a href="#file-1" class="seealso">file/1</a>.</p> </div></div> <div class="function"> <h3 id="join_lines/1">join_lines(Lines::[CommentLine]) -&gt; [Comment]</h3> <ul class="type"> <li><code>CommentLine = {Line, Column, Indent, string()}</code></li><li><code>Line = integer()</code></li><li><code>Column = integer()</code></li><li><code>Indent = integer()</code></li><li><code>Comment = {Line, Column, Indent, Text}</code></li><li><code>Text = [string()]</code></li></ul> <div class="description"><span id="join_lines-1"> </span> <p>Joins individual comment lines into multi-line comments. The input is a list of entries representing individual comment lines, <em>in order of decreasing line-numbers</em>; see <a href="#scan_lines-1" class="seealso">scan_lines/1</a> for details. The result is a list of entries representing <em>multi-line</em> comments, <em>still listed in order of decreasing line-numbers</em>, but where for each entry, <code>Text</code> is a list of consecutive comment lines in order of <em>increasing</em> line-numbers (i.e., top-down). </p> <p><em>See also:</em> <a href="#scan_lines-1" class="seealso">scan_lines/1</a>.</p> </div></div></div> <authors> <aname>Richard Carlsson</aname> <email>carlsson.richard@gmail.com</email></authors>
