string()</h3> <div class="description"><span id="format-1"> </span> <p>Equivalent to <a href="#format-2" class="seealso">format(Tree, [])</a>.</p> </div></div> <div class="function"> <h3 id="format/1-1">format(Tree::syntaxTree(), Options::[term()]) -&gt; string()</h3> <ul class="type"> <li><code>syntaxTree() (see module erl_syntax)</code></li></ul> <div class="description"><span id="format-2"> </span> <p>Prettyprint-formats an abstract Erlang syntax tree as text. For example, if you have a <code>.beam</code> file that has been compiled with <code>debug_info</code>, the following should print the source code for the module (as it looks in the debug info representation): </p><pre class="sh_erlang"> {ok,{_,[{abstract_code,{_,AC}}]}} = beam_lib:chunks("myfile.beam",[abstract_code]), io:put_chars(erl_prettypr:format(erl_syntax:form_list(AC)))</pre><p> Available options: </p><taglist> <dt>{hook, none | <a href="#type-hook" class="seealso">hook()</a>}</dt> <item><p>Unless the value is <code>none</code>, the given function is called for each node whose list of annotations is not empty; see below for details. The default value is <code>none</code>.</p></item> <dt>{paper, integer()}</dt> <item><p>Specifies the preferred maximum number of characters on any line, including indentation. The default value is 80.</p></item> <dt>{ribbon, integer()}</dt> <item><p>Specifies the preferred maximum number of characters on any line, not counting indentation. The default value is 65.</p></item> <dt>{user, term()}</dt> <item><p>User-specific data for use in hook functions. The default value is <code>undefined</code>.</p></item> <dt>{encoding, epp:source_encoding()}</dt> <item><p>Specifies the encoding of the generated file.</p></item> </taglist><p> A hook function (cf. the <a href="#type-hook" class="seealso">hook()</a> type) is passed the current syntax tree node, the context, and a continuation. The context can be examined and manipulated by functions such as <code>get_ctxt_user/1</code> and <code>set_ctxt_user/2</code>. The hook must return a "document" data structure (see <a href="#layout-2" class="seealso">layout/2</a> and <a href="#best-2" class="seealso">best/2</a>); this may be constructed in part or in whole by applying the continuation function. For example, the following is a trivial hook: </p><pre class="sh_erlang"> fun (Node, Ctxt, Cont) -&gt; Cont(Node, Ctxt) end</pre><p> which yields the same result as if no hook was given. The following, however: </p><pre class="sh_erlang"> fun (Node, Ctxt, Cont) -&gt; Doc = Cont(Node, Ctxt), prettypr:beside(prettypr:text("&lt;b&gt;"), prettypr:beside(Doc, prettypr:text("&lt;/b&gt;"))) end</pre><p> will place the text of any annotated node (regardless of the annotation data) between HTML "boldface begin" and "boldface end" tags. </p> <p><em>See also:</em> <a href="erl_syntax.html" class="seealso">erl_syntax</a>, <a href="#best-2" class="seealso">best/2</a>, <a href="#format-1" class="seealso">format/1</a>, <a href="#get_ctxt_user-1" class="seealso">get_ctxt_user/1</a>, <a href="#layout-2" class="seealso">layout/2</a>, <a href="#set_ctxt_user-2" class="seealso">set_ctxt_user/2</a>.</p> </div></div> <div class="function"> <h3 id="best/1">best(Tree::syntaxTree()) -&gt; empty | document()</h3> <div class="description"><span id="best-1"> </span> <p>Equivalent to <a href="#best-2" class="seealso">best(Tree, [])</a>.</p> </div></div> <div class="function"> <h3 id="best/1-1">best(Tree::syntaxTree(), Options::[term()]) -&gt; empty | document()</h3> <div class="description"><span id="best-2"> </span> <p>Creates a fixed "best" abstract layout for a syntax tree. This is similar to the <code>layout/2</code> function, except that here, the final layout has been selected with respect to the given options. The atom <code>empty</code> is returned if no such layout could be produced. For information on the options, see the <code>format/2</code> function. </p> <p><em>See also:</em> <a href="#best-1" class="seealso">best/1</a>, <a href="#format-2" class="seealso">format/2</a>, <a href="#layout-2" class="seealso">layout/2</a>, <a href="prettypr.html#best-3" class="seealso">prettypr:best/3</a>.</p> </div></div> <div class="function"> <h3 id="layout/1">layout(Tree::syntaxTree()) -&gt; document()</h3> <div class="description"><span id="layout-1"> </span> <p>Equivalent to <a href="#layout-2" class="seealso">layout(Tree, [])</a>.</p> </div></div> <div class="function"> <h3 id="layout/1-1">layout(Tree::syntaxTree(), Options::[term()]) -&gt; document()</h3> <ul class="type"> <li><code>document() (see module prettypr)</code></li></ul> <div class="description"><span id="layout-2"> </span> <p>Creates an abstract document layout for a syntax tree. The result represents a set of possible layouts (cf. module <code>prettypr</code>). For information on the options, see <a href="#format-2" class="seealso">format/2</a>; note, however, that the <code>paper</code> and <code>ribbon</code> options are ignored by this function.</p> <p>This function provides a low-level interface to the pretty printer, returning a flexible representation of possible layouts, independent of the paper width eventually to be used for formatting. This can be included as part of another document and/or further processed directly by the functions in the <code>prettypr</code> module, or used in a hook function (see <code>format/2</code> for details). </p> <p><em>See also:</em> <a href="prettypr.html" class="seealso">prettypr</a>, <a href="#format-2" class="seealso">format/2</a>, <a href="#layout-1" class="seealso">layout/1</a>.</p> </div></div></div> <authors> <aname>Richard Carlsson</aname> <email>carlsson.richard@gmail.com</email></authors>
