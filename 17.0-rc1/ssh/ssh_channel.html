<h1>ssh_channel</h1> <h2 class="modsummary">-behaviour(ssh_channel). </h2> <div class="description"> <p>SSH services (clients and servers) are implemented as channels that are multiplexed over an SSH connection and communicates via the <url href="http://www.ietf.org/rfc/rfc4254.txt"> SSH Connection Protocol</url>. This module provides a callback API that takes care of generic channel aspects such as flow control and close messages and lets the callback functions take care of the service (application) specific parts. This behavior also ensures that the channel process honors the principal of an OTP-process so that it can be part of a supervisor tree. This is a requirement of channel processes implementing a subsystem that will be added to the SSH applications supervisor tree. </p> <div class="note"><h2>Note!</h2> <p>When implementing a SSH subsystem use the <code>-behaviour(ssh_daemon_channel).</code> instead of <code>-behaviour(ssh_channel).</code> as the only relevant callback functions for subsystems are init/1, handle_ssh_msg/2, handle_msg/2 and terminate/2, so the ssh_daemon_channel behaviour is limited version of the ssh_channel behaviour. </p> </div> </div> <div class="section"> <h4>DATA TYPES </h4> <p>Type definitions that are used more than once in this module and/or abstractions to indicate the intended use of the data type:</p> <p><code>boolean() = true | false </code></p> <p><code>string() = list of ASCII characters</code></p> <p><code>timeout() = infinity | integer() - in milliseconds.</code></p> <p><code>ssh_connection_ref() - opaque to the user returned by ssh:connect/3 or sent to an SSH channel process</code></p> <p><code>ssh_channel_id() = integer() </code></p> <p><code>ssh_data_type_code() = 1 ("stderr") | 0 ("normal") are currently valid values see <url href="http://www.ietf.org/rfc/rfc4254.txt">RFC 4254 </url> section 5.2.</code></p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="call/2">call(ChannelRef, Msg) -&gt;</h3> <h3 id="call/3">call(ChannelRef, Msg, Timeout) -&gt; Reply | {error, Reason}</h3> <ul class="type"> <li><code>ChannelRef = pid() </code></li> <d>As returned by start_link/4 </d> <li><code>Msg = term() </code></li> <li><code>Timeout = timeout() </code></li> <li><code>Reply = term() </code></li> <li><code>Reason = closed | timeout </code></li> </ul> <div class="description"> <p>Makes a synchronous call to the channel process by sending a message and waiting until a reply arrives or a timeout occurs. If the channel process does not exist <code>{error, closed}</code> is returned. </p> </div> </div> <div class="function"> <h3 id="cast/2">cast(ChannelRef, Msg) -&gt; ok </h3> <ul class="type"> <li><code>ChannelRef = pid() </code></li> <d>As returned by start_link/4 </d> <li><code>Msg = term() </code></li> </ul> <div class="description"> <p>Sends an asynchronous message to the channel process and returns ok immediately, ignoring if the destination node or channel process does not exist.
