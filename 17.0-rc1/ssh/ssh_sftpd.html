<h1>ssh_sftpd</h1> <h2 class="modsummary">Specifies the channel process to handle an sftp subsystem.</h2> <div class="description"> <p>Specifies a channel process to handle a sftp subsystem.</p> </div> <div class="section"> <h4>DATA TYPES </h4> <p><code>subsystem_spec() = {subsystem_name(), {channel_callback(), channel_init_args()}} </code></p> <p><code>subsystem_name() = "sftp"</code></p> <p><code>channel_callback() = atom()</code> - Name of the erlang module implementing the subsystem using the ssh_channel behavior see <a href="ssh_channel.html" class="seealso">ssh_channel(3)</a></p> <p><code> channel_init_args() = list() - The one given as argument to function subsystem_spec/1.</code></p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="subsystem_spec/1">subsystem_spec(Options) -&gt; subsystem_spec()</h3> <ul class="type"> <li><code>Options = [{Option, Value}]</code></li> </ul> <div class="description"> <p>Should be used together with ssh:daemon/[1,2,3]</p> <p>Options are:</p> <taglist> <dt><code>{cwd, String}</code></dt> <item> <p>Sets the initial current working directory for the server.</p> </item> <dt><code>{file_handler, CallbackModule}</code></dt> <item> <p>Determines which module to call for accessing the file server. The default value is <code>ssh_sftpd_file</code> that uses the <a href="../kernel/file.html" class="seealso">file</a> and <a href="../stdlib/filelib.html" class="seealso">filelib</a> API:s to access the standard OTP file server. This option may be used to plug in other file servers.</p> </item> <dt><code>{max_files, Integer}</code></dt> <item> <p>The default value is <code>0</code>, which means that there is no upper limit. If supplied, the number of filenames returned to the sftp client per <code>READDIR</code> request is limited to at most the given value.</p> </item> <dt><code>{root, String}</code></dt> <item> <p>Sets the sftp root directory. The user will then not be able to see any files above this root. If for instance the root is set to <code>/tmp</code> the user will see this directory as <code>/</code> and if the user does cd <code>/etc</code> the user will end up in <code>/tmp/etc</code>. </p> </item> <dt><code>{sftpd_vsn, integer()}</code></dt> <item> <p>Sets the sftp version to use, defaults to 5. Version 6 is under development and limited.</p> </item> </taglist> </div> </div> </div>
