<h1>snmpa_mib_storage</h1> <h2 class="modsummary"> Behaviour module for the SNMP agent mib storage. </h2> <div class="description"> <p>This module defines the behaviour of the SNMP agent mib storage. </p> <p>The mib storage is used by the agent to store internal mib- related information. The mib storage module is used by several entities, not just the mib-server. </p> <p>A <code>snmpa_mib_storage</code> compliant module must export the following functions: </p> <list type="bulleted"> <item> <a href="#open" class="seealso">open/5</a> </item> <item> <a href="#close" class="seealso">close/1</a> </item> <item> <a href="#read" class="seealso">read/2</a> </item> <item> <a href="#write" class="seealso">write/2</a> </item> <item> <a href="#delete1" class="seealso">delete/1</a> </item> <item> <a href="#delete2" class="seealso">delete/2</a> </item> <item> <a href="#match_object" class="seealso">match_object/2</a> </item> <item> <a href="#match_delete" class="seealso">match_delete/2</a> </item> <item> <a href="#tab2list" class="seealso">tab2list/1</a> </item> <item> <a href="#info" class="seealso">info/1</a> </item> <item> <a href="#sync" class="seealso">sync/1</a> </item> <item> <a href="#backup" class="seealso">backup/2</a> </item> </list> <p>The semantics of them and their exact signatures are explained below. </p> </div> <div class="section"> <h4>CALLBACK FUNCTIONS</h4> <p>The following functions must be exported from a <code>mib-server</code> data callback module: </p> <span id="open"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="open/5">Module:open(Name, RecordName, Fields, Type, Options) -&gt;
