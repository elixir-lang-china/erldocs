<h1>snmp_notification_mib</h1> <h2 class="modsummary">Instrumentation Functions for SNMP-NOTIFICATION-MIB</h2> <div class="description"> <p>The module <code>snmp_notification_mib</code> implements the instrumentation functions for the SNMP-NOTIFICATION-MIB, and functions for configuring the database. </p> <p>The configuration files are described in the SNMP User's Manual.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="configure/1">configure(ConfDir) -&gt; void()</h3> <ul class="type"> <li><code>ConfDir = string()</code></li> </ul> <div class="description"> <p>This function is called from the supervisor at system start-up. </p> <p>Inserts all data in the configuration files into the database and destroys all old rows with StorageType <code>volatile</code>. The rows created from the configuration file will have StorageType <code>nonVolatile</code>. </p> <p>If an error is found in the configuration file, it is reported using the function <code>config_err/2</code> of the error report module, and the function fails with reason <code>configuration_error</code>. </p> <p><code>ConfDir</code> is a string which points to the directory where the configuration files are found. </p> <p>The configuration file read is: <code>notify.conf</code>. </p> </div> </div> <div class="function"> <h3 id="reconfigure/1">reconfigure(ConfDir) -&gt; void()</h3> <ul class="type"> <li><code>ConfDir = string()</code></li> </ul> <div class="description"> <p>Inserts all data in the configuration files into the database and destroys all old data, including the rows with StorageType <code>nonVolatile</code>. The rows created from the configuration file will have StorageType <code>nonVolatile</code>. </p> <p>Thus, the data in the SNMP-NOTIFICATION-MIB, after this function has been called, is from the configuration files. </p> <p>If an error is found in the configuration file, it is reported using the function <code>config_err/2</code> of the error report module, and the function fails with reason <code>configuration_error</code>. </p> <p><code>ConfDir</code> is a string which points to the directory where the configuration files are found. </p> <p>The configuration file read is: <code>notify.conf</code>. <span id="add_notify"> </span> </p> </div> </div> <div class="function"> <h3 id="add_notify/3">add_notify(Name, Tag, Type) -&gt; Ret</h3> <ul class="type"> <li><code>Name = string()</code></li> <li><code>Tag = string()</code></li> <li><code>Type = trap | inform</code></li> <li><code>Ret = {ok, Key} | {error, Reason}</code></li> <li><code>Key = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Adds a notify definition to the agent config. Equivalent to one line in the <code>notify.conf</code> file.</p> <span id="delete_notify"> </span> </div> </div> <div class="function"> <h3 id="delete_notify/1">delete_notify(Key) -&gt; Ret</h3> <ul class="type"> <li><code>Key = term()</code></li> <li><code>Ret = ok | {error, Reason}</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Delete a notify definition from the agent config.</p> </div> </div> </div>
