<h1>snmp_target_mib</h1> <h2 class="modsummary">Instrumentation Functions for SNMP-TARGET-MIB</h2> <div class="description"> <p>The module <code>snmp_target_mib</code> implements the instrumentation functions for the SNMP-TARGET-MIB, and functions for configuring the database. </p> <p>The configuration files are described in the SNMP User's Manual.</p> <span id="types"> </span> </div> <div class="section"> <h4>DATA TYPES</h4> <pre class="sh_erlang"> transportDomain() = transportDomainUdpIpv4 | transportDomainUdpIpv6 transportAddressIPv4() = [integer()], length 4 transportAddressIPv6() = [integer()], length 8 transportAddressMask() = [integer()], length 0 (default), 6 (IPv4) or 10 (IPv6) </pre> <span id="configure"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="configure/1">configure(ConfDir) -&gt; void()</h3> <ul class="type"> <li><code>ConfDir = string()</code></li> </ul> <div class="description"> <p>This function is called from the supervisor at system start-up. </p> <p>Inserts all data in the configuration files into the database and destroys all old rows with StorageType <code>volatile</code>. The rows created from the configuration file will have StorageType <code>nonVolatile</code>. </p> <p>All <code>snmp</code> counters are set to zero. </p> <p>If an error is found in the configuration file, it is reported using the function <code>config_err/2</code> of the error report module, and the function fails with the reason <code>configuration_error</code>. </p> <p><code>ConfDir</code> is a string which points to the directory where the configuration files are found. </p> <p>The configuration files read are: <code>target_addr.conf</code> and <code>target_params.conf</code>. </p> <span id="reconfigure"> </span> </div> </div> <div class="function"> <h3 id="reconfigure/1">reconfigure(ConfDir) -&gt; void()</h3> <ul class="type"> <li><code>ConfDir = string()</code></li> </ul> <div class="description"> <p>Inserts all data in the configuration files into the database and destroys all old data, including the rows with StorageType <code>nonVolatile</code>. The rows created from the configuration file will have StorageType <code>nonVolatile</code>. </p> <p>Thus, the data in the SNMP-TARGET-MIB, after this function has been called, is the data from the configuration files. </p> <p>All <code>snmp</code> counters are set to zero.</p> <p>If an error is found in the configuration file, it is reported using the function <code>config_err/2</code> of the , and the function fails with the reason <code>configuration_error</code>.</p> <p><code>ConfDir</code> is a string which points to the directory where the configuration files are found. </p> <p>The configuration files read are: <code>target_addr.conf</code> and <code>target_params.conf</code>. </p> <span id="set_target_engine_id"> </span> </div> </div> <div class="function"> <h3 id="set_target_engine_id/2">set_target_engine_id(TargetAddrName, EngineId) -&gt; boolean()</h3> <ul class="type"> <li><code>TargetAddrName = string()</code></li> <li><code>EngineId = string()</code></li> </ul> <div class="description"> <p>Changes the engine id for a target in the <code>snmpTargetAddrTable</code>.
