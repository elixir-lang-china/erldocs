<h1>snmp_framework_mib</h1> <h2 class="modsummary">Instrumentation Functions for SNMP-FRAMEWORK-MIB</h2> <div class="description"> <p>The module <code>snmp_framework_mib</code> implements instrumentation functions for the SNMP-FRAMEWORK-MIB, and functions for initializing and configuring the database. </p> <p>The configuration files are described in the SNMP User's Manual.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="configure/1">configure(ConfDir) -&gt; void()</h3> <ul class="type"> <li><code>ConfDir = string()</code></li> </ul> <div class="description"> <p>This function is called from the supervisor at system start-up. </p> <p>Inserts all data in the configuration files into the database and destroys all old data. </p> <p>Thus, the data in the SNMP-FRAMEWORK-MIB, after this function has been called, is from the configuration files. </p> <p>All <code>snmp</code> counters are set to zero. </p> <p>If an error is found in the configuration file, it is reported using the function <code>config_err/2</code> of the error report module, and the function fails with reason <code>configuration_error</code>. </p> <p><code>ConfDir</code> is a string which points to the directory where the configuration files are found. </p> <p>The configuration file read is: <code>context.conf</code>. </p> </div> </div> <div class="function"> <h3 id="init/0">init() -&gt; void()</h3> <div class="description"> <p>This function is called from the supervisor at system start-up. </p> <p>Creates the necessary objects in the database if they do not exist. It does not destroy any old values. <span id="add_context"> </span> </p> </div> </div> <div class="function"> <h3 id="add_context/1">add_context(Ctx) -&gt; Ret</h3> <ul class="type"> <li><code>Ctx = string()</code></li> <li><code>Ret = {ok, Key} | {error, Reason}</code></li> <li><code>Key = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Adds a context to the agent config. Equivalent to one line in the <code>context.conf</code> file.</p> <span id="delete_context"> </span> </div> </div> <div class="function"> <h3 id="delete_context/1">delete_context(Key) -&gt; Ret</h3> <ul class="type"> <li><code>Key = term()</code></li> <li><code>Ret = ok | {error, Reason}</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Delete a context from the agent config.</p> </div> </div> </div>
