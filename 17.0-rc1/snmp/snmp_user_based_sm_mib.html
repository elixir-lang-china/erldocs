<h1>snmp_user_based_sm_mib</h1> <h2 class="modsummary">Instrumentation Functions for SNMP-USER-BASED-SM-MIB</h2> <div class="description"> <p>The module <code>snmp_user_based_sm_mib</code> implements the instrumentation functions for the SNMP-USER-BASED-SM-MIB, and functions for configuring the database. </p> <p>The configuration files are described in the SNMP User's Manual.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="configure/1">configure(ConfDir) -&gt; void()</h3> <ul class="type"> <li><code>ConfDir = string()</code></li> </ul> <div class="description"> <p>This function is called from the supervisor at system start-up. </p> <p>Inserts all data in the configuration files into the database and destroys all old rows with StorageType <code>volatile</code>. The rows created from the configuration file will have StorageType <code>nonVolatile</code>. </p> <p>All <code>snmp</code> counters are set to zero. </p> <p>If an error is found in the configuration file, it is reported using the function <code>config_err/2</code> of the error report module, and the function fails with the reason <code>configuration_error</code>. </p> <p><code>ConfDir</code> is a string which points to the directory where the configuration files are found. </p> <p>The configuration file read is: <code>usm.conf</code>. </p> </div> </div> <div class="function"> <h3 id="reconfigure/1">reconfigure(ConfDir) -&gt; void()</h3> <ul class="type"> <li><code>ConfDir = string()</code></li> </ul> <div class="description"> <p>Inserts all data in the configuration files into the database and destroys all old data, including the rows with StorageType <code>nonVolatile</code>. The rows created from the configuration file will have StorageType <code>nonVolatile</code>. </p> <p>Thus, the data in the SNMP-USER-BASED-SM-MIB, after this function has been called, is the data from the configuration files. </p> <p>All <code>snmp</code> counters are set to zero. </p> <p>If an error is found in the configuration file, it is reported using the function <code>config_err/2</code> of the error report module, and the function fails with the reason <code>configuration_error</code>. </p> <p><code>ConfDir</code> is a string which points to the directory where the configuration files are found. </p> <p>The configuration file read is: <code>usm.conf</code>. <span id="add_user"> </span> </p> </div> </div> <div class="function"> <h3 id="add_user/13">add_user(EngineID, Name, SecName, Clone, AuthP, AuthKeyC, OwnAuthKeyC, PrivP, PrivKeyC, OwnPrivKeyC, Public, AuthKey, PrivKey) -&gt; Ret</h3> <ul class="type"> <li><code>EngineID = string()</code></li> <li><code>Name = string()</code></li> <li><code>SecName = string()</code></li> <li><code>Clone = zeroDotZero | [integer()]</code></li> <li><code>AuthP = usmNoAuthProtocol | usmHMACMD5AuthProtocol | usmHMACSHAAuthProtocol</code></li> <li><code>AuthKeyC = string()</code></li> <li><code>OwnAuthKeyC = string()</code></li> <li><code>PrivP = usmNoPrivProtocol | usmDESPrivProtocol</code></li> <li><code>PrivKeyC = string()</code></li> <li><code>OwnPrivKeyC = string()</code></li> <li><code>Public = string()</code></li> <li><code>AuthKey = string()</code></li> <li><code>PrivKey = string()</code></li> <li><code>Ret = {ok, Key} | {error, Reason}</code></li> <li><code>Key = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Adds a USM security data (user) to the agent config. Equivalent to one line in the <code>usm.conf</code> file.</p> <span id="delete_user"> </span> </div> </div> <div class="function"> <h3 id="delete_user/1">delete_user(Key) -&gt; Ret</h3> <ul class="type"> <li><code>Key = term()</code></li> <li><code>Ret = ok | {error, Reason}</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Delete a USM security data (user) from the agent config.</p> </div> </div> </div>
