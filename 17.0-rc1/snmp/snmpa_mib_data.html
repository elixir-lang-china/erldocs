<h1>snmpa_mib_data</h1> <h2 class="modsummary">Behaviour module for the SNMP agent mib-server data module.</h2> <div class="description"> <p>This module defines the behaviour of the SNMP agent mib-server data module. A <code>snmpa_mib_data</code> compliant module must export the following functions: </p> <list type="bulleted"> <item> <a href="#new" class="seealso">new/1</a> </item> <item> <a href="#close" class="seealso">close/1</a> </item> <item> <a href="#sync" class="seealso">sync/1</a> </item> <item> <a href="#load_mib" class="seealso">load_mib/4</a> </item> <item> <a href="#unload_mib" class="seealso">unload_mib/4</a> </item> <item> <a href="#lookup" class="seealso">lookup/2</a> </item> <item> <a href="#next" class="seealso">next/3</a> </item> <item> <a href="#register_subagent" class="seealso">register_subagent/3</a> </item> <item> <a href="#unregister_subagent" class="seealso">unregister_subagent/2</a> </item> <item> <a href="#which_mib" class="seealso">which_mib/2</a> </item> <item> <a href="#which_mibs" class="seealso">which_mibs/1</a> </item> <item> <a href="#whereis_mib" class="seealso">whereis_mib/2</a> </item> <item> <a href="#dump" class="seealso">dump/2</a> </item> <item> <a href="#info" class="seealso">info/1</a> </item> <item> <a href="#backup" class="seealso">backup/2</a> </item> <item> <a href="#code_change" class="seealso">code_change/4</a> </item> </list> <p>The semantics of them and their exact signatures are explained below. </p> <p>Note that the data extracted from the imported (loaded) mibs are stored partly by the mib-server and partly by the symbolic-store server. See the default mib-server data module, <code>snmpa_mib_data_tttn</code> for details. </p> </div> <div class="section"> <h4>CALLBACK FUNCTIONS</h4> <p>The following functions must be exported from a <code>mib-server</code> data callback module: </p> <span id="new"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="new/1">Module:new(Storage) -&gt; State</h3> <ul class="type"> <li><code>Storage = mib_storage()</code></li> <li><code>State = term()</code></li> </ul> <div class="description"> <p>Create a new mib-server data instance. </p> <span id="close"> </span> </div> </div> <div class="function"> <h3 id="close/1">Module:close(State) -&gt; void()</h3> <ul class="type"> <li><code>State = term()</code></li> </ul> <div class="description"> <p>Close the mib-storage.</p> <span id="sync"> </span> </div> </div> <div class="function"> <h3 id="sync/1">Module:sync(State) -&gt; void()</h3> <ul class="type"> <li><code>State = term()</code></li> </ul> <div class="description"> <p>Synchronize (write to disc, if possible) the mib-server data. This depends on the <code>mib_storage</code> option, and will only have an effect if the mib-storage option has an actual disc component (such as dets, or ets with a file). </p> <span id="load_mib"> </span> </div> </div> <div class="function"> <h3 id="load_mib/4">Module:load_mib(State, Filename, MeOverride, TeOverride) -&gt; {ok, NewState} | {error, Reason}</h3> <ul class="type"> <li><code>State = NewState = term()</code></li> <li><code>Filename = filename()</code></li> <li><code>MeOverride = boolean()</code></li> <li><code>TeOverride = boolean()</code></li> <li><code>Reason = already_loaded | term()</code></li> </ul> <div class="description"> <p>Load the mib specified by the <code>Filename</code> argument into the mib-server. The <code>MeOverride</code> and <code>TeOverride</code> arguments specifies how the mib-server shall handle duplicate mib- and trap- entries. </p> <span id="unload_mib"> </span> </div> </div> <div class="function"> <h3 id="unload_mib/2">Module:unload_mib(State, Filename) -&gt; {ok, NewState} | {error, Reason}</h3> <ul class="type"> <li><code>State = NewState = term()</code></li> <li><code>Filename = filename()</code></li> <li><code>Reason = not_loaded | term()</code></li> </ul> <div class="description"> <p>Unload the mib specified by the <code>Filename</code> argument from the mib-server. </p> <span id="lookup"> </span> </div> </div> <div class="function"> <h3 id="lookup/2">Module:lookup(State, Oid) -&gt; Reply</h3> <ul class="type"> <li><code>State = term()</code></li> <li><code>Reply = {variable, ME} | {table_column, ME, TEOid} | {subagent, SAPid, SAOid} | {false, Reason}</code></li> <li><code>Oid = TEOid = SAOid = oid()</code></li> <li><code>SAPid = pid()</code></li> <li><code>ME = me()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Find the mib-entry corresponding to the <code>Oid</code>. If it is a variable, the <code>Oid</code> must be &lt;Oid for var&gt;.0 and if it is a table, <code>Oid</code> must be &lt;table&gt;.&lt;entry&gt;.&lt;col&gt;.&lt;any&gt;.</p> <span id="next"> </span> </div> </div> <div class="function"> <h3 id="next/3">Module:next(State, Oid, MibView) -&gt; Reply</h3> <ul class="type"> <li><code>State = term()</code></li> <li><code>Reply = false | endOfTable | {subagent, SAPid, SAOid} | {variable, ME, VarOid} | {table, TableOid, TableRestOid, ME}</code></li> <li><code>Oid = SAOid = VarOid = TableOid = TableRestOid = oid()</code></li> <li><code>SAPid = pid()</code></li> <li><code>ME = me()</code></li> </ul> <div class="description"> <p>Finds the lexicographically next oid. </p> <span id="register_subagent"> </span> </div> </div> <div class="function"> <h3 id="register_subagent/3">Module:register_subagent(State, Oid, Pid) -&gt; Reply</h3> <ul class="type"> <li><code>State = NewState = term()</code></li> <li><code>Reply = {ok, NewState} | {error, Reason}</code></li> <li><code>Oid = oid()</code></li> <li><code>Pid = pid()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Register the subagent, process, handling part of the mib-tree. </p> <span id="unregister_subagent"> </span> </div> </div> <div class="function"> <h3 id="unregister_subagent/2">Module:unregister_subagent(State, PidOrOid) -&gt; Reply</h3> <ul class="type"> <li><code>State = NewState = term()</code></li> <li><code>Reply = {ok, NewState} | {ok, NewState, Pid} | {error, Reason}</code></li> <li><code>PidOrOid = pid() | oid()</code></li> <li><code>Pid = pid()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Unregister the subagent, handling part of the mib-tree, as specified by the <code>oid()</code> or <code>pid()</code> (<code>PidOrOid</code>). </p> <p>When unregister the subagent using an <code>oid()</code>, the <code>pid()</code> of the process handling the sub-tree is also returned. </p> <span id="dump"> </span> </div> </div> <div class="function"> <h3 id="dump/2">Module:dump(State, Destination) -&gt; Reply</h3> <ul class="type"> <li><code>State = term()</code></li> <li><code>Reply = ok | {error, Reason}</code></li> <li><code>Destination = io | filename()</code></li> <li><code>Pid = pid()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Dump the mib-server data to <code>stdio</code> (Destination = <code>io</code>) or the specified file. </p> <span id="which_mib"> </span> </div> </div> <div class="function"> <h3 id="which_mib/2">Module:which_mib(State, Oid) -&gt; Reply</h3> <ul class="type"> <li><code>State = term()</code></li> <li><code>Reply = {ok, MibFile} | {error, Reason}</code></li> <li><code>Oid = oid()</code></li> <li><code>MibFile = string()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Retrieve the mib-file to which an given <code>oid()</code> belongs. </p> <span id="which_mibs"> </span> </div> </div> <div class="function"> <h3 id="which_mibs/1">Module:which_mibs(State) -&gt; Reply</h3> <ul class="type"> <li><code>State = term()</code></li> <li><code>Reply = [{MibName, Filename}]</code></li> <li><code>MibName = atom()</code></li> <li><code>Filename = string()</code></li> </ul> <div class="description"> <p>Retrieve all loaded mib-files. </p> <span id="whereis_mib"> </span> </div> </div> <div class="function"> <h3 id="whereis_mib/2">Module:whereis_mib(State, MibName) -&gt; Reply</h3> <ul class="type"> <li><code>State = term()</code></li> <li><code>MibName = atom()</code></li> <li><code>Reply = {ok, Filename} | {error, Reason}</code></li> <li><code>Filename = string()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Retrieve the mib file for the mib. </p> <span id="info"> </span> </div> </div> <div class="function"> <h3 id="info/1">Module:info(State) -&gt; Reply</h3> <ul class="type"> <li><code>State = term()</code></li> <li><code>Reply = {ok, Filename} | {error, Reason}</code></li> <li><code>Filename = string()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Retrieve misc info for the mib data. </p> <p>This is a utility function used to inspect, for instance, memory usage, in a simple way. </p> <span id="backup"> </span> </div> </div> <div class="function"> <h3 id="backup/2">Module:backup(State, BackupDir) -&gt; Reply</h3> <ul class="type"> <li><code>State = term()</code></li> <li><code>Reply = ok | {error, Reason}</code></li> <li><code>BackupDir = string()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Perform a backup of the mib-server data. </p> <p>Note that its implementation dependant (and also dependent on mib-storage is used) if a backup is possible. </p> <span id="code_change"> </span> </div> </div> <div class="function"> <h3 id="code_change/4">Module:code_change(Destination, Vsn, Extra, State) -&gt; NewState</h3> <ul class="type"> <li><code>Destination = up | down</code></li> <li><code>Vsn = term()</code></li> <li><code>Extra = term()</code></li> <li><code>State = NewState = term()</code></li> </ul> <div class="description"> <p>Perform a code-change (upgrade or downgrade). </p> <p>See <a href="../stdlib/gen_server.html" class="seealso">gen_server</a> for more info regarding the <code>Vsn</code> and <code>Extra</code> arguments. </p> </div> </div> </div>
