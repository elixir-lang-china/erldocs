<h1>snmp_pdus</h1> <h2 class="modsummary">Encode and Decode Functions for SNMP PDUs</h2> <div class="description"> <p>RFC1157, RFC1905 and/or RFC2272 should be studied carefully before using this module, <code>snmp_pdus</code>. </p> <p>The module <code>snmp_pdus</code> contains functions for encoding and decoding of SNMP protocol data units (PDUs). In short, this module converts a list of bytes to Erlang record representations and vice versa. The record definitions can be found in the file <code>snmp/include/snmp_types.hrl</code>. If snmpv3 is used, the module that includes <code>snmp_types.hrl</code> must define the constant <code>SNMP_USE_V3</code> before the header file is included. Example:</p> <pre class="sh_erlang"> -define(SNMP_USE_V3, true). -include_lib("snmp/include/snmp_types.hrl"). </pre> <p>Encoding and decoding must be done explicitly when writing your own Net if process. </p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="dec_message/1">dec_message([byte()]) -&gt; Message</h3> <ul class="type"> <li><code>Message = #message</code></li> </ul> <div class="description"> <p>Decodes a list of bytes into an SNMP Message. Note, if there is a v3 message, the <code>msgSecurityParameters</code> are not decoded. They must be explicitly decoded by a call to a security model specific decoding function, e.g. <code>dec_usm_security_parameters/1</code>. Also note, if the <code>scopedPDU</code> is encrypted, the OCTET STRING encoded <code>encryptedPDU</code> will be present in the <code>data</code> field.</p> </div> </div> <div class="function"> <h3 id="dec_message_only/1">dec_message_only([byte()]) -&gt; Message</h3> <ul class="type"> <li><code>Message = #message</code></li> </ul> <div class="description"> <p>Decodes a list of bytes into an SNMP Message, but does not decode the data part of the Message. That means, data is still a list of bytes, normally an encoded <code>PDU</code> (v1 and V2) or an encoded and possibly encrypted <code>scopedPDU</code> (v3).</p> </div> </div> <div class="function"> <h3 id="dec_pdu/1">dec_pdu([byte()]) -&gt; Pdu</h3> <ul class="type"> <li><code>Pdu = #pdu</code></li> </ul> <div class="description"> <p>Decodes a list of bytes into an SNMP Pdu.</p> </div> </div> <div class="function"> <h3 id="dec_scoped_pdu/1">dec_scoped_pdu([byte()]) -&gt; ScopedPdu</h3> <ul class="type"> <li><code>ScopedPdu = #scoped_pdu</code></li> </ul> <div class="description"> <p>Decodes a list of bytes into an SNMP ScopedPdu.</p> </div> </div> <div class="function"> <h3 id="dec_scoped_pdu_data/1">dec_scoped_pdu_data([byte()]) -&gt; ScopedPduData</h3> <ul class="type"> <li><code>ScopedPduData = #scoped_pdu | EncryptedPDU</code></li> <li><code>EncryptedPDU = [byte()]</code></li> </ul> <div class="description"> <p>Decodes a list of bytes into either a scoped pdu record, or - if the scoped pdu was encrypted - to a list of bytes.</p> </div> </div> <div class="function"> <h3 id="dec_usm_security_parameters/1">dec_usm_security_parameters([byte()]) -&gt; UsmSecParams</h3> <ul class="type"> <li><code>UsmSecParams = #usmSecurityParameters</code></li> </ul> <div class="description"> <p>Decodes a list of bytes into an SNMP UsmSecurityParameters</p> </div> </div> <div class="function"> <h3 id="enc_encrypted_scoped_pdu/1">enc_encrypted_scoped_pdu(EncryptedScopedPdu) -&gt; [byte()]</h3> <ul class="type"> <li><code>EncryptedScopedPdu = [byte()]</code></li> </ul> <div class="description"> <p>Encodes an encrypted SNMP ScopedPdu into an OCTET STRING that can be used as the <code>data</code> field in a <code>message</code> record, that later can be encoded with a call to <code>enc_message_only/1</code>. </p> <p>This function should be used whenever the <code>ScopedPDU</code> is encrypted.</p> </div> </div> <div class="function"> <h3 id="enc_message/1">enc_message(Message) -&gt; [byte()]</h3> <ul class="type"> <li><code>Message = #message</code></li> </ul> <div class="description"> <p>Encodes a message record to a list of bytes.</p> </div> </div> <div class="function"> <h3 id="enc_message_only/1">enc_message_only(Message) -&gt; [byte()]</h3> <ul class="type"> <li><code>Message = #message</code></li> </ul> <div class="description"> <p><code>Message</code> is a record where the <code>data</code> field is assumed to be encoded (a list of bytes). If there is a v1 or v2 message, the <code>data</code> field is an encoded <code>PDU</code>, and if there is a v3 message, <code>data</code> is an encoded and possibly encrypted <code>scopedPDU</code>.</p> </div> </div> <div class="function"> <h3 id="enc_pdu/1">enc_pdu(Pd) -&gt; [byte()]</h3> <ul class="type"> <li><code>Pdu = #pdu</code></li> </ul> <div class="description"> <p>Encodes an SNMP Pdu into a list of bytes.</p> </div> </div> <div class="function"> <h3 id="enc_scoped_pdu/1">enc_scoped_pdu(ScopedPdu) -&gt; [byte()]</h3> <ul class="type"> <li><code>ScopedPdu = #scoped_pdu</code></li> </ul> <div class="description"> <p>Encodes an SNMP ScopedPdu into a list of bytes, which can be encrypted, and after encryption, encoded with a call to <code>enc_encrypted_scoped_pdu/1</code>; or it can be used as the <code>data</code> field in a <code>message</code> record, which then can be encoded with <code>enc_message_only/1</code>.</p> </div> </div> <div class="function"> <h3 id="enc_usm_security_parameters/1">enc_usm_security_parameters(UsmSecParams) -&gt; [byte()]</h3> <ul class="type"> <li><code>UsmSecParams = #usmSecurityParameters</code></li> </ul> <div class="description"> <p>Encodes SNMP UsmSecurityParameters into a list of bytes.</p> </div> </div> </div>
