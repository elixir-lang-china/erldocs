<h1>snmpa_local_db</h1> <h2 class="modsummary">The SNMP built-in database</h2> <div class="description"> <p>The module <code>snmpa_local_db</code> contains functions for implementing tables (and variables) using the SNMP built-in database. The database exists in two instances, one volatile and one persistent. The volatile database is implemented with ets. The persistent database is implemented with dets. </p> <p>There is a scaling problem with this database. </p> <list type="bulleted"> <item>Insertions and deletions are inefficient for large tables. </item> </list> <p>This problem is best solved by using Mnesia instead. </p> <p>The following functions describe the interface to <code>snmpa_local_db</code>. Each function has a Mnesia equivalent. The argument <code>NameDb</code> is a tuple <code>{Name, Db}</code> where <code>Name</code> is the symbolic name of the managed object (as defined in the MIB), and <code>Db</code> is either <code>volatile</code> or <code>persistent</code>. <code>mnesia</code> is not possible since all these functions are <code>snmpa_local_db</code> specific. </p> </div> <div class="section"> <h4>Common Data Types</h4> <p>In the functions defined below, the following types are used: </p> <list type="bulleted"> <item> <p><code>NameDb = {Name, Db}</code></p> </item> <item> <p><code>Name = atom(), Db = volatile | persistent</code></p> </item> <item> <p><code>RowIndex = [int()]</code></p> </item> <item> <p><code>Cols = [Col] | [{Col, Value}], Col = int(), Value = term()</code></p> </item> </list> <p>where <code>RowIndex</code> denotes the last part of the OID, that specifies the index of the row in the table. <code>Cols</code> is a list of column numbers in case of a get operation, and a list of column numbers and values in case of a set operation. </p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="dump/0">dump() -&gt; ok | {error, Reason}</h3> <ul class="type"> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>This function can be used to manually dump the database to file.</p> </div> </div> <div class="function"> <h3 id="match/2">match(NameDb, Pattern)</h3> <div class="description"> <p>Performs an ets/dets matching on the table. See Stdlib documentation, module ets, for a description of <code>Pattern</code> and the return values.</p> </div> </div> <div class="function"> <h3 id="print/0">print()</h3> <h3 id="print/1">print(TableName)</h3> <h3 id="print/2">print(TableName, Db)</h3> <ul class="type"> <li><code>TableName = atom()</code></li> </ul> <div class="description"> <p>Prints the contents of the database on screen. This is useful for debugging since the <code>STANDARD-MIB</code> and <code>OTP-SNMPEA-MIB</code> (and maybe your own MIBs) are stored in <code>snmpa_local_db</code>. </p> <p><code>TableName</code> is an atom for a table in the database. When no name is supplied, the whole database is shown.</p> </div> </div> <div class="function"> <h3 id="table_create/1">table_create(NameDb) -&gt; bool()</h3> <div class="description"> <p>Creates a table.
