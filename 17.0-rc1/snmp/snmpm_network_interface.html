<h1>snmpm_network_interface</h1> <h2 class="modsummary">Behaviour module for the SNMP manager network interface.</h2> <div class="description"> <p>This module defines the behaviour of the manager network interface. A <code>snmpm_network_interface</code> compliant module must export the following functions: </p> <list type="bulleted"> <item> <p><a href="#start_link" class="seealso">start_link/2</a></p> </item> <item> <p><a href="#stop" class="seealso">stop/1</a></p> </item> <item> <p><a href="#send_pdu" class="seealso">send_pdu/7</a></p> </item> <item> <p><a href="#inform_response" class="seealso">inform_response/4</a></p> </item> <item> <p><a href="#note_store" class="seealso">note_store/2</a></p> </item> <item> <p><a href="#info" class="seealso">info/1</a></p> </item> <item> <p><a href="#get_log_type" class="seealso">get_log_type/1</a></p> </item> <item> <p><a href="#set_log_type" class="seealso">set_log_type/2</a></p> </item> <item> <p><a href="#verbosity" class="seealso">verbosity/2</a></p> </item> </list> <p>The semantics of them and their exact signatures are explained below. </p> <span id="start_link"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="start_link/2">start_link(Server, NoteStore) -&gt; {ok, Pid} | {error, Reason}</h3> <ul class="type"> <li><code>Server = pid()</code></li> <li><code>NoteStore = pid()</code></li> </ul> <div class="description"> <p>Start-link the network interface process.</p> <p><code>Server</code> is the pid of the managing process.</p> <p><code>NoteStore</code> is the pid of the note-store process.</p> <span id="stop"> </span> </div> </div> <div class="function"> <h3 id="stop/1">stop(Pid) -&gt; void()</h3> <ul class="type"> <li><code>Pid = pid()</code></li> </ul> <div class="description"> <p>Stop the network interface process.</p> <span id="send_pdu"> </span> </div> </div> <div class="function"> <h3 id="send_pdu/7">send_pdu(Pid, Pdu, Vsn, MsgData, Addr, Port, ExtraInfo) -&gt; void()</h3> <ul class="type"> <li><code>Pid = pid()</code></li> <li><code>Pdu = pdu()</code></li> <li><code>Vsn = 'version-1' | 'version-2' | 'version-3'</code></li> <li><code>MsgData = term()</code></li> <li><code>Addr = address()</code></li> <li><code>Port = integer()</code></li> <li><code>ExtraInfo = term()</code></li> </ul> <div class="description"> <p>Request the network interface process (<code>Pid</code>) to send this pdu (<code>Pdu</code>).</p> <p><code>ExtraInfo</code> is some opaque data that is passed to the net-if process. It originates from the <code>ExtraInfo</code> parameter in the calls to the <a href="snmpm.html#sync_get" class="seealso">synchronous get-request</a>, <a href="snmpm.html#async_get" class="seealso">asynchronous get-request</a>, <a href="snmpm.html#sync_get_next" class="seealso">synchronous get-next-request</a>, <a href="snmpm.html#async_get_next" class="seealso">asynchronous get-next-request</a>, <a href="snmpm.html#sync_set" class="seealso">synchronous set-request</a> and <a href="snmpm.html#async_set" class="seealso">asynchronous set-request</a> functions. Whether the net-if process chooses to use this is implementation dependent. The net-if process included in this application ignores it. </p> <span id="inform_response"> </span> </div> </div> <div class="function"> <h3 id="inform_response/4">inform_response(Pid, Ref, Addr, Port) -&gt; void()</h3> <ul class="type"> <li><code>Pid = pid()</code></li> <li><code>Ref = term()</code></li> <li><code>Addr = address()</code></li> <li><code>Port = integer()</code></li> </ul> <div class="description"> <p>Instruct the network interface process to send the response (acknowledgment) to an inform-request.</p> <p><code>Ref</code> is something that can be used to identify the inform-request, e.g. request-id of the inform-request.</p> <p><code>Addr</code> and <code>Port</code> identifies the agent, from which the inform-request originated.</p> <span id="note_store"> </span> </div> </div> <div class="function"> <h3 id="note_store/2">note_store(Pid, NoteStore) -&gt; void()</h3> <ul class="type"> <li><code>Pid = pid()</code></li> <li><code>NoteStore = pid()</code></li> </ul> <div class="description"> <p>Change the pid of the note-store process. This is used when the server re-starts the note_store (e.g. after a crach).</p> <span id="info"> </span> </div> </div> <div class="function"> <h3 id="info/1">info(Pid) -&gt; [{Key, Value}]</h3> <ul class="type"> <li><code>Pid = pid()</code></li> </ul> <div class="description"> <p>The info returned is basically up to the implementer to decide. The implementation provided by this application provides info about memory allocation and various socket information.</p> <p>The info returned by this function is returned together with other info collected by the manager when the <a href="snmpm.html#info" class="seealso">info</a> function is called (tagged with the key <code>net_if</code>).</p> <span id="verbosity"> </span> </div> </div> <div class="function"> <h3 id="verbosity/2">verbosity(Pid, Verbosity) -&gt; void()</h3> <ul class="type"> <li><code>Pid = pid()</code></li> <li><code>Verbosity = verbosity()</code></li> </ul> <div class="description"> <p>Change the verbosity of the network interface process.</p> <span id="get_log_type"> </span> </div> </div> <div class="function"> <h3 id="get_log_type/1">get_log_type(Pid) -&gt; {ok, LogType} | {error, Reason}</h3> <ul class="type"> <li><code>Pid = pid()</code></li> <li><code>LogType = atl_type()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>The Audit Trail Log is managed by the network interface process. So, it is this process that has to return the actual log-type. </p> <span id="set_log_type"> </span> </div> </div> <div class="function"> <h3 id="set_log_type/2">set_log_type(Pid, NewType) -&gt; {ok, OldType} | {error, Reason}</h3> <ul class="type"> <li><code>Pid = pid()</code></li> <li><code>NewType = OldType = atl_type()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>The Audit Trail Log is managed by the network interface process. So, it is this process that has to do the actual changing of the type. </p> <p>See <a href="snmpm.html#set_log_type" class="seealso">set_log_type</a> for more info. </p> </div> </div> </div>
