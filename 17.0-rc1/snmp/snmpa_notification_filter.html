<h1>snmpa_notification_filter</h1> <h2 class="modsummary">Behaviour module for the SNMP agent notification filters.</h2> <div class="description"> <p>This module defines the behaviour of the agent notification filters. A <code>snmpa_notification_filter</code> compliant module must export the following functions: </p> <list type="bulleted"> <item> <p>handle_notification/2</p> </item> </list> <p>The semantics of them and their exact signatures are explained below. </p> <p>The purpose of notification filters is to allow for modification and/or suppression of a notification.</p> <p>A misbehaving filter will be removed.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="handle_notification/2">handle_notification(Notif, Data) -&gt; Reply</h3> <ul class="type"> <li><code>Reply = send | {send, NewNotif} | dont_send</code></li> <li><code>Notif = NewNotif = notification() | trap()</code></li> <li><code>Data = term()</code></li> </ul> <div class="description"> <p>Handle a notification to be sent. The filter can either accept the notification as is, return <code>send</code>, modify the notification, return <code>{send, NewNotif}</code> or suppress the notification, return <code>dont_send</code>.</p> <p><code>Data</code> is supplied at filter registration time, see <a href="snmpa.html#register_notification_filter" class="seealso">register_notification_filter</a>. </p> </div> </div> </div>
