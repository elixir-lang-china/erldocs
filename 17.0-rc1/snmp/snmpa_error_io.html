<h1>snmpa_error_io</h1> <h2 class="modsummary">Functions for Reporting SNMP Errors on stdio</h2> <div class="description"> <p>The module <code>snmpa_error_io</code> implements the <code>snmp_error_report</code> behaviour (see <a href="snmpa_error_report.html" class="seealso">snmpa_error_report</a>) containing two callback functions which are called in order to report SNMP errors. </p> <p>This module provides a simple mechanism for reporting SNMP errors. Errors are written to stdout using the <code>io</code> module. It is provided as an simple example. </p> <p>This module needs to be explicitly configured, see <a href="snmpa_error.html#desc" class="seealso">snmpa_error</a> and <a href="snmp_config.html#configuration_params" class="seealso">configuration parameters</a>. </p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="config_err/2">config_err(Format, Args) -&gt; void()</h3> <ul class="type"> <li><code>Format = string()</code></li> <li><code>Args = list()</code></li> </ul> <div class="description"> <p>The function is called if an error occurs during the configuration phase, for example if a syntax error is found in a configuration file. </p> <p><code>Format</code> and <code>Args</code> are as in <code>io:format(Format, Args)</code>.</p> </div> </div> <div class="function"> <h3 id="user_err/2">user_err(Format, Args) -&gt; void()</h3> <ul class="type"> <li><code>Format = string()</code></li> <li><code>Args = list()</code></li> </ul> <div class="description"> <p>The function is called if a user related error occurs at run-time, for example if a user defined instrumentation function returns erroneous. </p> <p><code>Format</code> and <code>Args</code> are as in <code>io:format(Format, Args)</code>.</p> </div> </div> </div>
