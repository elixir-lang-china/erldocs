<h1>snmpa_mpd</h1> <h2 class="modsummary">Message Processing and Dispatch module for the SNMP agent</h2> <div class="description"> <p>The module <code>snmpa_mpd</code> implements the version independent Message Processing and Dispatch functionality in SNMP for the agent. It is supposed to be used from a Network Interface process (<a href="snmp_agent_netif.html" class="seealso">Definition of Agent Net if</a>). </p> <span id="init"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="init/1">init(Vsns) -&gt; mpd_state()</h3> <ul class="type"> <li><code>Vsns = [Vsn]</code></li> <li><code>Vsn = v1 | v2 | v3</code></li> </ul> <div class="description"> <p>This function can be called from the net_if process at start-up. The options list defines which versions to use. </p> <p>It also initializes some SNMP counters. </p> <span id="process_packet"> </span> </div> </div> <div class="function"> <h3 id="process_packet/6">process_packet(Packet, TDomain, TAddress, State, NoteStore, Log) -&gt; {ok, Vsn, Pdu, PduMS, ACMData} | {discarded, Reason} | {discovery, DiscoPacket}</h3> <h3 id="process_packet/7">process_packet(Packet, TDomain, TAddress, LocalEngineID, State, NoteStore, Log) -&gt; {ok, Vsn, Pdu, PduMS, ACMData} | {discarded, Reason} | {discovery, DiscoPacket}</h3> <ul class="type"> <li><code>Packet = binary()</code></li> <li><code>TDomain = snmpUDPDomain</code></li> <li><code>TAddress = {Ip, Udp}</code></li> <li><code>LocalEngineID = string()</code></li> <li><code>Ip = {integer(), integer(), integer(), integer()}</code></li> <li><code>Udp = integer()</code></li> <li><code>State = mpd_state()</code></li> <li><code>NoteStore = pid()</code></li> <li><code>Log = snmp_log()</code></li> <li><code>Vsn = 'version-1' | 'version-2' | 'version-3'</code></li> <li><code>Pdu = #pdu</code></li> <li><code>PduMs = integer()</code></li> <li><code>ACMData = acm_data()</code></li> <li><code>Reason = term()</code></li> <li><code>DiscoPacket = binary()</code></li> </ul> <div class="description"> <p>Processes an incoming packet. Performs authentication and decryption as necessary. The return values should be passed the agent.</p> <div class="note"><h2>Note!</h2> <p>Note that the use of the LocalEngineID argument is only intended for special cases, if the agent is to "emulate" multiple EngineIDs! By default, the agent uses the value of <code>SnmpEngineID</code> (see SNMP-FRAMEWORK-MIB). </p> </div> <span id="generate_response_msg"> </span> </div> </div> <div class="function"> <h3 id="generate_response_msg/5">generate_response_msg(Vsn, RePdu, Type, ACMData, Log) -&gt; {ok, Packet} | {discarded, Reason}</h3> <h3 id="generate_response_msg/6">generate_response_msg(Vsn, RePdu, Type, ACMData, LocalEngineID, Log) -&gt; {ok, Packet} | {discarded, Reason}</h3> <ul class="type"> <li><code>Vsn = 'version-1' | 'version-2' | 'version-3'</code></li> <li><code>RePdu = #pdu</code></li> <li><code>Type = atom()</code></li> <li><code>ACMData = acm_data()</code></li> <li><code>LocalEngineID = string()</code></li> <li><code>Packet = binary()</code></li> </ul> <div class="description"> <p>Generates a possibly encrypted response packet to be sent to the network. <code>Type</code> is the <code>#pdu.type</code> of the original request.</p> <div class="note"><h2>Note!</h2> <p>Note that the use of the LocalEngineID argument is only intended for special cases, if the agent is to "emulate" multiple EngineIDs! By default, the agent uses the value of <code>SnmpEngineID</code> (see SNMP-FRAMEWORK-MIB). </p> </div> <span id="generate_msg"> </span> </div> </div> <div class="function"> <h3 id="generate_msg/5">generate_msg(Vsn, NoteStore, Pdu, MsgData, To) -&gt; {ok, PacketsAndAddresses} | {discarded, Reason}</h3> <h3 id="generate_msg/6">generate_msg(Vsn, NoteStore, Pdu, MsgData, LocalEngineID, To) -&gt; {ok, PacketsAndAddresses} | {discarded, Reason}</h3> <ul class="type"> <li><code>Vsn = 'version-1' | 'version-2' | 'version-3'</code></li> <li><code>NoteStore = pid()</code></li> <li><code>Pdu = #pdu</code></li> <li><code>MsgData = msg_data()</code></li> <li><code>LocalEngineID = string()</code></li> <li><code>To = [dest_addrs()]</code></li> <li><code>PacketsAndAddresses = [{TDomain, TAddress, Packet}]</code></li> <li><code>TDomain = snmpUDPDomain</code></li> <li><code>TAddress = {Ip, Udp}</code></li> <li><code>Ip = {integer(), integer(), integer(), integer()}</code></li> <li><code>Udp = integer()</code></li> <li><code>Packet = binary()</code></li> </ul> <div class="description"> <p>Generates a possibly encrypted request packet to be sent to the network. </p> <p><code>MsgData</code> is the message specific data used in the SNMP message. In SNMPv3, it is the context information. <code>To</code> is a list of the destination addresses and their corresponding security parameters. This value is also received from the requests mentioned above. </p> <div class="note"><h2>Note!</h2> <p>Note that the use of the LocalEngineID argument is only intended for special cases, if the agent is to "emulate" multiple EngineIDs! This function can be used when the net_if process receives a <code>discarded_pdu</code> message from the agent. </p> </div> </div> </div>
