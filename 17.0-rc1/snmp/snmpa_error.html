<h1>snmpa_error</h1> <h2 class="modsummary">Functions for Reporting SNMP Errors</h2> <div class="description"> <span id="desc"> </span> <p>The module <code>snmpa_error</code> contains two callback functions which are called if an error occurs at different times during agent operation. These functions in turn calls the corresponding function in the configured error report module, which implements the actual report functionality. </p> <p>Two simple implementation(s) is provided with the toolkit; the modules <a href="snmpa_error_logger.html" class="seealso">snmpa_error_logger</a> which is the default and <a href="snmpa_error_io.html" class="seealso">snmpa_error_io</a>. </p> <p>The error report module is configured using the directive <code>error_report_mod</code>, see <a href="snmp_config.html#configuration_params" class="seealso">configuration parameters</a>. </p> <span id="config_err"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="config_err/2">config_err(Format, Args) -&gt; void()</h3> <ul class="type"> <li><code>Format = string()</code></li> <li><code>Args = list()</code></li> </ul> <div class="description"> <p>The function is called if an error occurs during the configuration phase, for example if a syntax error is found in a configuration file. </p> <p><code>Format</code> and <code>Args</code> are as in <code>io:format(Format, Args)</code>.</p> <span id="user_err"> </span> </div> </div> <div class="function"> <h3 id="user_err/2">user_err(Format, Args) -&gt; void()</h3> <ul class="type"> <li><code>Format = string()</code></li> <li><code>Args = list()</code></li> </ul> <div class="description"> <p>The function is called if a user related error occurs at run-time, for example if a user defined instrumentation function returns erroneous. </p> <p><code>Format</code> and <code>Args</code> are as in <code>io:format(Format, Args)</code>.</p> </div> </div> </div>
