<h1>snmp</h1> <h2 class="modsummary">Interface functions to the SNMP toolkit</h2> <div class="description"> <p>The module <code>snmp</code> contains interface functions to the SNMP toolkit.</p> </div> <div class="section"> <h4>Common Data Types</h4> <p>The following data-types are used in the functions below: </p> <list type="bulleted"> <item> <p><code>datetime() = {date(), time()}</code></p> <p>See <a href="../stdlib/calendar.html" class="seealso">calendar</a> for more info.</p> </item> </list> <span id="config"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="config/0">config() -&gt; ok | {error, Reason}</h3> <div class="description"> <p>A simple interactive configuration tool. Simple configuration files can be generated, but more complex configurations still have to be edited manually. </p> <p>The tool is a textual based tool that asks some questions and generates <code>sys.config</code> and <code>*.conf</code> files. </p> <p><em>Note</em> that if the application shall support version 3, then the crypto app must be started before running this function (password generation).</p> <p><em>Note</em> also that some of the configuration files for the agent and manager share the same names. This means that they have to be stored in <em>different</em> directories!</p> <span id="start"> </span> </div> </div> <div class="function"> <h3 id="start/0">start() -&gt; ok | {error, Reason}</h3> <h3 id="start/1">start(Type) -&gt; ok | {error, Reason}</h3> <ul class="type"> <li><code>Type = start_type()</code></li> </ul> <div class="description"> <p>Starts the SNMP application.</p> <p>See <a href="../kernel/application.html" class="seealso">application</a> for more info.</p> <span id="start_agent"> </span> </div> </div> <div class="function"> <h3 id="start_agent/0">start_agent() -&gt; ok | {error, Reason}</h3> <h3 id="start_agent/1">start_agent(Type) -&gt; ok | {error, Reason}</h3> <ul class="type"> <li><code>Type = start_type()</code></li> </ul> <div class="description"> <p>The SNMP application consists of several entities, of which the agent is one. {ok, mib()} | {error, Reason}</h3> <ul class="type"> <li><code>FileName = string()</code></li> <li><code>mib() = #mib{}</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Read a compiled mib.</p> <span id="log_to_txt"> </span> </div> </div> <div class="function"> <h3 id="log_to_txt/5">log_to_txt(LogDir, Mibs, OutFile, LogName, LogFile) -&gt; ok | {error, Reason}</h3> <h3 id="log_to_txt/6">log_to_txt(LogDir, Mibs, OutFile, LogName, LogFile, Block | Start) -&gt; ok | {error, Reason}</h3> <h3 id="log_to_txt/7">log_to_txt(LogDir, Mibs, OutFile, LogName, LogFile, Start, Block | Stop) -&gt; ok | {error, Reason}</h3> <h3 id="log_to_txt/8">log_to_txt(LogDir, Mibs, OutFile, LogName, LogFile, Start, Stop, Block) -&gt; ok | {error, Reason}</h3> <ul class="type"> <li><code>LogDir = string()</code></li> <li><code>Mibs = [MibName]</code></li> <li><code>OutFile = string()</code></li> <li><code>MibName = string()</code></li> <li><code>LogName = string()</code></li> <li><code>LogFile = string()</code></li> <li><code>Start = Stop = null | datetime() | {local_time,datetime()} | {universal_time,datetime()} </code></li> <li><code>Block = boolean()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Converts an Audit Trail Log to a readable text file, where each item has a trailing TAB character, and any TAB character in the body of an item has been replaced by ESC TAB. </p> <p>The function can be used on a running system, or by copying the entire log directory and calling this function. SNMP must be running in order to provide MIB information. </p> <p><code>LogDir</code> is the name of the directory where the audit trail log is stored. <code>Mibs</code> is a list of Mibs to be used. The function uses the information in the Mibs to convert for example object identifiers to their symbolic name. <code>OutFile</code> is the name of the generated text-file. <code>LogName</code> is the name of the log, <code>LogFile</code> is the name of the log file. <code>Start</code> is the start (first) date and time from which log events will be converted and <code>Stop</code> is the stop (last) date and time to which log events will be converted. The <code>Block</code> argument indicates if the log should be blocked during conversion. This could be usefull when converting large logs (when otherwise the log could wrap during conversion). Defaults to <code>true</code>. </p> <p>The format of an audit trail log text item is as follows: </p> <p><code>Tag Addr - Community [TimeStamp] Vsn</code><br /> <code>PDU</code></p> <p>where <code>Tag</code> is <code>request</code>, <code>response</code>, <code>report</code>, <code>trap</code> or <code>inform</code>; Addr is <code>IP:Port</code> (or comma space separated list of such); <code>Community</code> is the community parameter (SNMP version v1 and v2), or <code>SecLevel:"AuthEngineID":"UserName"</code> (SNMP v3); <code>TimeStamp</code> is a date and time stamp, and <code>Vsn</code> is the SNMP version. <code>PDU</code> is a textual version of the protocol data unit. There is a new line between <code>Vsn</code> and <code>PDU</code>.</p> <span id="log_to_io"> </span> </div> </div> <div class="function"> <h3 id="log_to_io/4">log_to_io(LogDir, Mibs, LogName, LogFile) -&gt; ok | {error, Reason}</h3> <h3 id="log_to_io/5">log_to_io(LogDir, Mibs, LogName, LogFile, Block | Start) -&gt; ok | {error, Reason}</h3> <h3 id="log_to_io/6">log_to_io(LogDir, Mibs, LogName, LogFile, Start, Block | Stop) -&gt; ok | {error, Reason}</h3> <h3 id="log_to_io/7">log_to_io(LogDir, Mibs, LogName, LogFile, Start, Stop, Block) -&gt; ok | {error, Reason}</h3> <ul class="type"> <li><code>LogDir = string()</code></li> <li><code>Mibs = [MibName]</code></li> <li><code>MibName = string()</code></li> <li><code>LogName = string()</code></li> <li><code>LogFile = string()</code></li> <li><code>Start = Stop = null | datetime() | {local_time,datetime()} | {universal_time,datetime()} </code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Converts an Audit Trail Log to a readable format and prints it on stdio.
