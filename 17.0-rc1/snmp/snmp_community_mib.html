<h1>snmp_community_mib</h1> <h2 class="modsummary">Instrumentation Functions for SNMP-COMMUNITY-MIB</h2> <div class="description"> <p>The module <code>snmp_community_mib</code> implements the instrumentation functions for the SNMP-COMMUNITY-MIB, and functions for configuring the database. </p> <p>The configuration files are described in the SNMP User's Manual.</p> <span id="configure"> </span> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="configure/1">configure(ConfDir) -&gt; void()</h3> <ul class="type"> <li><code>ConfDir = string()</code></li> </ul> <div class="description"> <p>This function is called from the supervisor at system start-up. </p> <p>Inserts all data in the configuration files into the database and destroys all old rows with StorageType <code>volatile</code>. The rows created from the configuration file will have StorageType <code>nonVolatile</code>. </p> <p>All <code>snmp</code> counters are set to zero. </p> <p>If an error is found in the configuration file, it is reported using the function <code>config_err/2</code> of the error, report module and the function fails with reason <code>configuration_error</code>. </p> <p><code>ConfDir</code> is a string which points to the directory where the configuration files are found. </p> <p>The configuration file read is: <code>community.conf</code>. </p> <span id="reconfigure"> </span> </div> </div> <div class="function"> <h3 id="reconfigure/1">reconfigure(ConfDir) -&gt; void()</h3> <ul class="type"> <li><code>ConfDir = string()</code></li> </ul> <div class="description"> <p>Inserts all data in the configuration files into the database and destroys all old data, including the rows with StorageType <code>nonVolatile</code>. The rows created from the configuration file will have StorageType <code>nonVolatile</code>. </p> <p>Thus, the data in the SNMP-COMMUNITY-MIB, after this function has been called, is from the configuration files. </p> <p>All <code>snmp</code> counters are set to zero. </p> <p>If an error is found in the configuration file, it is reported using the function <code>config_err/2</code> of the error report module, and the function fails with reason <code>configuration_error</code>. </p> <p><code>ConfDir</code> is a string which points to the directory where the configuration files are found. </p> <p>The configuration file read is: <code>community.conf</code>.</p> <span id="add_community"> </span> </div> </div> <div class="function"> <h3 id="add_community/5">add_community(Idx, CommName, SecName, CtxName, TransportTag) -&gt; Ret</h3> <h3 id="add_community/6">add_community(Idx, CommName, SecName, EngineId, CtxName, TransportTag) -&gt; Ret</h3> <ul class="type"> <li><code>Idx = string()</code></li> <li><code>CommName = string()</code></li> <li><code>SecName = string()</code></li> <li><code>EngineId = string()</code></li> <li><code>CtxName = string()</code></li> <li><code>TransportTag = string()</code></li> <li><code>Ret = {ok, Key} | {error, Reason}</code></li> <li><code>Key = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Adds a community to the agent config. Equivalent to one line in the <code>community.conf</code> file.</p> <p>With the <code>EngineId</code> argument it is possible to override the configured engine-id (SNMP-FRAMEWORK-MIB).</p> <span id="delete_community"> </span> </div> </div> <div class="function"> <h3 id="delete_community/1">delete_community(Key) -&gt; Ret</h3> <ul class="type"> <li><code>Key = term()</code></li> <li><code>Ret = ok | {error, Reason}</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>Delete a community from the agent config.</p> </div> </div> </div>
