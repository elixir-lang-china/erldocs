<h1>snmpa_error_report</h1> <h2 class="modsummary">Behaviour module for reporting SNMP agent errors</h2> <div class="description"> <span id="desc"> </span> <p>This module defines the behaviour of the agent error reporting. A <code>snmpa_error_report</code> compliant module must export the following functions: </p> <list type="bulleted"> <item> <p>config_err/2</p> </item> <item> <p>user_err/2</p> </item> </list> <p>The semantics of them and their exact signatures are explained below. </p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="config_err/2">config_err(Format, Args) -&gt; void()</h3> <ul class="type"> <li><code>Format = string()</code></li> <li><code>Args = list()</code></li> </ul> <div class="description"> <p>The function is called if an error occurs during the configuration phase, for example if a syntax error is found in a configuration file. </p> <p><code>Format</code> and <code>Args</code> are as in <code>io:format(Format, Args)</code>.</p> </div> </div> <div class="function"> <h3 id="user_err/2">user_err(Format, Args) -&gt; void()</h3> <ul class="type"> <li><code>Format = string()</code></li> <li><code>Args = list()</code></li> </ul> <div class="description"> <p>The function is called if a user related error occurs at run-time, for example if a user defined instrumentation function returns erroneous. </p> <p><code>Format</code> and <code>Args</code> are as in <code>io:format(Format, Args)</code>.</p> </div> </div> </div>
