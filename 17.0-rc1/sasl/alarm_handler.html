<h1>alarm_handler</h1> <h2 class="modsummary">An Alarm Handling Process</h2> <div class="description"> <p>The alarm handler process is a <code>gen_event</code> event manager process which receives alarms in the system. This process is not intended to be a complete alarm handler. It defines a place to which alarms can be sent. One simple event handler is installed in the alarm handler at start-up, but users are encouraged to write and install their own handlers. </p> <p>The simple event handler sends all alarms as info reports to the error logger, and saves all of them in a list which can be passed to a user defined event handler, which may be installed at a later stage. The list can grow large if many alarms are generated. So it is a good reason to install a better user defined handler. </p> <p>There are functions to set and clear alarms. The format of alarms are defined by the user. For example, an event handler for SNMP could be defined, together with an alarm MIB. </p> <p>The alarm handler is part of the SASL application. </p> <p>When writing new event handlers for the alarm handler, the following events must be handled: </p> <taglist> <dt><code>{set_alarm, {AlarmId, AlarmDescr}}</code></dt> <item> <p>This event is generated by <code>alarm_handler:set_alarm({AlarmId, AlarmDecsr})</code>. </p> </item> <dt><code>{clear_alarm, AlarmId}</code></dt> <item> <p>This event is generated by <code>alarm_handler:clear_alarm(AlarmId)</code>. </p> </item> </taglist> <p>The default simple handler is called <code>alarm_handler</code> and it may be exchanged by calling <code>gen_event:swap_handler/3</code> as <code>gen_event:swap_handler(alarm_handler, {alarm_handler, swap}, {NewHandler, Args})</code>. <code>NewHandler:init({Args, {alarm_handler, Alarms}})</code> is called. Refer to gen_event(3) for further details. </p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="clear_alarm/1">clear_alarm(AlarmId) -&gt; void()</h3> <ul class="type"> <li><code>AlarmId = term()</code></li> </ul> <div class="description"> <p>Clears all alarms with id <code>AlarmId</code>. </p> </div> </div> <div class="function"> <h3 id="get_alarms/0">get_alarms() -&gt; [alarm()]</h3> <div class="description"> <p>Returns a list of all active alarms.
