<h1>rb</h1> <h2 class="modsummary">The Report Browser Tool</h2> <div class="description"> <p>The Report Browser (RB) tool makes it possible to browse and format error reports written by the error logger handler <code>log_mf_h</code>. </p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="filter/1">filter(Filters)</h3> <h3 id="filter/2">filter(Filters, Dates)</h3> <ul class="type"> <li><code>Filters = [filter()]</code></li> <li><code>filter() = {Key, Value} | {Key, Value, no} | {Key, RegExp, re} | {Key, RegExp, re, no}</code></li> <li><code>Key = term()</code></li> <li><code>Value = term()</code></li> <li><code>RegExp = string() | {string, Options} | mp(), {mp(), Options}</code></li> <li><code>Dates = {DateFrom, DateTo} | {DateFrom, from} | {DateTo, to}</code></li> <li><code>DateFrom = DateTo = {date(), time()}</code></li> <li><code>date() and time() are the same type as in the <code>calendar</code> module</code></li> </ul> <div class="description"> <p>This function displays the reports that match the provided filters.</p> <p> When a filter includes the <code>no</code> atom it will exclude the reports that match that filter. </p> <p> The reports are matched using the <code>proplists</code> module. The report must be a proplist to be matched against any of the <code>filters()</code>. </p> <p> If the filter is of the form <code>{Key, RegExp, re}</code> the report must contain an element with <code>key = Key</code> and <code>Value</code> must match the RegExp regular expression. </p> <p> If the Dates parameter is provided, then the reports are filtered according to the date when they occurred. If Dates is of the form <code>{DateFrom, from}</code> then reports that occurred after DateFrom are displayed. </p> <p> If Dates is of the form <code>{DateTo, to}</code> then reports that occurred before DateTo are displayed. </p> <p> If two Dates are provided, then reports that occurred between those dates are returned. </p> <p> If you only want to filter only by dates, then you can provide the empty list as the Filters parameter. </p> <p> See <code>rb:grep/1</code> for more information on the RegExp parameter. </p> </div> </div> <div class="function"> <h3 id="grep/1">grep(RegExp)</h3> <ul class="type"> <li><code>RegExp = string() | {string, Options} | mp(), {mp(), Options}</code></li> </ul> <div class="description"> <p>All reports containing the regular expression <code>RegExp</code> are printed. </p> <p><code>RegExp</code> can be a string containing the regular expression; a tuple with the string and the options for compilation; a compiled regular expression; a compiled regular expression and the options for running it. Refer to the module <code>re</code> and specially the function <code>re:run/3</code> for a definition of valid regular expressions and options. </p> </div> </div> <div class="function"> <h3 id="h/0">h()</h3> <h3 id="help/0">help()</h3> <div class="description"> <p>Prints the on-line help information. </p> </div> </div> <div class="function"> <h3 id="list/0">list()</h3> <h3 id="list/1">list(Type)</h3> <ul class="type"> <li><code>Type = type()</code></li> <li><code>type() = error | error_report | info_msg | info_report | warning_msg | warning_report | crash_report | supervisor_report | progress</code></li> </ul> <div class="description"> <p>This function lists all reports loaded in the <code>rb_server</code>. Each report is given a unique number that can be used as a reference to the report in the <code>show/1</code> function. </p> <p>If no <code>Type</code> is given, all reports are listed. </p> </div> </div> <div class="function"> <h3 id="log_list/0">log_list()</h3> <h3 id="log_list/1">log_list(Type)</h3> <ul class="type"> <li><code>Type = type()</code></li> <li><code>type() = error | error_report | info_msg | info_report | warning_msg | warning_report | crash_report | supervisor_report | progress</code></li> </ul> <div class="description"> <p>Same as <code>list/0</code> or <code>list/1</code> functions but result is printed to logfile, if set, otherwise to standard_io. </p> <p>If no <code>Type</code> is given, all reports are listed. </p> </div> </div> <div class="function"> <h3 id="rescan/0">rescan()</h3> <h3 id="rescan/1">rescan(Options)</h3> <ul class="type"> <li><code>Options = [opt()]</code></li> </ul> <div class="description"> <p>Rescans the report directory. <code>Options</code> is the same as for <code>start()</code>. </p> </div> </div> <div class="function"> <h3 id="show/0">show()</h3> <h3 id="show/1">show(Report)</h3> <ul class="type"> <li><code>Report = int() | type()</code></li> </ul> <div class="description"> <p>If a type argument is given, all loaded reports of this type are printed. If an integer argument is given, the report with this reference number is printed. If no argument is given, all reports are shown. </p> </div> </div> <div class="function"> <h3 id="start/0">start()</h3> <h3 id="start/1">start(Options)</h3> <ul class="type"> <li><code>Options = [opt()]</code></li> <li><code>opt() = {start_log, FileName} | {max, MaxNoOfReports} | {report_dir, DirString} | {type, ReportType} | {abort_on_error, Bool}</code></li> <li><code>FileName = string() | atom() | pid()</code></li> <li><code>MaxNoOfReports = int() | all</code></li> <li><code>DirString = string()</code></li> <li><code>ReportType = type() | [type()] | all</code></li> <li><code>Bool = true | false</code></li> </ul> <div class="description"> <p>The function <code>start/1</code> starts the <code>rb_server</code> with the specified options, while <code>start/0</code> starts with default options. The <code>rb_server</code> must be started before reports can be browsed. When the <code>rb_server</code> is started, the files in the specified directory are scanned. The other functions assume that the server has started. </p> <p><code>{start_log, FileName}</code> starts logging to file, registered name or io_device. All reports will be printed to the named file. The default is <code>standard_io</code>. The option {start_log, standard_error} is not allowed and will be replaced by default standard_io. </p> <p><code>{max, MaxNoOfReports}</code>. Controls how many reports the <code>rb_server</code> should read on start-up. This option is useful as the directory may contain 20.000 reports. If this option is given, the <code>MaxNoOfReports</code> latest reports will be read. The default is 'all'. </p> <p><code>{report_dir, DirString}</code>. Defines the directory where the error log files are located. The default is <code>{sasl, error_logger_mf_dir}</code>. </p> <p><code>{type, ReportType}</code>. Controls what kind of reports the <code>rb_server</code> should read on start-up. <code>ReportType</code> is a supported type, 'all', or a list of supported types. The default is 'all'. </p> <p><code>{abort_on_error, Bool}</code>. This option specifies whether or not logging should be aborted if rb encounters an unprintable report. (You may get a report on incorrect form if the <code>error_logger</code> function <code>error_msg</code> or <code>info_msg</code> has been called with an invalid format string). If <code>Bool</code> is <code>true</code>, rb will stop logging (and print an error message to stdout) if it encounters a badly formatted report. If logging to file is enabled, an error message will be appended to the log file as well. If <code>Bool</code> is <code>false</code> (which is the default value), rb will print an error message to stdout for every bad report it encounters, but the logging process is never aborted. All printable reports will be written. If logging to file is enabled, rb prints <code>* UNPRINTABLE REPORT *</code> in the log file at the location of an unprintable report. </p> </div> </div> <div class="function"> <h3 id="start_log/1">start_log(FileName)</h3> <ul class="type"> <li><code>FileName = string() | atom() | pid()</code></li> </ul> <div class="description"> <p>Redirects all report output from the RB tool to the specified file, registered name or io_device. </p> </div> </div> <div class="function"> <h3 id="stop/0">stop()</h3> <div class="description"> <p>Stops the <code>rb_server</code>. </p> </div> </div> <div class="function"> <h3 id="stop_log/0">stop_log()</h3> <div class="description"> <p>Closes the log file. The output from the RB tool will be directed to <code>standard_io</code>. </p> </div> </div> </div>
