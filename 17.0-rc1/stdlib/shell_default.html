<h1>shell_default</h1> <h2 class="modsummary">Customizing the Erlang Environment</h2> <div class="description"> <p>The functions in <code>shell_default</code> are called when no module name is given in a shell command. </p> <p>Consider the following shell dialogue:</p> <pre class="sh_erlang"> 1 &gt; <code>lists:reverse("abc").</code> "cba" 2 &gt; <code>c(foo).</code> {ok, foo} </pre> <p>In command one, the module <code>lists</code> is called. In command two, no module name is specified. The shell searches the modules <code>user_default</code> followed by <code>shell_default</code> for the function <code>foo/1</code>. </p> <p><code>shell_default</code> is intended for "system wide" customizations to the shell. <code>user_default</code> is intended for "local" or individual user customizations.</p> </div> <div class="section"> <h4>Hint</h4> <p>To add your own commands to the shell, create a module called <code>user_default</code> and add the commands you want. Then add the following line as the <em>first</em> line in your <code>.erlang</code> file in your home directory. </p> <pre class="sh_erlang"> code:load_abs("$PATH/user_default"). </pre> <p><code>$PATH</code> is the directory where your <code>user_default</code> module can be found.</p> </div>
