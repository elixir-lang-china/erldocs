<h1>zip</h1> <h2 class="modsummary">Utility for reading and creating 'zip' archives.</h2> <div class="description"> <p>The <code>zip</code> module archives and extracts files to and from a zip archive. The zip format is specified by the "ZIP Appnote.txt" file available on PKWare's website www.pkware.com.</p> <p>The zip module supports zip archive versions up to 6.1. However, password-protection and Zip64 are not supported.</p> <p>By convention, the name of a zip file should end in "<code>.zip</code>". To abide to the convention, you'll need to add "<code>.zip</code>" yourself to the name.</p> <p>Zip archives are created with the <a href="#zip_2" class="seealso">zip/2</a> or the <a href="#zip_2" class="seealso">zip/3</a> function. (They are also available as <code>create</code>, to resemble the <code>erl_tar</code> module.)</p> <p>To extract files from a zip archive, use the <a href="#unzip_1" class="seealso">unzip/1</a> or the <a href="#unzip_2" class="seealso">unzip/2</a> function. (They are also available as <code>extract</code>.)</p> <p>To fold a function over all files in a zip archive, use the <a href="#foldl_3" class="seealso">foldl_3</a> function.</p> <p>To return a list of the files in a zip archive, use the <a href="#list_dir_1" class="seealso">list_dir/1</a> or the <a href="#list_dir_2" class="seealso">list_dir/2</a> function. (They are also available as <code>table</code>.)</p> <p>To print a list of files to the Erlang shell, use either the <a href="#t_1" class="seealso">t/1</a> or <a href="#tt_1" class="seealso">tt/1</a> function.</p> <p>In some cases, it is desirable to open a zip archive, and to unzip files from it file by file, without having to reopen the archive. The functions <a href="#zip_open" class="seealso">zip_open</a>, <a href="#zip_get" class="seealso">zip_get</a>, <a href="#zip_list_dir" class="seealso">zip_list_dir</a> and <a href="#zip_close" class="seealso">zip_close</a> do this.</p> </div> <div class="section"> <h4>LIMITATIONS</h4> <p>Zip64 archives are not currently supported.</p> <p>Password-protected and encrypted archives are not currently supported</p> <p>Only the DEFLATE (zlib-compression) and the STORE (uncompressed data) zip methods are supported.</p> <p>The size of the archive is limited to 2 G-byte (32 bits).</p> <p>Comments for individual files is not supported when creating zip archives. The zip archive comment for the whole zip archive is supported.</p> <p>There is currently no support for altering an existing zip archive. To add or remove a file from an archive, the whole archive must be recreated.</p> </div> <datatypes> <datatype> <name name="zip_comment" /> <div class="description"> <p>The record <code>zip_comment</code> just contains the archive comment for a zip archive</p> </div> </datatype> <datatype> <name name="zip_file" /> <div class="description"> <p>The record <code>zip_file</code> contains the following fields.</p> <taglist> <dt><code>name</code></dt> <item> <p>the name of the file</p> </item> <dt><code>info</code></dt> <item> <p>file info as in <a href="../kernel/file.html#read_file_info/1" class="seealso">file:read_file_info/1</a></p> </item> <dt><code>comment</code></dt> <item> <p>the comment for the file in the zip archive</p> </item> <dt><code>offset</code></dt> <item> <p>the offset of the file in the zip archive (used internally)</p> </item> <dt><code>comp_size</code></dt> <item> <p>the compressed size of the file (the uncompressed size is found in <code>info</code>)</p> </item> </taglist> </div> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="zip/2">zip/2</h3> <h3 id="zip/3">zip/3</h3> <h3 id="create/2">create/2</h3> <h3 id="create/3">create/3</h3> <div class="description"> <p>The <span id="zip_2"> </span><code>zip</code> function creates a zip archive containing the files specified in <code><anno>FileList</anno></code>.</p> <p>As synonyms, the functions <code>create/2</code> and <code>create/3</code> are provided, to make it resemble the <code>erl_tar</code> module.</p> <p>The file-list is a list of files, with paths relative to the current directory, they will be stored with this path in the archive. Files may also be specified with data in binaries, to create an archive directly from data.</p> <p>Files will be compressed using the DEFLATE compression, as described in the Appnote.txt file. However, files will be stored without compression if they already are compressed. The <code>zip/2</code> and <code>zip/3</code> functions check the file extension to see whether the file should be stored without compression. Files with the following extensions are not compressed: <code>.Z</code>, <code>.zip</code>, <code>.zoo</code>, <code>.arc</code>, <code>.lzh</code>, <code>.arj</code>.</p> <p>It is possible to override the default behavior and explicitly control what types of files that should be compressed by using the <code>{compress, <anno>What</anno>}</code> and <code>{uncompress, <anno>What</anno>}</code> options. It is possible to have several <code>compress</code> and <code>uncompress</code> options. In order to trigger compression of a file, its extension must match with the <code>compress</code> condition and must not match the <code>uncompress</code> condition. For example if <code>compress</code> is set to <code>["gif", "jpg"]</code> and <code>uncompress</code> is set to <code>["jpg"]</code>, only files with <code>"gif"</code> as extension will be compressed. No other files will be compressed.</p> <p>The following options are available:</p> <taglist> <dt><code>cooked</code></dt> <item> <p>By default, the <code>open/2</code> function will open the zip file in <code>raw</code> mode, which is faster but does not allow a remote (erlang) file server to be used. Adding <code>cooked</code> to the mode list will override the default and open the zip file without the <code>raw</code> option. The same goes for the files added.</p> </item> <dt><code>verbose</code></dt> <item> <p>Print an informational message about each file being added.</p> </item> <dt><code>memory</code></dt> <item> <p>The output will not be to a file, but instead as a tuple <code>{<anno>FileName</anno>, binary()}</code>. The binary will be a full zip archive with header, and can be extracted with for instance <code>unzip/2</code>.</p> </item> <dt><code>{comment, <anno>Comment</anno>}</code></dt> <item> <p>Add a comment to the zip-archive.</p> </item> <dt><code>{cwd, <anno>CWD</anno>}</code></dt> <item> <p>Use the given directory as current directory, it will be prepended to file names when adding them, although it will not be in the zip-archive. (Acting like a file:set_cwd/1, but without changing the global cwd property.)</p> </item> <dt><code>{compress, <anno>What</anno>}</code></dt> <item> <p>Controls what types of files will be compressed. It is by default set to <code>all</code>. The following values of <code>What</code> are allowed:</p> <taglist> <dt><code>all</code></dt> <item><p> means that all files will be compressed (as long as they pass the <code>uncompress</code> condition).</p></item> <dt><code>[<anno>Extension</anno>]</code></dt> <item><p>means that only files with exactly these extensions will be compressed.</p></item> <dt><code>{add,[<anno>Extension</anno>]}</code></dt> <item><p>adds these extensions to the list of compress extensions.</p></item> <dt><code>{del,[<anno>Extension</anno>]}</code></dt> <item><p>deletes these extensions from the list of compress extensions.</p></item> </taglist> </item> <dt><code>{uncompress, <anno>What</anno>}</code></dt> <item> <p>Controls what types of files will be uncompressed. It is by default set to <code>[".Z", ".zip", ".zoo", ".arc", ".lzh", ".arj"]</code>. The following values of <code>What</code> are allowed:</p> <taglist> <dt><code>all</code></dt> <item><p> means that no files will be compressed.</p></item> <dt><code>[<anno>Extension</anno>]</code></dt> <item><p>means that files with these extensions will be uncompressed.</p></item> <dt><code>{add,[<anno>Extension</anno>]}</code></dt> <item><p>adds these extensions to the list of uncompress extensions.</p></item> <dt><code>{del,[<anno>Extension</anno>]}</code></dt> <item><p>deletes these extensions from the list of uncompress extensions.</p></item> </taglist> </item> </taglist> </div> </div> <div class="function"> <h3 id="unzip/1">unzip/1</h3> <h3 id="unzip/2">unzip/2</h3> <h3 id="extract/1">extract/1</h3> <h3 id="extract/2">extract/2</h3> <div class="description"> <p>The <span id="unzip_1"> </span><code>unzip/1</code> function extracts all files from a zip archive. The <span id="unzip_2"> </span><code>unzip/2</code> function provides options to extract some files, and more.</p> <p>If the <code><anno>Archive</anno></code> argument is given as a binary, the contents of the binary is assumed to be a zip archive, otherwise it should be a filename.</p> <p>The following options are available:</p> <taglist> <dt><code>{file_list, <anno>FileList</anno>}</code></dt> <item> <p>By default, all files will be extracted from the zip archive. With the <code>{file_list, <anno>FileList</anno>}</code> option, the <code>unzip/2</code> function will only extract the files whose names are included in <code><anno>FileList</anno></code>. The full paths, including the names of all sub directories within the zip archive, must be specified.</p> </item> <dt><code>cooked</code></dt> <item> <p>By default, the <code>open/2</code> function will open the zip file in <code>raw</code> mode, which is faster but does not allow a remote (erlang) file server to be used. Adding <code>cooked</code> to the mode list will override the default and open the zip file without the <code>raw</code> option. The same goes for the files extracted.</p> </item> <dt><code>keep_old_files</code></dt> <item> <p>By default, all existing files with the same name as file in the zip archive will be overwritten. With the <code>keep_old_files</code> option, the <code>unzip/2</code> function will not overwrite any existing files. Note that even with the <code>memory</code> option given, which means that no files will be overwritten, files existing will be excluded from the result.</p> </item> <dt><code>verbose</code></dt> <item> <p>Print an informational message as each file is being extracted.</p> </item> <dt><code>memory</code></dt> <item> <p>Instead of extracting to the current directory, the <code>memory</code> option will give the result as a list of tuples <code>{Filename, Binary}</code>, where <code>Binary</code> is a binary containing the extracted data of the file named <code>Filename</code> in the zip archive.</p> </item> <dt><code>{cwd, CWD}</code></dt> <item> <p>Use the given directory as current directory, it will be prepended to file names when extracting them from the zip-archive. (Acting like a file:set_cwd/1, but without changing the global cwd property.)</p> </item> </taglist> </div> </div> <div class="function"> <h3 id="foldl/3">foldl/3</h3> <div class="description"> <p>The <span id="foldl_3"> </span> <code>foldl/3</code> function calls <code><anno>Fun</anno>(<anno>FileInArchive</anno>, <anno>GetInfo </anno>, <anno>GetBin</anno>, <anno>AccIn</anno>)</code> on successive files in the <code>Archive</code>, starting with <code><anno>AccIn</anno> == <anno>Acc0</anno></code>. <code><anno>FileInArchive</anno></code> is the name that the file has in the archive. <code><anno>GetInfo</anno></code> is a fun that returns info about the the file. <code><anno>GetBin</anno></code> returns the contents of the file. Both <code><anno>GetInfo</anno></code> and <code><anno>GetBin</anno></code> must be called within the <code><anno>Fun</anno></code>. Their behavior is undefined if they are called outside the context of the <code><anno>Fun</anno></code>. The <code><anno>Fun</anno></code> must return a new accumulator which is passed to the next call. <code>foldl/3</code> returns the final value of the accumulator. <code><anno>Acc0</anno></code> is returned if the archive is empty. It is not necessary to iterate over all files in the archive. The iteration may be ended prematurely in a controlled manner by throwing an exception.</p> <p>For example:</p> <pre class="sh_erlang"> &gt; <code>Name = "dummy.zip".</code> "dummy.zip" &gt; <code>{ok, {Name, Bin}} = zip:create(Name, [{"foo", &lt;&lt;"FOO"&gt;&gt;}, {"bar", &lt;&lt;"BAR"&gt;&gt;}], [memory]).</code> {ok,{"dummy.zip", &lt;&lt;80,75,3,4,20,0,0,0,0,0,74,152,97,60,171,39,212,26,3,0, 0,0,3,0,0,...&gt;&gt;}} &gt; <code>{ok, FileSpec} = zip:foldl(fun(N, I, B, Acc) -&gt; [{N, B(), I()} | Acc] end, [], {Name, Bin}).</code> {ok,[{"bar",&lt;&lt;"BAR"&gt;&gt;, {file_info,3,regular,read_write, {{2010,3,1},{19,2,10}}, {{2010,3,1},{19,2,10}}, {{2010,3,1},{19,2,10}}, 54,1,0,0,0,0,0}}, {"foo",&lt;&lt;"FOO"&gt;&gt;, {file_info,3,regular,read_write, {{2010,3,1},{19,2,10}}, {{2010,3,1},{19,2,10}}, {{2010,3,1},{19,2,10}}, 54,1,0,0,0,0,0}}]} &gt; <code>{ok, {Name, Bin}} = zip:create(Name, lists:reverse(FileSpec), [memory]).</code> {ok,{"dummy.zip", &lt;&lt;80,75,3,4,20,0,0,0,0,0,74,152,97,60,171,39,212,26,3,0, 0,0,3,0,0,...&gt;&gt;}} &gt; <code>catch zip:foldl(fun("foo", _, B, _) -&gt; throw(B()); (_,_,_,Acc) -&gt; Acc end, [], {Name, Bin}). </code> &lt;&lt;"FOO"&gt;&gt; </pre> </div> </div> <div class="function"> <h3 id="list_dir/1">list_dir/1</h3> <h3 id="list_dir/2">list_dir/2</h3> <h3 id="table/1">table/1</h3> <h3 id="table/2">table/2</h3> <div class="description"> <p>The <span id="list_dir_1"> </span><code>list_dir/1</code> function retrieves the names of all files in the zip archive <code><anno>Archive</anno></code>. The <span id="list_dir_2"> </span> <code>list_dir/2</code> function provides options.</p> <p>As synonyms, the functions <code>table/2</code> and <code>table/3</code> are provided, to make it resemble the <code>erl_tar</code> module.</p> <p>The result value is the tuple <code>{ok, List}</code>, where <code>List</code> contains the zip archive comment as the first element.</p> <p>The following options are available:</p> <taglist> <dt><code>cooked</code></dt> <item> <p>By default, the <code>open/2</code> function will open the zip file in <code>raw</code> mode, which is faster but does not allow a remote (erlang) file server to be used. Adding <code>cooked</code> to the mode list will override the default and open the zip file without the <code>raw</code> option.</p> </item> </taglist> </div> </div> <div class="function"> <h3 id="t/1">t/1</h3> <div class="description"> <p>The <span id="t_1"> </span><code>t/1</code> function prints the names of all files in the zip archive <code><anno>Archive</anno></code> to the Erlang shell. (Similar to "<code>tar&nbsp;t</code>".)</p> </div> </div> <div class="function"> <h3 id="tt/1">tt/1</h3> <div class="description"> <p>The <span id="tt_1"> </span><code>tt/1</code> function prints names and information about all files in the zip archive <code><anno>Archive</anno></code> to the Erlang shell. (Similar to "<code>tar tv</code>".)</p> </div> </div> <div class="function"> <h3 id="zip_open/1">zip_open/1</h3> <h3 id="zip_open/2">zip_open/2</h3> <div class="description"> <p>The <span id="zip_open"> </span><code>zip_open</code> function opens a zip archive, and reads and saves its directory. This means that subsequently reading files from the archive will be faster than unzipping files one at a time with <code>unzip</code>.</p> <p>The archive must be closed with <code>zip_close/1</code>.</p> </div> </div> <div class="function"> <h3 id="zip_list_dir/1">zip_list_dir/1</h3> <div class="description"> <p>The <span id="zip_list_dir"> </span> <code>zip_list_dir/1</code> function returns the file list of an open zip archive. The first returned element is the zip archive comment.</p> </div> </div> <div class="function"> <h3 id="zip_get/1">zip_get/1</h3> <h3 id="zip_get/2">zip_get/2</h3> <div class="description"> <p>The <span id="zip_get"> </span><code>zip_get</code> function extracts one or all files from an open archive.</p> <p>The files will be unzipped to memory or to file, depending on the options given to the <code>zip_open</code> function when the archive was opened.</p> </div> </div> <div class="function"> <h3 id="zip_close/1">zip_close/1</h3> <div class="description"> <p>The <span id="zip_close"> </span><code>zip_close/1</code> function closes a zip archive, previously opened with <code>zip_open</code>.
