<h1>erl_parse</h1> <h2 class="modsummary">The Erlang Parser</h2> <div class="description"> <p>This module is the basic Erlang parser which converts tokens into the abstract form of either forms (i.e., top-level constructs), expressions, or terms. The Abstract Format is described in the <a href="../erts/absform.html" class="seealso">ERTS User's Guide</a>. Note that a token list must end with the <em>dot</em> token in order to be acceptable to the parse functions (see <a href="erl_scan.html" class="seealso">erl_scan(3)</a>).</p> </div> <datatypes> <datatype> <name name="abstract_clause" /> <div class="description"><p>Parse tree for Erlang clause.</p> </div> </datatype> <datatype> <name name="abstract_expr" /> <div class="description"><p>Parse tree for Erlang expression.</p> </div> </datatype> <datatype> <name name="abstract_form" /> <div class="description"><p>Parse tree for Erlang form.</p> </div> </datatype> <datatype> <name name="error_description" /> </datatype> <datatype> <name name="error_info" /> </datatype> <datatype> <name name="token" /> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="parse_form/1">parse_form/1</h3> <div class="description"> <p>This function parses <code><anno>Tokens</anno></code> as if it were a form. It returns:</p> <taglist> <dt><code>{ok, <anno>AbsForm</anno>}</code></dt> <item> <p>The parsing was successful. <code><anno>AbsForm</anno></code> is the abstract form of the parsed form.</p> </item> <dt><code>{error, <anno>ErrorInfo</anno>}</code></dt> <item> <p>An error occurred.</p> </item> </taglist> </div> </div> <div class="function"> <h3 id="parse_exprs/1">parse_exprs/1</h3> <div class="description"> <p>This function parses <code><anno>Tokens</anno></code> as if it were a list of expressions. It returns:</p> <taglist> <dt><code>{ok, <anno>ExprList</anno>}</code></dt> <item> <p>The parsing was successful. <code><anno>ExprList</anno></code> is a list of the abstract forms of the parsed expressions.</p> </item> <dt><code>{error, <anno>ErrorInfo</anno>}</code></dt> <item> <p>An error occurred.</p> </item> </taglist> </div> </div> <div class="function"> <h3 id="parse_term/1">parse_term/1</h3> <div class="description"> <p>This function parses <code><anno>Tokens</anno></code> as if it were a term. It returns:</p> <taglist> <dt><code>{ok, <anno>Term</anno>}</code></dt> <item> <p>The parsing was successful. <code><anno>Term</anno></code> is the Erlang term corresponding to the token list.</p> </item> <dt><code>{error, ErrorInfo}</code></dt> <item> <p>An error occurred.</p> </item> </taglist> </div> </div> <div class="function"> <h3 id="format_error/1">format_error(ErrorDescriptor) -&gt; Chars</h3> <ul class="type"> <li><code>ErrorDescriptor = <a href="#type-error_info" class="seealso">error_description()</a></code></li> <li><code>Chars = [char() | Chars]</code></li> </ul> <div class="description"> <p>Uses an <code>ErrorDescriptor</code> and returns a string which describes the error. This function is usually called implicitly when an <code>ErrorInfo</code> structure is processed (see below).</p> </div> </div> <div class="function"> <h3 id="tokens/1">tokens/1</h3> <h3 id="tokens/2">tokens/2</h3> <div class="description"> <p>This function generates a list of tokens representing the abstract form <code><anno>AbsTerm</anno></code> of an expression. Optionally, it appends <code><anno>MoreTokens</anno></code>.</p> </div> </div> <div class="function"> <h3 id="normalise/1">normalise/1</h3> <div class="description"> <p>Converts the abstract form <code><anno>AbsTerm</anno></code> of a term into a conventional Erlang data structure (i.e., the term itself). This is the inverse of <code>abstract/1</code>.</p> </div> </div> <div class="function"> <h3 id="abstract/1">abstract/1</h3> <div class="description"> <p>Converts the Erlang data structure <code><anno>Data</anno></code> into an abstract form of type <code><anno>AbsTerm</anno></code>. This is the inverse of <code>normalise/1</code>.</p> <p><code>erl_parse:abstract(T)</code> is equivalent to <code>erl_parse:abstract(T, 0)</code>.</p> </div> </div> <div class="function"> <h3 id="abstract/2">abstract/2</h3> <div class="description"> <p>Converts the Erlang data structure <code><anno>Data</anno></code> into an abstract form of type <code><anno>AbsTerm</anno></code>.</p> <p>The <code><anno>Line</anno></code> option is the line that will be assigned to each node of the abstract form.</p> <p>The <code><anno>Encoding</anno></code> option is used for selecting which integer lists will be considered as strings. The default is to use the encoding returned by <a href="epp.html#default_encoding/0" class="seealso"> <code>epp:default_encoding/0</code></a></p> </div> </div> </div> <div class="section"> <h4>Error Information</h4> <p>The <code>ErrorInfo</code> mentioned above is the standard <code>ErrorInfo</code> structure which is returned from all IO modules. It has the format: </p> <pre class="sh_erlang"> {ErrorLine, Module, ErrorDescriptor} </pre> <p>A string which describes the error is obtained with the following call: </p> <pre class="sh_erlang"> Module:format_error(ErrorDescriptor) </pre> </div> <div class="section"> <h4>See Also</h4> <p><a href="io.html" class="seealso">io(3)</a>, <a href="erl_scan.html" class="seealso">erl_scan(3)</a>, ERTS User's Guide</p> </div>
