<h1>erl_scan</h1> <h2 class="modsummary">The Erlang Token Scanner</h2> <div class="description"> <p>This module contains functions for tokenizing characters into Erlang tokens.</p> </div> <datatypes> <datatype> <name name="attribute_info" /> </datatype> <datatype> <name name="attributes" /> </datatype> <datatype> <name name="attributes_data" /> </datatype> <datatype> <name name="category" /> </datatype> <datatype> <name name="column" /> </datatype> <datatype> <name name="error_description" /> </datatype> <datatype> <name name="error_info" /> </datatype> <datatype> <name name="info_line" /> </datatype> <datatype> <name name="info_location" /> </datatype> <datatype> <name name="line" /> </datatype> <datatype> <name name="location" /> </datatype> <datatype> <name name="option" /> </datatype> <datatype> <name name="options" /> </datatype> <datatype> <name name="symbol" /> </datatype> <datatype> <name name="resword_fun" /> </datatype> <datatype> <name name="token" /> </datatype> <datatype> <name name="token_info" /> </datatype> <datatype> <name name="tokens" /> </datatype> <datatype> <name name="tokens_result" /> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="string/1">string/1</h3> <h3 id="string/2">string/2</h3> <h3 id="string/3">string/3</h3> <div class="description"> <p>Takes the list of characters <code><anno>String</anno></code> and tries to scan (tokenize) them. Returns <code>{ok, <anno>Tokens</anno>, <anno>EndLocation</anno>}</code>, where <code><anno>Tokens</anno></code> are the Erlang tokens from <code><anno>String</anno></code>. <code><anno>EndLocation</anno></code> is the first location after the last token.</p> <p><code>{error, <anno>ErrorInfo</anno>, <anno>ErrorLocation</anno>}</code> is returned if an error occurs. <code><anno>ErrorLocation</anno></code> is the first location after the erroneous token.</p> <p><code>string(<anno>String</anno>)</code> is equivalent to <code>string(<anno>String</anno>, 1)</code>, and <code>string(<anno>String</anno>, <anno>StartLocation</anno>)</code> is equivalent to <code>string(<anno>String</anno>, <anno>StartLocation</anno>, [])</code>.</p> <p><code><anno>StartLocation</anno></code> indicates the initial location when scanning starts. If <code><anno>StartLocation</anno></code> is a line <code>attributes()</code> as well as <code><anno>EndLocation</anno></code> and <code><anno>ErrorLocation</anno></code> will be lines. If <code><anno>StartLocation</anno></code> is a pair of a line and a column <code>attributes()</code> takes the form of an opaque compound data type, and <code><anno>EndLocation</anno></code> and <code><anno>ErrorLocation</anno></code> will be pairs of a line and a column. The <em>token attributes</em> contain information about the column and the line where the token begins, as well as the text of the token (if the <code>text</code> option is given), all of which can be accessed by calling <a href="#token_info/1" class="seealso">token_info/1,2</a> or <a href="#attributes_info/1" class="seealso">attributes_info/1,2</a>.</p> <p>A <em>token</em> is a tuple containing information about syntactic category, the token attributes, and the actual terminal symbol. For punctuation characters (e.g. <code>;</code>, <code>|</code>) and reserved words, the category and the symbol coincide, and the token is represented by a two-tuple. Three-tuples have one of the following forms: <code>{atom, Info, atom()}</code>, <code>{char, Info, integer()}</code>, <code>{comment, Info, string()}</code>, <code>{float, Info, float()}</code>, <code>{integer, Info, integer()}</code>, <code>{var, Info, atom()}</code>, and <code>{white_space, Info, string()}</code>.</p> <p>The valid options are:</p> <taglist> <dt><code>{reserved_word_fun, reserved_word_fun()}</code></dt> <item><p>A callback function that is called when the scanner has found an unquoted atom. If the function returns <code>true</code>, the unquoted atom itself will be the category of the token; if the function returns <code>false</code>, <code>atom</code> will be the category of the unquoted atom.</p> </item> <dt><code>return_comments</code></dt> <item><p>Return comment tokens.</p> </item> <dt><code>return_white_spaces</code></dt> <item><p>Return white space tokens. By convention, if there is a newline character, it is always the first character of the text (there cannot be more than one newline in a white space token).</p> </item> <dt><code>return</code></dt> <item><p>Short for <code>[return_comments, return_white_spaces]</code>.</p> </item> <dt><code>text</code></dt> <item><p>Include the token's text in the token attributes. The text is the part of the input corresponding to the token.</p> </item> </taglist> </div> </div> <div class="function"> <h3 id="tokens/3">tokens/3</h3> <h3 id="tokens/4">tokens/4</h3> <type name="char_spec" /> <type name="return_cont" /> <type_desc name="return_cont">An opaque continuation</type_desc> <div class="description"> <p>This is the re-entrant scanner which scans characters until a <em>dot</em> ('.' followed by a white space) or <code>eof</code> has been reached. It returns:</p> <taglist> <dt><code>{done, <anno>Result</anno>, <anno>LeftOverChars</anno>}</code> </dt> <item> <p>This return indicates that there is sufficient input data to get a result. <code><anno>Result</anno></code> is:</p> <taglist> <dt><code>{ok, Tokens, EndLocation}</code> </dt> <item> <p>The scanning was successful. <code>Tokens</code> is the list of tokens including <em>dot</em>.</p> </item> <dt><code>{eof, EndLocation}</code></dt> <item> <p>End of file was encountered before any more tokens.</p> </item> <dt><code>{error, ErrorInfo, EndLocation}</code> </dt> <item> <p>An error occurred. <code><anno>LeftOverChars</anno></code> is the remaining characters of the input data, starting from <code>EndLocation</code>.</p> </item> </taglist> </item> <dt><code>{more, <anno>Continuation1</anno>}</code></dt> <item> <p>More data is required for building a term. <code><anno>Continuation1</anno></code> must be passed in a new call to <code>tokens/3,4</code> when more data is available.</p> </item> </taglist> <p>The <code><anno>CharSpec</anno></code> <code>eof</code> signals end of file. <code><anno>LeftOverChars</anno></code> will then take the value <code>eof</code> as well.</p> <p><code>tokens(<anno>Continuation</anno>, <anno>CharSpec</anno>, <anno>StartLocation</anno>)</code> is equivalent to <code>tokens(<anno>Continuation</anno>, <anno>CharSpec</anno>, <anno>StartLocation</anno>, [])</code>.</p> <p>See <a href="#string/3" class="seealso">string/3</a> for a description of the various options.</p> </div> </div> <div class="function"> <h3 id="reserved_word/1">reserved_word/1</h3> <div class="description"> <p>Returns <code>true</code> if <code><anno>Atom</anno></code> is an Erlang reserved word, otherwise <code>false</code>.</p> </div> </div> <div class="function"> <h3 id="token_info/1">token_info/1</h3> <div class="description"> <p>Returns a list containing information about the token <code><anno>Token</anno></code>. The order of the <code><anno>TokenInfoTuple</anno></code>s is not defined. See <a href="#token_info/2" class="seealso">token_info/2</a> for information about specific <code><anno>TokenInfoTuple</anno></code>s.</p> <p>Note that if <code>token_info(Token, TokenItem)</code> returns <code>undefined</code> for some <code>TokenItem</code>, the item is not included in <code><anno>TokenInfo</anno></code>.</p> </div> </div> <div class="function"> <name name="token_info" arity="2" clause_i="1" /> <name name="token_info" arity="2" clause_i="2" /> <type name="token_item" /> <type name="attribute_item" /> <div class="description"> <p>Returns a list containing information about the token <code><anno>Token</anno></code>. If one single <code><anno>TokenItem</anno></code> is given the returned value is the corresponding <code>TokenInfoTuple</code>, or <code>undefined</code> if the <code>TokenItem</code> has no value. If a list of <code><anno>TokenItem</anno></code>s is given the result is a list of <code><anno>TokenInfoTuple</anno></code>. The <code><anno>TokenInfoTuple</anno></code>s will appear with the corresponding <code><anno>TokenItem</anno></code>s in the same order as the <code><anno>TokenItem</anno></code>s appear in the list of <code>TokenItem</code>s. <code><anno>TokenItem</anno></code>s with no value are not included in the list of <code><anno>TokenInfoTuple</anno></code>.</p> <p>The following <code><anno>TokenInfoTuple</anno></code>s with corresponding <code><anno>TokenItem</anno></code>s are valid:</p> <taglist> <dt><code>{category, <a href="#type-category" class="seealso"> category()</a>}</code></dt> <item><p>The category of the token.</p> </item> <dt><code>{column, <a href="#type-column" class="seealso"> column()</a>}</code></dt> <item><p>The column where the token begins.</p> </item> <dt><code>{length, integer() &gt; 0}</code></dt> <item><p>The length of the token's text.</p> </item> <dt><code>{line, <a href="#type-line" class="seealso"> line()</a>}</code></dt> <item><p>The line where the token begins.</p> </item> <dt><code>{location, <a href="#type-location" class="seealso"> location()</a>}</code></dt> <item><p>The line and column where the token begins, or just the line if the column unknown.</p> </item> <dt><code>{symbol, <a href="#type-symbol" class="seealso"> symbol()</a>}</code></dt> <item><p>The token's symbol.</p> </item> <dt><code>{text, string()}</code></dt> <item><p>The token's text.</p> </item> </taglist> </div> </div> <div class="function"> <h3 id="attributes_info/1">attributes_info/1</h3> <div class="description"> <p>Returns a list containing information about the token attributes <code><anno>Attributes</anno></code>.
