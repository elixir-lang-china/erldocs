<h1>log_mf_h</h1> <h2 class="modsummary">An Event Handler which Logs Events to Disk</h2> <div class="description"> <p>The <code>log_mf_h</code> is a <code>gen_event</code> handler module which can be installed in any <code>gen_event</code> process. It logs onto disk all events which are sent to an event manager. Each event is written as a binary which makes the logging very fast. However, a tool such as the <code>Report Browser</code> (<code>rb</code>) must be used in order to read the files. The events are written to multiple files. When all files have been used, the first one is re-used and overwritten. The directory location, the number of files, and the size of each file are configurable. The directory will include one file called <code>index</code>, and report files <code>1, 2, ....</code>. </p> </div> <datatypes> <datatype> <name name="args" /> <div class="description"><p>Term to be sent to <a href="gen_event.html#add_handler/3" class="seealso"> gen_event:add_handler/3</a>.</p></div> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="init/3">init/3</h3> <h3 id="init/4">init/4</h3> <div class="description"> <p>Initiates the event handler. This function returns <code><anno>Args</anno></code>, which should be used in a call to <code>gen_event:add_handler(EventMgr, log_mf_h, <anno>Args</anno>)</code>. </p> <p><code><anno>Dir</anno></code> specifies which directory to use for the log files. <code><anno>MaxBytes</anno></code> specifies the size of each individual file. <code><anno>MaxFiles</anno></code> specifies how many files are used. <code><anno>Pred</anno></code> is a predicate function used to filter the events. If no predicate function is specified, all events are logged.</p> </div> </div> </div> <div class="section"> <h4>See Also</h4> <p><a href="gen_event.html" class="seealso">gen_event(3)</a>, rb(3) </p> </div>
