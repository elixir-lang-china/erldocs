<h1>gb_trees</h1> <h2 class="modsummary">General Balanced Trees</h2> <div class="description"> <p>An efficient implementation of Prof. Arne Andersson's General Balanced Trees. These have no storage overhead compared to unbalanced binary trees, and their performance is in general better than AVL trees.</p> <p>This module considers two keys as different if and only if they do not compare equal (<code>==</code>).</p> </div> <div class="section"> <h4>Data structure</h4> <p>Data structure:</p> <pre class="sh_erlang"> - {Size, Tree}, where `Tree' is composed of nodes of the form: - {Key, Value, Smaller, Bigger}, and the "empty tree" node: - nil.</pre> <p>There is no attempt to balance trees after deletions. Since deletions do not increase the height of a tree, this should be OK.</p> <p>Original balance condition <em>h(T) &lt;= ceil(c * log(|T|))</em> has been changed to the similar (but not quite equivalent) condition <em>2 ^ h(T) &lt;= |T| ^ c</em>. This should also be OK.</p> <p>Performance is comparable to the AVL trees in the Erlang book (and faster in general due to less overhead); the difference is that deletion works for these trees, but not for the book's trees. Behaviour is logarithmic (as it should be).</p> </div> <datatypes> <datatype> <div class="description"><p>A GB tree.</p></div> </datatype> <datatype> <name name="iter" /> <div class="description"><p>A GB tree iterator.</p></div> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="balance/1">balance/1</h3> <div class="description"> <p>Rebalances <code><anno>Tree1</anno></code>. Note that this is rarely necessary, but may be motivated when a large number of nodes have been deleted from the tree without further insertions. Rebalancing could then be forced in order to minimise lookup times, since deletion only does not rebalance the tree.</p> </div> </div> <div class="function"> <h3 id="delete/2">delete/2</h3> <div class="description"> <p>Removes the node with key <code><anno>Key</anno></code> from <code><anno>Tree1</anno></code>; returns new tree. Assumes that the key is present in the tree, crashes otherwise.</p> </div> </div> <div class="function"> <h3 id="delete_any/2">delete_any/2</h3> <div class="description"> <p>Removes the node with key <code><anno>Key</anno></code> from <code><anno>Tree1</anno></code> if the key is present in the tree, otherwise does nothing; returns new tree.</p> </div> </div> <div class="function"> <h3 id="empty/0">empty/0</h3> <div class="description"> <p>Returns a new empty tree</p> </div> </div> <div class="function"> <h3 id="enter/3">enter/3</h3> <div class="description"> <p>Inserts <code><anno>Key</anno></code> with value <code><anno>Val</anno></code> into <code><anno>Tree1</anno></code> if the key is not present in the tree, otherwise updates <code><anno>Key</anno></code> to value <code><anno>Val</anno></code> in <code><anno>Tree1</anno></code>. Returns the new tree.</p> </div> </div> <div class="function"> <h3 id="from_orddict/1">from_orddict/1</h3> <div class="description"> <p>Turns an ordered list <code><anno>List</anno></code> of key-value tuples into a tree. The list must not contain duplicate keys.</p> </div> </div> <div class="function"> <h3 id="get/2">get/2</h3> <div class="description"> <p>Retrieves the value stored with <code><anno>Key</anno></code> in <code><anno>Tree</anno></code>. Assumes that the key is present in the tree, crashes otherwise.</p> </div> </div> <div class="function"> <h3 id="insert/3">insert/3</h3> <div class="description"> <p>Inserts <code><anno>Key</anno></code> with value <code><anno>Val</anno></code> into <code><anno>Tree1</anno></code>; returns the new tree. Assumes that the key is not present in the tree, crashes otherwise.</p> </div> </div> <div class="function"> <h3 id="is_defined/2">is_defined/2</h3> <div class="description"> <p>Returns <code>true</code> if <code><anno>Key</anno></code> is present in <code><anno>Tree</anno></code>, otherwise <code>false</code>.</p> </div> </div> <div class="function"> <h3 id="is_empty/1">is_empty/1</h3> <div class="description"> <p>Returns <code>true</code> if <code><anno>Tree</anno></code> is an empty tree, and <code>false</code> otherwise.</p> </div> </div> <div class="function"> <h3 id="iterator/1">iterator/1</h3> <div class="description"> <p>Returns an iterator that can be used for traversing the entries of <code><anno>Tree</anno></code>; see <code>next/1</code>. The implementation of this is very efficient; traversing the whole tree using <code>next/1</code> is only slightly slower than getting the list of all elements using <code>to_list/1</code> and traversing that. The main advantage of the iterator approach is that it does not require the complete list of all elements to be built in memory at one time.</p> </div> </div> <div class="function"> <h3 id="keys/1">keys/1</h3> <div class="description"> <p>Returns the keys in <code><anno>Tree</anno></code> as an ordered list.</p> </div> </div> <div class="function"> <h3 id="largest/1">largest/1</h3> <div class="description"> <p>Returns <code>{<anno>Key</anno>, <anno>Val</anno>}</code>, where <code><anno>Key</anno></code> is the largest key in <code><anno>Tree</anno></code>, and <code><anno>Val</anno></code> is the value associated with this key. Assumes that the tree is nonempty.</p> </div> </div> <div class="function"> <h3 id="lookup/2">lookup/2</h3> <div class="description"> <p>Looks up <code><anno>Key</anno></code> in <code><anno>Tree</anno></code>; returns <code>{value, <anno>Val</anno>}</code>, or <code>none</code> if <code><anno>Key</anno></code> is not present.</p> </div> </div> <div class="function"> <h3 id="map/2">map/2</h3> <div class="description"><p>Maps the function F(<anno>K</anno>, <anno>V1</anno>) -&gt; <anno>V2</anno> to all key-value pairs of the tree <code><anno>Tree1</anno></code> and returns a new tree <code><anno>Tree2</anno></code> with the same set of keys as <code><anno>Tree1</anno></code> and the new set of values <code><anno>V2</anno></code>.</p> </div> </div> <div class="function"> <h3 id="next/1">next/1</h3> <div class="description"> <p>Returns <code>{<anno>Key</anno>, <anno>Val</anno>, <anno>Iter2</anno>}</code> where <code><anno>Key</anno></code> is the smallest key referred to by the iterator <code><anno>Iter1</anno></code>, and <code><anno>Iter2</anno></code> is the new iterator to be used for traversing the remaining nodes, or the atom <code>none</code> if no nodes remain.</p> </div> </div> <div class="function"> <h3 id="size/1">size/1</h3> <div class="description"> <p>Returns the number of nodes in <code><anno>Tree</anno></code>.</p> </div> </div> <div class="function"> <h3 id="smallest/1">smallest/1</h3> <div class="description"> <p>Returns <code>{<anno>Key</anno>, <anno>Val</anno>}</code>, where <code><anno>Key</anno></code> is the smallest key in <code><anno>Tree</anno></code>, and <code><anno>Val</anno></code> is the value associated with this key.
