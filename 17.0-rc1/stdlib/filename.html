<h1>filename</h1> <h2 class="modsummary">Filename Manipulation Functions</h2> <div class="description"> <p>The module <code>filename</code> provides a number of useful functions for analyzing and manipulating file names. These functions are designed so that the Erlang code can work on many different platforms with different formats for file names. With file name is meant all strings that can be used to denote a file. They can be short relative names like <code>foo.erl</code>, very long absolute name which include a drive designator and directory names like <code>D:\usr/local\bin\erl/lib\tools\foo.erl</code>, or any variations in between.</p> <p>In Windows, all functions return file names with forward slashes only, even if the arguments contain back slashes. Use <code>join/1</code> to normalize a file name by removing redundant directory separators.</p> <p>The module supports raw file names in the way that if a binary is present, or the file name cannot be interpreted according to the return value of <a href="../kernel/file.html#native_name_encoding/0" class="seealso">file:native_name_encoding/0</a>, a raw file name will also be returned. For example filename:join/1 provided with a path component being a binary (and also not being possible to interpret under the current native file name encoding) will result in a raw file name being returned (the join operation will have been performed of course). For more information about raw file names, see the <a href="../kernel/file.html" class="seealso">file</a> module.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="absname/1">absname/1</h3> <div class="description"> <p>Converts a relative <code><anno>Filename</anno></code> and returns an absolute name. No attempt is made to create the shortest absolute name, because this can give incorrect results on file systems which allow links.</p> <p>Unix examples:</p> <pre class="sh_erlang"> 1&gt; <code>pwd().</code> "/usr/local" 2&gt; <code>filename:absname("foo").</code> "/usr/local/foo" 3&gt; <code>filename:absname("../x").</code> "/usr/local/../x" 4&gt; <code>filename:absname("/").</code> "/"</pre> <p>Windows examples:</p> <pre class="sh_erlang"> 1&gt; <code>pwd().</code> "D:/usr/local" 2&gt; <code>filename:absname("foo").</code> "D:/usr/local/foo" 3&gt; <code>filename:absname("../x").</code> "D:/usr/local/../x" 4&gt; <code>filename:absname("/").</code> "D:/"</pre> </div> </div> <div class="function"> <h3 id="absname/2">absname/2</h3> <div class="description"> <p>This function works like <code>absname/1</code>, except that the directory to which the file name should be made relative is given explicitly in the <code><anno>Dir</anno></code> argument.</p> </div> </div> <div class="function"> <h3 id="absname_join/2">absname_join/2</h3> <div class="description"> <p>Joins an absolute directory with a relative filename. Similar to <code>join/2</code>, but on platforms with tight restrictions on raw filename length and no support for symbolic links (read: VxWorks), leading parent directory components in <code><anno>Filename</anno></code> are matched against trailing directory components in <code><anno>Dir</anno></code> so they can be removed from the result - minimizing its length.</p> </div> </div> <div class="function"> <h3 id="basename/1">basename/1</h3> <div class="description"> <p>Returns the last component of <code><anno>Filename</anno></code>, or <code><anno>Filename</anno></code> itself if it does not contain any directory separators.</p> <pre class="sh_erlang"> 5&gt; <code>filename:basename("foo").</code> "foo" 6&gt; <code>filename:basename("/usr/foo").</code> "foo" 7&gt; <code>filename:basename("/").</code> []</pre> </div> </div> <div class="function"> <h3 id="basename/2">basename/2</h3> <div class="description"> <p>Returns the last component of <code><anno>Filename</anno></code> with the extension <code><anno>Ext</anno></code> stripped. This function should be used to remove a specific extension which might, or might not, be there. Use <code>rootname(basename(Filename))</code> to remove an extension that exists, but you are not sure which one it is.</p> <pre class="sh_erlang"> 8&gt; <code>filename:basename("~/src/kalle.erl", ".erl").</code> "kalle" 9&gt; <code>filename:basename("~/src/kalle.beam", ".erl").</code> "kalle.beam" 10&gt; <code>filename:basename("~/src/kalle.old.erl", ".erl").</code> "kalle.old" 11&gt; <code>filename:rootname(filename:basename("~/src/kalle.erl")).</code> "kalle" 12&gt; <code>filename:rootname(filename:basename("~/src/kalle.beam")).</code> "kalle"</pre> </div> </div> <div class="function"> <h3 id="dirname/1">dirname/1</h3> <div class="description"> <p>Returns the directory part of <code><anno>Filename</anno></code>.</p> <pre class="sh_erlang"> 13&gt; <code>filename:dirname("/usr/src/kalle.erl").</code> "/usr/src" 14&gt; <code>filename:dirname("kalle.erl").</code> "." 5&gt; <code>filename:dirname("\\usr\\src/kalle.erl").</code> % Windows "/usr/src"</pre> </div> </div> <div class="function"> <h3 id="extension/1">extension/1</h3> <div class="description"> <p>Returns the file extension of <code><anno>Filename</anno></code>, including the period. Returns an empty string if there is no extension.</p> <pre class="sh_erlang"> 15&gt; <code>filename:extension("foo.erl").</code> ".erl" 16&gt; <code>filename:extension("beam.src/kalle").</code> []</pre> </div> </div> <div class="function"> <h3 id="flatten/1">flatten/1</h3> <div class="description"> <p>Converts a possibly deep list filename consisting of characters and atoms into the corresponding flat string filename.</p> </div> </div> <div class="function"> <h3 id="join/1">join/1</h3> <div class="description"> <p>Joins a list of file name <code><anno>Components</anno></code> with directory separators. If one of the elements of <code><anno>Components</anno></code> includes an absolute path, for example <code>"/xxx"</code>, the preceding elements, if any, are removed from the result.</p> <p>The result is "normalized":</p> <list type="bulleted"> <item>Redundant directory separators are removed.</item> <item>In Windows, all directory separators are forward slashes and the drive letter is in lower case.</item> </list> <pre class="sh_erlang"> 17&gt; <code>filename:join(["/usr", "local", "bin"]).</code> "/usr/local/bin" 18&gt; <code>filename:join(["a/b///c/"]).</code> "a/b/c" 6&gt; <code>filename:join(["B:a\\b///c/"]).</code> % Windows "b:a/b/c"</pre> </div> </div> <div class="function"> <h3 id="join/2">join/2</h3> <div class="description"> <p>Joins two file name components with directory separators. Equivalent to <code>join([<anno>Name1</anno>, <anno>Name2</anno>])</code>.</p> </div> </div> <div class="function"> <h3 id="nativename/1">nativename/1</h3> <div class="description"> <p>Converts <code><anno>Path</anno></code> to a form accepted by the command shell and native applications on the current platform. On Windows, forward slashes is converted to backward slashes. On all platforms, the name is normalized as done by <code>join/1</code>.</p> <pre class="sh_erlang"> 19&gt; <code>filename:nativename("/usr/local/bin/").</code> % Unix "/usr/local/bin" 7&gt; <code>filename:nativename("/usr/local/bin/").</code> % Windows "\\usr\\local\\bin"</pre> </div> </div> <div class="function"> <h3 id="pathtype/1">pathtype/1</h3> <div class="description"> <p>Returns the type of path, one of <code>absolute</code>, <code>relative</code>, or <code>volumerelative</code>.</p> <taglist> <dt><code>absolute</code></dt> <item> <p>The path name refers to a specific file on a specific volume.</p> <p>Unix example: <code>/usr/local/bin</code></p> <p>Windows example: <code>D:/usr/local/bin</code></p> </item> <dt><code>relative</code></dt> <item> <p>The path name is relative to the current working directory on the current volume.</p> <p>Example: <code>foo/bar, ../src</code></p> </item> <dt><code>volumerelative</code></dt> <item> <p>The path name is relative to the current working directory on a specified volume, or it is a specific file on the current working volume.</p> <p>Windows example: <code>D:bar.erl, /bar/foo.erl</code></p> </item> </taglist> </div> </div> <div class="function"> <h3 id="rootname/1">rootname/1</h3> <h3 id="rootname/2">rootname/2</h3> <div class="description"> <p>Remove a filename extension. <code>rootname/2</code> works as <code>rootname/1</code>, except that the extension is removed only if it is <code><anno>Ext</anno></code>.</p> <pre class="sh_erlang"> 20&gt; <code>filename:rootname("/beam.src/kalle").</code> /beam.src/kalle" 21&gt; <code>filename:rootname("/beam.src/foo.erl").</code> "/beam.src/foo" 22&gt; <code>filename:rootname("/beam.src/foo.erl", ".erl").</code> "/beam.src/foo" 23&gt; <code>filename:rootname("/beam.src/foo.beam", ".erl").</code> "/beam.src/foo.beam"</pre> </div> </div> <div class="function"> <h3 id="split/1">split/1</h3> <div class="description"> <p>Returns a list whose elements are the path components of <code><anno>Filename</anno></code>.</p> <pre class="sh_erlang"> 24&gt; <code>filename:split("/usr/local/bin").</code> ["/","usr","local","bin"] 25&gt; <code>filename:split("foo/bar").</code> ["foo","bar"] 26&gt; <code>filename:split("a:\\msdev\\include").</code> ["a:/","msdev","include"]</pre> </div> </div> <div class="function"> <h3 id="find_src/1">find_src/1</h3> <h3 id="find_src/2">find_src/2</h3> <div class="description"> <p>Finds the source filename and compiler options for a module. The result can be fed to <code>compile:file/2</code> in order to compile the file again.</p> <div class="warning"><h2>Warning!</h2><p>We don't recommend using this function. If possible, use <a href="beam_lib.html" class="seealso">beam_lib(3)</a> to extract the abstract code format from the BEAM file and compile that instead.</p></div> <p>The <code><anno>Beam</anno></code> argument, which can be a string or an atom, specifies either the module name or the path to the source code, with or without the <code>".erl"</code> extension. In either case, the module must be known by the code server, i.e. <code>code:which(<anno>Module</anno>)</code> must succeed.</p> <p><code><anno>Rules</anno></code> describes how the source directory can be found, when the object code directory is known. It is a list of tuples <code>{<anno>BinSuffix</anno>, <anno>SourceSuffix</anno>}</code> and is interpreted as follows: If the end of the directory name where the object is located matches <code><anno>BinSuffix</anno></code>, then the source code directory has the same name, but with <code><anno>BinSuffix</anno></code> replaced by <code><anno>SourceSuffix</anno></code>. <code><anno>Rules</anno></code> defaults to:</p> <pre class="sh_erlang"> [{"", ""}, {"ebin", "src"}, {"ebin", "esrc"}]</pre> <p>If the source file is found in the resulting directory, then the function returns that location together with <code><anno>Options</anno></code>. Otherwise, the next rule is tried, and so on.</p> <p>The function returns <code>{<anno>SourceFile</anno>, <anno>Options</anno>}</code> if it succeeds. <code><anno>SourceFile</anno></code> is the absolute path to the source file without the <code>".erl"</code> extension. <code><anno>Options</anno></code> include the options which are necessary to recompile the file with <code>compile:file/2</code>, but excludes options such as <code>report</code> or <code>verbose</code> which do not change the way code is generated. The paths in the <code>{outdir, <anno>Path</anno>}</code> and <code>{i, Path}</code> options are guaranteed to be absolute.</p> </div> </div> </div>
