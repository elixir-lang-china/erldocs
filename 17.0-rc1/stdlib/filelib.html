<h1>filelib</h1> <h2 class="modsummary">File utilities, such as wildcard matching of filenames</h2> <div class="description"> <p>This module contains utilities on a higher level than the <code>file</code> module.</p> <p>This module does not support "raw" file names (i.e. files whose names do not comply with the expected encoding). Such files will be ignored by the functions in this module.</p> <p>For more information about raw file names, see the <a href="../kernel/file.html" class="seealso">file</a> module.</p> </div> <datatypes> <datatype> <name name="filename" /> </datatype> <datatype> <name name="dirname" /> </datatype> <datatype> <name name="dirname_all" /> </datatype> <datatype> <name name="filename_all" /> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="ensure_dir/1">ensure_dir/1</h3> <div class="description"> <p>The <code>ensure_dir/1</code> function ensures that all parent directories for the given file or directory name <code><anno>Name</anno></code> exist, trying to create them if necessary.</p> <p>Returns <code>ok</code> if all parent directories already exist or could be created, or <code>{error, <anno>Reason</anno>}</code> if some parent directory does not exist and could not be created for some reason.</p> </div> </div> <div class="function"> <h3 id="file_size/1">file_size/1</h3> <div class="description"> <p>The <code>file_size</code> function returns the size of the given file.</p> </div> </div> <div class="function"> <h3 id="fold_files/5">fold_files/5</h3> <div class="description"> <p>The <code>fold_files/5</code> function folds the function <code><anno>Fun</anno></code> over all (regular) files <code><anno>F</anno></code> in the directory <code><anno>Dir</anno></code> that match the regular expression <code><anno>RegExp</anno></code> (see the <a href="re.html" class="seealso">re</a> module for a description of the allowed regular expressions). If <code><anno>Recursive</anno></code> is true all sub-directories to <code>Dir</code> are processed. The regular expression matching is done on just the filename without the directory part.</p> <p>If Unicode file name translation is in effect and the file system is completely transparent, file names that cannot be interpreted as Unicode may be encountered, in which case the <code>fun()</code> must be prepared to handle raw file names (i.e. binaries). If the regular expression contains codepoints beyond 255, it will not match file names that do not conform to the expected character encoding (i.e. are not encoded in valid UTF-8).</p> <p>For more information about raw file names, see the <a href="../kernel/file.html" class="seealso">file</a> module.</p> </div> </div> <div class="function"> <h3 id="is_dir/1">is_dir/1</h3> <div class="description"> <p>The <code>is_dir/1</code> function returns <code>true</code> if <code><anno>Name</anno></code> refers to a directory, and <code>false</code> otherwise.</p> </div> </div> <div class="function"> <h3 id="is_file/1">is_file/1</h3> <div class="description"> <p>The <code>is_file/1</code> function returns <code>true</code> if <code><anno>Name</anno></code> refers to a file or a directory, and <code>false</code> otherwise.</p> </div> </div> <div class="function"> <h3 id="is_regular/1">is_regular/1</h3> <div class="description"> <p>The <code>is_regular/1</code> function returns <code>true</code> if <code><anno>Name</anno></code> refers to a file (regular file), and <code>false</code> otherwise.</p> </div> </div> <div class="function"> <h3 id="last_modified/1">last_modified/1</h3> <div class="description"> <p>The <code>last_modified/1</code> function returns the date and time the given file or directory was last modified, or 0 if the file does not exist.</p> </div> </div> <div class="function"> <h3 id="wildcard/1">wildcard/1</h3> <div class="description"> <p>The <code>wildcard/1</code> function returns a list of all files that match Unix-style wildcard-string <code><anno>Wildcard</anno></code>.</p> <p>The wildcard string looks like an ordinary filename, except that certain "wildcard characters" are interpreted in a special way. The following characters are special: </p> <taglist> <dt>?</dt> <item> <p>Matches one character.</p> </item> <dt>*</dt> <item> <p>Matches any number of characters up to the end of the filename, the next dot, or the next slash.</p> </item> <dt>**</dt> <item> <p>Two adjacent <code>*</code>'s used as a single pattern will match all files and zero or more directories and subdirectories.</p> </item> <dt>[Character1,Character2,...]</dt> <item> <p>Matches any of the characters listed. Two characters separated by a hyphen will match a range of characters. Example: <code>[A-Z]</code> will match any uppercase letter.</p> </item> <dt>{Item,...}</dt> <item> <p>Alternation. Matches one of the alternatives.</p> </item> </taglist> <p>Other characters represent themselves. Only filenames that have exactly the same character in the same position will match. (Matching is case-sensitive; i.e. "a" will not match "A"). </p> <p>Note that multiple "*" characters are allowed (as in Unix wildcards, but opposed to Windows/DOS wildcards). </p> <p>Examples:</p> <p>The following examples assume that the current directory is the top of an Erlang/OTP installation. </p> <p>To find all <code>.beam</code> files in all applications, the following line can be used:</p> <pre class="sh_erlang"> filelib:wildcard("lib/*/ebin/*.beam"). </pre> <p>To find either <code>.erl</code> or <code>.hrl</code> in all applications <code>src</code> directories, the following</p> <pre class="sh_erlang"> filelib:wildcard("lib/*/src/*.?rl") </pre> <p>or the following line</p> <pre class="sh_erlang"> filelib:wildcard("lib/*/src/*.{erl,hrl}") </pre> <p>can be used.</p> <p>To find all <code>.hrl</code> files in either <code>src</code> or <code>include</code> directories, use:</p> <pre class="sh_erlang"> filelib:wildcard("lib/*/{src,include}/*.hrl"). </pre> <p>To find all <code>.erl</code> or <code>.hrl</code> files in either <code>src</code> or <code>include</code> directories, use:</p> <pre class="sh_erlang"> filelib:wildcard("lib/*/{src,include}/*.{erl,hrl}") </pre> <p>To find all <code>.erl</code> or <code>.hrl</code> files in any subdirectory, use:</p> <pre class="sh_erlang"> filelib:wildcard("lib/**/*.{erl,hrl}") </pre> </div> </div> <div class="function"> <h3 id="wildcard/2">wildcard/2</h3> <div class="description"> <p>The <code>wildcard/2</code> function works like <code>wildcard/1</code>, except that instead of the actual working directory, <code><anno>Cwd</anno></code> will be used.</p> </div> </div> </div>
