<h1>erl_id_trans</h1> <h2 class="modsummary">An Identity Parse Transform</h2> <div class="description"> <p>This module performs an identity parse transformation of Erlang code. It is included as an example for users who may wish to write their own parse transformers. If the option <code>{parse_transform,Module}</code> is passed to the compiler, a user written function <code>parse_transform/2</code> is called by the compiler before the code is checked for errors.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="parse_transform/2">parse_transform(Forms, Options) -&gt; Forms</h3> <ul class="type"> <li><code>Forms = [<a href="erl_parse.html#type-abstract_form" class="seealso">erl_parse:abstract_form()</a>]</code></li> <li><code>Options = [<a href="compile.html#type-option" class="seealso">compile:option()</a>]</code></li> </ul> <div class="description"> <p>Performs an identity transformation on Erlang forms, as an example.</p> </div> </div> </div> <div class="section"> <h4>Parse Transformations</h4> <p>Parse transformations are used if a programmer wants to use Erlang syntax, but with different semantics. The original Erlang code is then transformed into other Erlang code. </p> <div class="note"><h2>Note!</h2> <p>Programmers are strongly advised not to engage in parse transformations and no support is offered for problems encountered.</p> </div> </div> <div class="section"> <h4>See Also</h4> <p><a href="erl_parse.html" class="seealso">erl_parse(3)</a>, <a href="../compiler/compile.html" class="seealso">compile(3)</a>.</p> </div>
