<h1>digraph_utils</h1> <h2 class="modsummary">Algorithms for Directed Graphs</h2> <div class="description"> <p>The <code>digraph_utils</code> module implements some algorithms based on depth-first traversal of directed graphs. See the <code>digraph</code> module for basic functions on directed graphs. </p> <p>A <span id="digraph"> </span><em>directed graph</em> (or just "digraph") is a pair (V,&nbsp;E) of a finite set V of <span id="vertex"> </span><em>vertices</em> and a finite set E of <span id="edge"> </span><em>directed edges</em> (or just "edges"). The set of edges E is a subset of V&nbsp;×&nbsp;V (the Cartesian product of V with itself). </p> <p>Digraphs can be annotated with additional information. Such information may be attached to the vertices and to the edges of the digraph. A digraph which has been annotated is called a <em>labeled digraph</em>, and the information attached to a vertex or an edge is called a <span id="label"> </span> <em>label</em>.</p> <p>An edge e&nbsp;=&nbsp;(v,&nbsp;w) is said to <span id="emanate"> </span><em>emanate</em> from vertex v and to be <span id="incident"> </span><em>incident</em> on vertex w. If there is an edge emanating from v and incident on w, then w is said to be an <span id="out_neighbour"> </span><em>out-neighbour</em> of v, and v is said to be an <span id="in_neighbour"> </span><em>in-neighbour</em> of w. A <span id="path"> </span><em>path</em> P from v[1] to v[k] in a digraph (V,&nbsp;E) is a non-empty sequence v[1],&nbsp;v[2],&nbsp;...,&nbsp;v[k] of vertices in V such that there is an edge (v[i],v[i+1]) in E for 1&nbsp;&lt;=&nbsp;i&nbsp;&lt;&nbsp;k. The <span id="length"> </span><em>length</em> of the path P is k-1. P is a <span id="cycle"> </span><em>cycle</em> if the length of P is not zero and v[1] = v[k]. A <span id="loop"> </span><em>loop</em> is a cycle of length one. An <span id="acyclic_digraph"> </span><em>acyclic digraph</em> is a digraph that has no cycles. </p> <p>A <span id="depth_first_traversal"> </span> <em>depth-first traversal</em> of a directed digraph can be viewed as a process that visits all vertices of the digraph. Initially, all vertices are marked as unvisited. The traversal starts with an arbitrarily chosen vertex, which is marked as visited, and follows an edge to an unmarked vertex, marking that vertex. The search then proceeds from that vertex in the same fashion, until there is no edge leading to an unvisited vertex. At that point the process backtracks, and the traversal continues as long as there are unexamined edges. If there remain unvisited vertices when all edges from the first vertex have been examined, some hitherto unvisited vertex is chosen, and the process is repeated. </p> <p>A <span id="partial_ordering"> </span><em>partial ordering</em> of a set S is a transitive, antisymmetric and reflexive relation between the objects of S. The problem of <span id="topsort"> </span><em>topological sorting</em> is to find a total ordering of S that is a superset of the partial ordering. A digraph G&nbsp;=&nbsp;(V,&nbsp;E) is equivalent to a relation E on V (we neglect the fact that the version of directed graphs implemented in the <code>digraph</code> module allows multiple edges between vertices). If the digraph has no cycles of length two or more, then the reflexive and transitive closure of E is a partial ordering. </p> <p>A <span id="subgraph"> </span><em>subgraph</em> G' of G is a digraph whose vertices and edges form subsets of the vertices and edges of G. G' is <em>maximal</em> with respect to a property P if all other subgraphs that include the vertices of G' do not have the property P. A <span id="strong_components"> </span> <em>strongly connected component</em> is a maximal subgraph such that there is a path between each pair of vertices. A <span id="components"> </span><em>connected component</em> is a maximal subgraph such that there is a path between each pair of vertices, considering all edges undirected. An <span id="arborescence"> </span><em>arborescence</em> is an acyclic digraph with a vertex V, the <span id="root"> </span><em>root</em>, such that there is a unique path from V to every other vertex of G. A <span id="tree"> </span><em>tree</em> is an acyclic non-empty digraph such that there is a unique path between every pair of vertices, considering all edges undirected.</p> </div> <datatypes> <datatype> <div class="description"><p>A digraph as returned by <code>digraph:new/0,1</code>.</p></div> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="arborescence_root/1">arborescence_root/1</h3> <div class="description"> <p>Returns <code>{yes, <anno>Root</anno>}</code> if <code><anno>Root</anno></code> is the <a href="#root" class="seealso">root</a> of the arborescence <code><anno>Digraph</anno></code>, <code>no</code> otherwise. </p> </div> </div> <div class="function"> <h3 id="components/1">components/1</h3> <div class="description"> <p>Returns a list of <a href="#components" class="seealso">connected components</a>. Each component is represented by its vertices. The order of the vertices and the order of the components are arbitrary. Each vertex of the digraph <code><anno>Digraph</anno></code> occurs in exactly one component. </p> </div> </div> <div class="function"> <h3 id="condensation/1">condensation/1</h3> <div class="description"> <p>Creates a digraph where the vertices are the <a href="#strong_components" class="seealso">strongly connected components</a> of <code><anno>Digraph</anno></code> as returned by <code>strong_components/1</code>. If X and Y are two different strongly connected components, and there exist vertices x and y in X and Y respectively such that there is an edge <a href="#emanate" class="seealso">emanating</a> from x and <a href="#incident" class="seealso">incident</a> on y, then an edge emanating from X and incident on Y is created. </p> <p>The created digraph has the same type as <code><anno>Digraph</anno></code>. All vertices and edges have the default <a href="#label" class="seealso">label</a> <code>[]</code>. </p> <p>Each and every <a href="#cycle" class="seealso">cycle</a> is included in some strongly connected component, which implies that there always exists a <a href="#topsort" class="seealso">topological ordering</a> of the created digraph.</p> </div> </div> <div class="function"> <h3 id="cyclic_strong_components/1">cyclic_strong_components/1</h3> <div class="description"> <p>Returns a list of <a href="#strong_components" class="seealso">strongly connected components</a>. Each strongly component is represented by its vertices. The order of the vertices and the order of the components are arbitrary. Only vertices that are included in some <a href="#cycle" class="seealso">cycle</a> in <code><anno>Digraph</anno></code> are returned, otherwise the returned list is equal to that returned by <code>strong_components/1</code>. </p> </div> </div> <div class="function"> <h3 id="is_acyclic/1">is_acyclic/1</h3> <div class="description"> <p>Returns <code>true</code> if and only if the digraph <code><anno>Digraph</anno></code> is <a href="#acyclic_digraph" class="seealso">acyclic</a>.</p> </div> </div> <div class="function"> <h3 id="is_arborescence/1">is_arborescence/1</h3> <div class="description"> <p>Returns <code>true</code> if and only if the digraph <code><anno>Digraph</anno></code> is an <a href="#arborescence" class="seealso">arborescence</a>.</p> </div> </div> <div class="function"> <h3 id="is_tree/1">is_tree/1</h3> <div class="description"> <p>Returns <code>true</code> if and only if the digraph <code><anno>Digraph</anno></code> is a <a href="#tree" class="seealso">tree</a>.</p> </div> </div> <div class="function"> <h3 id="loop_vertices/1">loop_vertices/1</h3> <div class="description"> <p>Returns a list of all vertices of <code><anno>Digraph</anno></code> that are included in some <a href="#loop" class="seealso">loop</a>.</p> </div> </div> <div class="function"> <h3 id="postorder/1">postorder/1</h3> <div class="description"> <p>Returns all vertices of the digraph <code><anno>Digraph</anno></code>. The order is given by a <a href="#depth_first_traversal" class="seealso">depth-first traversal</a> of the digraph, collecting visited vertices in postorder. More precisely, the vertices visited while searching from an arbitrarily chosen vertex are collected in postorder, and all those collected vertices are placed before the subsequently visited vertices. </p> </div> </div> <div class="function"> <h3 id="preorder/1">preorder/1</h3> <div class="description"> <p>Returns all vertices of the digraph <code><anno>Digraph</anno></code>. The order is given by a <a href="#depth_first_traversal" class="seealso">depth-first traversal</a> of the digraph, collecting visited vertices in pre-order.</p> </div> </div> <div class="function"> <h3 id="reachable/2">reachable/2</h3> <div class="description"> <p>Returns an unsorted list of digraph vertices such that for each vertex in the list, there is a <a href="#path" class="seealso">path</a> in <code><anno>Digraph</anno></code> from some vertex of <code><anno>Vertices</anno></code> to the vertex. In particular, since paths may have length zero, the vertices of <code><anno>Vertices</anno></code> are included in the returned list. </p> </div> </div> <div class="function"> <h3 id="reachable_neighbours/2">reachable_neighbours/2</h3> <div class="description"> <p>Returns an unsorted list of digraph vertices such that for each vertex in the list, there is a <a href="#path" class="seealso">path</a> in <code><anno>Digraph</anno></code> of length one or more from some vertex of <code><anno>Vertices</anno></code> to the vertex. As a consequence, only those vertices of <code><anno>Vertices</anno></code> that are included in some <a href="#cycle" class="seealso">cycle</a> are returned. </p> </div> </div> <div class="function"> <h3 id="reaching/2">reaching/2</h3> <div class="description"> <p>Returns an unsorted list of digraph vertices such that for each vertex in the list, there is a <a href="#path" class="seealso">path</a> from the vertex to some vertex of <code><anno>Vertices</anno></code>. In particular, since paths may have length zero, the vertices of <code><anno>Vertices</anno></code> are included in the returned list. </p> </div> </div> <div class="function"> <h3 id="reaching_neighbours/2">reaching_neighbours/2</h3> <div class="description"> <p>Returns an unsorted list of digraph vertices such that for each vertex in the list, there is a <a href="#path" class="seealso">path</a> of length one or more from the vertex to some vertex of <code><anno>Vertices</anno></code>. As a consequence, only those vertices of <code><anno>Vertices</anno></code> that are included in some <a href="#cycle" class="seealso">cycle</a> are returned. </p> </div> </div> <div class="function"> <h3 id="strong_components/1">strong_components/1</h3> <div class="description"> <p>Returns a list of <a href="#strong_components" class="seealso">strongly connected components</a>. Each strongly component is represented by its vertices. The order of the vertices and the order of the components are arbitrary. Each vertex of the digraph <code><anno>Digraph</anno></code> occurs in exactly one strong component. </p> </div> </div> <div class="function"> <h3 id="subgraph/2">subgraph/2</h3> <h3 id="subgraph/3">subgraph/3</h3> <div class="description"> <p>Creates a maximal <a href="#subgraph" class="seealso">subgraph</a> of <code>Digraph</code> having as vertices those vertices of <code><anno>Digraph</anno></code> that are mentioned in <code><anno>Vertices</anno></code>. </p> <p>If the value of the option <code>type</code> is <code>inherit</code>, which is the default, then the type of <code><anno>Digraph</anno></code> is used for the subgraph as well. Otherwise the option value of <code>type</code> is used as argument to <code>digraph:new/1</code>. </p> <p>If the value of the option <code>keep_labels</code> is <code>true</code>, which is the default, then the <a href="#label" class="seealso">labels</a> of vertices and edges of <code><anno>Digraph</anno></code> are used for the subgraph as well. If the value is <code>false</code>, then the default label, <code>[]</code>, is used for the subgraph's vertices and edges. </p> <p><code>subgraph(<anno>Digraph</anno>, <anno>Vertices</anno>)</code> is equivalent to <code>subgraph(<anno>Digraph</anno>, <anno>Vertices</anno>, [])</code>. </p> <p>There will be a <code>badarg</code> exception if any of the arguments are invalid. </p> </div> </div> <div class="function"> <h3 id="topsort/1">topsort/1</h3> <div class="description"> <p>Returns a <a href="#topsort" class="seealso">topological ordering</a> of the vertices of the digraph <code><anno>Digraph</anno></code> if such an ordering exists, <code>false</code> otherwise. For each vertex in the returned list, there are no <a href="#out_neighbour" class="seealso">out-neighbours</a> that occur earlier in the list.</p> </div> </div> </div> <div class="section"> <h4>See Also</h4> <p><a href="digraph.html" class="seealso">digraph(3)</a></p> </div>
