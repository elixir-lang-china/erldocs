<h1>io_lib</h1> <h2 class="modsummary">IO Library Functions</h2> <div class="description"> <p>This module contains functions for converting to and from strings (lists of characters). They are used for implementing the functions in the <code>io</code> module. There is no guarantee that the character lists returned from some of the functions are flat, they can be deep lists. <code>lists:flatten/1</code> can be used for flattening deep lists.</p> </div> <datatypes> <datatype> <name name="chars" /> </datatype> <datatype> <name name="continuation" /> <div class="description"><p>A continuation as returned by <a href="#fread/3" class="seealso"><code>fread/3</code></a>.</p> </div> </datatype> <datatype> <name name="depth" /> </datatype> <datatype> <name name="fread_error" /> </datatype> <datatype> <name name="fread_item" /> </datatype> <datatype> <name name="latin1_string" /> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="nl/0">nl/0</h3> <div class="description"> <p>Returns a character list which represents a new line character.</p> </div> </div> <div class="function"> <h3 id="write/1">write/1</h3> <h3 id="write/2">write/2</h3> <div class="description"> <p>Returns a character list which represents <code><anno>Term</anno></code>. The <code><anno>Depth</anno></code> (-1) argument controls the depth of the structures written. When the specified depth is reached, everything below this level is replaced by "...". For example:</p> <pre class="sh_erlang"> 1&gt; <code>lists:flatten(io_lib:write({1,[2],[3],[4,5],6,7,8,9})).</code> "{1,[2],[3],[4,5],6,7,8,9}" 2&gt; <code>lists:flatten(io_lib:write({1,[2],[3],[4,5],6,7,8,9}, 5)).</code> "{1,[2],[3],[...],...}"</pre> </div> </div> <div class="function"> <h3 id="print/1">print/1</h3> <h3 id="print/4">print/4</h3> <div class="description"> <p>Also returns a list of characters which represents <code><anno>Term</anno></code>, but breaks representations which are longer than one line into many lines and indents each line sensibly. It also tries to detect and output lists of printable characters as strings. <code><anno>Column</anno></code> is the starting column (1), <code><anno>LineLength</anno></code> the maximum line length (80), and <code><anno>Depth</anno></code> (-1) the maximum print depth.</p> </div> </div> <div class="function"> <h3 id="fwrite/2">fwrite/2</h3> <h3 id="format/2">format/2</h3> <div class="description"> <p>Returns a character list which represents <code><anno>Data</anno></code> formatted in accordance with <code><anno>Format</anno></code>. See <a href="io.html#fwrite/1" class="seealso">io:fwrite/1,2,3</a> for a detailed description of the available formatting options. A fault is generated if there is an error in the format string or argument list.</p> <p>If (and only if) the Unicode translation modifier is used in the format string (i.e. ~ts or ~tc), the resulting list may contain characters beyond the ISO-latin-1 character range (in other words, numbers larger than 255). If so, the result is not an ordinary Erlang string(), but can well be used in any context where Unicode data is allowed.</p> </div> </div> <div class="function"> <h3 id="fread/2">fread/2</h3> <div class="description"> <p>Tries to read <code><anno>String</anno></code> in accordance with the control sequences in <code><anno>Format</anno></code>. See <a href="io.html#fread/3" class="seealso">io:fread/3</a> for a detailed description of the available formatting options. It is assumed that <code><anno>String</anno></code> contains whole lines. It returns:</p> <taglist> <dt><code>{ok, <anno>InputList</anno>, <anno>LeftOverChars</anno>}</code></dt> <item> <p>The string was read. <code><anno>InputList</anno></code> is the list of successfully matched and read items, and <code><anno>LeftOverChars</anno></code> are the input characters not used.</p> </item> <dt><code>{more, <anno>RestFormat</anno>, <anno>Nchars</anno>, <anno>InputStack</anno>}</code></dt> <item> <p>The string was read, but more input is needed in order to complete the original format string. <code><anno>RestFormat</anno></code> is the remaining format string, <code><anno>Nchars</anno></code> the number of characters scanned, and <code><anno>InputStack</anno></code> is the reversed list of inputs matched up to that point.</p> </item> <dt><code>{error, <anno>What</anno>}</code></dt> <item> <p>The read operation failed and the parameter <code><anno>What</anno></code> gives a hint about the error.</p> </item> </taglist> <p>Example:</p> <pre class="sh_erlang"> 3&gt; <code>io_lib:fread("~f~f~f", "15.6 17.3e-6 24.5").</code> {ok,[15.6,1.73e-5,24.5],[]}</pre> </div> </div> <div class="function"> <h3 id="fread/3">fread/3</h3> <div class="description"> <p>This is the re-entrant formatted reader. The continuation of the first call to the functions must be <code>[]</code>. Refer to Armstrong, Virding, Williams, 'Concurrent Programming in Erlang', Chapter 13 for a complete description of how the re-entrant input scheme works.</p> <p>The function returns:</p> <taglist> <dt><code>{done, <anno>Result</anno>, <anno>LeftOverChars</anno>}</code></dt> <item> <p>The input is complete. The result is one of the following:</p> <taglist> <dt><code>{ok, <anno>InputList</anno>}</code></dt> <item> <p>The string was read. <code><anno>InputList</anno></code> is the list of successfully matched and read items, and <code><anno>LeftOverChars</anno></code> are the remaining characters.</p> </item> <dt><code>eof</code></dt> <item> <p>End of file has been encountered. <code><anno>LeftOverChars</anno></code> are the input characters not used.</p> </item> <dt><code>{error, <anno>What</anno>}</code></dt> <item> <p>An error occurred and the parameter <code><anno>What</anno></code> gives a hint about the error.</p> </item> </taglist> </item> <dt><code>{more, <anno>Continuation</anno>}</code></dt> <item> <p>More data is required to build a term. <code><anno>Continuation</anno></code> must be passed to <code>fread/3</code>, when more data becomes available.</p> </item> </taglist> </div> </div> <div class="function"> <h3 id="write_atom/1">write_atom/1</h3> <div class="description"> <p>Returns the list of characters needed to print the atom <code><anno>Atom</anno></code>.</p> </div> </div> <div class="function"> <h3 id="write_string/1">write_string/1</h3> <div class="description"> <p>Returns the list of characters needed to print <code><anno>String</anno></code> as a string.</p> </div> </div> <div class="function"> <h3 id="write_string_as_latin1/1">write_string_as_latin1/1</h3> <div class="description"> <p>Returns the list of characters needed to print <code><anno>String</anno></code> as a string. Non-Latin-1 characters are escaped.</p> </div> </div> <div class="function"> <h3 id="write_latin1_string/1">write_latin1_string/1</h3> <div class="description"> <p>Returns the list of characters needed to print <code><anno>Latin1String</anno></code> as a string.</p> </div> </div> <div class="function"> <h3 id="write_char/1">write_char/1</h3> <div class="description"> <p>Returns the list of characters needed to print a character constant in the Unicode character set.</p> </div> </div> <div class="function"> <h3 id="write_char_as_latin1/1">write_char_as_latin1/1</h3> <div class="description"> <p>Returns the list of characters needed to print a character constant in the Unicode character set. Non-Latin-1 characters are escaped.</p> </div> </div> <div class="function"> <h3 id="write_latin1_char/1">write_latin1_char/1</h3> <div class="description"> <p>Returns the list of characters needed to print a character constant in the ISO-latin-1 character set.</p> </div> </div> <div class="function"> <h3 id="indentation/2">indentation/2</h3> <div class="description"> <p>Returns the indentation if <code><anno>String</anno></code> has been printed, starting at <code><anno>StartIndent</anno></code>.</p> </div> </div> <div class="function"> <h3 id="char_list/1">char_list/1</h3> <div class="description"> <p>Returns <code>true</code> if <code><anno>Term</anno></code> is a flat list of characters in the Unicode range, otherwise it returns <code>false</code>.</p> </div> </div> <div class="function"> <h3 id="latin1_char_list/1">latin1_char_list/1</h3> <div class="description"> <p>Returns <code>true</code> if <code><anno>Term</anno></code> is a flat list of characters in the ISO-latin-1 range, otherwise it returns <code>false</code>.</p> </div> </div> <div class="function"> <h3 id="deep_char_list/1">deep_char_list/1</h3> <div class="description"> <p>Returns <code>true</code> if <code><anno>Term</anno></code> is a, possibly deep, list of characters in the Unicode range, otherwise it returns <code>false</code>.</p> </div> </div> <div class="function"> <h3 id="deep_latin1_char_list/1">deep_latin1_char_list/1</h3> <div class="description"> <p>Returns <code>true</code> if <code><anno>Term</anno></code> is a, possibly deep, list of characters in the ISO-latin-1 range, otherwise it returns <code>false</code>.</p> </div> </div> <div class="function"> <h3 id="printable_list/1">printable_list/1</h3> <div class="description"> <p>Returns <code>true</code> if <code><anno>Term</anno></code> is a flat list of printable characters, otherwise it returns <code>false</code>.</p> <p>What is a printable character in this case is determined by the <code>+pc</code> start up flag to the Erlang VM. See <a href="io.html#printable_range/0" class="seealso">io:printable_range/0</a> and <a href="../erts/erl.html#erl" class="seealso">erl(1)</a>.</p> </div> </div> <div class="function"> <h3 id="printable_latin1_list/1">printable_latin1_list/1</h3> <div class="description"> <p>Returns <code>true</code> if <code><anno>Term</anno></code> is a flat list of printable ISO-latin-1 characters, otherwise it returns <code>false</code>.</p> </div> </div> <div class="function"> <h3 id="printable_unicode_list/1">printable_unicode_list/1</h3> <div class="description"> <p>Returns <code>true</code> if <code><anno>Term</anno></code> is a flat list of printable Unicode characters, otherwise it returns <code>false</code>.</p> </div> </div> </div>
