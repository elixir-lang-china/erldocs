<h1>erl_tar</h1> <h2 class="modsummary">Unix 'tar' utility for reading and writing tar archives</h2> <div class="description"> <p>The <code>erl_tar</code> module archives and extract files to and from a tar file. The tar file format is the POSIX extended tar file format specified in IEEE Std 1003.1 and ISO/IEC&nbsp;9945-1. That is the same format as used by <code>tar</code> program on Solaris, but is not the same as used by the GNU tar program.</p> <p>By convention, the name of a tar file should end in "<code>.tar</code>". To abide to the convention, you'll need to add "<code>.tar</code>" yourself to the name.</p> <p>Tar files can be created in one operation using the <a href="#create_2" class="seealso">create/2</a> or <a href="#create_3" class="seealso">create/3</a> function.</p> <p>Alternatively, for more control, the <a href="#open" class="seealso">open</a>, <a href="#add" class="seealso">add/3,4</a>, and <a href="#close" class="seealso">close/1</a> functions can be used.</p> <p>To extract all files from a tar file, use the <a href="#extract_1" class="seealso">extract/1</a> function. To extract only some files or to be able to specify some more options, use the <a href="#extract_2" class="seealso">extract/2</a> function.</p> <p>To return a list of the files in a tar file, use either the <a href="#table_1" class="seealso">table/1</a> or <a href="#table_2" class="seealso">table/2</a> function. To print a list of files to the Erlang shell, use either the <a href="#t_1" class="seealso">t/1</a> or <a href="#tt_1" class="seealso">tt/1</a> function.</p> <p>To convert an error term returned from one of the functions above to a readable message, use the <a href="#format_error_1" class="seealso">format_error/1</a> function.</p> </div> <div class="section"> <h4>LIMITATIONS</h4> <p>For maximum compatibility, it is safe to archive files with names up to 100 characters in length. Such tar files can generally be extracted by any <code>tar</code> program.</p> <p>If filenames exceed 100 characters in length, the resulting tar file can only be correctly extracted by a POSIX-compatible <code>tar</code> program (such as Solaris <code>tar</code>), not by GNU tar.</p> <p>File have longer names than 256 bytes cannot be stored at all.</p> <p>The filename of the file a symbolic link points is always limited to 100 characters.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="add/3">add(TarDescriptor, Filename, Options) -&gt; RetValue</h3> <ul class="type"> <li><code>TarDescriptor = term()</code></li> <li><code>Filename = filename()</code></li> <li><code>Options = [Option]</code></li> <li><code>Option = dereference|verbose</code></li> <li><code>RetValue = ok|{error,{Filename,Reason}}</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>The <span id="add"> </span><code>add/3</code> function adds a file to a tar file that has been opened for writing by <a href="#open" class="seealso">open/1</a>.</p> <taglist> <dt><code>dereference</code></dt> <item> <p>By default, symbolic links will be stored as symbolic links in the tar file. The files may either be read from disk or given as binaries.</p> <p>The options in <code>OptionList</code> modify the defaults as follows. </p> <taglist> <dt><code>compressed</code></dt> <item> <p>The entire tar file will be compressed, as if it has been run through the <code>gzip</code> program. To abide to the convention that a compressed tar file should end in "<code>.tar.gz</code>" or "<code>.tgz</code>", you'll need to add the appropriate extension yourself.</p> </item> <dt><code>cooked</code></dt> <item> <p>By default, the <code>open/2</code> function will open the tar file in <code>raw</code> mode, which is faster but does not allow a remote (erlang) file server to be used. Adding <code>cooked</code> to the mode list will override the default and open the tar file without the <code>raw</code> option.</p> </item> <dt><code>dereference</code></dt> <item> <p>By default, symbolic links will be stored as symbolic links in the tar file. Use the <code>dereference</code> option to override the default and store the file that the symbolic link points to into the tar file.</p> </item> <dt><code>verbose</code></dt> <item> <p>Print an informational message about each file being added.</p> </item> </taglist> </div> </div> <div class="function"> <h3 id="extract/1">extract(Name) -&gt; RetValue</h3> <ul class="type"> <li><code>Name = </span><code>extract/1</code> function extracts all files from a tar archive.</p> <p>If the <code>Name</code> argument is given as "<code>{binary,Binary}</code>", the contents of the binary is assumed function. </p> <p>Otherwise, <code>Name</code> should be a filename.</p> </div> </div> <div class="function"> <h3 id="extract/2">extract(Name, OptionList)</h3> <ul class="type"> <li><code>Name = filename() | {binary,Binary} | {file,Fd} </code></li> <li><code>Binary = binary()</code></li> <li><code>Fd = file_descriptor()</code></li> <li><code>OptionList = [Option]</code></li> <li><code>Option = {cwd,Cwd}|{files,FileList}|keep_old_files|verbose|memory</code></li> <li><code>Cwd = [dirname()]</code></li> <li><code>FileList = [filename()]</code></li> <li><code>RetValue = ok|MemoryRetValue|{error,{Name,Reason}}</code></li> <li><code>MemoryRetValue = {ok, [{NameInArchive,binary()}]}</code></li> <li><code>NameInArchive = filename()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>The <span id="extract_2"> </span><code>extract/2</code> function extracts files from a tar archive.</p> <p>If the <code>Name</code> argument is given as "<code>{binary,Binary}</code>", the contents of the binary is assumed to be a tar archive. </p> <p>If the <code>Name</code> argument is given as "<code>{file,Fd}</code>", <code>Fd</code> is assumed to be a file descriptor returned from the <code>file:open/2</code> function. </p> <p>Otherwise, <code>Name</code> should be a filename. </p> <p>The following options modify the defaults for the extraction as follows.</p> <taglist> <dt><code>{cwd,Cwd}</code></dt> <item> <p>Files with relative filenames will by default be extracted to the current working directory. Given the <code>{cwd,Cwd}</code> option, the <code>extract/2</code> function will extract into the directory <code>Cwd</code> instead of to the current working directory.</p> </item> <dt><code>{files,FileList}</code></dt> <item> <p>By default, all files will be extracted from the tar file. Given the <code>{files,Files}</code> option, the <code>extract/2</code> function will only extract the files whose names are included in <code>FileList</code>.</p> </item> <dt><code>compressed</code></dt> <item> <p>Given the <code>compressed</code> option, the <code>extract/2</code> function will uncompress the file while extracting If the tar file is not actually compressed, the <code>compressed</code> will effectively be ignored.</p> </item> <dt><code>cooked</code></dt> <item> <p>By default, the <code>open/2</code> function will open the tar file in <code>raw</code> mode, which is faster but does not allow a remote (erlang) file server to be used. Adding <code>cooked</code> to the mode list will override the default and open the tar file without the <code>raw</code> option.</p> </item> <dt><code>memory</code></dt> <item> <p>Instead of extracting to a directory, the memory option will give the result as a list of tuples {Filename, Binary}, where Binary is a binary containing the extracted data of the file named Filename in the tar file.</p> </item> <dt><code>keep_old_files</code></dt> <item> <p>By default, all existing files with the same name as file in the tar file will be overwritten Given the <code>keep_old_files</code> option, the <code>extract/2</code> function will not overwrite any existing files.</p> </item> <dt><code>verbose</code></dt> <item> <p>Print an informational message as each file is being extracted.</p> </item> </taglist> </div> </div> <div class="function"> <h3 id="format_error/1">format_error(Reason) -&gt; string()</h3> <ul class="type"> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>The <span id="format_error_1"> </span><code>format_error/1</code> function converts an error reason term to a human-readable error message string.</p> </div> </div> <div class="function"> <h3 id="open/2">open(Name, OpenModeList) -&gt; RetValue</h3> <ul class="type"> <li><code>Name = filename()</code></li> <li><code>OpenModeList = [OpenMode]</code></li> <li><code>Mode = write|compressed|cooked</code></li> <li><code>RetValue = {ok,TarDescriptor}|{error,{Name,Reason}}</code></li> <li><code>TarDescriptor = term()</code></li> <li><code>Reason = term()</code></li> </ul> <div class="description"> <p>The <span id="open"> </span><code>open/2</code> function creates a tar file for writing.
