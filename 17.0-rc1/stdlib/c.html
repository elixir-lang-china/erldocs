<h1>c</h1> <h2 class="modsummary">Command Interface Module</h2> <div class="description"> <p>The <code>c</code> module enables users to enter the short form of some commonly used commands.</p> <div class="note"><h2>Note!</h2> <p>These functions are are intended for interactive use in the Erlang shell only. The module prefix may be omitted.</p> </div> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="bt/1">bt/1</h3> <div class="description"> <p>Stack backtrace for a process. Equivalent to <code>erlang:process_display(<anno>Pid</anno>, backtrace)</code>.</p> </div> </div> <div class="function"> <h3 id="c/1">c/1</h3> <h3 id="c/2">c/2</h3> <div class="description"> <p><code>c/1,2</code> compiles and then purges and loads the code for a file. <code><anno>Options</anno></code> defaults to []. Compilation is equivalent to:</p> <pre class="sh_erlang"> compile:file(<anno>File</anno>, <anno>Options</anno> ++ [report_errors, report_warnings])</pre> <p>Note that purging the code means that any processes lingering in old code for the module are killed without warning. See <code>code/3</code> for more information.</p> </div> </div> <div class="function"> <h3 id="cd/1">cd/1</h3> <div class="description"> <p>Changes working directory to <code><anno>Dir</anno></code>, which may be a relative name, and then prints the name of the new working directory.</p> <pre class="sh_erlang"> 2&gt; <code>cd("../erlang").</code> /home/ron/erlang</pre> </div> </div> <div class="function"> <h3 id="flush/0">flush/0</h3> <div class="description"> <p>Flushes any messages sent to the shell.</p> </div> </div> <div class="function"> <h3 id="help/0">help/0</h3> <div class="description"> <p>Displays help information: all valid shell internal commands, and commands in this module.</p> </div> </div> <div class="function"> <h3 id="i/0">i/0</h3> <h3 id="ni/0">ni/0</h3> <div class="description"> <p><code>i/0</code> displays information about the system, listing information about all processes. <code>ni/0</code> does the same, but for all nodes the network.</p> </div> </div> <div class="function"> <h3 id="i/3">i/3</h3> <div class="description"> <p>Displays information about a process, Equivalent to <code>process_info(pid(<anno>X</anno>, <anno>Y</anno>, <anno>Z</anno>))</code>, but location transparent.</p> </div> </div> <div class="function"> <h3 id="l/1">l/1</h3> <div class="description"> <p>Purges and loads, or reloads, a module by calling <code>code:purge(<anno>Module</anno>)</code> followed by <code>code:load_file(<anno>Module</anno>)</code>.</p> <p>Note that purging the code means that any processes lingering in old code for the module are killed without warning. See <code>code/3</code> for more information.</p> </div> </div> <div class="function"> <h3 id="lc/1">lc(Files) -&gt; ok</h3> <ul class="type"> <li><code>Files = [File]</code></li> <li><code>File = <a href="file.html#type-filename" class="seealso">file:filename() </a></code></li> </ul> <div class="description"> <p>Compiles a list of files by calling <code>compile:file(File, [report_errors, report_warnings])</code> for each <code>File</code> in <code>Files</code>.</p> </div> </div> <div class="function"> <h3 id="ls/0">ls/0</h3> <div class="description"> <p>Lists files in the current directory.</p> </div> </div> <div class="function"> <h3 id="ls/1">ls/1</h3> <div class="description"> <p>Lists files in directory <code><anno>Dir</anno></code> or, if Dir is a file, only list it.</p> </div> </div> <div class="function"> <h3 id="m/0">m/0</h3> <div class="description"> <p>Displays information about the loaded modules, including the files from which they have been loaded.</p> </div> </div> <div class="function"> <h3 id="m/1">m/1</h3> <div class="description"> <p>Displays information about <code><anno>Module</anno></code>.</p> </div> </div> <div class="function"> <h3 id="memory/0">memory/0</h3> <div class="description"> <p>Memory allocation information. Equivalent to <a href="../erts/erlang.html#memory/0" class="seealso"><code>erlang:memory/0</code> </a>.</p> </div> </div> <div class="function"> <name name="memory" arity="1" clause_i="1" /> <name name="memory" arity="1" clause_i="2" /> <div class="description"> <p>Memory allocation information. Equivalent to <a href="../erts/erlang.html#memory/1" class="seealso"><code>erlang:memory/1</code> </a>.</p> </div> </div> <div class="function"> <h3 id="nc/1">nc/1</h3> <h3 id="nc/2">nc/2</h3> <div class="description"> <p>Compiles and then loads the code for a file on all nodes. <code><anno>Options</anno></code> defaults to []. Compilation is equivalent to:</p> <pre class="sh_erlang"> compile:file(<anno>File</anno>, <anno>Options</anno> ++ [report_errors, report_warnings])</pre> </div> </div> <div class="function"> <h3 id="nl/1">nl/1</h3> <div class="description"> <p>Loads <code><anno>Module</anno></code> on all nodes.</p> </div> </div> <div class="function"> <h3 id="pid/3">pid/3</h3> <div class="description"> <p>Converts <code><anno>X</anno></code>, <code><anno>Y</anno></code>, <code><anno>Z</anno></code> to the pid <code>&lt;X.Y.Z&gt;</code>. This function should only be used when debugging.</p> </div> </div> <div class="function"> <h3 id="pwd/0">pwd/0</h3> <div class="description"> <p>Prints the name of the working directory.</p> </div> </div> <div class="function"> <h3 id="q/0">q/0</h3> <div class="description"> <p>This function is shorthand for <code>init:stop()</code>, that is, it causes the node to stop in a controlled fashion.</p> </div> </div> <div class="function"> <h3 id="regs/0">regs/0</h3> <h3 id="nregs/0">nregs/0</h3> <div class="description"> <p><code>regs/0</code> displays information about all registered processes. <code>nregs/0</code> does the same, but for all nodes in the network.</p> </div> </div> <div class="function"> <h3 id="xm/1">xm(ModSpec) -&gt; void()</h3> <ul class="type"> <li><code>ModSpec = Module | Filename</code></li> <li><code>&nbsp;Module = atom()</code></li> <li><code>&nbsp;Filename = string()</code></li> </ul> <div class="description"> <p>This function finds undefined functions, unused functions, and calls to deprecated functions in a module by calling <code>xref:m/1</code>.</p> </div> </div> <div class="function"> <h3 id="y/1">y(File) -&gt; YeccRet</h3> <ul class="type"> <li><code>File = name() -- see filename(3)</code></li> <li><code>YeccRet = -- see yecc:file/2</code></li> </ul> <div class="description"> <p>Generates an LALR-1 parser.
