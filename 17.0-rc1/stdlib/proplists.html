<h1>proplists</h1> <h2 class="modsummary">Support functions for property lists</h2> <div class="description"> <p>Property lists are ordinary lists containing entries in the form of either tuples, whose first elements are keys used for lookup and insertion, or atoms, which work as shorthand for tuples <code>{Atom, true}</code>. (Other terms are allowed in the lists, but are ignored by this module.) If there is more than one entry in a list for a certain key, the first occurrence normally overrides any later (irrespective of the arity of the tuples).</p> <p>Property lists are useful for representing inherited properties, such as options passed to a function where a user may specify options overriding the default settings, object properties, annotations, etc.</p> <p>Two keys are considered equal if they match (<code>=:=</code>). In other words, numbers are compared literally rather than by value, so that, for instance, <code>1</code> and <code>1.0</code> are different keys.</p> </div> <datatypes> <datatype> <name name="property" /> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="append_values/2">append_values/2</h3> <div class="description"> <p>Similar to <code>get_all_values/2</code>, but each value is wrapped in a list unless it is already itself a list, and the resulting list of lists is concatenated. This is often useful for "incremental" options; e.g., <code>append_values(a, [{a, [1,2]}, {b, 0}, {a, 3}, {c, -1}, {a, [4]}])</code> will return the list <code>[1,2,3,4]</code>.</p> </div> </div> <div class="function"> <h3 id="compact/1">compact/1</h3> <div class="description"> <p>Minimizes the representation of all entries in the list. This is equivalent to <code>[property(P) || P &lt;- ListIn]</code>.</p> <p>See also: <code>property/1</code>, <code>unfold/1</code>.</p> </div> </div> <div class="function"> <h3 id="delete/2">delete/2</h3> <div class="description"> <p>Deletes all entries associated with <code><anno>Key</anno></code> from <code><anno>List</anno></code>.</p> </div> </div> <div class="function"> <h3 id="expand/2">expand/2</h3> <div class="description"> <p>Expands particular properties to corresponding sets of properties (or other terms). For each pair <code>{<anno>Property</anno>, <anno>Expansion</anno>}</code> in <code><anno>Expansions</anno></code>, if <code>E</code> is the first entry in <code><anno>ListIn</anno></code> with the same key as <code><anno>Property</anno></code>, and <code>E</code> and <code><anno>Property</anno></code> have equivalent normal forms, then <code>E</code> is replaced with the terms in <code><anno>Expansion</anno></code>, and any following entries with the same key are deleted from <code><anno>ListIn</anno></code>.</p> <p>For example, the following expressions all return <code>[fie, bar, baz, fum]</code>:</p> <pre class="sh_erlang"> expand([{foo, [bar, baz]}], [fie, foo, fum]) expand([{{foo, true}, [bar, baz]}], [fie, foo, fum]) expand([{{foo, false}, [bar, baz]}], [fie, {foo, false}, fum])</pre> <p>However, no expansion is done in the following call:</p> <pre class="sh_erlang"> expand([{{foo, true}, [bar, baz]}], [{foo, false}, fie, foo, fum])</pre> <p>because <code>{foo, false}</code> shadows <code>foo</code>.</p> <p>Note that if the original property term is to be preserved in the result when expanded, it must be included in the expansion list. The inserted terms are not expanded recursively. If <code><anno>Expansions</anno></code> contains more than one property with the same key, only the first occurrence is used.</p> <p>See also: <code>normalize/2</code>.</p> </div> </div> <div class="function"> <h3 id="get_all_values/2">get_all_values/2</h3> <div class="description"> <p>Similar to <code>get_value/2</code>, but returns the list of values for <em>all</em> entries <code>{Key, Value}</code> in <code><anno>List</anno></code>. If no such entry exists, the result is the empty list.</p> <p>See also: <code>get_value/2</code>.</p> </div> </div> <div class="function"> <h3 id="get_bool/2">get_bool/2</h3> <div class="description"> <p>Returns the value of a boolean key/value option. If <code>lookup(<anno>Key</anno>, <anno>List</anno>)</code> would yield <code>{<anno>Key</anno>, true}</code>, this function returns <code>true</code>; otherwise <code>false</code> is returned.</p> <p>See also: <code>get_value/2</code>, <code>lookup/2</code>.</p> </div> </div> <div class="function"> <h3 id="get_keys/1">get_keys/1</h3> <div class="description"> <p>Returns an unordered list of the keys used in <code><anno>List</anno></code>, not containing duplicates.</p> </div> </div> <div class="function"> <h3 id="get_value/2">get_value/2</h3> <div class="description"> <p>Equivalent to <code>get_value(<anno>Key</anno>, <anno>List</anno>, undefined)</code>.</p> </div> </div> <div class="function"> <h3 id="get_value/3">get_value/3</h3> <div class="description"> <p>Returns the value of a simple key/value property in <code><anno>List</anno></code>. If <code>lookup(<anno>Key</anno>, <anno>List</anno>)</code> would yield <code>{<anno>Key</anno>, Value}</code>, this function returns the corresponding <code>Value</code>, otherwise <code><anno>Default</anno></code> is returned.</p> <p>See also: <code>get_all_values/2</code>, <code>get_bool/2</code>, <code>get_value/2</code>, <code>lookup/2</code>.</p> </div> </div> <div class="function"> <h3 id="is_defined/2">is_defined/2</h3> <div class="description"> <p>Returns <code>true</code> if <code><anno>List</anno></code> contains at least one entry associated with <code><anno>Key</anno></code>, otherwise <code>false</code> is returned.</p> </div> </div> <div class="function"> <h3 id="lookup/2">lookup/2</h3> <div class="description"> <p>Returns the first entry associated with <code><anno>Key</anno></code> in <code><anno>List</anno></code>, if one exists, otherwise returns <code>none</code>. For an atom <code>A</code> in the list, the tuple <code>{A, true}</code> is the entry associated with <code>A</code>.</p> <p>See also: <code>get_bool/2</code>, <code>get_value/2</code>, <code>lookup_all/2</code>.</p> </div> </div> <div class="function"> <h3 id="lookup_all/2">lookup_all/2</h3> <div class="description"> <p>Returns the list of all entries associated with <code><anno>Key</anno></code> in <code><anno>List</anno></code>. If no such entry exists, the result is the empty list.</p> <p>See also: <code>lookup/2</code>.</p> </div> </div> <div class="function"> <h3 id="normalize/2">normalize/2</h3> <div class="description"> <p>Passes <code><anno>ListIn</anno></code> through a sequence of substitution/expansion stages. For an <code>aliases</code> operation, the function <code>substitute_aliases/2</code> is applied using the given list of aliases; for a <code>negations</code> operation, <code>substitute_negations/2</code> is applied using the given negation list; for an <code>expand</code> operation, the function <code>expand/2</code> is applied using the given list of expansions. The final result is automatically compacted (cf. <code>compact/1</code>).</p> <p>Typically you want to substitute negations first, then aliases, then perform one or more expansions (sometimes you want to pre-expand particular entries before doing the main expansion). You might want to substitute negations and/or aliases repeatedly, to allow such forms in the right-hand side of aliases and expansion lists.</p> <p>See also: <code>compact/1</code>, <code>expand/2</code>, <code>substitute_aliases/2</code>, <code>substitute_negations/2</code>.</p> </div> </div> <div class="function"> <h3 id="property/1">property/1</h3> <div class="description"> <p>Creates a normal form (minimal) representation of a property. If <code><anno>PropertyIn</anno></code> is <code>{Key, true}</code> where <code>Key</code> is an atom, this returns <code>Key</code>, otherwise the whole term <code><anno>PropertyIn</anno></code> is returned.</p> <p>See also: <code>property/2</code>.</p> </div> </div> <div class="function"> <h3 id="property/2">property/2</h3> <div class="description"> <p>Creates a normal form (minimal) representation of a simple key/value property. Returns <code><anno>Key</anno></code> if <code><anno>Value</anno></code> is <code>true</code> and <code><anno>Key</anno></code> is an atom, otherwise a tuple <code>{<anno>Key</anno>, <anno>Value</anno>}</code> is returned.</p> <p>See also: <code>property/1</code>.</p> </div> </div> <div class="function"> <h3 id="split/2">split/2</h3> <div class="description"> <p>Partitions <code><anno>List</anno></code> into a list of sublists and a remainder. <code><anno>Lists</anno></code> contains one sublist for each key in <code><anno>Keys</anno></code>, in the corresponding order. The relative order of the elements in each sublist is preserved from the original <code><anno>List</anno></code>. <code><anno>Rest</anno></code> contains the elements in <code><anno>List</anno></code> that are not associated with any of the given keys, also with their original relative order preserved.</p> <p>Example: split([{c, 2}, {e, 1}, a, {c, 3, 4}, d, {b, 5}, b], [a, b, c])</p> <p>returns</p> <p>{[[a], [{b, 5}, b],[{c, 2}, {c, 3, 4}]], [{e, 1}, d]}</p> </div> </div> <div class="function"> <h3 id="substitute_aliases/2">substitute_aliases/2</h3> <div class="description"> <p>Substitutes keys of properties. For each entry in <code><anno>ListIn</anno></code>, if it is associated with some key <code>K1</code> such that <code>{K1, K2}</code> occurs in <code><anno>Aliases</anno></code>, the key of the entry is changed to <code>K2</code>. If the same <code>K1</code> occurs more than once in <code><anno>Aliases</anno></code>, only the first occurrence is used.</p> <p>Example: <code>substitute_aliases([{color, colour}], L)</code> will replace all tuples <code>{color, ...}</code> in <code>L</code> with <code>{colour, ...}</code>, and all atoms <code>color</code> with <code>colour</code>.</p> <p>See also: <code>normalize/2</code>, <code>substitute_negations/2</code>.</p> </div> </div> <div class="function"> <h3 id="substitute_negations/2">substitute_negations/2</h3> <div class="description"> <p>Substitutes keys of boolean-valued properties and simultaneously negates their values. For each entry in <code><anno>ListIn</anno></code>, if it is associated with some key <code>K1</code> such that <code>{K1, K2}</code> occurs in <code><anno>Negations</anno></code>, then if the entry was <code>{K1, true}</code> it will be replaced with <code>{K2, false}</code>, otherwise it will be replaced with <code>{K2, true}</code>, thus changing the name of the option and simultaneously negating the value given by <code>get_bool(ListIn)</code>. If the same <code>K1</code> occurs more than once in <code><anno>Negations</anno></code>, only the first occurrence is used.</p> <p>Example: <code>substitute_negations([{no_foo, foo}], L)</code> will replace any atom <code>no_foo</code> or tuple <code>{no_foo, true}</code> in <code>L</code> with <code>{foo, false}</code>, and any other tuple <code>{no_foo, ...}</code> with <code>{foo, true}</code>.</p> <p>See also: <code>get_bool/2</code>, <code>normalize/2</code>, <code>substitute_aliases/2</code>.</p> </div> </div> <div class="function"> <h3 id="unfold/1">unfold/1</h3> <div class="description"> <p>Unfolds all occurrences of atoms in <code><anno>ListIn</anno></code> to tuples <code>{Atom, true}</code>.</p> </div> </div> </div>
