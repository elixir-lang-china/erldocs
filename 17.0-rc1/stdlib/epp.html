<h1>epp</h1> <h2 class="modsummary">An Erlang Code Preprocessor</h2> <div class="description"> <p>The Erlang code preprocessor includes functions which are used by <code>compile</code> to preprocess macros and include files before the actual parsing takes place.</p> <p>The Erlang source file <marker id="encoding"><em>encoding</em></marker> is selected by a comment in one of the first two lines of the source file. The first string that matches the regular expression <code>coding\s*[:=]\s*([-a-zA-Z0-9])+</code> selects the encoding. If the matching string is not a valid encoding it is ignored. The valid encodings are <code>Latin-1</code> and <code>UTF-8</code> where the case of the characters can be chosen freely. Examples:</p> <pre class="sh_erlang"> %% coding: utf-8 %% For this file we have chosen encoding = Latin-1 %% -*- coding: latin-1 -*-</pre> </div> <datatypes> <datatype> <name name="macros" /> </datatype> <datatype> <name name="epp_handle" /> <div class="description"><p>Handle to the epp server.</p></div> </datatype> <datatype> <name name="source_encoding" /> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="open/2">open/2</h3> <h3 id="open/3">open/3</h3> <div class="description"> <p>Opens a file for preprocessing.</p> </div> </div> <div class="function"> <h3 id="close/1">close/1</h3> <div class="description"> <p>Closes the preprocessing of a file.</p> </div> </div> <div class="function"> <h3 id="parse_erl_form/1">parse_erl_form/1</h3> <div class="description"> <p>Returns the next Erlang form from the opened Erlang source file. The tuple <code>{eof, <anno>Line</anno>}</code> is returned at end-of-file. The first form corresponds to an implicit attribute <code>-file(File,1).</code>, where <code>File</code> is the name of the file.</p> </div> </div> <div class="function"> <h3 id="parse_file/3">parse_file/3</h3> <div class="description"> <p>Preprocesses and parses an Erlang source file. Note that the tuple <code>{eof, <anno>Line</anno>}</code> returned at end-of-file is included as a "form".</p> </div> </div> <div class="function"> <h3 id="default_encoding/0">default_encoding/0</h3> <div class="description"> <p>Returns the default encoding of Erlang source files.</p> </div> </div> <div class="function"> <h3 id="encoding_to_string/1">encoding_to_string/1</h3> <div class="description"> <p>Returns a string representation of an encoding. The string is recognized by <code>read_encoding/1,2</code>, <code>read_encoding_from_binary/1,2</code>, and <code>set_encoding/1</code> as a valid encoding.</p> </div> </div> <div class="function"> <h3 id="read_encoding/1">read_encoding/1</h3> <h3 id="read_encoding/2">read_encoding/2</h3> <div class="description"> <p>Read the <a href="#encoding" class="seealso">encoding</a> from a file. Returns the read encoding, or <code>none</code> if no valid encoding was found.</p> <p>The option <code>in_comment_only</code> is <code>true</code> by default, which is correct for Erlang source files. If set to <code>false</code> the encoding string does not necessarily have to occur in a comment.</p> </div> </div> <div class="function"> <h3 id="read_encoding_from_binary/1">read_encoding_from_binary/1</h3> <h3 id="read_encoding_from_binary/2">read_encoding_from_binary/2</h3> <div class="description"> <p>Read the <a href="#encoding" class="seealso">encoding</a> from a binary. Returns the read encoding, or <code>none</code> if no valid encoding was found.</p> <p>The option <code>in_comment_only</code> is <code>true</code> by default, which is correct for Erlang source files. If set to <code>false</code> the encoding string does not necessarily have to occur in a comment.</p> </div> </div> <div class="function"> <h3 id="set_encoding/1">set_encoding/1</h3> <div class="description"> <p>Reads the <a href="#encoding" class="seealso">encoding</a> from an IO device and sets the encoding of the device accordingly. The position of the IO device referenced by <code><anno>File</anno></code> is not affected. If no valid encoding can be read from the IO device the encoding of the IO device is set to the default encoding.</p> <p>Returns the read encoding, or <code>none</code> if no valid encoding was found.</p> </div> </div> <div class="function"> <h3 id="format_error/1">format_error/1</h3> <div class="description"> <p>Takes an <code><anno>ErrorDescriptor</anno></code> and returns a string which describes the error or warning. This function is usually called implicitly when processing an <code>ErrorInfo</code> structure (see below).</p> </div> </div> </div> <div class="section"> <h4>Error Information</h4> <p>The <code>ErrorInfo</code> mentioned above is the standard <code>ErrorInfo</code> structure which is returned from all IO modules. It has the following format: </p> <pre class="sh_erlang"> {ErrorLine, Module, ErrorDescriptor} </pre> <p>A string which describes the error is obtained with the following call: </p> <pre class="sh_erlang"> Module:format_error(ErrorDescriptor) </pre> </div> <div class="section"> <h4>See Also</h4> <p><a href="erl_parse.html" class="seealso">erl_parse(3)</a></p> </div>
