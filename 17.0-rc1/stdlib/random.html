<h1>random</h1> <h2 class="modsummary">Pseudo random number generation</h2> <div class="description"> <p>Random number generator. The method is attributed to B.A. Wichmann and I.D.Hill, in 'An efficient and portable pseudo-random number generator', Journal of Applied Statistics. AS183. 1982. Also Byte March 1987. </p> <p>The current algorithm is a modification of the version attributed to Richard A O'Keefe in the standard Prolog library.</p> <p>Every time a random number is requested, a state is used to calculate it, and a new state produced. The state can either be implicit (kept in the process dictionary) or be an explicit argument and return value. In this implementation, the state (the type <code>ran()</code>) consists of a tuple of three integers.</p> <p>It should be noted that this random number generator is not cryptographically strong. If a strong cryptographic random number generator is needed for example <code>crypto:rand_bytes/1</code> could be used instead.</p> </div> <datatypes> <datatype> <name name="ran" /> <div class="description"><p>The state.</p></div> </datatype> </datatypes> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="seed/0">seed/0</h3> <div class="description"> <p>Seeds random number generation with default (fixed) values in the process dictionary, and returns the old state.</p> </div> </div> <div class="function"> <h3 id="seed/3">seed/3</h3> <div class="description"> <p>Seeds random number generation with integer values in the process dictionary, and returns the old state.</p> <p>One way of obtaining a seed is to use the BIF <code>now/0</code>:</p> <pre class="sh_erlang"> ... {A1,A2,A3} = now(), random:seed(A1, A2, A3), ...</pre> </div> </div> <div class="function"> <h3 id="seed/1">seed/1</h3> <div class="description"> <p> <code>seed({<anno>A1</anno>, <anno>A2</anno>, <anno>A3</anno>})</code> is equivalent to <code>seed(<anno>A1</anno>, <anno>A2</anno>, <anno>A3</anno>)</code>. </p> </div> </div> <div class="function"> <h3 id="seed0/0">seed0/0</h3> <div class="description"> <p>Returns the default state.</p> </div> </div> <div class="function"> <h3 id="uniform/0">uniform/0</h3> <div class="description"> <p>Returns a random float uniformly distributed between <code>0.0</code> and <code>1.0</code>, updating the state in the process dictionary.</p> </div> </div> <div class="function"> <h3 id="uniform/1">uniform/1</h3> <div class="description"> <p>Given an integer <code><anno>N</anno> &gt;= 1</code>, <code>uniform/1</code> returns a random integer uniformly distributed between <code>1</code> and <code><anno>N</anno></code>, updating the state in the process dictionary.</p> </div> </div> <div class="function"> <h3 id="uniform_s/1">uniform_s/1</h3> <div class="description"> <p>Given a state, <code>uniform_s/1</code>returns a random float uniformly distributed between <code>0.0</code> and <code>1.0</code>, and a new state.</p> </div> </div> <div class="function"> <h3 id="uniform_s/2">uniform_s/2</h3> <div class="description"> <p>Given an integer <code><anno>N</anno> &gt;= 1</code> and a state, <code>uniform_s/2</code> returns a random integer uniformly distributed between <code>1</code> and <code><anno>N</anno></code>, and a new state.</p> </div> </div> </div> <div class="section"> <h4>Note</h4> <p>Some of the functions use the process dictionary variable <code>random_seed</code> to remember the current seed.</p> <p>If a process calls <code>uniform/0</code> or <code>uniform/1</code> without setting a seed first, <code>seed/0</code> is called automatically.</p> <p>The implementation changed in R15. Upgrading to R15 will break applications that expect a specific output for a given seed. The output is still deterministic number series, but different compared to releases older than R15. The seed <code>{0,0,0}</code> will for example no longer produce a flawed series of only zeros.</p> </div>
