<h1>gs</h1> <h2 class="modsummary">The Graphics System for Erlang.</h2> <div class="description"> <div class="warning"><h2>Warning!</h2> <p> GS is not recommended for use in new applications. Instead we recommend WX for applications that need a graphical user interface. </p> <p> GS is deprecated and will be removed in the R16 release. </p> </div> <p>The Graphics System, GS, is easy to learn and designed to be portable to many different platforms.</p> <p>In the description below, the type <code>gsobj()</code> denotes a reference to a graphical object created with GS. Such a reference is either a GS object identifier or the name of the object (an atom), if such a name exists. The functions all return the specified values or <code>{error,Reason}</code> if an error occurs.</p> <p>Please refer to the GS User's Guide for a description of the different object types and possible options.</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="config/2">config(GSObj, Options) -&gt; ok</h3> <ul class="type"> <li><code>GSOBj = gsobj()</code></li> <li><code>Options = [Option] | Option</code></li> <li><code>&nbsp;Option = {Key,Value}</code></li> </ul> <div class="description"> <p>Configures a graphical object according to the specified options.</p> </div> </div> <div class="function"> <h3 id="create/2">create(ObjType, Parent) -&gt; ObjId</h3> <h3 id="create/3">create(ObjType, Parent, Options) -&gt; ObjId</h3> <h3 id="create/4">create(ObjType, Name, Parent, Options) -&gt; ObjId</h3> <ul class="type"> <li><code>ObjType = atom()</code></li> <li><code>Parent = gsobj()</code></li> <li><code>Name = atom()</code></li> <li><code>Options = [Option] | Option</code></li> <li><code>&nbsp;Option = {Key,Value}</code></li> </ul> <div class="description"> <p>Creates a new graphical object of the specified type as a child to the specified parent object. The object is configured according to the options and its identifier is returned. If no options are provided, default option values are used.</p> <p>If a name is provided, this name can be used to reference the object instead of the object identifier. The name is local to the process which creates the object.</p> <p>The following object types exist: <code>window | button | radiobutton | checkbutton | label | frame | entry | listbox | canvas | arc | image | line | oval | polygon | rectangle | text | menubar | menubutton | menu | menuitem | grid | gridline | editor | scale</code></p> </div> </div> <div class="function"> <h3 id="create_tree/2">create_tree(Parent, Tree) -&gt; ok</h3> <ul class="type"> <li><code>Parent = gsobj()</code></li> <li><code>Tree = [Object]</code></li> <li><code>&nbsp;Object = {ObjType,Options} | {ObjType,Options,Tree} | {ObjType,Name,Options,Tree}</code></li> </ul> <div class="description"> <p>Creates a hierarchy of graphical objects.</p> </div> </div> <div class="function"> <h3 id="destroy/1">destroy(GSObj) -&gt; void()</h3> <ul class="type"> <li><code>GSObj = gsobj()</code></li> </ul> <div class="description"> <p>Destroys a graphical object and all its children.</p> </div> </div> <div class="function"> <h3 id="ObjType/1">ObjType(Parent)</h3> <h3 id="ObjType/2">ObjType(Parent, Options)</h3> <h3 id="ObjType/3">ObjType(Name, Parent, Options)</h3> <div class="description"> <p>These functions are shorthand equivalents of <code>create/2</code>, <code>create/3</code>, and <code>create/4</code>, respectively.</p> </div> </div> <div class="function"> <h3 id="read/2">read(GSObj, Key) -&gt; Value</h3> <ul class="type"> <li><code>GSObj = gsobj()</code></li> <li><code>Key = atom()</code></li> <li><code>Value = term()</code></li> </ul> <div class="description"> <p>Returns the value of an option key for the specified graphical object.</p> </div> </div> <div class="function"> <h3 id="start/0">start() -&gt; ObjId</h3> <div class="description"> <p>Starts GS, unless it is already started, and returns its object identifier.</p> </div> </div> <div class="function"> <h3 id="stop/0">stop() -&gt; void()</h3> <div class="description"> <p>Stops GS and closes all windows. This function is not the opposite of <code>start/0</code> as it will cause <em>all</em> applications to lose their GS objects.</p> </div> </div> </div>
