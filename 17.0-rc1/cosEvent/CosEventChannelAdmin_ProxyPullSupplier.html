<h1>CosEventChannelAdmin_ProxyPullSupplier</h1> <h2 class="modsummary">This module implements a ProxyPullSupplier interface which acts as a middleman between pull consumer and the event channel.</h2> <div class="description"> <p>The ProxyPullSupplier interface defines the second step for connecting pull consumers to the event channel. A proxy supplier is similar to a normal supplier, but includes an additional method for connecting a consumer to the proxy supplier.</p> <p>To get access to all definitions, e.g., exceptions, include necessary <code>hrl</code> files by using:<br /><code>-include_lib("cosEvent/include/*.hrl").</code></p> <p>Any object that possesses an object reference that supports the ProxyPullSupplier interface can perform the following operations:</p> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="connect_pull_consumer/2">connect_pull_consumer(Object, PullConsumer) -&gt; Return</h3> <ul class="type"> <li><code>Object = #objref</code></li> <li><code>PullConsumer = #objref of PullConsumer type</code></li> <li><code>Return = ok | {'EXCEPTION', #'CosEventChannelAdmin_AlreadyConnected'{}}</code></li> </ul> <div class="description"> <p>This operation connects PullConsumer object to the ProxyPullSupplier object. A nil object reference can be passed to this operation. If so a channel cannot invoke the disconnect_pull_consumer operation on the consumer; the consumer may be disconnected from the channel without being informed. If the ProxyPullSupplier is already connected to a PullConsumer, then the <code>CosEventChannelAdmin_AlreadyConnected</code> exception is raised.</p> </div> </div> <div class="function"> <h3 id="disconnect_pull_supplier/1">disconnect_pull_supplier(Object) -&gt; Return</h3> <ul class="type"> <li><code>Object = #objref</code></li> <li><code>Return = ok</code></li> </ul> <div class="description"> <p>This operation disconnects proxy pull supplier from the event channel. It sends a notification about the loss of the connection to the pull consumer attached to it, unless nil object reference was passed at the connection time.</p> </div> </div> <div class="function"> <h3 id="pull/1">pull(Object) -&gt; Return</h3> <ul class="type"> <li><code>Object = #objref</code></li> <li><code>Return = any</code></li> </ul> <div class="description"> <p>This operation blocks until the event data is available or the <code>CosEventComm_Disconnected</code> exception is raised. It returns the event data to the consumer.</p> </div> </div> <div class="function"> <h3 id="try_pull/1">try_pull(Object) -&gt; Return</h3> <ul class="type"> <li><code>Object = #objref</code></li> <li><code>Return = {any, bool()}</code></li> </ul> <div class="description"> <p>This operation does not block: if the event data is available, it returns the event data and sets the data availability flag to true; otherwise it returns a long with an undefined value and sets the data availability to false. If the event communication has already been disconnected, the <code>CosEventComm_Disconnected</code> exception is raised.</p> </div> </div> </div>
