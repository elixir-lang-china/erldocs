<h1>eldap</h1> <h2 class="modsummary">Eldap Functions</h2> <div class="description"> <p>This module provides a client api to the Lightweight Directory Access Protocol (LDAP). </p> <p>References:</p> <list type="bulleted"> <item> <p>RFC 4510 - RFC 4519</p> </item> <item> <p>RFC 2830</p> </item> </list> <p>The above publications can be found at <url href="http://www.ietf.org">IETF</url>. </p> <p><em>Types</em></p> <pre class="sh_erlang"> handle() Connection handle attribute() {Type = string(), Values=[string()]} modify_op() See mod_add/2, mod_delete/2, mod_replace/2 scope() See baseObject/0, singleLevel/0, wholeSubtree/0 dereference() See neverDerefAliases/0, derefInSearching/0, derefFindingBaseObj/0, derefAlways/0 filter() See present/1, substrings/2, equalityMatch/2, greaterOrEqual/2, lessOrEqual/2, approxMatch/2, 'and'/1, 'or'/1, 'not'/1. </pre> <p /> </div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="open/1">open([Host]) -&gt; ok | {error,Error}</h3> <div class="description"> <p>Same as start_tls(Handle, Options, infinity)</p> </div> </div> <div class="function"> <h3 id="start_tls/3">start_tls(Handle, Options, Timeout) -&gt; ok | {error,Error}</h3> <ul class="type"> <li><code>Handle = handle()</code></li> <li><code>Options = ssl:ssl_options()</code></li> <li><code>Timeout = inifinity | positive_integer()</code></li> </ul> <div class="description"> <p>Upgrade the connection associated with <code>Handle</code> to a tls connection if possible.</p> <p>The upgrade is done in two phases: first the server is asked for permission to upgrade. Second, if the request is acknowledged, the upgrade is performed.</p> <p>Error responese from phase one will not affect the current encryption state of the connection. Those responses are:</p> <taglist> <dt><code>tls_already_started</code></dt> <item>The connection is already encrypted. The connection is not affected.</item> <dt><code>{response,ResponseFromServer}</code></dt> <item>The upgrade was refused by the LDAP server. The <code>ResponseFromServer</code> is an atom delivered byt the LDAP server explained in section 2.3 of rfc 2830. The connection is not affected, so it is still un-encrypted.</item> </taglist> <p>Errors in the seconde phase will however end the connection:</p> <taglist> <dt><code>Error</code></dt> <item>Any error responded from ssl:connect/3</item> </taglist> </div> </div> <div class="function"> <h3 id="simple_bind/3">simple_bind(Handle, Dn, Password) -&gt; ok | {error, Reason}</h3> <ul class="type"> <li><code>Handle = handle()</code></li> <li><code>Dn = string()</code></li> <li><code>Password = string()</code></li> </ul> <div class="description"> <p>Authenticate the connection using simple authentication.</p> </div> </div> <div class="function"> <h3 id="add/3">add(Handle, Dn, [Attribute]) -&gt; ok | {error, Reason}</h3> <ul class="type"> <li><code>Handle = handle()</code></li> <li><code>Dn = string()</code></li> <li><code>Attribute = attribute()</code></li> </ul> <div class="description"> <p> Add an entry.
