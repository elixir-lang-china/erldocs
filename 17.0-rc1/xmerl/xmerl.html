<h1>xmerl</h1> <h2 class="modsummary">Functions for exporting XML data to an external format.</h2> <div class="description"> <p>Functions for exporting XML data to an external format. </p></div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="export/2">export(Content, Callback) -&gt; ExportedFormat</h3> <div class="description"><span id="export-2"> </span> <p>Equivalent to <a href="#export-3" class="seealso">export(Data, Callback, [])</a>.</p> </div></div> <div class="function"> <h3 id="export/3">export(Content, Callback, RootAttributes) -&gt; ExportedFormat</h3> <ul class="type"> <li><code>Content = [Element]</code></li><li><code>Callback = atom()</code></li><li><code>RootAttributes = [XmlAttributes]</code></li></ul> <div class="description"><span id="export-3"> </span> <p>Exports normal, well-formed XML content, using the specified callback-module. </p><p><code>Element</code> is any of:</p> <list> <item><p><code>#xmlText{}</code></p></item> <item><p><code>#xmlElement{}</code></p></item> <item><p><code>#xmlPI{}</code></p></item> <item><p><code>#xmlComment{}</code></p></item> <item><p><code>#xmlDecl{}</code></p></item> </list> <p>(See <code>xmerl.hrl</code> for the record definitions.) Text in <code>#xmlText{}</code> elements can be deep lists of characters and/or binaries.</p> <p><code>RootAttributes</code> is a list of <code>#xmlAttribute{}</code> attributes for the <code>#root#</code> element, which implicitly becomes the parent of the given <code>Content</code>. The tag-handler function for <code>#root#</code> is thus called with the complete exported data of <code>Content</code>. Root attributes can be used to specify e.g. encoding or other metadata of an XML or HTML document.</p> <p>The <code>Callback</code> module should contain hook functions for all tags present in the data structure. A hook function must have the following format:</p> <pre class="sh_erlang"> Tag(Data, Attributes, Parents, E)</pre> <p>where <code>E</code> is the corresponding <code>#xmlElement{}</code>, <code>Data</code> is the already-exported contents of <code>E</code> and <code>Attributes</code> is the list of <code>#xmlAttribute{}</code> records of <code>E</code>. Finally, <code>Parents</code> is the list of parent nodes of <code>E</code>, on the form <code>[{ParentTag::atom(), ParentPosition::integer()}]</code>.</p> <p>The hook function should return either the data to be exported, or a tuple <code>{'#xml-alias#', NewTag::atom()}</code>, or a tuple <code>{'#xml-redefine#', Content}</code>, where <code>Content</code> is a content list (which can be on simple-form; see <code>export_simple/2</code> for details).</p> <p>A callback module can inherit definitions from other callback modules, through the required function <code>'#xml-interitance#() -&gt; [ModuleName::atom()]</code>.</p> <p><em>See also:</em> <a href="#export-2" class="seealso">export/2</a>, <a href="#export_simple-3" class="seealso">export_simple/3</a>.</p> </div></div> <div class="function"> <h3 id="export_simple/2">export_simple(Content, Callback) -&gt; ExportedFormat</h3> <div class="description"><span id="export_simple-2"> </span> <p>Equivalent to <a href="#export_simple-3" class="seealso">export_simple(Content, Callback, [])</a>.</p> </div></div> <div class="function"> <h3 id="export_simple/3">export_simple(Content, Callback, RootAttrs::RootAttributes) -&gt; ExportedFormat</h3> <ul class="type"> <li><code>Content = [Element]</code></li><li><code>Callback = atom()</code></li><li><code>RootAttributes = [XmlAttributes]</code></li></ul> <div class="description"><span id="export_simple-3"> </span> <p>Exports "simple-form" XML content, using the specified callback-module. </p><p><code>Element</code> is any of:</p> <list> <item><p><code>{Tag, Attributes, Content}</code></p></item> <item><p><code>{Tag, Content}</code></p></item> <item><p><code>Tag</code></p></item> <item><p><code>IOString</code></p></item> <item><p><code>#xmlText{}</code></p></item> <item><p><code>#xmlElement{}</code></p></item> <item><p><code>#xmlPI{}</code></p></item> <item><p><code>#xmlComment{}</code></p></item> <item><p><code>#xmlDecl{}</code></p></item> </list> <p>where</p> <list> <item><p><code>Tag = atom()</code></p></item> <item><p><code>Attributes = [{Name, Value}]</code></p></item> <item><p><code>Name = atom()</code></p></item> <item><p><code>Value = IOString | atom() | integer()</code></p></item> </list> <p>Normal-form XML elements can thus be included in the simple-form representation. Note that content lists must be flat. An <code>IOString</code> is a (possibly deep) list of characters and/or binaries.</p> <p><code>RootAttributes</code> is a list of:</p> <list> <item><p><code>XmlAttributes = #xmlAttribute{}</code></p></item> </list> <p>See <code>export/3</code> for details on the callback module and the root attributes. The XML-data is always converted to normal form before being passed to the callback module.</p> <p><em>See also:</em> <a href="#export-3" class="seealso">export/3</a>, <a href="#export_simple-2" class="seealso">export_simple/2</a>.</p> </div></div> <div class="function"> <h3 id="export_simple_content/2">export_simple_content(Content, Callback) -&gt;
