<h1>xmerl_scan</h1> <h2 class="modsummary">This module is the interface to the XML parser, it handles XML 1.0.</h2> <div class="description"> <p>This module is the interface to the XML parser, it handles XML 1.0. The XML parser is activated through <code>xmerl_scan:string/[1,2]</code> or <code>xmerl_scan:file/[1,2]</code>. It returns records of the type defined in xmerl.hrl. See also <url href="xmerl_examples.html">tutorial</url> on customization functions.</p></div> <div class="section"><h4>DATA TYPES</h4><span id="types"> </span> <taglist> <dt><code>document() = xmlElement() | xmlDocument()</code></dt> <item><span id="type-document"> </span> <p> The document returned by <code>xmerl_scan:string/[1,2]</code> and <code>xmerl_scan:file/[1,2]</code>. The type of the returned record depends on the value of the document option passed to the function. </p> </item> <dt><code>global_state()</code></dt> <item><span id="type-global_state"> </span> <p> The global state of the scanner, represented by the #xmerl_scanner{} record. </p> </item> <dt><code>option_list()</code></dt> <item><span id="type-option_list"> </span> <p>Options allow to customize the behaviour of the scanner. See also <url href="xmerl_examples.html">tutorial</url> on customization functions. </p> <p> Possible options are: </p> <taglist> <dt><code>{acc_fun, Fun}</code></dt> <item><p>Call back function to accumulate contents of entity.</p></item> <dt><code>{continuation_fun, Fun} | {continuation_fun, Fun, ContinuationState}</code></dt> <item><p>Call back function to decide what to do if the scanner runs into EOF before the document is complete.</p></item> <dt><code>{event_fun, Fun} | {event_fun, Fun, EventState}</code></dt> <item><p>Call back function to handle scanner events.</p></item> <dt><code>{fetch_fun, Fun} | {fetch_fun, Fun, FetchState}</code></dt> <item><p>Call back function to fetch an external resource.</p></item> <dt><code>{hook_fun, Fun} | {hook_fun, Fun, HookState}</code></dt> <item><p>Call back function to process the document entities once identified.</p></item> <dt><code>{close_fun, Fun}</code></dt> <item><p>Called when document has been completely parsed.</p></item> <dt><code>{rules, ReadFun, WriteFun, RulesState} | {rules, Rules}</code></dt> <item><p>Handles storing of scanner information when parsing.</p></item> <dt><code>{user_state, UserState}</code></dt> <item><p>Global state variable accessible from all customization functions</p></item> <dt><code>{fetch_path, PathList}</code></dt> <item><p>PathList is a list of directories to search when fetching files. If the file in question is not in the fetch_path, the URI will be used as a file name.</p></item> <dt><code>{space, Flag}</code></dt> <item><p>'preserve' (default) to preserve spaces, 'normalize' to accumulate consecutive whitespace and replace it with one space.</p></item> <dt><code>{line, Line}</code></dt> <item><p>To specify starting line for scanning in document which contains fragments of XML.</p></item> <dt><code>{namespace_conformant, Flag}</code></dt> <item><p>Controls whether to behave as a namespace conformant XML parser, 'false' (default) to not otherwise 'true'.</p></item> <dt><code>{validation, Flag}</code></dt> <item><p>Controls whether to process as a validating XML parser: 'off' (default) no validation, or validation 'dtd' by DTD or 'schema' by XML Schema. 'false' and 'true' options are obsolete (i.e. they may be removed in a future release), if used 'false' equals 'off' and 'true' equals 'dtd'.</p></item> <dt><code>{schemaLocation, [{Namespace,Link}|...]}</code></dt> <item><p>Tells explicitly which XML Schema documents to use to validate the XML document. Used together with the <code>{validation,schema}</code> option.</p></item> <dt><code>{quiet, Flag}</code></dt> <item><p>Set to 'true' if xmerl should behave quietly and not output any information to standard output (default 'false').</p></item> <dt><code>{doctype_DTD, DTD}</code></dt> <item><p>Allows to specify DTD name when it isn't available in the XML document. This option has effect only together with <code>{validation,'dtd'</code> option.</p></item> <dt><code>{xmlbase, Dir}</code></dt> <item><p>XML Base directory. If using string/1 default is current directory. If using file/1 default is directory of given file.</p></item> <dt><code>{encoding, Enc}</code></dt> <item><p>Set default character set used (default UTF-8). This character set is used only if not explicitly given by the XML declaration. </p></item> <dt><code>{document, Flag}</code></dt> <item><p>Set to 'true' if xmerl should return a complete XML document as an xmlDocument record (default 'false').</p></item> <dt><code>{comments, Flag}</code></dt> <item><p>Set to 'false' if xmerl should skip comments otherwise they will be returned as xmlComment records (default 'true').</p></item> <dt><code>{default_attrs, Flag}</code></dt> <item><p>Set to 'true' if xmerl should add to elements missing attributes with a defined default value (default 'false').</p></item> </taglist> </item> </taglist></div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="user_state/1">user_state(S::global_state()) -&gt;
