<h1>xmerl_xs</h1> <h2 class="modsummary"> Erlang has similarities to XSLT since both languages have a functional programming approach.</h2> <div class="description"> <p> Erlang has similarities to XSLT since both languages have a functional programming approach. Using <code>xmerl_xpath</code> it is possible to write XSLT like transforms in Erlang.</p> <p>XSLT stylesheets are often used when transforming XML documents, to other XML documents or (X)HTML for presentation. XSLT contains quite many functions and learning them all may take some effort. This document assumes a basic level of understanding of XSLT. </p> <p>Since XSLT is based on a functional programming approach with pattern matching and recursion it is possible to write similar style sheets in Erlang. At least for basic transforms. This document describes how to use the XPath implementation together with Erlangs pattern matching and a couple of functions to write XSLT like transforms.</p> <p>This approach is probably easier for an Erlanger but if you need to use real XSLT stylesheets in order to "comply to the standard" there is an adapter available to the Sablotron XSLT package which is written i C++. See also the <url href="xmerl_xs_examples.html">Tutorial</url>. </p></div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="xslapply/2">xslapply(Fun::Function, EList::list()) -&gt; List</h3> <ul class="type"> <li><code>Function = () -&gt; list()</code></li></ul> <div class="description"><span id="xslapply-2"> </span> <p>xslapply is a wrapper to make things look similar to xsl:apply-templates.</p> <p>Example, original XSLT:</p><br /><pre class="sh_erlang"> &lt;xsl:template match="doc/title"&gt; &lt;h1&gt; &lt;xsl:apply-templates/&gt; &lt;/h1&gt; &lt;/xsl:template&gt; </pre> <p>becomes in Erlang:</p><br /><pre class="sh_erlang"> template(E = #xmlElement{ parents=[{'doc',_}|_], name='title'}) -&gt; ["&lt;h1&gt;", xslapply(fun template/1, E), "&lt;/h1&gt;"]; </pre> </div></div> <div class="function"> <h3 id="value_of/1">value_of(E) -&gt; List</h3> <ul class="type"> <li><code>E = unknown()</code></li></ul> <div class="description"><span id="value_of-1"> </span> <p>Concatenates all text nodes within the tree.</p> <p>Example:</p><br /><pre class="sh_erlang"> &lt;xsl:template match="title"&gt; &lt;div align="center"&gt; &lt;h1&gt;&lt;xsl:value-of select="." /&gt;&lt;/h1&gt; &lt;/div&gt; &lt;/xsl:template&gt; </pre> <p>becomes:</p><br /> <pre class="sh_erlang"> template(E = #xmlElement{name='title'}) -&gt; ["&lt;div align="center"&gt;&lt;h1&gt;", value_of(select(".", E)), "&lt;/h1&gt;&lt;/div&gt;"] </pre> </div></div> <div class="function"> <h3 id="select/1">select(String::string(), E) -&gt; E</h3> <div class="description"><span id="select-2"> </span> <p>Extracts the nodes from the xml tree according to XPath.</p> <p><em>See also:</em> <a href="#value_of-1" class="seealso">value_of/1</a>.</p> </div></div> <div class="function"> <h3 id="built_in_rules/2">built_in_rules(Fun, E) -&gt; List</h3> <div class="description"><span id="built_in_rules-2"> </span> <p>The default fallback behaviour.
