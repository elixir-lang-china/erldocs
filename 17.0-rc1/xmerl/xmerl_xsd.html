<h1>xmerl_xsd</h1> <h2 class="modsummary">Interface module for XML Schema vlidation.</h2> <div class="description"> <p>Interface module for XML Schema vlidation. It handles the W3.org <url href="http://www.w3.org/XML/Schema#dev">specifications</url> of XML Schema second edition 28 october 2004. For an introduction to XML Schema study <url href="http://www.w3.org/TR/xmlschema-0/">part 0.</url> An XML structure is validated by xmerl_xsd:validate/[2,3].</p></div> <div class="section"><h4>DATA TYPES</h4><span id="types"> </span> <taglist> <dt><code>global_state()</code></dt> <item><span id="type-global_state"> </span> <p>The global state of the validator. It is representated by the <code>#xsd_state{}</code> record. </p> </item> <dt><code>option_list()</code></dt> <item><span id="type-option_list"> </span> <p>Options allow to customize the behaviour of the validation. </p> <p> Possible options are : </p> <taglist> <dt><code>{tab2file,boolean()}</code></dt> <item><p>Enables saving of abstract structure on file for debugging purpose.</p></item> <dt><code>{xsdbase,filename()}</code></dt> <item><p>XSD Base directory.</p></item> <dt><code>{fetch_fun,FetchFun}</code></dt> <item><p>Call back function to fetch an external resource.</p></item> <dt><code>{fetch_path,PathList}</code></dt> <item><p>PathList is a list of directories to search when fetching files. If the file in question is not in the fetch_path, the URI will be used as a file name.</p></item> <dt><code>{state,State}</code></dt> <item><p>It is possible by this option to provide a state with process information from an earlier validation.</p></item> </taglist> </item> </taglist></div> <div class="functions"><h4>Functions</h4><hr /> <div class="function"> <h3 id="validate/2">validate(Xml::Element, State) -&gt; Result</h3> <div class="description"><span id="validate-2"> </span> <p>Equivalent to <a href="#validate-3" class="seealso">validate(Element, State, [])</a>.</p> </div></div> <div class="function"> <h3 id="validate/3">validate(Xml::Element, State, Opts::Options) -&gt; Result</h3> <ul class="type"> <li><code>Element = XmlElement</code></li><li><code>Options = option_list()</code></li><li><code>Result = {ValidElement, global_state()} | {error, Reasons}</code></li><li><code>ValidElement = XmlElement</code></li><li><code>State = global_state()</code></li><li><code>Reasons = [ErrorReason] | ErrorReason</code></li></ul> <div class="description"><span id="validate-3"> </span> <p>Validates a parsed well-formed XML element (Element). </p><p>A call to validate/2 or validate/3 must provide a well formed parsed XML element <code>#xmlElement{}</code> and a State, <code>global_state()</code>, which holds necessary information from an already processed schema. Thus validate enables reuse of the schema information and therefore if one shall validate several times towards the same schema it reduces time consumption.</p> <p>The result, ValidElement, is the valid element that conforms to the post-schema-validation infoset. When the validator finds an error it tries to continue and reports a list of all errors found. In those cases an unexpected error is found it may cause a single error reason. </p> <p> Usage example:</p> <p> <code>1&gt;{E,_} = xmerl_scan:file("my_XML_document.xml").</code><br /> <code>2&gt;{ok,S} = xmerl_xsd:process_schema("my_XML_Schema.xsd").</code><br /> <code>3&gt;{E2,_} = xmerl_xsd:validate(E,S).</code> </p> <p> Observe that E2 may differ from E if for instance there are default values defined in <code>my_XML_Schema.xsd</code>.</p> </div></div> <div class="function"> <h3 id="state2file/1">state2file(S::State) -&gt; ok | {error, Reason}</h3> <div class="description"><span id="state2file-1"> </span> <p>Same as state2file(State,SchemaName)</p> <p>The name of the saved file is the same as the name of the schema, but with <code>.xss</code> extension.</p> </div></div> <div class="function"> <h3 id="state2file/2">state2file(S::State, FileName) -&gt; ok | {error, Reason}</h3> <ul class="type"> <li><code>State = global_state()</code></li><li><code>FileName = filename()</code></li></ul> <div class="description"><span id="state2file-2"> </span> <p>Saves the schema state with all information of the processed schema in a file. You can provide the file name for the saved state. FileName is saved with the <code>.xss</code> extension added.</p> </div></div> <div class="function"> <h3 id="file2state/1">file2state(FileName) -&gt; {ok, State} | {error, Reason}</h3> <ul class="type"> <li><code>State = global_state()</code></li><li><code>FileName = filename()</code></li></ul> <div class="description"><span id="file2state-1"> </span> <p>Reads the schema state with all information of the processed schema from a file created with <code>state2file/[1,2]</code>. The format of this file is internal. The state can then be used validating an XML document.</p> </div></div> <div class="function"> <h3 id="process_validate/2">process_validate(Schema, Xml::Element) -&gt; Result</h3> <div class="description"><span id="process_validate-2"> </span> <p>Equivalent to <a href="#process_validate-3" class="seealso">process_validate(Schema, Xml, [])</a>.</p> </div></div> <div class="function"> <h3 id="process_validate/3">process_validate(Schema, Xml::Element, Opts::Options) -&gt; Result</h3> <ul class="type"> <li><code>Schema = filename()</code></li><li><code>Element = XmlElement</code></li><li><code>Options = option_list()</code></li><li><code>Result = {ValidXmlElement, State} | {error, Reason}</code></li><li><code>Reason = [ErrorReason] | ErrorReason</code></li></ul> <div class="description"><span id="process_validate-3"> </span> <p>Validates a parsed well-formed XML element towards an XML schema. </p><p> Validates in two steps. First it processes the schema, saves the type and structure info in an ets table and then validates the element towards the schema.</p> <p> Usage example:</p> <p> <code>1&gt;{E,_} = xmerl_scan:file("my_XML_document.xml").</code><br /> <code>2&gt;{E2,_} = xmerl_xsd:validate("my_XML_Schema.xsd",E).</code> </p> <p> Observe that E2 may differ from E if for instance there are default values defined in <code>my_XML_Schema.xsd</code>.</p> </div></div> <div class="function"> <h3 id="process_schema/1">process_schema(Schema) -&gt; Result</h3> <div class="description"><span id="process_schema-1"> </span> <p>Equivalent to <a href="#process_schema-2" class="seealso">process_schema(Schema, [])</a>.</p> </div></div> <div class="function"> <h3 id="process_schema/2">process_schema(Schema, Options) -&gt; Result</h3> <ul class="type"> <li><code>Schema = filename()</code></li><li><code>Result = {ok, State} | {error, Reason}</code></li><li><code>State = global_state()</code></li><li><code>Reason = [ErrorReason] | ErrorReason</code></li><li><code>Options = option_list()</code></li></ul> <div class="description"><span id="process_schema-2"> </span> <p>Reads the referenced XML schema and checks that it is valid. Returns the <code>global_state()</code> with schema info or an error reason. The error reason may be a list of several errors or a single error encountered during the processing.</p> </div></div> <div class="function"> <h3 id="process_schemas/1">process_schemas(Schemas) -&gt; Result</h3> <div class="description"><span id="process_schemas-1"> </span> <p>Equivalent to <a href="#process_schema-2" class="seealso">process_schema(Schemas, [])</a>.</p> </div></div> <div class="function"> <h3 id="process_schemas/2">process_schemas(Schemas, Options) -&gt; Result</h3> <ul class="type"> <li><code>Schemas = [{NameSpace, filename()} | Schemas] | []</code></li><li><code>Result = {ok, State} | {error, Reason}</code></li><li><code>Reason = [ErrorReason] | ErrorReason</code></li><li><code>Options = option_list()</code></li></ul> <div class="description"><span id="process_schemas-2"> </span> <p>Reads the referenced XML schemas and controls they are valid.
